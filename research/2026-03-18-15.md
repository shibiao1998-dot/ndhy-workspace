## 二、分方向详细发现

### 2.1 AI Agent 组织管理

#### AGENTS.md 规范正式定稿（⭐⭐⭐ 重要）
- **来源**：[asdlc.io/practices/agents-md-spec](https://asdlc.io/practices/agents-md-spec/)（2026-03-16 发布）
- **核心发现**：
  - Gloaguen et al. (2026) 对 138 个真实代码库的研究发现：**LLM 生成的上下文文件降低 Agent 任务成功率，同时增加推理成本 20%+**
  - 人类手写的上下文文件仅带来 +4% 的边际改善——且只在**最精简**时有效
  - "不必要的需求主动伤害 Agent 表现，不是因为 Agent 忽略它们，而是因为 Agent 忠实执行它们"
  - **Pink Elephant Problem（粉色大象问题）**：告诉 LLM"不要用 tRPC"反而让 tRPC 在注意力机制中被高度激活。AGENTS.md 中的每条"禁止"指令都是代码库摩擦的信号
  - 规范推荐：Mission（2-4 句）→ Toolchain Registry（指向配置文件，不重复描述）→ Judgment Boundaries（NEVER/PREFER/MAY 三级）
- **对我们的影响**：
  - ✅ 验证我们"设定=宪法，精简不浪费一行"的理念
  - ⚠️ 我们 AGENTS.md + CLAUDE.md 可能存在冗余——需要审计哪些约束已被工具链覆盖
  - ⚠️ 我们的"禁止操作"清单可能触发粉色大象效应——考虑改为"只允许"式白名单

#### Claude Code Agent Teams 生态爆发（⭐⭐⭐ 重要）
- **来源**：claudefa.st、medium.com、ksred.com 等多个深度指南（本周密集发布）
- **核心发现**：
  - Agent Teams 与 Subagents 的**本质区别是通信**：Subagents 只能向父 Agent 报告结果，不能互相通信；Agent Teams 通过共享任务列表+信箱直接协调
  - "Subagents 是你派出去跑腿的承包商。Agent Teams 是坐在同一间会议室的项目组"
  - **成本画像明确**：Subagent 4-7x，Agent Teams ~15x（Anthropic 官方数据）
  - **最佳使用场景**：竞争性假设调试（各自测试不同理论并互相反驳）、跨层协调（前端/后端/测试各自拥有）、辩论与共识（多方论证后收敛）
  - **不适用**：顺序任务、同文件编辑、紧耦合依赖
  - `--dangerously-skip-permissions` + `--worktree` 是实战标配（解决权限瓶颈 + git worktree 隔离）
- **对我们的影响**：
  - ✅ 验证我们 ACP-First + Agent Team 宪法级决策
  - 🔧 需要建立**成本决策矩阵**：什么时候用 subagent（4-7x）vs Agent Team（15x）vs 单 session
  - 🔧 考虑在指令包中增加 `--worktree` 使用指引，减少 Agent 间文件冲突

#### Deep Agents: Harness Engineering 实战指南
- **来源**：[agentnativedev.medium.com](https://agentnativedev.medium.com/)（2026-03-17 发布）
- **核心观点**：LangChain 团队仅改 harness 不改模型，Terminal Bench 2.0 从 52.8% 跳到 66.5%（Top 30 → Top 5）
- **对我们的影响**：再次验证"Harness > Model"——我们的 SOUL+STANDARDS+Skills 架构就是 Harness 实践

### 2.2 上下文工程 / Context Engineering

#### Prompt Engineering → Context Design 行业共识巩固
- **来源**：[SDG Group](https://www.sdggroup.com/en/insights/blog/the-evolution-of-prompt-engineering-to-context-design-in-2026)（2026-03-12）
- **核心观点**：
  - 2022 = Prompt（单条指令）→ 2024 = Memory & History → 2025 = Context Linking → **2026 = Context Design（动态信息生态系统）**
  - "Context Engineer 不再是设置语气的人，而是信息架构师"
  - Context Design = 编排来自多个数据源的信息流，随交互展开动态更新
  - 无编程知识的专业人员也能生成高度复杂的 AI 驱动解决方案
- **对我们的影响**：✅ 我们的四层架构（HOT/WARM/项目级/应急）+ 渐进式披露 = 行业定义的 Context Design 实践

#### OpenViking 持续升温（15.4K⭐ → 媒体密集报道）
- **来源**：MarkTechPost、byteiota、decisioncrafters、Reddit 等（本周 5+ 篇报道）
- **新信息**：
  - 被定位为"解决 AI Agent 健忘症"的方案
  - 宣称可降低 95% Agent 记忆成本

  - L0（元数据/摘要）→ L1（核心内容）→ L2（完整细节）三级按需加载
- **对我们的影响**：
  - 我们的 HOT/WARM/项目级/应急 四层与 L0/L1/L2 高度同构
  - 🔧 值得评估是否引入 OpenViking 替代手动记忆管理——但当前我们的文件系统方案更轻量、更可控

## 三、对组织有价值的 5 条经验/发现

### 1. 🔴 AGENTS.md 越精简越好——冗余指令反伤 Agent（研究实证）
**Gloaguen 2026 研究**证实 LLM 生成的上下文文件降低任务成功率 20%+。我们需要审计 AGENTS.md + CLAUDE.md + 各角色 STANDARDS.md，剔除已被工具链覆盖的重复约束。

### 2. 🔴 粉色大象效应——"禁止"指令需改为"只允许"
研究发现告诉 Agent"不要用 X"反而激活 X 的注意力权重。我们指令包中的"禁止操作清单"应考虑改为白名单式（"只操作这些目录/文件"）。

### 3. 🟡 Agent Teams vs Subagent 的成本决策需量化
行业共识：Subagent 4-7x，Agent Teams 15x。我们应建立成本基线，明确何时用 subagent（独立任务、无需互通）vs Agent Teams（需要直接协调、竞争性假设）。

### 4. 🟢 Context Engineering 行业共识已稳固——我们的架构领先
从 SDG Group 到 ASDLC 到 Epsilla，2026 行业共识是"竞争优势不属于拥有最大模型的人，而属于拥有最有效 Harness 的人"。我们的四层记忆+渐进式披露+角色化指令包 = 行业领先实践。

### 5. 🟡 教育 AI 进入政策驱动期——窗口期缩短
中国两会政协委员公开引用 AI 教育平台，SAT 培训行业被 AI 颠覆的报道登上主流媒体。教育 AI 从实验阶段进入政策推动+商业验证阶段，华渔需要加速。
