## 一、AI Agent 组织管理

### 1.1 行业趋势：Harness Engineering 成为新焦点

**核心信号：** 行业重心正从"模型能力"转向"Agent 控制系统"——即 Harness Engineering。

- **LangChain 发文《The Anatomy of an Agent Harness》**（2026-03-16）
  - 核心公式：**Agent = Model + Harness**
  - Harness 定义：模型之外的一切——系统提示、工具描述、执行基础设施、编排逻辑、Hooks/中间件
  - 关键组件：文件系统（最基础的 Harness 原语）→ Bash/代码执行 → 沙箱 → 记忆与搜索
  - **与 NDHY 关联**：我们的 AGENTS.md + SOUL.md + Skill 系统本质上就是一个 Harness 架构

- **Epsilla 发文《Harness Engineering: Why the Focus is Shifting from Models to Agent Control Systems》**（2026-03-12）
  - 引用 OpenAI 2026-02 论文：小团队用 Agent 构建了 100 万行代码的产品，~1500 个自动化 PR
  - **核心警告**：AGENTS.md 文件 ≠ 充分的 Harness。一个静态 Markdown 文件是脆弱的，会随系统演化而腐化
  - **更强的 Agent 需要更严格的约束，而非更宽松的环境**
  - 缺失的关键组件：功能正确性验证（当前 Harness 过度关注代码质量，忽视了业务行为正确性）
  - 建议 4 个补充组件：用户价值 E2E 验证、关键路径 SLO、对抗性测试数据集、意图偏差检测

- **HumanLayer 发文《Harness Engineering for Coding Agents》**（2026-03-12）
  - "Harness Engineering" 是 "Context Engineering" 的子集，专注于编码 Agent 的上下文管理
  - Context Engineering 是 Prompt Engineering 的超集

## 二、Context Engineering / Harness Engineering

### 2.1 从 Prompt Engineering 到 Context Engineering 的演进

- **SDG Group《The Evolution of Prompt Engineering to Context Design in 2026》**（2026-03-12）
  - 2026 年 prompt engineer 角色转型：不再是写好提示词，而是**帮助 Agent 理解上下文并高效自动化任务**
  - Prompt Engineer → Context Designer

- **Instinctools《Context Engineering in AI》**（2026-03-11）
  - Context Engineering 核心策略：上下文注入、中间件基础设施层、状态管理
  - 与 Prompt Engineering 的区别：Prompt 是一次性的，Context 是系统性的

- **ArXiv 论文《Building Effective AI Coding Agents for the Terminal》**（2026-03-05）
  - OPENDEV 框架：强制显式推理阶段 + 优先上下文效率
  - 提供了终端优先 AI 辅助的蓝图

### 2.2 关键洞察

**Medium 文章《The State of AI Coding Agents 2026》**指出：
> "与编码 Agent 合作的真正技能不再是 prompt design，而是 context engineering。"

LangChain 的 Harness 文章预测：
> "模型会在规划、自我验证和长期连贯性方面原生变好，因此需要更少的上下文注入。但就像 prompt engineering 至今仍有价值一样，harness engineering 对构建好的 Agent 将持续有用。"

## 六、对 NDHY 的关键发现与建议

### 🔑 发现 1：我们的架构方向是正确的，但需要进化

Epsilla 文章直接点名了 AGENTS.md 文件的局限性——静态 Markdown 会腐化。我们的 AGENTS.md + SOUL.md + Skill 系统比单文件 AGENTS.md 更完善，但仍需关注**可验证性**问题。

**建议**：
- 增加"意图偏差检测"机制——子 Agent 产出不仅要通过形式检查，还要验证是否符合业务意图
- 对应专家：**Leader**（流程改进）

### 🔑 发现 2：Harness > Model 已成行业共识

OpenAI 论文 + LangChain + Epsilla + HumanLayer 同时指向同一个结论：**竞争优势不在最大的模型，而在最有效的 Harness**。我们的 Skill 系统、指令包机制、分层质量检查本质上都是 Harness Engineering。

**建议**：
- 将 NDHY 的 Harness 架构文档化为可分享的最佳实践
- 对应专家：**架构师**（技术架构）

### 🔑 发现 3：OpenPencil 是体验设计的革命

这是首个让 AI Agent 直接操作设计稿的开源工具。内置 MCP 服务器意味着 Claude Code 可以直接编辑 UI 设计——这打通了"设计→开发"的最后一公里。

**建议**：
- 评估 OpenPencil 在 NDHY 设计流程中的可行性
- 对应专家：**体验设计专家**（研究）、**前端开发专家**（集成评估）

### 🔑 发现 4：Agent Teams 已是生产力标配

Claude Code Agent Teams 已从研究预览进入实战，配套工具（Antfarm、Agent-Teams-Lite、ClaudeFast）快速成熟。我们应该深入研究这些工具的编排模式，可能优化现有的 ACP 调度逻辑。

**建议**：
- 研究 Agent-Teams-Lite 的零依赖 Spec-Driven 模式，可能简化我们的指令包组装
- 研究 Antfarm 是否适合替代/补充现有的 ACP 调度
- 对应专家：**Leader**（组织进化）

### 🔑 发现 5：教育 AI 的深层方向——Agency over AI

Washington Post 和 Guardian 的报道指向一个重要转变：教育 AI 不应仅关注"用 AI 提效"，更要关注"让学生对 AI 有主动权"。这对教育产品的定位有战略意义。

**建议**：
- 在产品定位中考虑"AI Agency"维度
- 对应专家：**教育领域专家**（理论研究）、**产品定义专家**（战略定位）
