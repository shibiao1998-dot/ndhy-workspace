## 二、Harness Engineering — 2026 年最重要的范式转变

### 2.1 核心概念

**Harness Engineering（线束工程）** 是 2026 年 Agent 领域最热门的新概念，由 HumanLayer 团队定义：

> "每次 Agent 犯错，就花时间工程化一个解决方案，让 Agent 永远不再犯同样的错。"
> — Mitchell Hashimoto

**层次关系**：
```
Context Engineering（上下文工程）
  └── Harness Engineering（线束工程）
        ├── System Prompt / CLAUDE.md / AGENTS.md
        ├── Tools / MCP Servers
        ├── Sub-agents
        ├── Skills
        ├── Memory
        └── Hooks / Back-pressure
```

**Harness 的四大定制杠杆**：
1. **系统提示词**（规则、角色、约束）
2. **工具与 MCP**（能力扩展）
3. **上下文管理**（信息流动）
4. **子 Agent 编排**（任务分解）

### 2.2 学术论文：OPENDEV

**论文**：[Building Effective AI Coding Agents for the Terminal](https://arxiv.org/abs/2603.05344)（2026-03-05，已更新至 v3）

开源终端编码 Agent，Rust 实现，核心创新：
- **双 Agent 架构**：规划 Agent 与执行 Agent 分离
- **惰性工具发现**：不预加载所有工具
- **自适应上下文压缩**：渐进式减少旧观察数据
- **自动记忆系统**：跨会话积累项目知识
- **事件驱动系统提醒**：对抗指令衰减（instruction fade-out）

**与 NDHY 的对应关系**：
| OPENDEV 概念 | NDHY 现有实现 |
|-------------|-------------|
| 双 Agent 架构 | Leader（规划）+ 专家（执行） |
| 自适应上下文压缩 | context-management 技能 |
| 自动记忆系统 | MEMORY.md + memory/*.md 分层 |
| 事件驱动提醒 | 心跳巡查机制 |
| 惰性工具发现 | 渐进式技能披露 |

### 2.3 行业文章：Prompt → Context Design 的进化

**SDG Group** 发表文章明确 2026 年趋势：
- Prompt Engineering → **Context Design**：从单条指令优化升级为动态信息生态系统设计
- Context Designer 不只设定语气和角色，而是**编排多源数据流**，让模型的上下文窗口成为持续演化的状态
- 预测：没有高级编程知识的专业人员也能生成复杂的 AI 驱动解决方案

### 2.4 AGENTS.md 最佳实践

**gtcode.com** 深度文章揭示 AGENTS.md 分层策略：
- OpenAI 内部仓库使用了 **88 个 AGENTS.md 文件**，每个子系统一个
- 根文件建立全局默认值，子目录文件覆盖本地规则
- 推荐章节：Build & Test、Architecture Overview、Security、Git Workflows、Conventions & Patterns

## 三、Agent 记忆系统

### 3.1 高 Star 项目

| 项目 | Star | 说明 |
|------|------|------|
| [cognee](https://github.com/topoteretes/cognee) | 14.2K ⭐ | 知识引擎：向量搜索 + 图数据库 + 认知科学，6 行代码接入 |
| [hindsight](https://github.com/vectorize-io/hindsight) | 4.3K ⭐ | Agent 记忆系统：不只记忆，还能学习。LongMemEval 基准 SOTA |
| [Agent-Memory-Paper-List](https://github.com/Shichun-Liu/Agent-Memory-Paper-List) | 1.5K ⭐ | "Memory in the Age of AI Agents" 论文集 |
| [A-mem](https://github.com/agiresearch/A-mem) | 888 ⭐ | NeurIPS 2025 论文"Agentic Memory for LLM Agents" |

### 3.2 关键洞察

**cognee** 的架构对我们有参考价值：
- 统一摄入任何格式数据 → 本地运行
- **持续学习 Agent**：从反馈中学习、跨 Agent 知识共享
- 实体隔离（per-user/per-tenant）+ 审计追踪

**hindsight** 声称是"最准确的 Agent 记忆系统"：
- 2 行代码即可为现有 Agent 添加记忆
- LLM Wrapper 模式：透明拦截 LLM 调用，自动存取记忆
- Docker 一键部署，支持多种 LLM 提供者

## 六、对 NDHY 的价值提取

### 🔑 5 条核心发现

#### 1. Harness Engineering 验证了我们的方向
NDHY 的 SOUL.md + AGENTS.md + Skill 体系 + 记忆分层 **完全对应** Harness Engineering 的四大杠杆。我们不是在"发明"什么，而是在实践行业正在收敛的最佳范式。**但我们可以更系统化**——学习 OpenAI 88 个 AGENTS.md 的分层策略，为每个专家目录部署专属 AGENTS.md。

#### 2. Claude Code Agent Teams 是我们 ACP 的天然升级
我们已经在用 ACP + Claude Code，Agent Teams 是下一步：
- 共享任务列表 → 替代 PROJECT.md 手动同步
- Agent-to-Agent 通信 → 减少 Leader 中转开销
- Git worktree 隔离 → 并行开发不冲突

#### 3. "指令衰减"是真实问题，OPENDEV 提出了解法
OPENDEV 论文明确描述了 **instruction fade-out**（指令在长上下文中逐渐被模型"遗忘"），并用**事件驱动系统提醒**对抗。我们的心跳机制部分解决了这个问题，但可以更系统——在长任务中定期注入关键指令提醒。

#### 4. Agent 记忆正在从"检索"走向"学习"
cognee 和 hindsight 都在做同一件事：让 Agent 不只是记住对话历史，而是**从交互中学习模式、提炼规则**。我们的 context-management 技能中的"信号捕获 → 模式晋升"流程是这个方向的早期实现。

#### 5. 教育 AI 是蓝海
高质量的教育 AI 开源项目几乎不存在。大量讨论停留在"AI 对教育的影响"层面，但少有人在做"AI 原生的教育产品"。这对 NDHY 的教育方向是好消息——竞争少，空间大。
