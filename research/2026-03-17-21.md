## 二、重点发现

### 1. 🤖 AI Agent 组织管理 — "编排层"成为新战场

**关键项目：**

| 项目 | 组织 | 核心能力 | 关注点 |
|------|------|---------|--------|
| **[ComposioHQ/agent-orchestrator](https://github.com/ComposioHQ/agent-orchestrator)** | Composio | 并行编码 Agent 编排，管理 30+ Agent 同时工作 | ⭐⭐⭐ 与我们架构高度相关 |
| **[bytedance/deer-flow v2.0](https://github.com/bytedance/deer-flow)** | 字节跳动 | SuperAgent Harness：子 Agent + 记忆 + 沙箱 + 技能 | ⭐⭐⭐ Context Engineering 概念 |
| **[microsoft/agent-framework](https://github.com/microsoft/agent-framework)** | 微软 | Python + .NET 双语言，图编排，OpenTelemetry 可观测性 | ⭐⭐ 企业级标准 |
| **[google/adk-python](https://github.com/google/adk-python)** | Google | Code-first Agent 开发，A2A 协议，Agent Config 无代码建 Agent | ⭐⭐ A2A 协议值得关注 |
| **[CopilotKit/CopilotKit](https://github.com/CopilotKit/CopilotKit)** | CopilotKit | AG-UI Protocol（Agent-UI 通信协议），Generative UI | ⭐⭐ 前端 Agent 集成 |

**核心洞察：**

1. **Agent Orchestrator 模式成熟化**：ComposioHQ 的 agent-orchestrator 实现了"30 个 Agent 并行开发"的编排——每个 Agent 独立 worktree、独立 branch、独立 PR，CI 失败自动修复，Review 评论自动处理。**这是 Agent Team 规模化的工业级实现。**

2. **DeerFlow v2.0 的 Context Engineering**：字节跳动将 DeerFlow 从 Deep Research 工具升级为 SuperAgent Harness，核心能力包括：技能系统（Skills & Tools）、子 Agent、沙箱文件系统、Context Engineering、长期记忆。**这跟我们的架构几乎同构。**

3. **协议层竞争激烈**：Google 的 A2A（Agent-to-Agent）、CopilotKit 的 AG-UI（Agent-to-UI）、MCP（Model-Context Protocol）三大协议正在争夺标准位。**多协议共存是趋势。**

4. **Microsoft Agent Framework 统一 Semantic Kernel + AutoGen**：提供了迁移指南，说明行业在做框架整合收敛。

### 2. 📐 上下文/提示词工程

**关键发现：**

- **DeerFlow v2.0 将 Context Engineering 作为核心特性单独列出**，说明"上下文工程"已从技巧升级为系统能力
- **Google ADK 的 Agent Config**：支持无代码构建 Agent（YAML 配置），是 Prompt Management 的产品化形态
- **CopilotKit 的 Shared State**：Agent 和 UI 组件共享状态层，实时同步，这是上下文在前端的新形态
- **learn-claude-code (shareAI-lab)**：从零构建类 Claude Code Agent，核心理念 "Bash is all you need"

**对我们的启示：**
- 我们的 SOUL.md + AGENTS.md + SKILL 三层知识架构（宪法-操作手册-参考材料）在行业中属于领先实践
- 需要关注 Context Engineering 作为独立能力域的发展

## 三、对 NDHY AI Agent Team 的价值提炼

### 🔑 5 条核心经验

1. **Agent Orchestrator 模式验证了我们的方向**
   - ComposioHQ 的 agent-orchestrator 证明"一个编排层 + N个并行 Agent"是可行且高效的。他们的 8 槽插件架构（Runtime/Agent/Workspace/Tracker/SCM/Notifier/Terminal/Lifecycle）值得借鉴。
   - **行动建议**：评估 agent-orchestrator 的插件模型，看是否能为我们的 ACP 调度层提供参考。

2. **Context Engineering 是下一个系统能力**
   - DeerFlow v2.0 将 Context Engineering 与 Skills、Sub-Agents、Memory 并列为核心能力。这不再是"提示词技巧"，而是一个独立的工程学科。
   - **行动建议**：将我们的三层知识架构（SOUL/AGENTS/SKILL）正式定义为 Context Engineering 框架，形成可复用方法论。

3. **协议标准化竞赛：A2A + AG-UI + MCP 三足鼎立**
   - Google A2A 管 Agent 间通信、CopilotKit AG-UI 管 Agent-UI 交互、MCP 管 Agent-工具调用。
   - **行动建议**：持续跟踪三大协议的采纳进展，特别是 MCP 已经最成熟。

4. **Presenton 是可直接落地的教育工具**
   - 开源（Apache 2.0）、支持 PPTX 模板、内置 MCP Server、Docker 部署。非常适合教育课件自动生成场景。
   - **行动建议**：安排技术调研专家深入评估 Presenton 的技术栈和定制能力。

5. **设计工具正在被 Agent 侵入**
   - TellFigma、Penpot MCP、Stitch-Kit、design-with-claude 等项目表明，AI Agent 直接操作设计工具已成现实。
   - **行动建议**：评估是否需要为体验设计专家配备设计工具 MCP 能力。
