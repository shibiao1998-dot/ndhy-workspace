## 二、AI Agent 组织管理

### 2.1 oh-my-claudecode（⭐ 10,097）
- **仓库**：https://github.com/Yeachan-Heo/oh-my-claudecode
- **核心**：Claude Code 多 Agent 编排框架，零学习曲线
- **5 种执行模式**：从 Team-backed 编排到 token 高效重构
- **关键能力**：
  - 自动并行化 — 复杂任务分发到专业 Agent
  - 持续执行 — 直到任务验证完成才停止
  - 成本优化 — 智能模型路由，节省 30-50% token
  - 经验学习 — 自动提取并复用问题解决模式
  - 实时可见性 — HUD 状态栏
- **deep-interview**：苏格拉底式提问，在写代码前厘清需求
- **启示**：其 "autopilot" 模式 + "deep-interview" 需求澄清 = 我们 Leader 工作流 + 需求挖掘 Skill 的自动化版本。可考虑研究其编排策略。

### 2.2 ClawPort（⭐ 550）
- **仓库**：https://github.com/JohnRiceML/clawport-ui
- **核心**：OpenClaw Agent 团队的可视化管理中心
- **功能矩阵**：
  - Org Chart — Agent 组织结构图
  - Agent Chat — 直接对话 Agent（支持视觉/语音）
  - Kanban Board — 跨 Agent 任务看板
  - Cron Monitoring — 定时任务监控
  - Cost Tracking — token 使用与成本分析
  - Live Logs — 实时日志流
  - Memory Browser — 团队记忆浏览器
- **技术**：直接连接 OpenClaw Gateway，无需额外 API Key
- **🔴 行动建议**：这是我们生态内的项目，应立即评估试用

### 2.3 HydraTeams（⭐ 39）
- **仓库**：https://github.com/Pickle-Pixel/HydraTeams
- **核心**：翻译代理，让 Claude Code Agent Teams 可用任意模型（GPT/Gemini/Ollama）
- **启示**：模型无关的 Agent 团队是趋势，我们目前锁定 Claude 单模型

### 2.4 agent-spec（⭐ 65）
- **仓库**：https://github.com/Superfleys/agent-spec

### 2.5 行业动态

## 三、上下文工程 / Context Engineering

### 3.1 论文：Context Engineering: From Prompts to Corporate Multi-Agent Architecture
- **来源**：arXiv:2603.09619（2026-03-10）
- **核心观点**：
  - Context Engineering（CE）是独立于 Prompt Engineering 的新学科
  - 5 个上下文质量标准：**相关性、充分性、隔离性、经济性、溯源性**
  - 上下文 = Agent 的操作系统
  - 提出 4 层成熟度金字塔：Prompt Engineering → Context Engineering → Intent Engineering → Specification Engineering
  - "控制 Agent 上下文的人控制其行为；控制其意图的人控制其策略；控制其规范的人控制其规模"
- **🔴 对 NDHY 的启示**：
  - 我们的 SOUL.md（Intent）+ STANDARDS.md（Specification）+ 三层记忆（Context）正好对应论文的金字塔模型
  - **建议**：按论文的 5 质量标准审查我们的指令包质量

### 3.2 论文：OPENDEV — Terminal Coding Agent 的上下文工程实践
- **来源**：arXiv:2603.05344（2026-03-05）
- **核心技术**：
  - 双 Agent 架构：Planning Agent + Execution Agent 分离
  - 自适应上下文压缩：渐进式减少旧观察
  - 懒加载工具发现
  - 自动记忆系统：跨 session 积累项目知识
  - 事件驱动系统提醒：对抗指令淡出
- **启示**：其"指令淡出对抗"机制 = 我们的心跳巡查 + 技能优先原则，方向一致

### 3.3 LangChain Deep Agents
- **发布**：2026-03-15
- **核心**：multi-step Agent 的结构化运行时
- **关键设计**：
  - `write_todos` — 规划和任务分解
  - 文件系统上下文管理 — 大上下文卸载到文件
  - Subagent 上下文隔离 — 主线程保持干净
  - 多种后端：StateBackend / FilesystemBackend / LocalShellBackend
- **启示**：与我们的 ACP + 增量写入文件策略高度吻合。"文件即记忆"不是我们独创，是行业共识

### 3.4 JetBrains Research：Efficient Context Management
- **发布**：2026-03-12
- **核心发现**：随着上下文窗口增长，LLM 利用信息的效率反而下降
- **启示**：验证了我们"渐进式披露"原则的正确性——不堆上下文

### 3.5 Nyne（TechCrunch 报道）
- **核心**：给 AI Agent 补充"人类上下文"（个人偏好、决策模式）
- **启示**：类似我们的 USER.md 理念，但做成了产品

## 八、对 NDHY 组织的 3-5 条核心经验

### 1. 🔴 我们的架构方向被行业验证
Context Engineering 论文的 4 层金字塔（Prompt → Context → Intent → Specification）与我们的组织设计高度吻合：
- SOUL.md = Intent Engineering
- STANDARDS.md = Specification Engineering
- 三层记忆 + 指令包 = Context Engineering

**行动**：用论文的 5 质量标准（相关性/充分性/隔离性/经济性/溯源性）审查现有指令包。

### 2. 🔴 细粒度任务拆分是行业共识
Reddit r/ClaudeCode 社区多个高赞帖确认：每个 Agent session 只做一件事，出错率显著降低。oh-my-claudecode 的 autopilot 模式本质也是自动化的细粒度任务分发。

**行动**：继续强化"拆得够细就没有做不到的事"原则，记录任务粒度经验库。

### 3. 🟡 可视化管理工具成熟了
ClawPort 提供了我们一直缺的：org chart、kanban、成本追踪、实时日志、记忆浏览器。基于 OpenClaw 生态，零配置接入。

**行动**：评估 ClawPort 是否满足日常管理需求。

### 4. 🟡 "文件即记忆"是行业共识
LangChain Deep Agents、OPENDEV、oh-my-claudecode 都采用了"把大上下文卸载到文件系统"的策略。我们的"增量写入 + 三层记忆文件"策略与行业最佳实践一致。

### 5. 🟢 教育 AI 进入应用深水区
从个性化到情绪智能，从硬件到场景化。中国政策重心转向"AI 教育基本普及"。智能学伴、智能教师是明确方向。

**行动**：教育领域专家更新知识库，关注政策导向和落地案例。
