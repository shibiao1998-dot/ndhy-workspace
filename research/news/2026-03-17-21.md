### 3. 🆕 Jeremy Howard 炮轰 Claude Code 路线："这条路错了"
- **来源**：[36氪](https://36kr.com/p/3725127329888647)（昨天 17:42）
- **摘要**：fast.ai 创始人、ULMFiT 论文作者 Jeremy Howard 在播客中系统性批评 Claude Code 及 AI 编程工具方向：
  - **核心观点**：Claude Code 让开发者与代码脱节，是"不人道的"方向
  - **点名批评**：Dario Amodei（Anthropic CEO）和马斯克"根本不懂现代软件工程"
  - **关键论断**：
    - "90% 的代码由模型代劳，但效率没有显著提升，因为编程从来不是效率瓶颈"
    - Claude Code 写的 C 编译器"本质上是风格迁移，不是真正突破"
    - AI 编程累积技术债，企业"既无法维护现有产品，也难以开发新产品"
  - **他的替代方案**：人机紧密互动的协作开发（类 Smalltalk/APL 理念），而非"输入 prompt 等输出"
- **对我们的影响**：🟡🟡 **必须认真对待的反面声音。** 我们深度依赖 Claude Code（ACP Runtime），Jeremy 的批评指向了一个真实风险：过度依赖 AI 生成代码 → 团队对系统理解降低 → 技术债累积。建议：
  - (1) 在代码审查流程中强化"人类可理解性"标准
  - (2) 关注代码库的可维护性，不只是"能跑就行"
  - (3) 但也要辩证看待——Jeremy 代表的是传统软件工程视角，Agent Team 模式的效率优势是真实的
