### 2. 🆕 "Agentic Context Management" — 新论文提出模型应自主管理上下文
- **来源**：[Dead Neurons Substack](https://deadneurons.substack.com/p/agentic-context-management-why-the)（4 小时前）
- **摘要**：深度分析文章探讨"谁应该管理 Agent 的上下文"。引用了 Zhang, Kraska & Khattab (2026) 提出的 **Recursive Language Models (RLM)** — 将用户 prompt 视为 Python REPL 环境中的变量而非直接进入上下文窗口的内容，被评为"设计空间中最激进的立场"。核心论点：让模型自己决定什么该记住、什么该丢弃。
- **对我们的影响**：🟡 **高度相关。** 我们的 context-management skill 做的就是类似的事——分层记忆 + 压缩协议 + 模式晋升。RLM 的 REPL 思路值得研究，可能启发我们的上下文管理进化方向。

### 5. 🆕 ArXiv 新研究："Prompt Injection as Role Confusion"
- **来源**：[arXiv:2603.12277](https://arxiv.org/abs/2603.12277)
- **摘要**：新论文将 Prompt Injection 攻击追溯到"角色混淆"——模型从内容中推断角色指令，导致安全训练失效。这是对 Agent 安全的底层机制研究。
- **对我们的影响**：🟡 与我们的 EXTERNAL_UNTRUSTED_CONTENT 安全标记机制直接相关。OpenClaw 的安全包裹和 NVIDIA OpenShell 的策略引擎本质上都在解决角色混淆问题。
