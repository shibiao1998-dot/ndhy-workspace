### 1. 🔴🆕 Claude Code v2.1.76 发布 — Voice Mode / /Loop / 1M Context / Opus 4.6 默认 / Ultrathink
- **来源**：[pasqualepillitteri.it](https://pasqualepillitteri.it/en/news/381/claude-code-march-2026-updates)（8 小时前）
- **摘要**：Claude Code 发布 v2.1.76 重大更新，核心变化：
  - **Voice Mode**：语音交互编程
  - **/Loop 命令**：类 cron 的定期自动执行（代码审查、依赖更新、例行维护）
  - **1M 上下文窗口**：完整 100 万 token 上下文支持
  - **Opus 4.6 默认模型**：从 Sonnet 升级到 Opus 作为默认
  - **Ultrathink**：深度推理模式
  - **MCP Elicitation**：MCP 服务器交互增强
  - 更新命令：`npm update -g @anthropic-ai/claude-code`
- **对我们的影响**：🔴🔴🔴 **紧急行动项。** 我们当前使用 v2.1.37，已落后 39 个版本！(1) Opus 4.6 默认 = ACP 任务质量大幅提升。(2) 1M 上下文 = 大型项目不再需要手动分片。(3) /Loop = 可能重新设计部分定时任务。(4) Voice Mode = 全新的人机交互范式。**建议立即升级。**

### 2. 🔴🆕 Anthropic 移除 1M Token 长上下文溢价 — Opus/Sonnet 4.6 全价格段统一
- **来源**：[The New Stack](https://thenewstack.io/claude-million-token-pricing/)（15 小时前）| [The New Stack](https://thenewstack.io/anthropic-doubles-claude-usage-outside-peak-hours/)（10 小时前）
- **摘要**：Anthropic 宣布 Claude Opus 4.6 和 Sonnet 4.6 的 1M token 上下文窗口正式 GA，关键变化：**移除了超过 200K token 的长上下文溢价**，全量 1M token 使用标准价格。此前超过 200K token 会触发 premium pricing。
- **对我们的影响**：🔴 **成本直降 + 能力扩展。** (1) 大型代码库的 ACP 任务成本显著降低。(2) 长上下文不再需要权衡成本，可以更自由地注入完整项目上下文。(3) 配合 Claude Code v2.1.76 的 1M 上下文支持，我们的 ACP 工作流将获得质的飞跃。
