#### 🔑 关键发现：GSD-2 的进化路径

GSD-2 是 Get Shit Done 的 v2，从 Claude Code 的 prompt 框架进化为独立 CLI。核心突破：
- **Fresh session per task**：每个任务启动全新 context window，不积累垃圾
- **铁律：一个 task 必须装进一个 context window，装不下就拆成两个**
- **状态机驱动的 auto mode**：读磁盘上的 .gsd/ 文件确定下一步，不靠 LLM 自循环
- **崩溃恢复**：lock 文件 + session 取证
- **成本追踪**：per-unit token/cost 看板
- **卡住检测**：重试一次后停止并输出诊断

> 💡 **对我们的启示**：GSD-2 的 "task 必须装进一个 context window" 铁律和我们 AGENTS.md 中的"拆得够细就没有做不到的事"高度一致。但 GSD-2 做到了**程序化控制**（TypeScript 状态机），而不是靠 LLM 自觉。这是一个重要的工程化方向。

#### 🔑 关键发现：OpenViking 的三级上下文架构

字节跳动开源的 OpenViking 提出了**文件系统范式**管理 Agent 上下文：
- **L0/L1/L2 三级结构**：按需加载，显著节省 token
- **目录递归检索**：结合目录定位 + 语义搜索
- **可视化检索轨迹**：可观测的 context 调用链
- **自动会话管理**：自动压缩对话、提取长期记忆

> 💡 **对我们的启示**：我们的 MEMORY.md + memory/YYYY-MM-DD.md + PROJECT.md 三级记忆已经是类似的分层思路。但 OpenViking 做到了**可视化检索轨迹**——这是我们缺少的。当 Agent 的回答不对时，很难知道是「记忆没加载对」还是「推理出了问题」。

#### Headroom 的 Context 压缩

Headroom 作为透明代理，在 Agent 和 LLM 之间压缩 context：
- **92% 压缩率**（代码搜索 100 结果：17,765 → 1,408 tokens）
- **精度不降**：GSM8K、TruthfulQA、SQuAD v2 评测均无损
- **核心原理**：不丢数据，压缩后保留摘要，给 LLM 工具按需取回原文
- 一行环境变量即可接入（`ANTHROPIC_BASE_URL=http://localhost:8787 claude`）

> 💡 **对我们的启示**：Headroom 可以直接用于我们的 Agent 系统，降低 token 消耗。特别是子 Agent 处理大量代码时，92% 的压缩率非常可观。

#### Memori 已有 OpenClaw 插件

Memori 已经有官方的 OpenClaw 插件（`openclaw plugins install @memorilabs/openclaw-memori`），可以自动捕获和注入跨 session 记忆。值得评估是否能补充我们现有的文件记忆系统。
