## 二、各方向详细调研

### 2.1 AI Agent 组织管理

#### 🔥 Karpathy Loop（"自主研究循环"）
- **来源**：Fortune 2026-03-17 / Karpathy 本人 X 帖
- **核心**：AI Agent 自主运行实验循环 → 度量结果 → 调整代码 → 再实验。700次实验/2天。
- **三要素**：① 可修改的单文件 ② 可客观测量的指标 ③ 固定时间限制
- **启发**：这个模式可以应用到我们的**任何可量化优化任务**——不只是ML训练，任何有明确指标的流程都可以"Karpathy Loop"化。
- **Karpathy 预言**："下一步是异步大规模协作Agent，目标不是模拟单个博士生，是模拟整个研究社区。"

### 2.2 上下文工程（Context Engineering）

#### 🔥 CE 论文：从提示词到企业多Agent架构
- **来源**：arXiv:2603.09619（2026-03-10）
- **核心观点**：Context Engineering 是一个**独立学科**，关注设计、结构化和管理 AI Agent 做决策时的整个信息环境
- **五大质量标准**：相关性(Relevance)、充分性(Sufficiency)、隔离性(Isolation)、经济性(Economy)、可溯源(Provenance)
- **四层金字塔成熟度模型**：
  1. Prompt Engineering（提示工程）
  2. Context Engineering（上下文工程）
  3. Intent Engineering（意图工程）— 编码组织目标、价值观、权衡层级
  4. Specification Engineering（规范工程）— 机器可读的企业策略语料库
- **与我们的关系**：我们的架构（SOUL.md → AGENTS.md → Skills → References）正是 CE 的实践。论文验证了我们的方向。

#### Pre-Action Gates（mcp-memory-gateway）
- **来源**：GitHub/IgorGanapolsky
- **核心**：local-first上下文工程 + 预操作门控
- **亮点**：
  - 捕获反馈 → 验证 → 记忆 → 预防 → 门控 → 审计 的完整循环
  - PreToolUse hooks 物理阻断已知失败模式
  - 推理追踪（Reasoning Trace）让黑盒失败透明化
- **与我们的关系**：我们的"纠正→写入记忆→永不重犯"模式与此高度一致。Pre-Action Gates 概念值得深入研究。

## 三、对NDHY组织的5条核心经验

### 1. 🔴 Karpathy Loop 可立即落地
**经验**：任何有"可量化指标 + 可修改代码 + 固定时间"的任务都可以用Agent循环自动优化。
**行动建议**：评估我们的哪些流程（Skill质量评分、编译时间、测试覆盖率）适合做 Karpathy Loop 式自动优化。

### 2. 🔴 我们的 Context Engineering 架构领先行业
**经验**：学术界刚正式定义CE为独立学科（arXiv 2026-03-10），我们的 SOUL.md/AGENTS.md/Skills/References 层级式渐进披露架构已经在实践中验证了这个理论。
**行动建议**：用论文的"五大质量标准"（相关性/充分性/隔离性/经济性/可溯源）回头审视我们的上下文架构，查找优化空间。

### 3. 🟡 GSD v2 的"每任务清空上下文"值得学习
**经验**：GSD v2 每个task都创建全新的Agent会话，预装所有需要的上下文，避免上下文污染。这比让Agent积累垃圾上下文效果好得多。
**行动建议**：我们通过 ACP 的 `sessions_spawn` 已实现类似效果（每个子Agent是独立会话），可以进一步学习 GSD 的"pre-inline context injection"技术。

### 4. 🟡 Pre-Action Gates 是记忆系统的下一步进化
**经验**：mcp-memory-gateway 的"预操作门控"机制，在Agent执行操作前物理阻断已知失败模式，比"写入记忆希望Agent遵守"更可靠。
**行动建议**：评估在我们的工作流中引入类似的自动门控机制（特别是部署、代码提交等高风险操作）。

### 5. 🟡 Agent Team 已成行业共识
**经验**：Claude Code Agent Teams 正式铺开，多框架/多文章都在讲多Agent并行开发。我们的 ACP Agent Team 策略走在正确的路上。
**行动建议**：持续深化Agent Team实战经验，关注 Claude Code 后续的正式发布和新功能。
