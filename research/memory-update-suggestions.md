## 建议写入 MEMORY.md 的关键条目

### Agent 协作架构（来自协作通信调研）

- **Orchestrator 中介模式已确认为当前最佳方案**：`maxSpawnDepth: 2` + 共享文件（黑板模式），零开发成本，利用 OpenClaw 原生能力。Leader→Orchestrator→Worker 三层结构。（2026-03-15）

- **共享文件规范是 Agent 协作的关键基础设施**：项目目录下需建立 `contracts/`（API 契约）、`issues/`（问题记录）、`decisions/`（设计决策）、`status/`（进度状态）四个标准目录，Agent 通过文件传递上下文而非消息重复。（2026-03-15）

- **sessions_send Ping-Pong 是未来直接通信的钥匙**：支持 5 轮自动来回对话，需 `tools.agentToAgent.enabled = true`。当前飞书不支持 thread binding 是主要阻碍。（2026-03-15）

- **子 Agent 每轮修复需重新 spawn，上下文重注入是主要 Token 消耗点**：这是当前架构的固有代价，优化方向是共享文件减少消息中的上下文重复。（2026-03-15）
