### 关键趋势

1. **Builder-Validator 模式**成为 Agent Team 开发的标准范式 — Claude 生成任务依赖图，每个组件分配 builder-validator 对，最后加 integration validator
2. **Agent Drift（漂移）问题被社区广泛认知** — Reddit 用户反馈 Agent 在数十次迭代后"tone slowly shifts"，需要定期人类审查
3. **"编排工具 vs 并行会话"之争** — 编码任务更适合并行会话 + worktrees（保持 human-in-the-loop），非编码自动化更适合编排器模式

## 二、提示词/上下文工程 — 从 Prompt Engineering 进化到 Context Engineering

### 核心发现

**2026 年行业关键词从 "Prompt Engineering" 切换为 "Context Engineering" 和 "Harness Engineering"。**

1. **SDG Group: "Prompt Engineer → Context Designer"** — 上下文工程不再是写一条静态指令，而是构建"动态信息生态系统"：编排多源数据流、维护用户偏好和历史消息、适应交互演进。这比传统提示工程的维度高了一个数量级。

2. **Epsilla: "Harness Engineering"** — 焦点从模型转向 Agent 控制系统。"竞争优势不属于拥有最大模型的人，而属于拥有最高效 harness 的人"。Harness = scaffolding + context management + tool orchestration。

3. **OPENDEV 论文 (arXiv 2603.05344)** — 开源 Rust 编写的终端 AI 编码 Agent，核心创新：
   - **Dual-agent 架构**：规划与执行分离
   - **Adaptive context compaction**：渐进压缩旧观察信息
   - **Automated memory system**：跨会话累积项目知识
   - **Event-driven system reminders**：对抗"指令淡忘"
   - **Workload-specialized model routing**：按任务类型选择模型

### 对我们的映射

| OPENDEV 概念 | 我们的对应 | 差距 |
|-------------|-----------|------|
| Dual-agent（规划/执行分离） | Leader（规划）+ 执行专家 | ✅ 已实现 |
| Adaptive context compaction | MEMORY.md + working-buffer | ⚠️ 可优化压缩算法 |
| Automated memory system | memory/YYYY-MM-DD.md | ✅ 已实现 |
| Event-driven reminders | 心跳巡查 | ✅ 已实现 |
| Model routing | 当前单模型 | ⚠️ 可探索多模型路由 |

## 六、对组织的价值经验（Top 5）

### 1. 🔴 跨模型对抗审查（来自 metaswarm）
metaswarm 用 Claude/Gemini/Codex 交叉审查代码——实现者用一个模型，审查者用另一个模型。这比同模型自审更能发现盲区。**建议**：在 code-review 流程中试点"跨模型审查"——让 Gemini 审查 Claude 产出的代码。

### 2. 🔴 Builder-Validator 对模式（来自 Claude Code Agent Teams 最佳实践）
每个组件/模块分配一个 Builder Agent 和一个 Validator Agent，最终有 Integration Validator。这是我们"对抗式协作"的具体落地模式。**建议**：在下一个开发项目中试验此模式。

### 3. 🟡 Agent Drift 防护（来自社区经验）
Agent 在长时间运行后会产生"风格漂移"。解法：定期人类审查 + event-driven 系统提醒（OPENDEV 的做法）。我们的心跳机制已覆盖部分场景，但长时间运行的 ACP 会话需要额外的"指令刷新"机制。

### 4. 🟡 设计工具革命即将到来
Google Stitch（5 月发布）+ Figma Make 将打通"设计 → 代码"全流程。**建议**：🎨 体验设计专家 应密切关注 Google I/O 2026（5.19-20），评估 Stitch 对我们设计流程的影响。

### 5. 🟢 Context Engineering > Prompt Engineering
行业已进入"上下文工程"时代。我们的 SOUL.md/AGENTS.md/MEMORY.md 体系本质上就是 Context Engineering 的实践，方向正确。可参考 OPENDEV 的 adaptive context compaction 进一步优化。
