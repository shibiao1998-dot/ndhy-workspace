## 一、AI Agent 组织管理 — 多Agent协作

### 1.1 Claude Code Agent Teams 正式落地（⭐ 重点关注）

Anthropic 在 Opus 4.6 版本中正式发布了 **Agent Teams** 实验性功能，标志着多 Agent 协作进入原生支持阶段。

**核心机制**：
- 多个独立 Claude 实例通过**共享任务列表 + 邮箱系统**协调
- 每个 teammate 有明确角色定义、允许/禁止操作、独立 prompt
- 支持 `blockedBy` 依赖管理，自动阻塞未就绪任务
- Team Lead 负责任务分解、分配和协调

**实战案例 — 5-Agent 游戏工作室**（dev.to, 2026-03-16）：
- 一个非工程师用 5 个 Agent（builder/designer/researcher/grower/shipper）+ 1 个 Team Lead 同时运行
- 单次 session 完成 9/17 个任务（53%），覆盖代码开发、设计资源、市场调研、数据采集、发布准备
- **关键经验**：
  - 任务依赖自动阻塞防止协调失败
  - 专业化分工消除文件冲突（每个 Agent 有"禁区"）
  - 并行执行速度约为串行的 1/5
  - `--dangerously-skip-permissions` 在多 Agent 场景几乎必须（权限瓶颈被放大）

**对我们的启示**：我们的 ACP Agent Team 架构与 Claude Code Agent Teams 理念高度一致。他们的 `blockedBy` 依赖管理和"禁区"机制可以直接借鉴到我们的指令包设计中。

## 二、Context Engineering / 提示词工程

### 2.1 Agent-Skills-for-Context-Engineering（⭐ 高 Star 项目）

- **GitHub**: muratcankoylan/Agent-Skills-for-Context-Engineering

### 2.2 GSD-2（Get Shit Done v2）（⭐ 重点关注）

- **GitHub**: gsd-build/gsd-2
- **Stars**: 1,741（更新至 2026-03-18）
- **定位**：从 prompt 框架进化为真正的编码 Agent——基于 Pi SDK 的 TypeScript CLI

**核心能力**（v1 → v2 的飞跃）：

| 能力 | v1（Prompt 框架） | v2（Agent 应用） |
|------|------------------|-----------------|
| 上下文管理 | 靠 LLM 自己管 | 每任务新 session，编程控制 |
| 自动模式 | LLM 自循环 | 状态机读 .gsd/ 文件 |
| 崩溃恢复 | 无 | Lock 文件 + session 取证 |
| Git 策略 | LLM 写 git 命令 | Worktree 隔离、顺序提交、squash 合并 |
| 成本追踪 | 无 | 每单元 token/cost 账本 + 仪表盘 |
| 卡死检测 | 无 | 重试一次后停止 + 诊断 |
| 超时监控 | 无 | 软/空闲/硬超时 + 恢复引导 |

**对我们的直接启示**：
1. **每任务新 session** — 我们的 ACP 任务也应该严格隔离上下文
2. **Token 优化 Profile** — 按任务复杂度选择不同 token 策略
3. **Worktree 隔离** — 多 Agent 并行开发时用 git worktree 避免冲突
4. **成本仪表盘** — 我们缺少对 ACP 任务成本的追踪机制
