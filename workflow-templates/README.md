# 工作流模板索引

> 这些模板是**参考手册**，不是强制流程。项目管理专家完全可以根据实际情况裁剪、合并、跳过步骤。

## 定位

NDHY AI Agent Team 的 26 位专家覆盖产品全链路。这组模板提供常见协作场景的**参考路径**，帮助项目管理专家快速编排多专家协作，减少从零设计流程的成本。

**核心理念**：
- 模板是起点，不是终点——每个项目都有独特约束
- 能跳过的就跳过，能并行的就并行
- 项目管理专家拥有最终编排权

## 模板一览

| 模板 | 文件 | 适用场景 | 典型规模 |
|------|------|----------|----------|
| 标准开发流 | `standard-dev.md` | 新产品/新功能的完整开发 | 8-15 位专家，2-8 周 |
| 调研分析流 | `research-analysis.md` | 市场调研、竞品分析、用户研究 | 3-6 位专家，1-3 周 |
| 快速修复流 | `quick-fix.md` | 线上 bug、小需求、热修复 | 2-4 位专家，1-3 天 |
| 组织变更流 | `org-change.md` | 角色创建/升级、Skill 开发、流程改进 | 2-4 位专家，1-5 天 |

## 选择决策树

```
收到任务 →
├─ 线上 bug / 小修复 / 紧急补丁？       → quick-fix.md
├─ 需要先调研市场/竞品/用户？            → research-analysis.md
├─ 组织内部变更（角色/Skill/流程）？      → org-change.md
└─ 新产品 / 新功能 / 重大改版？           → standard-dev.md
```

## 裁剪原则

1. **按输入裁剪**：上游产出物已有 → 跳过对应阶段（如已有 PRD → 跳过需求分析）
2. **按规模裁剪**：小功能 → 合并设计与开发阶段；大系统 → 每阶段独立
3. **按风险裁剪**：高风险 → 增加审查节点；低风险 → 精简审查
4. **按时间裁剪**：紧急 → 最大化并行、最小化审查层级；从容 → 完整流程

## 符号说明

| 符号 | 含义 |
|------|------|
| `→` | 串行依赖（前一步完成后才能开始） |
| `∥` | 可并行执行 |
| `[ ]` | 并行组（组内节点可同时进行） |
| `⬡` | 里程碑检查点 |
