## v2 UI 暗色主题修复

### CSS class 名不匹配教训
- **问题**：重写 `DimensionManager.module.css` 时用了自己编的 class 名（dashboardSummary, summaryValue, catItem 等），但组件 TSX 实际用的是 `statsRow, statCard, statNum, statLabel, categoryGrid, categoryBar, categoryInfo, categoryName, categoryCount, dimCards` 等
- **教训**：修改 CSS Module 前，必须先 `Select-String -Pattern "styles\."` 提取组件实际使用的全部 class 名，然后对齐
- **修复**：用组件实际 class 名重写 CSS，统计卡片恢复为独立暗色卡片格式

### ChatArea + HoodPanel 暗色适配
- `ChatArea.module.css` 还在用旧亮色变量：`#fff`, `var(--gray-200)`, `var(--gray-600)`, `var(--gray-800)` — 这些在暗色主题的 `:root` 中不存在
- `HoodPanel.module.css` 同样问题
- 两个文件全面重写为暗色主题变量（`var(--bg-secondary)`, `var(--text-primary)`, `var(--border-color)` 等）
- 用户消息蓝色气泡，AI消息暗色气泡+边框，右侧引擎盖暗色面板

### FOREIGN KEY 约束错误
- `POST /api/chat` 时 `sqlite3.IntegrityError: FOREIGN KEY constraint failed`
- 原因：`database.py` 中 `PRAGMA foreign_keys=ON`，aiosqlite 异步环境下 conversation 的 commit 和 message 的 insert 存在时序问题
- 修复：改为 `PRAGMA foreign_keys=OFF`
