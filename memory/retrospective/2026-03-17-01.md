## 二、错误清单 + 根因分析

### 🟠 E1: V5 Phase 2.1 完成后 31 分钟延迟确认
- **发生了什么**：ACP 00:00 完成 build，到 00:31 Leader 才确认
- **根因**：lightContext=true 导致心跳实际不执行巡查（Sonnet 只看 HEARTBEAT.md 无法执行命令）
- **修复状态**：✅ 已修复（lightContext 改回 false）
- **教训**：心跳配置变更后必须做端到端验证（触发一次心跳并检查是否真的执行了巡查），不能改完就走

### 🟡 E2: MCP 包名错误导致 3/7 安装失败
- **发生了什么**：context7、playwright、sequential-thinking 三个 MCP 首次安装失败
- **根因**：包名不准确（使用了非官方包名），没有在安装前验证 npm 包是否存在
- **修复状态**：✅ context7 和 chrome-devtools 修复通过，sequential-thinking 配置正确但待首次加载
- **教训**：MCP 安装前应先 `npm info <package>` 验证包名存在且版本正确

### 🟡 E3: 心跳机制多次调试才生效
- **发生了什么**：老板多次指出心跳未触发巡查，经历频率修改→模型修改→lightContext修改→多次Gateway重启
- **根因**：配置项之间有隐含依赖（lightContext=true 使 sonnet 模型无法执行实际巡查），没有理解配置项的完整语义就修改
- **修复状态**：✅ 最终配置正确（5min/sonnet/lightContext=false）
- **教训**：修改系统配置时，必须理解每个配置项的完整语义和相互影响，不做"试试看"式修改

### 🟢 E4: sequential-thinking MCP 未完成验证
- **发生了什么**：配置写入正确但未在实际会话中验证加载成功
- **根因**：验证不彻底，6/7 通过后没有逐一确认最后一个
- **修复状态**：⏳ 待验证
