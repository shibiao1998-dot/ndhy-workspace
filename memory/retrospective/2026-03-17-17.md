## 二、错误清单 + 根因分析

### 🟢 E0: 本窗口无严重错误

这是自 3/16 以来**首个零严重错误的 8 小时窗口**。

### 🟡 E1: 心跳 session 无法看到主会话的子 Agent
- **发生了什么**：所有心跳巡查中 `subagents list` 始终返回 0，因为 ACP 任务由 Lark 主会话 spawn，心跳 session 是独立 session
- **影响**：心跳只能通过文件系统间接判断任务状态，无法直接管理子 Agent（send/steer/kill）
- **根因**：OpenClaw 的 session 隔离设计——每个 session 的 subagents 是独立的
- **缓解措施**：已通过文件系统巡查（检查目标目录文件变化）成功替代
- **建议**：这是架构限制，暂无修复方案。当前的文件系统巡查方式已证明有效

### 🟡 E2: PowerShell `$_` 变量在 exec 中持续被吞
- **发生了什么**：心跳巡查中多次使用 `Where-Object { $_.Name ... }` 的 PowerShell 命令失败
- **影响**：浪费 1-2 次 tool call 重试
- **根因**：exec 工具传递命令时 `$` 前缀被处理掉（已知问题，09:00 复盘已记录）
- **修复状态**：⏳ 未根治，通过使用简化命令或避免 `$_` 绕过
- **建议**：建立 heartbeat 巡查标准命令模板库，避免每次重写命令踩坑
