## 错误与教训

### 🔴 错误 1：ACP 连续修改未做中间 commit
- **事件**：V6 官网第一轮 ACP 完成 S3-S7 后，直接启动第二轮修复
- **后果**：第二轮导致质量下降，想回滚时 git stash 把两轮改动一起回滚，丧失第一轮成果的独立快照
- **根因**：Leader 在 ACP 轮次间缺少"commit 检查点"流程
- **已修正**：写入 MEMORY.md 宪法级规则——每轮 ACP 修改完成后必须先 git commit 再做下一轮
- **评价**：✅ 教训已正确捕获和落盘。这是一个经典的"没有存档点"错误。

### 🟡 错误 2：ACP 浏览器验证陷入 GSAP pin zone 死循环
- **事件**：ACP 在 GSAP ScrollTrigger pin zone 中尝试滚动验证，反复尝试无法穿透
- **后果**：浪费时间在无法自动化的验证上
- **根因**：GSAP pin 交互不适合 headless 浏览器自动化
- **已修正**：写入教训——ACP 浏览器自动化不测试 GSAP pin/scrub，改用 DOM 验证 + build 检查
- **评价**：✅ 合理。GSAP 高级交互确实需要人工走查。

### 🟢 亮点：git push 失败处理得当
- 凌晨 push 到 GitHub 因网络问题失败，本地 commit 安全，日志清楚记录"push pending"
- 没有反复重试浪费时间
