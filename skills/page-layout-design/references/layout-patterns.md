# 布局模式库

> 不重复发明布局——从模式库选择，根据内容类型和用户任务匹配最佳模式。

## 模式速查

| 模式名 | 适用内容 | 适用页面 | 视觉效果 |
|--------|---------|---------|---------|
| 英雄区 | 品牌展示、核心卖点 | Landing Page、产品首页 | 大面积、高冲击 |
| 双栏对比 | 功能对比、左文右图 | 产品介绍、定价页 | 对称、平衡 |
| 卡片网格 | 同类内容集合 | 列表页、Dashboard、图库 | 整齐、可扫描 |
| 数据仪表盘 | 统计指标+图表 | Dashboard、分析页 | 信息密集 |
| 时间线 | 有序事件、历程 | 活动记录、版本日志 | 线性、时序 |
| 瀑布流 | 不等高内容 | 图片库、社交信息流 | 动态、丰富 |
| 全宽展示 | 沉浸式内容 | 详情页、文章页 | 大气、聚焦 |
| 分栏表单 | 结构化输入 | 创建/编辑页 | 有组织、清晰 |
| 主从布局 | 列表+详情 | 邮箱、消息、设置 | 高效、联动 |

## 模式详解

### 1. 英雄区（Hero Section）

**结构**：

```
┌─────────────────────────────────────────────────┐
│                                                 │
│   大标题（H1, 36-48px）                          │
│   副标题/描述（16-18px）                          │
│   [主按钮]  [次按钮]                              │
│                                    [Hero图片]    │
│                                                 │
└─────────────────────────────────────────────────┘
```

**设计要点**：
- 标题简洁有力（≤12 字）
- 最多 2 个 CTA 按钮
- 背景可用纯色 / 渐变 / 图片
- 最小高度：60vh-80vh（桌面端）
- 移动端：图片移到上方或作为背景

**响应式变换**：Desktop 左文右图 → Mobile 图上文下（或隐藏图片）

---

### 2. 双栏对比（Two-Column Compare）

**结构**：

```
┌───────────────────┬───────────────────┐
│                   │                   │
│    左侧内容        │    右侧内容        │
│  （文字/表单）      │  （图片/代码）      │
│                   │                   │
└───────────────────┴───────────────────┘
```

**设计要点**：
- 6:6 或 5:7 比例（内容侧宽一点）
- 视觉上两栏高度平衡
- 垂直居中对齐

**响应式变换**：Desktop 左右 → Mobile 上下堆叠

---

### 3. 卡片网格（Card Grid）

**结构**：

```
┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
│  Card   │ │  Card   │ │  Card   │ │  Card   │
│  1      │ │  2      │ │  3      │ │  4      │
└─────────┘ └─────────┘ └─────────┘ └─────────┘
┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
│  Card   │ │  Card   │ │  Card   │ │  Card   │
│  5      │ │  6      │ │  7      │ │  8      │
└─────────┘ └─────────┘ └─────────┘ └─────────┘
```

**设计要点**：
- 等宽等高卡片，内容截断到固定行数
- Desktop 3-4 列 / Tablet 2 列 / Mobile 1 列
- 卡片间距：16-24px
- 每张卡片结构一致：图片 + 标题 + 描述 + 操作

**响应式变换**：列数递减（4→2→1）

---

### 4. 数据仪表盘（Dashboard）

**结构**：

```
┌──────────┬──────────┬──────────┬──────────┐
│  Stat 1  │  Stat 2  │  Stat 3  │  Stat 4  │
└──────────┴──────────┴──────────┴──────────┘
┌────────────────────────────┬───────────────┐
│                            │               │
│      主图表（8 列）         │  辅助信息      │
│                            │  （4 列）      │
└────────────────────────────┴───────────────┘
┌─────────────────────────────────────────────┐
│  数据表格 / 列表（12 列）                      │
└─────────────────────────────────────────────┘
```

**设计要点**：
- 顶部：关键指标卡片（4 个最佳，不超过 6 个）
- 中部：主图表占大面积（8 列），辅助信息（4 列）
- 底部：详细数据表格
- 每个区块可独立加载（loading skeleton）

**响应式变换**：指标卡片 4→2→1，图表全宽，表格转卡片列表

---

### 5. 主从布局（Master-Detail）

**结构**：

```
┌──────────────┬──────────────────────────────┐
│              │                              │
│   列表       │                              │
│  （Master）  │      详情（Detail）            │
│              │                              │
│  · Item 1   │                              │
│  · Item 2 ← │                              │
│  · Item 3   │                              │
│              │                              │
└──────────────┴──────────────────────────────┘
```

**设计要点**：
- 列表区固定宽度（240-320px），详情区弹性伸缩
- 列表区支持搜索和筛选
- 当前选中项高亮
- 详情区跟随列表选择动态更新

**响应式变换**：Desktop 左右分栏 → Mobile 列表和详情分页切换（点击进入/返回）

---

### 6. 分栏表单（Sectioned Form）

**结构**：

```
┌─────────────────────────────────────────────┐
│  表单标题 + 描述                               │
├─────────────────────────────────────────────┤
│  Section 1: 基本信息                          │
│  ┌──────────────┐ ┌──────────────┐          │
│  │  Field 1     │ │  Field 2     │          │
│  └──────────────┘ └──────────────┘          │
│  ┌──────────────┐ ┌──────────────┐          │
│  │  Field 3     │ │  Field 4     │          │
│  └──────────────┘ └──────────────┘          │
├─────────────────────────────────────────────┤
│  Section 2: 详细设置                          │
│  ┌─────────────────────────────────┐        │
│  │  Field 5（全宽）                  │        │
│  └─────────────────────────────────┘        │
├─────────────────────────────────────────────┤
│              [取消]  [保存]                    │
└─────────────────────────────────────────────┘
```

**设计要点**：
- 表单最大宽度 640px，居中
- 相关字段分组（Section），每组有标题
- Desktop 可双列排列短字段
- Mobile 全部单列
- 提交按钮始终可见（fixed bottom 或跟随内容）

## 模式选择决策树

```
页面目标是什么？
├── 展示品牌/产品 → 英雄区 + 卡片网格
├── 数据概览 → 数据仪表盘
├── 管理列表数据 → 卡片网格 或 数据表格
├── 查看详情 → 全宽展示 或 主从布局
├── 填写信息 → 分栏表单
├── 浏览信息流 → 时间线 或 瀑布流
└── 对比内容 → 双栏对比
```
