# 页面布局设计质量检查清单

> 逐项检查，全部 ✅ 才能交付。

## 信息架构检查

| # | 检查项 | 标准 |
|---|--------|------|
| I1 | 内容分组 | 所有内容按功能/主题分组，无游离元素 |
| I2 | 优先级排序 | L1/L2/L3 三级信息层级明确 |
| I3 | 导航清晰 | 用户从任何页面都能找到回主页/上一级的路 |
| I4 | 面包屑 | 层级 ≥ 3 的页面有面包屑导航 |

## 栅格与间距检查

| # | 检查项 | 标准 |
|---|--------|------|
| G1 | 栅格对齐 | 所有内容在栅格线上对齐，无随意偏移 |
| G2 | 列间距一致 | 同一页面的 gutter 统一 |
| G3 | 页面边距 | Desktop ≥ 32px / Tablet ≥ 24px / Mobile ≥ 16px |
| G4 | 最大宽度 | 内容区有 max-width 约束（长文本 ≤ 720px） |
| G5 | 间距档位 | 只使用 Design Token 定义的间距值，无非标值 |
| G6 | 留白层级 | 区块间 > 模块间 > 元素间，层次分明 |

## 布局模式检查

| # | 检查项 | 标准 |
|---|--------|------|
| L1 | 模式选择 | 布局模式与内容类型匹配（参考模式选择决策树） |
| L2 | 一屏一焦点 | 每个视口高度内只有一个主视觉焦点 |
| L3 | 视觉重心 | 核心操作/信息在视觉焦点位置 |
| L4 | 阅读模式 | 文字页面遵循 F 型，展示页面遵循 Z 型 |

## 响应式检查

| # | 检查项 | 标准 |
|---|--------|------|
| R1 | Mobile First | CSS 从小屏幕写起，用 `min-width` 向上扩展 |
| R2 | 320px 可用 | 在 320px 宽度下内容不溢出、可操作、可阅读 |
| R3 | 断点切换 | 768px / 1024px 处布局切换平滑 |
| R4 | 大屏约束 | 2560px 下内容不过度拉伸 |
| R5 | Sidebar 适配 | Desktop 展开 / Tablet 折叠 / Mobile 抽屉 |
| R6 | 表格适配 | Mobile 下表格转卡片列表或水平滚动 |
| R7 | 表单适配 | Mobile 下全部单列 |
| R8 | 图片适配 | 使用 `max-width: 100%`，不横向溢出 |

## 全状态检查

| # | 检查项 | 标准 |
|---|--------|------|
| S1 | Loading Skeleton | 有骨架屏，形状与正常内容布局一致 |
| S2 | Empty State | 有空状态设计，含说明文字和行动引导 |
| S3 | Error State | 有错误状态设计，含错误说明和操作出路 |
| S4 | First-time | 首次使用有引导（空状态引导 / 步骤向导 / 提示） |
| S5 | No Permission | 无权限有说明和操作出路 |
| S6 | Partial Load | 部分加载失败时，正常部分不受影响 |

## 可访问性检查

| # | 检查项 | 标准 |
|---|--------|------|
| A1 | 对比度 | 普通文本 ≥ 4.5:1，大文本 ≥ 3:1 |
| A2 | 焦点可见 | `:focus-visible` 焦点环清晰可见 |
| A3 | 焦点顺序 | Tab 顺序与视觉阅读顺序一致 |
| A4 | 语义标签 | 使用 `<nav>/<main>/<aside>/<header>/<footer>` |
| A5 | 标题层级 | h1-h6 不跳级，每页一个 h1 |
| A6 | 触摸目标 | 可点击区域 ≥ 44×44px（移动端） |
| A7 | 不依赖颜色 | 错误/成功不仅靠颜色（加图标/文字） |

## 反模式速查

| ❌ 反模式 | 后果 | ✅ 正确做法 |
|----------|------|-----------|
| 从桌面端开始设计再缩小 | 移动端体验差，信息丢失 | Mobile First |
| 留白大小随意 | 界面混乱，看不出分组 | 用 Token 定义的间距档位 |
| 只设计正常状态 | 用户遇到 loading/error 时困惑 | 全状态覆盖 |
| 焦点环被 CSS reset 移除 | 键盘用户无法操作 | 保留 `:focus-visible` |
| 用 `<div>` 模拟所有结构 | 屏幕阅读器无法识别 | 语义化 HTML |
| 内容无最大宽度 | 大屏幕上一行 200 字，不可读 | 设置 max-width |
| 信息密度不分场景 | 数据页太松、展示页太挤 | 按场景选择密度档位 |
