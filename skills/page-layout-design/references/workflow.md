# 7步标准流程详解

## Step 1：需求分析

**目标**：明确页面要解决什么问题、服务什么用户任务。

**必须确认的信息**：

| 维度 | 需要了解 | 示例 |
|------|---------|------|
| 页面目标 | 这个页面的核心任务是什么 | 展示数据概览 / 引导用户注册 / 管理列表数据 |
| 用户任务 | 用户来这个页面要完成什么 | 查看今日概览 → 找到异常 → 进入详情 |
| 内容类型 | 页面展示什么类型的内容 | 统计数字 / 数据表格 / 表单 / 富文本 / 图片 |
| 内容量级 | 内容多少、是否动态 | 固定 4 张卡片 / 不定数量列表 / 长表单 |
| 设备分布 | 用户主要用什么设备 | 80% 桌面 / 移动优先 / 平板教学场景 |
| 上下文页面 | 从哪来、到哪去 | 从列表页点击进入，完成后返回列表 |

**输出**：页面需求摘要（一段话 + 关键要素表）

---

## Step 2：信息架构

**目标**：对页面内容进行分组和优先级排序。

**方法**：

1. **内容清单**：列出页面需要展示的所有内容元素
2. **分组**：按功能/主题/用户任务将内容分组
3. **排序**：按重要性和使用频率排序（核心 → 辅助 → 补充）
4. **层级**：定义 1-3 级信息层级

**信息层级模型**：

| 层级 | 定义 | 视觉表现 | 示例 |
|------|------|---------|------|
| **L1 核心信息** | 页面主要目标对应的内容 | 最大字号、最高对比度、首屏可见 | Dashboard 的关键指标 |
| **L2 支撑信息** | 辅助理解核心信息的内容 | 中等字号、次要色彩 | 图表、趋势说明 |
| **L3 补充信息** | 可选查看的附加内容 | 小字号、低对比度、可折叠 | 详细日志、元数据 |

**输出**：信息架构图（树状结构 + 层级标注）

---

## Step 3：区块划分

**目标**：将页面划分为功能明确的区块。

**常见区块结构**：

```
┌─────────────────────────────────────────┐
│  Header / Navbar（全局导航）              │
├──────────┬──────────────────────────────┤
│          │  Page Header（页面标题+操作）   │
│ Sidebar  ├──────────────────────────────┤
│（可选）   │  Content Area（主内容区）      │
│          ├──────────────────────────────┤
│          │  Footer / Action Bar（底部操作）│
└──────────┴──────────────────────────────┘
```

**区块划分原则**：
- 每个区块有且只有一个职责
- 区块之间用明确的间距或分割线区分
- 固定区块（Header/Sidebar）和滚动区块（Content）分离
- 操作按钮（ActionBar）靠近操作对象

**输出**：区块划分线框图（ASCII 或 Markdown 描述）

---

## Step 4：栅格布局

**目标**：将区块在栅格系统中定位。

> 📖 详见 [grid-system.md](grid-system.md)

**执行要点**：
1. 选择栅格列数（Desktop 12列 / Tablet 8列 / Mobile 4列）
2. 为每个区块分配列数
3. 定义列间距（gutter）和页面边距（margin）
4. 确定内容最大宽度（max-width）

---

## Step 5：留白与视觉重心

**目标**：通过留白和视觉手段引导用户注意力。

> 📖 详见 [visual-hierarchy.md](visual-hierarchy.md)

**执行要点**：
1. 定义三级留白（区块间 / 模块间 / 元素间）
2. 选择阅读模式（F 型 / Z 型）
3. 设计视觉焦点（每屏一个主焦点）
4. 控制信息密度（紧凑型 / 舒适型 / 宽松型）

---

## Step 6：响应式适配

**目标**：确保布局在所有断点下可用。

> 📖 断点策略详见 [grid-system.md](grid-system.md)

**响应式适配策略**：

| 策略 | 说明 | 示例 |
|------|------|------|
| **列数折叠** | 减少栅格列数 | 3 列卡片 → 2 列 → 1 列 |
| **堆叠重排** | 水平布局变垂直 | 左右分栏 → 上下堆叠 |
| **隐藏/折叠** | 次要内容隐藏或折叠到菜单 | Sidebar → 汉堡菜单 |
| **尺寸缩放** | 保持布局，缩小元素 | 大标题 → 中标题 |
| **替换模式** | 用不同组件展示 | DataTable → 卡片列表 |

---

## Step 7：全状态 + 可访问性

**目标**：为非正常状态设计布局，确保可访问性。

> 📖 全状态方案见 [state-layouts.md](state-layouts.md)
> 📖 可访问性规范见 [accessibility.md](accessibility.md)

**执行要点**：
1. 设计 Loading Skeleton（与正常布局骨架一致）
2. 设计 Empty State（引导用户创建第一条内容）
3. 设计 Error State（错误说明 + 重试操作）
4. 确认焦点顺序与视觉顺序一致
5. 检查文字与背景的对比度

---

## 深度分级详细说明

### L1 快速布局
- **适用**：已有设计系统、常规 CRUD 页面
- **范围**：Step 3（区块划分） + Step 4（栅格布局） + Step 6（响应式）
- **输出**：布局描述 + 响应式断点方案

### L2 标准布局
- **适用**：新模块、新页面类型
- **范围**：完整 Step 1-7
- **输出**：信息架构 + 区块划分 + 栅格方案 + 全状态布局

### L3 深度布局
- **适用**：全新产品、复杂信息架构
- **范围**：Step 1-7 + 用户动线分析 + 多端极限测试
- **额外输出**：
  - 用户动线图（眼动路径模拟）
  - 极限测试报告（320px/2560px/超长内容/无内容）
  - 可访问性审查报告
