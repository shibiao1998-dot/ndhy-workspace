# 全状态布局方案

> 页面不只有"数据正常显示"一种状态——每种异常状态都需要被设计。

## 状态清单

| 状态 | 触发条件 | 用户感知 |
|------|---------|---------|
| **Loading** | 数据加载中 | "正在获取" |
| **Empty** | 无数据 | "这里什么都没有" |
| **Error** | 请求失败 | "出了点问题" |
| **Partial** | 部分加载成功 | "有一些内容，另一些加载失败" |
| **First-time** | 首次使用 | "我该怎么开始" |
| **No Permission** | 无权限 | "我没有权限看这个" |
| **Offline** | 网络断开 | "我断网了" |

---

## Loading 状态

### Skeleton（骨架屏）

**原则**：骨架屏的形状应与实际内容布局一致，让用户知道"东西要来了"。

```
卡片 Skeleton：
┌─────────────────────────┐
│  ██████████ (标题占位)    │
│  ████████████████ (描述)  │
│  ██████ (标签)            │
└─────────────────────────┘

表格 Skeleton：
┌──────┬──────┬──────┬──────┐
│ ████ │ ████ │ ████ │ ████ │  ← 表头
├──────┼──────┼──────┼──────┤
│ ████ │ ████ │ ████ │ ████ │  ← 行占位
│ ████ │ ████ │ ████ │ ████ │
│ ████ │ ████ │ ████ │ ████ │
└──────┴──────┴──────┴──────┘
```

**设计要点**：
- 使用 `--color-bg-tertiary` 作为骨架色
- 添加 shimmer 动画（从左到右的光泽扫过）
- 骨架行数 = 预期内容行数（通常 3-5 行）
- 不要用 spinner 替代 skeleton（spinner 不传达布局信息）

### 何时用 Skeleton vs Spinner

| 场景 | 推荐 | 原因 |
|------|------|------|
| 页面首次加载 | Skeleton | 预示内容结构 |
| 区域内容刷新 | Skeleton | 保持布局稳定 |
| 按钮操作等待 | Button Spinner | 操作级反馈 |
| 全屏跳转 | 进度条（Top Bar） | 页面级反馈 |
| 后台提交 | Toast + Spinner | 不阻塞操作 |

## Empty 状态

### 结构模板

```
┌─────────────────────────────┐
│                             │
│         [  插图  ]           │
│                             │
│    还没有任何数据              │
│    描述为什么为空+引导行动     │
│                             │
│    [ 创建第一个 XXX ]         │
│                             │
└─────────────────────────────┘
```

**设计要点**：
- **居中展示**：在内容区垂直水平居中
- **插图/图标**：用轻量插图或大号图标，传达"空但不是错"
- **说明文字**：告诉用户为什么空 + 怎么不空
- **行动按钮**：引导用户创建第一条内容
- **不留完全空白**：空白页面会让用户以为页面坏了

### Empty 状态分类

| 类型 | 说明 | 引导语示例 |
|------|------|-----------|
| 首次使用 | 从未创建过内容 | "创建你的第一个项目" |
| 筛选无结果 | 筛选条件过窄 | "没有符合条件的结果，试试调整筛选" |
| 搜索无结果 | 搜索无匹配 | "没有找到相关内容，换个关键词试试" |
| 已清空 | 用户删除了所有内容 | "所有项目已归档/删除" |

## Error 状态

### 结构模板

```
┌─────────────────────────────┐
│                             │
│         [  错误图标  ]       │
│                             │
│    加载失败                   │
│    简要描述发生了什么          │
│                             │
│    [ 重试 ]  [ 返回首页 ]     │
│                             │
└─────────────────────────────┘
```

**设计要点**：
- 使用 `--color-error` 强调错误图标/标题
- 描述用用户能理解的语言（不显示错误码/堆栈）
- **必须提供操作出路**：重试按钮 / 返回按钮 / 联系支持
- 区分全页错误和局部错误

### 错误级别

| 级别 | 布局 | 示例 |
|------|------|------|
| **全页错误** | 替换整个内容区 | 404 / 500 / 未授权 |
| **区块错误** | 替换单个区块内容 | 图表加载失败（其他正常） |
| **行内错误** | 表单字段下方提示 | 输入验证失败 |
| **全局错误** | Toast 通知 | 网络请求超时 |

## First-time 状态（首次引导）

### 引导模式

| 模式 | 适用 | 说明 |
|------|------|------|
| **空状态引导** | 功能简单 | Empty 状态中内嵌操作引导 |
| **步骤引导** | 需要配置 | 分步向导（Step 1/2/3） |
| **提示气泡** | 界面直觉差 | 关键按钮/区域添加提示 |
| **示例数据** | 理解成本高 | 预填示例数据，用户可清除 |

```
┌─────────────────────────────┐
│                             │
│         [  锁图标  ]         │
│                             │
│    你没有权限查看此内容        │
│    请联系管理员获取权限        │
│                             │
│    [ 返回 ]  [ 申请权限 ]     │
│                             │
└─────────────────────────────┘
```
