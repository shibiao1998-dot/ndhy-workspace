# 栅格系统与断点策略

> 栅格是布局的坐标系——让元素对齐、有节奏、可预测。

## 栅格基础参数

| 参数 | Desktop (≥1024px) | Tablet (≥768px) | Mobile (<768px) |
|------|-------------------|-----------------|-----------------|
| 列数 | 12 列 | 8 列 | 4 列 |
| 列间距（Gutter） | 24px | 16px | 16px |
| 页面边距（Margin） | 32px-64px | 24px | 16px |
| 最大内容宽度 | 1280px | 100% | 100% |

## 断点策略

### Mobile First 原则

```css
/* 基础样式 = Mobile（最小屏幕） */
.container { /* 4列布局 */ }

/* 向上扩展 */
@media (min-width: 640px)  { /* sm: 大屏手机横屏 */ }
@media (min-width: 768px)  { /* md: 平板竖屏 */ }
@media (min-width: 1024px) { /* lg: 平板横屏/小笔记本 */ }
@media (min-width: 1280px) { /* xl: 桌面 */ }
@media (min-width: 1536px) { /* 2xl: 大桌面 */ }
```

### 标准断点表

| 断点名 | 最小宽度 | 设备类型 | 栅格 | 典型场景 |
|--------|---------|---------|------|---------|
| `base` | 0px | 小屏手机 | 4 列 | iPhone SE (375px) |
| `sm` | 640px | 大屏手机横屏 | 4 列 | 手机横屏 |
| `md` | 768px | 平板竖屏 | 8 列 | iPad 竖屏 (768px) |
| `lg` | 1024px | 平板横屏/笔记本 | 12 列 | iPad 横屏 (1024px) |
| `xl` | 1280px | 桌面 | 12 列 | 普通显示器 |
| `2xl` | 1536px | 大桌面 | 12 列 | 大显示器/外接屏 |

### 极限测试宽度

| 宽度 | 用途 | 必须验证 |
|------|------|---------|
| **320px** | 最小可用宽度 | 内容不溢出、可操作、可阅读 |
| **375px** | iPhone SE/Mini | 主流最小手机 |
| **768px** | 断点临界值 | 布局切换正确 |
| **1024px** | 断点临界值 | 布局切换正确 |
| **1440px** | 设计稿常用宽度 | 理想呈现 |
| **2560px** | 大屏/4K | 内容不过度拉伸、有最大宽度约束 |

## 栅格使用规范

### 列分配常见模式

#### Desktop（12 列）

```
全宽内容：          [============12============]
左侧导航+主内容：    [===3===][=========9=========]
双栏对比：           [======6======][======6======]
三等分：             [====4====][====4====][====4====]
内容+侧边栏：       [========8========][====4====]
居中窄内容：         [--][========8========][--]
```

#### Tablet（8 列）

```
全宽内容：          [========8========]
双栏：               [====4====][====4====]
主内容+侧边：       [=====5=====][===3===]
居中窄内容：         [--][======6======][--]
```

#### Mobile（4 列）

```
全宽内容：          [====4====]
（移动端几乎全部是单列全宽布局）
```

### 列间距的用法

```css
/* Tailwind CSS 实现 */
.grid-container {
  display: grid;
  gap: var(--spacing-6);          /* Desktop: 24px gutter */
  padding-inline: var(--spacing-8); /* Desktop: 32px margin */
  max-width: 1280px;
  margin-inline: auto;
}

@media (max-width: 1023px) {
  .grid-container {
    gap: var(--spacing-4);          /* Tablet: 16px gutter */
    padding-inline: var(--spacing-6); /* Tablet: 24px margin */
  }
}

@media (max-width: 767px) {
  .grid-container {
    gap: var(--spacing-4);          /* Mobile: 16px gutter */
    padding-inline: var(--spacing-4); /* Mobile: 16px margin */
  }
}
```

## 响应式布局变换规则

### 常见变换模式

| 模式 | Desktop | Tablet | Mobile |
|------|---------|--------|--------|
| **卡片网格** | 4 列 | 2 列 | 1 列 |
| **左右分栏** | 左+右 | 左+右（窄化） | 上+下堆叠 |
| **侧边导航** | 展开侧栏 | 折叠侧栏(图标) | 汉堡菜单 |
| **数据表格** | 完整表格 | 水平滚动 | 卡片列表 |
| **表单布局** | 双列表单 | 双列 | 单列 |
| **统计卡片** | 一行 4 个 | 一行 2 个 | 一行 1 个 |
| **英雄区** | 左文右图 | 左文右图 | 图上文下 |

### Sidebar 响应式方案

```
Desktop (≥1024px)：固定展开侧栏（width: 240px）
Tablet (≥768px)：  折叠侧栏仅图标（width: 64px），hover 展开
Mobile (<768px)：  隐藏，顶部汉堡菜单 → 侧滑抽屉
```

### DataTable 响应式方案

```
Desktop：完整表格，所有列可见
Tablet： 隐藏次要列，显示核心列，水平可滚动
Mobile： 转为卡片列表，每条数据一张卡片
```

## 内容最大宽度

| 内容类型 | 推荐最大宽度 | 原因 |
|---------|-------------|------|
| 长文本（文章/文档） | 680-720px | 每行 60-80 字符最佳可读性 |
| Dashboard/数据页 | 1280px | 需要利用宽度展示更多信息 |
| 表单 | 560-640px | 表单不宜过宽 |
| 营销/Landing Page | 1200-1440px | 视觉冲击力 |
| 设置页 | 720-800px | 操作界面不宜过宽 |
