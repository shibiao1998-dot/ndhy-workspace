# 分析方法库

14 种分析方法，按分析层次分为 4 类：描述性（4）、诊断性（4）、预测性（3）、建议性（3）。

---

## 一、描述性分析（发生了什么？）

### 1. 趋势分析

**用途**：看一个指标随时间的变化趋势。

**关注三要素**：
- **方向**：上升还是下降？
- **速度**：变化快还是慢？是否在加速/减速？
- **拐点**：什么时候趋势发生了变化？那个时间点发生了什么？

**教育场景注意**：
- 对齐教学日历——学期初/中/末、考试周、假期
- 工作日 vs 周末差异巨大——教师使用集中在周一到周五
- 寒暑假活跃骤降是正常现象

**操作步骤**：
1. 确定指标和时间粒度（日/周/月）
2. 绘制时间序列图
3. 标注关键事件节点（版本发布、运营活动、假期等）
4. 识别趋势方向和拐点
5. 与预期/目标线对比

### 2. 构成分析

**用途**：把一个总量拆成各部分的占比，了解结构。

**常见拆法**：
- 活跃用户中，小学/初中/高中教师各占多少？
- 使用时间中，备课/教学/查看数据各占多少？
- 新增用户中，自然增长/邀请/活动各占多少？

**操作步骤**：
1. 确定总量指标和拆分维度
2. 计算各部分的绝对值和占比
3. 与历史占比对比——结构是否在变化？
4. 识别优势部分和薄弱环节

### 3. 对比分析

**用途**：通过比较发现差异和变化。

**四种对比维度**：

| 对比类型 | 示例 | 发现 |
|---------|------|------|
| 时间对比 | 和上周/上月/去年同期比 | 发现变化趋势 |
| 群体对比 | 活跃用户 vs 非活跃用户 | 发现行为差异 |
| 区域对比 | A 区域 vs B 区域 | 发现最佳实践 |
| 版本对比 | 新版本 vs 旧版本 | 评估改版效果 |

**关键原则**：确保"同类可比"——不能拿学期初和学期末比，不能拿大校和小校直接比总量。

### 4. 分布分析

**用途**：不只看平均值，看整体分布。

**分布形态的解读**：
- **正态分布偏右**：大部分人都用得不错
- **两极分化**：产品可能只满足了一部分人的需求
- **长尾分布**：少数重度用户贡献了大部分价值
- **集中在左端**：大部分人用得很少，需要关注激活

**操作步骤**：
1. 绘制直方图或箱线图
2. 计算关键分位数：中位数、P25、P75、P90
3. 与平均值对比——差距大说明分布不均
4. 按群体细分分布——不同群体的分布是否不同？

## 二、诊断性分析（为什么发生？）

### 5. 漏斗分析

**用途**：把用户行为路径拆成步骤，找到流失最严重的环节。

**标准漏斗示例**：
```
访问注册页（100%）
  -> 开始填写（70%）    # 30% 的人连表单都不填——注册页有问题？
  -> 完成注册（50%）    # 20% 的人填到一半放弃——流程太复杂？
  -> 首次核心操作（20%）# 30% 的人注册了不知道干啥——引导有问题？
```

**操作步骤**：
1. 定义漏斗步骤（关键路径的每一步）
2. 计算每步的绝对人数和转化率
3. 找到"流失最大的环节"——这就是最大的优化机会
4. 按维度拆分（渠道/设备/区域）看哪里漏得最多
5. 优先优化流失最大的环节——投入产出比最高

### 6. 同期群分析（Cohort Analysis）

**用途**：把同一时期进来的用户归为一组，追踪后续表现。

**核心用途**：
- 判断留存是否在改善——后来的用户比早期用户留存更好吗？
- 找到流失的关键时间点——大部分用户是在第几周流失的？
- 评估产品改进效果——某次改版后进来的用户，留存有没有好于之前？

**同期群表示例**：

| 注册月份 | 第 1 周 | 第 2 周 | 第 3 周 | 第 4 周 | 第 8 周 |
|---------|--------|--------|--------|--------|--------|
| 2025 年 9 月 | 100% | 58% | 42% | 35% | 22% |
| 2025 年 10 月 | 100% | 62% | 48% | 40% | 28% |
| 2025 年 11 月 | 100% | 67% | 53% | 46% | -- |

**解读**：每个月的用户第 1 周到第 2 周留存都在改善 -> 说明激活优化有效果。

**操作步骤**：
1. 确定分组维度（注册时间/首次使用时间）
2. 确定追踪指标（留存率/活跃度/付费率）
3. 确定时间粒度（日/周/月）
4. 绘制同期群表
5. 横向对比（同一群不同时间）——看衰减速度
6. 纵向对比（不同群同一时间点）——看是否改善

### 7. 归因分析

**用途**：某个指标变了，到底是什么导致的？

**三种拆解方法**：

| 拆解维度 | 方法 | 示例 |
|---------|------|------|
| 按维度拆 | 是哪个区域/学段/功能模块的变化导致的？ | 发现 A 区域下降 25%，其他持平 |
| 按时间拆 | 从什么时间点开始变的？那时发生了什么？ | 从某次版本发布后开始下降 |
| 按行为拆 | 变化的用户和不变的用户，行为有什么差异？ | 流失用户都是近期新增的低活跃用户 |

**归因案例**：
```
现象："活跃用户数下降 10%"
  -> 按区域拆：A 区域下降 25%，其他区域持平
  -> 深入 A 区域：骨干教师 XXX 被调走了
  -> 结论：单点人员依赖问题，不是产品问题
  -> 建议：减少对单一骨干的依赖，建立教师社群
```

### 8. 交叉分析

**用途**：两个维度交叉，发现单一维度看不到的洞察。

**常见交叉维度**：
- 学科 x 使用频次 -> 发现数学老师用得最多，语文老师最少
- 学校规模 x 渗透率 -> 发现小规模学校渗透率反而更高
- 教师年龄 x 功能偏好 -> 发现年轻教师更爱用 AI 功能
- 注册渠道 x 留存率 -> 发现邀请来的用户留存远高于活动拉新

**操作步骤**：
1. 选择两个有业务意义的维度
2. 构建交叉表（矩阵）
3. 寻找"反直觉"的数据组合
4. 交叉后问"为什么"——这才是洞察

## 三、预测性分析（将要发生什么？）

### 9. 趋势外推

**用途**：基于历史趋势预测未来。

**适用条件**：
- 短期预测（1-3 个月）
- 趋势稳定的指标
- 无重大外部变化（如政策变动）

**不适用**：
- 有重大变化发生时（新学期政策变化、产品大改版）
- 长期预测（不确定性太大）

**教育场景注意**：必须考虑学期的周期性，不能简单线性外推跨越假期。

### 10. 留存曲线拟合

**用途**：用已有的留存数据，预测长期留存率。

**关键知识**：
- 教育产品的留存曲线通常在 8-12 周后趋于稳定
- 稳定留存率 = 真正的核心用户占比
- 如果稳定留存率低于 15%，产品需要重新审视核心价值

**操作方法**：
1. 收集至少 8 周的留存数据
2. 绘制留存曲线
3. 用对数或幂函数拟合
4. 外推至稳定点，预测长期留存率

### 11. 用户生命周期价值（LTV）估算

**用途**：估算每个用户在整个生命周期内贡献的价值。

**基本公式**：
```
LTV = 平均每用户收入 (ARPU) x 平均生命周期
```

**教育产品特点**：
- 生命周期通常以"学期"为单位
- B 端产品以"合同周期"为单位
- LTV / CAC > 3 是健康的标志
- LTV / CAC < 1 说明获客模式不可持续

## 四、建议性分析（应该做什么？）

### 12. 机会量化

**用途**：把改进机会量化为具体的业务影响，帮助决策优先级。

**公式示例**：
```
月新增注册 x (目标激活率 - 当前激活率) = 月增活跃用户数
1000 x (30% - 20%) = 100 个/月
```

**操作步骤**：
1. 识别可改进环节（通常来自漏斗分析）
2. 估算合理的改进目标（参考行业基准或历史最优）
3. 计算改进后的业务增量
4. 按增量大小排列优先级

### 13. 敏感性分析

**用途**：哪个环节改善 1%，对最终结果影响最大？

**示例**：
```
问题：获取量 +10% vs 激活率 +10% vs 留存率 +10%，哪个对年末活跃用户数影响最大？
通常答案：留存率的敏感性最高——因为它是复利
```

**操作步骤**：
1. 建立指标间的数学关系模型
2. 逐一调整每个输入变量（+1%、+5%、+10%）
3. 观察输出变量的变化幅度
4. 找到"杠杆点"——小投入大产出的环节

### 14. 优先级矩阵

**用途**：将改进机会按"潜力 x 难度"排序。

**四象限**：

| | 潜力大 | 潜力小 |
|---|--------|--------|
| **难度低** | 优先做（Quick Win） | 可以做（填充型） |
| **难度高** | 规划做（战略型） | 不做（放弃） |

**操作步骤**：
1. 列出所有改进机会
2. 用机会量化评估每个机会的潜力
3. 评估每个机会的实施难度（资源/时间/技术）
4. 放入矩阵，确定优先级

## 方法选择决策树

```
收到分析需求
  |- 想知道"现状如何"
  |   -> 趋势分析 + 构成分析 + 分布分析
  |
  |- 想知道"为什么变了"
  |   -> 归因分析 + 交叉分析
  |   |- 涉及用户路径 -> + 漏斗分析
  |   |- 涉及时间维度 -> + 同期群分析
  |
  |- 想知道"未来会怎样"
  |   -> 趋势外推
  |   |- 涉及留存 -> + 留存曲线拟合
  |   |- 涉及商业价值 -> + LTV 估算
  |
  +- 想知道"应该做什么"
      -> 机会量化 + 敏感性分析 + 优先级矩阵
```
