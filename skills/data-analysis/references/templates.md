# 分析输出模板

---

## 模板 1：分析报告

```markdown
# [分析主题]

## 一、核心结论（30 秒版本）

> 1-3 句话总结最重要的发现和建议。管理层只看这一段也能做决策。

- **核心发现**：[一句话结论]
- **关键数字**：[支撑结论的 1-2 个数字]
- **建议行动**：[最重要的 1 个建议]

## 二、分析背景

- **分析目的**：为什么做这个分析？要回答什么问题？
- **决策场景**：分析结论将用于什么决策？
- **数据范围**：[数据来源]，时间范围：[起止日期]
- **关键定义**：[涉及的指标定义说明]

## 三、关键发现

### 发现 1：[结论标题]

[一句话结论]

[数据/图表支撑]

**解读**：[这个发现意味着什么]

### 发现 2：[结论标题]

[同上结构]

### 发现 3：[结论标题]

[同上结构]

> 每个发现遵循"结论先行"原则——先说结论，再给证据。

## 四、深度分析（如适用）

- 详细的分析过程和方法
- 支撑性的数据和图表
- 排除的其他假设
- 事实 vs 推测的标注

## 五、行动建议

| 优先级 | 建议 | 预期影响 | 所需资源 |
|--------|------|---------|---------|
| P0 | [具体建议] | [量化的预期影响] | [时间/人力] |
| P1 | [具体建议] | [量化的预期影响] | [时间/人力] |
| P2 | [具体建议] | [量化的预期影响] | [时间/人力] |

## 六、局限与后续

- **局限性**：[坦诚说明本次分析的局限]
- **不确定性**：[标注哪些结论确定度较低]
- **后续计划**：
  - [ ] [需要进一步研究的问题]
  - [ ] [效果追踪计划和时间点]
  - [ ] [数据补充采集计划]
```

## 模板 2：指标定义文档

```markdown
# 指标定义文档

## 基本信息

| 字段 | 内容 |
|------|------|
| 指标名称 | [中文名] |
| 英文名/缩写 | [英文名] |
| 指标分类 | 获取 / 激活 / 留存 / 活跃 / 营收 |
| 负责人 | [谁负责这个指标] |
| 创建日期 | [YYYY-MM-DD] |
| 最后更新 | [YYYY-MM-DD] |

## 定义与口径

| 字段 | 内容 |
|------|------|
| 定义描述 | [用一句话说清这个指标衡量什么] |
| 计算公式 | [明确的计算公式] |
| 包含 | [哪些行为/用户被纳入计算] |
| 不包含 | [哪些行为/用户被排除] |
| 特殊处理 | [去重规则、边界情况处理] |

## 数据与展示

| 字段 | 内容 |
|------|------|
| 数据来源 | [具体的数据表/系统] |
| 刷新频率 | [每日/每周/实时] |
| 呈现方式 | [趋势图/柱状图 + 对比方式] |
| 预警阈值 | [偏离基线多少触发预警] |

## 关联指标

- 上游指标：[影响本指标的指标]
- 下游指标：[本指标影响的指标]
- 对照指标：[辅助解读本指标的指标]

## 使用注意

- [指标解读时需要注意的事项]
- [常见误读和正确解读]
- [周期性影响说明]
```

## 模板 3：数据异常报告

```markdown
# 数据异常报告

## 异常概述

| 字段 | 内容 |
|------|------|
| 异常指标 | [什么指标出了异常] |
| 异常程度 | [偏离正常值多少？绝对值和百分比] |
| 发现时间 | [YYYY-MM-DD HH:MM] |
| 影响范围 | [影响了多少用户/学校？影响了哪些业务指标？] |

## 时间线

| 时间 | 事件 |
|------|------|
| [时间点] | 异常开始 |
| [时间点] | [相关事件：版本发布/运营活动/其他] |
| [时间点] | 发现异常 |
| [时间点] | [后续动作] |

- 异常类型：突变 / 渐变
- 是否还在持续：是 / 否

## 初步归因

### 排查的可能原因

| # | 可能原因 | 排查结果 | 证据 |
|---|---------|---------|------|
| 1 | [原因 A] | 排除 / 确认 / 待验证 | [数据证据] |
| 2 | [原因 B] | 排除 / 确认 / 待验证 | [数据证据] |

### 最可能的原因

[基于排查结果的判断，标注确定性级别]

## 当前状态

- 异常是否已恢复：是 / 否
- 已采取的措施：[列出已执行的动作]

## 建议行动

| 类型 | 行动 | 负责人 | 时间 |
|------|------|--------|------|
| 短期（止损） | [具体措施] | [谁] | [什么时候] |
| 长期（根因） | [具体措施] | [谁] | [什么时候] |

## 后续追踪

- [ ] [追踪项 1 + 预期时间]
- [ ] [追踪项 2 + 预期时间]
```

## 模板 4：实验设计方案

```markdown
# 实验设计方案：[实验名称]

## 背景与假设

- **业务背景**：[为什么要做这个实验]
- **假设**：我们认为做了 [X 改变]，[Y 指标] 会 [提升/降低] [Z%]，因为 [原因]

## 实验设计

| 字段 | 内容 |
|------|------|
| 实验类型 | A/B 测试 / 灰度发布 / 前后对比 |
| 分组方式 | 随机个人 / 按学校 / 按区域 |
| 对照组 | [当前版本/方案描述] |
| 实验组 | [新版本/方案描述] |
| 预计样本量 | [每组需要的样本数] |
| 实验时长 | [开始日期] 至 [结束日期]（共 N 天/周） |

## 指标

| 类型 | 指标 | 当前基线 | 目标 |
|------|------|---------|------|
| 主指标 | [1 个] | [当前值] | [期望值] |
| 辅助指标 | [2-3 个] | [当前值] | -- |
| 护栏指标 | [1-2 个] | [当前值] | 不恶化 |

## 风险与预案

- [风险 1]：[应对措施]
- [风险 2]：[应对措施]
- 紧急停止条件：[什么情况下停止实验]

## 结果判定标准

- 显著性标准：p < 0.05
- 最小可检测效应：[MDE]
- 判定流程：[谁在什么时候看结果并做决策]
```
