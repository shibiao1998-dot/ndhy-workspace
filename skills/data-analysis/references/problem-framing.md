# 数据思维与问题框架

## 数据分析的四个层次

数据分析不是 Excel / SQL / Python / 做图表，而是**结构化地思考问题，然后用数据验证思考**。

| 层次 | 类型 | 核心问题 | 示例 | 价值 |
|------|------|---------|------|------|
| **Level 1** | 描述型 | 发生了什么？ | "上月活跃用户数是多少？" | 报表级——大多数人停在这层 |
| **Level 2** | 诊断型 | 为什么发生？ | "活跃用户为什么下降了 15%？" | 开始算分析——找原因、找归因 |
| **Level 3** | 预测型 | 将要发生什么？ | "按目前趋势，下学期续费率会是多少？" | 基于历史规律做预判 |
| **Level 4** | 建议型 | 应该做什么？ | "要把续费率从 60% 提升到 80%，最应该做的 3 件事？" | 从数据到行动——终极价值 |

**关键认知**：能提出正确的问题，比能操作工具重要 100 倍。产品经理不需要亲自跑 SQL，但需要具备 Level 2-4 的思维能力。

## 业务问题到数据问题的转化

### 转化路径

```
业务问题 -> 分析框架 -> 数据需求 -> 分析方法 -> 结论 -> 行动
```

### 转化示例

**业务问题**："这个新功能上线后效果怎么样？"

这个问题太模糊，需要拆解：

| 拆解维度 | 模糊问题 | 数据问题 |
|---------|---------|---------|
| "效果"指什么？ | 效果好不好 | 新功能使用率是多少？（覆盖面） |
| "怎么样"对比什么？ | 效果怎么样 | 使用 vs 未使用的用户，留存率差异多大？（效果） |
| 因果还是相关？ | 功能有效吗 | 差异是功能带来的，还是活跃用户更爱尝新？（归因） |
| 机会在哪？ | 怎么优化 | 哪些用户群体使用率最高/最低？为什么？（机会） |

### 常犯的三大错误

| 错误 | 表现 | 修正 |
|------|------|------|
| **带着结论找数据** | "帮我找数据证明这个功能有效" | 这不是分析，是自欺。应该是"帮我验证这个功能是否有效" |
| **问题太大** | "分析一下我们的用户" | 分析什么？从哪个角度？要回答什么？先具体化 |
| **不明确行动导向** | "给我看看数据" | 看了之后要做什么决策？决策场景决定分析方向 |

## SMART 数据问题框架

每个分析问题在开始前用 SMART 校验：

| 维度 | 含义 | 反例 | 正例 |
|------|------|------|------|
| **S** (Specific) | 具体 | "用户活跃情况" | "教师周均使用产品的天数" |
| **M** (Measurable) | 可量化 | "用户满意不满意" | "NPS 评分分布" |
| **A** (Actionable) | 可行动 | "看看行业趋势" | "我们的留存率 vs 行业基准，差距在哪" |
| **R** (Relevant) | 与目标相关 | "分析一下页面加载速度" | "首屏加载 >3s 的用户流失率是否显著更高" |
| **T** (Time-bound) | 有时间框架 | "看看留存率" | "2025 秋季学期新注册用户的 30 日留存率" |

### SMART 校验流程

```
收到分析需求
  |- S：问题是否具体？-> 不具体 -> 追问"具体要看什么指标？"
  |- M：是否可量化？-> 不可量化 -> 追问"怎么衡量？有没有数据？"
  |- A：结论能指导行动吗？-> 不能 -> 追问"分析完你要做什么决策？"
  |- R：和当前最重要目标相关吗？-> 不相关 -> 建议调整优先级
  |- T：有时间框架吗？-> 没有 -> 追问"最近一学期还是上线以来？"
  |
  +-> 全部通过 -> 进入 Step 2
```

## 问题明确后的快速框架选择

| 问题类型 | 推荐框架 | 典型分析方法 |
|---------|---------|-------------|
| "X 指标现在怎么样？" | 现状描述 | 趋势分析 + 构成分析 + 分布分析 |
| "为什么 X 指标变了？" | 原因诊断 | 归因分析 + 交叉分析 + 漏斗分析 |
| "X 功能/活动效果如何？" | 效果评估 | 对比分析 + 同期群 + A/B 测试 |
| "下一步应该做什么？" | 决策建议 | 机会量化 + 敏感性分析 + 优先级矩阵 |
| "未来趋势如何？" | 趋势预测 | 趋势外推 + 留存拟合 + LTV 估算 |
| "指标体系怎么设计？" | 体系构建 | 北极星指标 + 三维体系 + 指标定义规范 |
