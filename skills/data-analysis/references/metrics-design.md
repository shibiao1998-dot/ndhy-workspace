# 指标体系设计

## 指标设计四原则

| # | 原则 | 核心要求 | 检验方法 |
|---|------|---------|---------|
| 1 | **为决策服务** | 每个指标回答："如果这个数字变了，我会做什么不同的决定？" | 回答不出来 -> 不需要这个指标 |
| 2 | **少即是多** | 核心指标控制在 5-7 个，细分指标按需查看 | 指标太多谁也看不过来 |
| 3 | **成体系不零散** | 指标之间有逻辑关系，能互相解释和校验 | 单独一个指标会误导 |
| 4 | **区分结果与过程** | 结果指标 = 最终目标；过程指标 = 关键动作 | 只看结果太滞后，只看过程会迷失 |

## 教育产品指标体系框架

### 北极星指标

**定义**：1 个，最能代表产品核心价值的指标。

**选择标准**：
- 能反映用户是否获得了产品的核心价值
- 所有团队都能理解且认同
- 能通过产品和运营手段影响
- 具有先导性（变化能预示业务健康度）

**教育产品典型北极星**：
- 周活跃教师数（WAT）
- 活跃学校数 x 校均渗透率
- 周完成核心教学任务的教师数

### 三维指标体系

```
                    北极星指标
                       |
         +-------------+-------------+
         |             |             |
      用户维度       使用维度       价值维度
         |             |             |
    - 新增用户数    - 日/周活跃率   - 续费率
    - 激活率       - 人均使用频次   - 扩展率
    - 留存率       - 核心功能使用率 - NPS
    - 流失率       - 使用深度      - 客诉率
    - 用户构成                      - ROI
         |             |             |
         +-------------+-------------+
                       |
              下钻/细分维度
         按学校/区域/学段/角色/功能模块拆解
```

### 结果指标 vs 过程指标

| 类型 | 定义 | 示例 | 特点 |
|------|------|------|------|
| **结果指标** | 最终要达到的目标 | 学期末活跃学校数、续费率 | 滞后——等看到时来不及调整 |
| **过程指标** | 通向目标的关键动作 | 本周完成培训的学校数、周新增激活教师数 | 先导——能及时调整方向 |

**最佳实践**：结果指标定方向，过程指标驱执行，两者搭配监控。

## 指标定义规范

每个指标必须有明确的定义文档，模板如下：

```
指标名称：周活跃教师数（WAT - Weekly Active Teachers）
定义：在统计周内（周一至周日）至少完成 1 次核心操作的教师账号数
核心操作定义：创建/编辑教案、布置作业、查看学情报告、使用 AI 工具
不包括：仅登录未操作、仅浏览页面未执行操作
统计口径：去重，同一教师一周内多次操作只计 1
数据来源：用户行为日志
刷新频率：每日更新（次日上午）
呈现方式：趋势图 + 周环比 + 同期对比
负责人：XX
```

**为什么要这么严格？**
- 指标定义模糊 -> 不同人看到的数据不一样 -> 开会吵架 -> 无法决策
- "活跃用户"如果没有统一定义，A 说有 1 万，B 说有 3 千，都没错——因为标准不同

## 5 大指标陷阱详解

### 陷阱 1：平均值陷阱

**表现**："平均使用时长 30 分钟"——可能是一半人用了 1 小时，另一半打开就走了。

**解法**：
- 看分布，不只看平均——用中位数、P25/P75 分位数
- 绘制直方图查看实际分布形状
- 两极分化时报告应呈现完整分布

**示例**：
```
平均值：30 分钟（看起来不错）
中位数：8 分钟（实际大多数人用得很少）
P75：55 分钟（少部分重度用户拉高了平均值）
-> 结论完全不同：产品只满足了少数重度用户
```

### 陷阱 2：比率陷阱

**表现**："激活率从 40% 提升到 60%"——听起来很好，但分母从 1000 缩小到 100 了。

**解法**：
- 比率和绝对值一起看
- 关注分母变化——分母缩小导致比率"虚假改善"
- 报告中同时展示：比率 + 分子绝对值 + 分母绝对值

### 陷阱 3：相关性 ≠ 因果性

**表现**："使用 AI 功能的教师留存率更高"——不代表 AI 功能导致了留存。可能是本来就活跃的教师更爱尝试新功能。

**解法**：
- 控制变量对比（控制用户活跃度后再比较）
- A/B 测试（随机分组消除选择偏差）
- 在报告中明确标注："相关关系"还是"因果关系"

### 陷阱 4：幸存者偏差

**表现**：只分析还在用的用户，忽略了已流失的用户。"用户满意度 90 分"——因为不满意的早走了。

**解法**：
- 流失用户也要分析——甚至比活跃用户更值得分析
- 满意度调查同时关注回收率——低回收率本身就是信号
- 对比流失用户和留存用户的行为差异

### 陷阱 5：季节性干扰

**表现**：寒暑假期间活跃度骤降——不是产品出了问题，是教师放假了。

**解法**：
- 用同比（和去年同期比）比环比（和上个月比）更有意义
- 建立教学日历对照表，标注开学、考试、假期节点
- 设置指标预警时纳入周期性因子——否则每个假期都会误报警
