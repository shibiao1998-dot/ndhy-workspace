# 3 步修正 Prompt 模板

> 用于修正已有 Skill。保留原始意图，按最佳实践重写并审计质量。

---

## Step 1：问题诊断

### 目标
对照最佳实践逐项检查现有 Skill，输出诊断报告和评分。

### 执行流程

**1.1 读取现有 Skill**

读取目标 Skill 的所有文件：
- SKILL.md（必须）
- references/（如有）
- scripts/（如有）
- templates/（如有）

**1.2 逐项检查清单**

| # | 检查项 | 状态 | 问题描述 |
|---|--------|------|----------|
| 1 | YAML frontmatter 只有 name + description | ✅/❌ | |
| 2 | name 为 hyphen-case | ✅/❌ | |
| 3 | description 包含 WHAT + WHEN | ✅/❌ | |
| 4 | description 30-200 词 | ✅/❌ | |
| 5 | SKILL.md < 500 行 | ✅/❌ | |
| 6 | 不含 AI 已知的常识内容 | ✅/❌ | |
| 7 | 有 ❌/✅ 反面案例 | ✅/❌ | |
| 8 | 祈使语气 | ✅/❌ | |
| 9 | 超 100 行内容已拆分 | ✅/❌ | |
| 10 | references 文件有加载说明 | ✅/❌ | |
| 11 | 无辅助文件（README 等） | ✅/❌ | |
| 12 | 引用深度 ≤ 1 层 | ✅/❌ | |

**1.3 知识缺口重新分析**

重新评估现有 Skill 的知识分布：

| 类别 | 现状 | 应调整为 |
|------|------|----------|
| 包含了 AI 已知内容 | {列出具体内容} | 删除 |
| 缺少 AI 不知道的内容 | {列出缺失内容} | 补充到 SKILL.md |
| 缺少 AI 易犯错的案例 | {列出缺失案例} | 补充 ❌/✅ 示例 |

**1.4 输出诊断报告**

```markdown
## 诊断报告：{Skill 名称}

### 总体评估
- **原始意图**：{这个 Skill 想解决什么问题}
- **当前状态**：{总体质量评级：优秀/良好/需改进/严重问题}
- **核心问题数**：{个}

### 问题清单（按严重度排序）
1. 🔴 严重：{问题描述}
2. 🟡 中等：{问题描述}
3. 🟢 轻微：{问题描述}

### 10 维度预评分
{使用 quality-audit.md 的评分标准}
| 维度 | 当前评分 | 主要问题 |
|------|----------|----------|
| ... | ... | ... |
```

---

## Step 2：SKILL.md 重写

### 目标
保留原始意图，按最佳实践完整重写。

### 执行原则

1. **保留原始意图** — 不改变 Skill 的核心功能定位
2. **按 7 步生成标准重写** — 即使不走完整 7 步，也要符合每步的产出标准
3. **知识缺口驱动** — 基于 Step 1 的知识缺口重新分析，重新组织内容
4. **结构优化** — 按最佳架构模式重新组织

### 重写清单

- [ ] 重写 description（WHAT + WHEN，30-200 词）
- [ ] 删除 AI 已知的常识内容
- [ ] 补充 AI 不知道的核心知识
- [ ] 添加/完善 ❌/✅ 反面案例
- [ ] 统一为祈使语气
- [ ] 超 100 行内容拆到 references/
- [ ] 为所有 references 文件添加加载说明
- [ ] 删除辅助文件（README.md 等）
- [ ] 确保 SKILL.md < 500 行
- [ ] 清理多余的 YAML frontmatter 字段

### 产出

输出完整重写后的：
1. SKILL.md（含 YAML frontmatter + 正文）
2. references/ 下的新文件或修改后的文件
3. 变更说明（改了什么、为什么改）

---

## Step 3：质量审计

### 目标
验证重写后的 Skill 质量达标。

### 执行流程

**3.1 对重写后的 Skill 执行 10 维度评分**

使用 [quality-audit.md](quality-audit.md) 的完整评分标准。

| 维度 | 重写前 | 重写后 | 改善幅度 |
|------|--------|--------|----------|
| 1. Description 触发精准度 | | | |
| 2. 知识增量 | | | |
| 3. 代码示例质量 | | | |
| 4. 反面案例覆盖 | | | |
| 5. 结构清晰度 | | | |
| 6. 渐进式披露 | | | |
| 7. 语气一致性 | | | |
| 8. 边界处理 | | | |
| 9. 可操作性 | | | |
| 10. 完整性 | | | |
| **总分** | | | |

**3.2 质量门禁**

- 所有维度必须 ≥ 8 分
- 总分必须 > 重写前总分
- 如有 < 8 分的维度，回到 Step 2 修复后重新审计

**3.3 输出修正总结**

```markdown
## 修正总结：{Skill 名称}

### 变更概要
- **修正问题数**：{个}
- **质量提升**：{重写前总分} → {重写后总分}

### 主要改进
1. {改进项 1}
2. {改进项 2}

### 变更文件
| 文件 | 变更类型 | 说明 |
|------|----------|------|
| SKILL.md | 重写 | {变更说明} |
| references/xxx.md | 新增/修改 | {说明} |

### 最终评分
{10 维度评分表}
```
