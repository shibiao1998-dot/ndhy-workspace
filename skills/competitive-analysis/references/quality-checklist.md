# 质量原则与防错清单

## 质量原则（8 条）

每次产出前用这个清单自检：

- [ ] **结论前置** — 先给答案再给论据，不要让读者翻到最后才知道结论
- [ ] **事实与观点分开** — 所有信息标注 `[事实]` 或 `[观点/推测]`，混在一起丧失可信度
- [ ] **不做功能清单搬运工** — 要回答"为什么这样做"和"这意味着什么"
- [ ] **每条分析落回我们** — "对我们意味着什么？"回答不了的分析删掉
- [ ] **承认信息局限** — 诚实标注"此信息未经验证"、"数据来源有限"
- [ ] **关注变化非静态** — 竞品的变化（新版本、战略转向、裁员扩招）比当前状态更有价值
- [ ] **结构化优于大段文字** — 优先用表格、列表、对比矩阵，避免大段描述
- [ ] **有态度地建议** — "综上建议 X，因为 Y" 而非 "以上供参考"

## 防错清单

每次调研完成前自检：

- [ ] 调研目的是否在步骤 1 中明确定义？
- [ ] 竞品清单是否覆盖了三个层级（直接/间接/潜在）？
- [ ] 信息采集是否标注了来源和日期？
- [ ] 事实与推测是否严格分开？
- [ ] 分析是否使用了合适的框架，而非随意罗列？
- [ ] 5 类核心产出是否都有覆盖？
- [ ] 结论是否明确回答了步骤 1 的调研命题？
- [ ] 建议是否有态度、可行动，而非空洞的"供参考"？
- [ ] 信息的局限性是否诚实标注？
