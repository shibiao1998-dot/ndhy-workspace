# 信息来源地图

竞品信息按可靠性分 4 圈，从内到外可靠性递减。采集时优先内圈，外圈用于补充和交叉验证。

---

## 第一圈：直接信息（最可靠）

直接接触竞品产品和官方渠道获得的一手信息。

| 来源 | 能获得什么 | 采集技巧 |
|------|-----------|----------|
| **竞品产品本身** | 功能、流程、体验、定价 | 注册试用账号、购买体验完整流程、录屏关键操作流程 |
| **官方网站** | 定位、卖点、目标客户、定价策略 | 重点看：首页话术（= 核心价值主张）、客户案例页（= 目标客户画像）、定价页（= 商业模式） |
| **官方文档/帮助中心** | 功能全貌、技术架构线索、API 能力 | 帮助文档往往比官网更真实——官网是营销语言，文档是工程语言 |
| **App Store/应用市场** | 版本更新记录、用户评价、评分趋势 | 更新日志 = 产品优先级的时间线；关注评分趋势而非绝对值 |

**采集要点**：
- 官网截图保存（官网会改版，截图是时间快照）
- 试用账号记录操作路径和关键页面
- 帮助文档中的 API 列表常暴露未公开功能

---

## 第二圈：公开商业信息（较可靠）

通过公开商业渠道获取的二手信息，可信度较高但可能有滞后。

| 来源 | 能获得什么 | 常用平台 |
|------|-----------|----------|
| **融资/财报信息** | 资金实力、投资方、估值变化、烧钱速度 | Crunchbase、天眼查、IT桔子、企查查 |
| **招聘信息** | 正在招什么人 = 正在布局什么方向 | Boss直聘、LinkedIn、拉勾 |
| **专利/论文** | 技术路线、未来产品方向的线索 | Google Patents、中国知识产权局、Google Scholar |
| **创始人/高管公开发言** | 战略思路、价值观、对行业的判断 | 演讲视频、播客、采访文章、Twitter/X |
| **行业报告** | 市场规模、格局、趋势、预测 | 艾瑞咨询、Gartner、CB Insights、IDC、QuestMobile |

**采集要点**：
- 招聘信息是最好的战略情报——大量招 AI 工程师 = 在做 AI；招国际化 = 出海
- 融资信息看投资方背景：战略投资（巨头入局信号）vs 财务投资
- 创始人发言注意区分"说给投资人听的"和"真实战略意图"

---

## 第三圈：用户侧信息（真实但零散）

来自用户的真实反馈，信息碎片化但真实度高。

| 来源 | 能获得什么 | 常用平台 |
|------|-----------|----------|
| **用户评价** | 真实口碑、核心优缺点、使用痛点 | G2、Capterra（海外）；知乎、小红书（国内） |
| **社区/论坛** | 用户痛点、功能诉求、吐槽、workaround | Reddit、GitHub Issues（开源产品）、贴吧、V2EX |
| **社交媒体** | 品牌声量、用户情绪、事件传播 | Twitter/X、微博、即刻 |
| **客户案例/成功故事** | 核心使用场景、价值主张验证、客户画像 | 竞品官网案例页、客户采访、会议演讲 |

**采集要点**：
- 差评比好评更有价值——差评暴露产品短板和用户未满足需求
- GitHub Issues/Discussions 是技术类产品的金矿（功能请求 = 用户需求排行榜）
- 客户案例要反向阅读：案例越强调什么，说明竞品越想抢什么市场

---

## 第四圈：隐性信息（需要推理）

不直接说明但可以通过分析推断的信息，需要归因推理能力。

| 来源 | 能获得什么 | 常用工具 |
|------|-----------|----------|
| **SEO/SEM 策略** | 竞品在抢什么关键词 = 在抢什么用户心智 | SimilarWeb、Ahrefs、SEMrush、5118（国内） |
| **内容营销策略** | 竞品在教育市场什么 = 在塑造什么认知 | 分析博客主题、白皮书方向、视频内容策略 |
| **合作伙伴/生态** | 谁跟谁结盟 = 行业未来的势力版图 | 官网合作伙伴页、集成市场、联合活动 |
| **离职员工评价** | 内部文化、真实困境、管理问题 | Glassdoor（海外）、脉脉（国内） |

**采集要点**：
- SEO 关键词变化反映战略转向——突然开始投某类词 = 在开新战线
- 内容营销的主题选择暴露竞品对市场教育的判断
- 合作伙伴选择暴露生态战略——跟谁合作 = 不跟谁竞争
- 离职评价注意时间窗口和可能的情绪偏差

---

## 按调研目的选择采集重点

不同目的侧重不同圈层：

| 调研目的 | 重点采集圈 | 说明 |
|----------|-----------|------|
| **A. 市场验证型** | 第二圈 + 第四圈 | 需要市场数据和趋势信号 |
| **B. 方案参考型** | 第一圈 + 第三圈 | 需要产品细节和用户反馈 |
| **C. 竞争定位型** | 第一圈 + 第二圈 | 需要产品和商业全貌 |
| **D. 趋势洞察型** | 第二圈 + 第四圈 | 需要战略信号和隐性趋势 |
