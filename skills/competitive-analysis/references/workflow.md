# 竞品调研 6 步 SOP 详细流程

## 步骤 1：明确调研目的

**目标**：定义"我们到底想搞清楚什么"。

操作：
1. 从需求方获取核心问题，复述确认
2. 判断调研目的类型（决定后续深度和侧重）：

| 类型 | 核心问题 | 侧重 |
|------|----------|------|
| **A. 市场验证型** | 这个市场值不值得进？ | 市场规模、格局、进入壁垒 |
| **B. 方案参考型** | 别人怎么解决这个问题？ | 功能拆解、技术路线、体验对比 |
| **C. 竞争定位型** | 我们跟对手差在哪？ | 横向对比、差异化、优劣势 |
| **D. 趋势洞察型** | 行业往哪走？ | 迭代轨迹、投融资、技术演进 |

3. 产出：**一句话调研命题** + 3-5 个待回答的具体问题

## 步骤 2：建立竞品清单

**目标**：确定调研对象，不遗漏关键玩家。

三层分类法：

| 层级 | 定义 | 数量建议 | 深度 |
|------|------|----------|------|
| **直接竞品** | 做同样的事给同样的人 | 3-5 个 | 深度拆解 |
| **间接竞品** | 解决同样问题但方案不同 | 2-3 个 | 思路差异分析 |
| **潜在竞品** | 现在不是但可能切进来 | 列出即可 | 简要关注 |

输出格式：竞品清单表（名称 + 分类 + 选入理由 + 一句话定位）

## 步骤 3：信息采集

**目标**：系统性收集每个竞品的关键信息。

→ 按 **4 圈信息来源地图** 系统性采集，读 `references/information-source-map.md`

每个竞品采集 5 类信息：

| 类别 | 包含内容 |
|------|----------|
| **基本信息** | 成立时间、团队规模、融资阶段、总部 |
| **产品信息** | 核心功能、定价、技术栈、版本更新频率 |
| **用户信息** | 目标用户、DAU/MAU（如可获取）、核心使用场景 |
| **商业信息** | 商业模式、营收（如可获取）、融资额、投资方 |
| **战略信息** | 招聘方向、合作伙伴、高管公开发言、专利布局 |

**采集纪律**（强制）：
- 事实和推测**必须分开标注**：`[事实]` vs `[推测]`
- 记录每条信息的来源 URL 或出处
- 标注采集日期

## 步骤 4：产品深度拆解

**目标**：把产品拆成零件级理解。

对每个直接竞品，从 9 个层面拆解：

1. **产品定位** — 一句话说清"给谁解决什么问题"
2. **核心功能** — 功能架构图 + 核心功能描述
3. **用户体验** — 关键流程的体验走查
4. **信息架构** — 导航结构、内容组织方式
5. **定价策略** — 价格梯度、免费/付费边界、计费模式
6. **技术路线** — 技术栈、架构思路（从公开信息推断）
7. **运营策略** — 获客渠道、内容策略、社区运营
8. **迭代轨迹** — 版本更新历史 → 产品优先级的时间线
9. **差异化点** — 与其他竞品最大的不同是什么

**关键**：不是 9 个都要写得一样深。根据步骤 1 的调研目的，确定重点层面。

## 步骤 5：比较分析与洞察提炼

**目标**：从事实中提炼出有判断力的结论。

→ 选用合适的分析框架，读 `references/analysis-frameworks.md`

**框架选择速查**：

| 需要做什么 | 用哪个框架 |
|------------|-----------|
| 对比各家有没有某功能 | 功能矩阵 |
| 看各家战略定位差异 | 定位地图 |
| 全面评估单个竞品 | SWOT |
| 找差异化切入点 | 价值曲线 |
| 对比用户体验 | 用户旅程对比 |
| 拆解商业逻辑 | 商业模式画布 |
| 对比技术方案 | 技术栈对比 |

从分析中必须提炼 **5 类核心产出**：

1. **行业共识** — 所有玩家都在做的（= 基本功，必须有）
2. **差异化方向** — 各家走的不同路（= 战略选择空间）
3. **市场空白** — 没人做或做得不好的（= 机会）
4. **最佳实践** — 某家做得特别好值得借鉴的
5. **常见陷阱** — 某家踩过的坑，我们要避开的

## 步骤 6：得出结论与建议

**目标**：回答调研命题，给出可行动的建议。

必须明确回答以下 5 个问题：

1. **竞争格局**：这个市场现在是什么状态？（红海/蓝海、集中/分散、成熟/新兴）
2. **我们的机会**：哪里有空间？为什么？
3. **应借鉴什么**：竞品做得好的哪些做法值得学？
4. **应避开什么**：哪些方向已经卷得没意义或有明确陷阱？
5. **差异化方向**：如果我们要做，应该走什么路线？

**关键**：给出**有态度的建议**，不是"以上供参考"。"综合以上分析，建议XXX，理由是XXX"。

→ 按标准模板组织报告，读 `references/report-template.md`
