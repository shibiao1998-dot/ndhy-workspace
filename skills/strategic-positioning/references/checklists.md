# 质量检查清单

## §1 交付前自检清单

战略定位文档交付前，逐条检查：

### 结构完整性

- [ ] 七个章节全部填写（轻量版可省略"方向选择"章节）
- [ ] 每个章节有实质内容，不是空模板
- [ ] Phase 1 输入文件已索引

### 铁律 1：定位可证伪

- [ ] 目标客群具体到可识别的人群（不是"所有人"或"广大用户"）
- [ ] 价值主张可用"场景-问题-程度"三元组描述
- [ ] 核心定位可以设计出验证/证伪的方法
- [ ] 定位语句中不包含以下大词（除非有具体解释）：
  - "领先的" / "一流的" / "最好的"
  - "全面的" / "一站式" / "综合"
  - "赋能" / "助力" / "引领"

### 铁律 2：选择有取舍

- [ ] "不做什么"清单**不为空**
- [ ] 每条放弃项有明确理由（不是"以后再说"）
- [ ] 放弃理由涉及战略判断（不只是资源不足）
- [ ] "暂不做"每条有明确的启动触发条件
- [ ] 做/不做/暂不做三个清单逻辑自洽（不矛盾）

### 铁律 3：逻辑链条完整

- [ ] 市场事实 → 机会判断 → 方向选择 → 具体定位，每步有依据
- [ ] 没有跳步推理（如直接从市场数据跳到具体功能）
- [ ] 推荐方向的理由追溯到 Phase 1 数据/洞察
- [ ] 放弃方向的理由同样有依据
- [ ] 战略叙事的每一句都可追溯到前文分析

### 价值主张质量

- [ ] 核心价值点 1-3 个（不超过 3 个）
- [ ] 每个价值点通过三重检验（真实性 + 差异性 + 重要性）
- [ ] 三重检验中有未通过项的已被替换或调整
- [ ] 价值主张层次结构清晰（核心 + 支撑）

### 验证设计质量

- [ ] 至少提取 3 个关键假设
- [ ] 每个假设有具体的验证方式（不是"后续验证"）
- [ ] 判断标准可操作（不是"看情况"）
- [ ] 预警信号具体且可监控
- [ ] 每个假设有 Plan B

### 整体质量

- [ ] 全文没有与 Phase 1 输出矛盾的判断
- [ ] 差异化至少达到"体验差异"层次
- [ ] 没有出现"假大空"表述（愿景口号替代因果逻辑）
- [ ] 产品架构师拿到后能明确知道"做什么、不做什么"

## §2 五大质量陷阱详细防范

### 陷阱 1：定位空泛

**表现**：
- "打造领先的智慧教育平台"
- "构建一站式教育解决方案"
- 使用大量修饰词但无具体内容

**根因**：没有做真正的选择，试图覆盖所有人和所有场景。

**防范措施**：
1. 写完定位后做"替换测试"：把公司名和产品名换成竞品名，如果依然成立 → 定位太空泛
2. 做"销售测试"：把定位发给销售，问他能否在一周内找到 10 个目标客户。不能 → 客群太模糊
3. 强制使用"场景-问题-程度"三元组格式：
   - ❌ "帮助教师提高效率"
   - ✅ "在英语课备课场景下，将个性化作业设计时间从 3 小时缩短到 30 分钟"

**修复方法**：回到竞争定位三问，逐个问题重新回答，每个答案必须具体到可验证。

### 陷阱 2：缺乏取舍

**表现**：
- 做什么清单很长，不做什么清单为空
- 所有功能都是 P0
- "我们要做 A，也要做 B，还要做 C"

**根因**：害怕错过机会，不敢做战略性放弃。

**防范措施**：
1. 硬规则："不做什么"清单条目数 ≥ "做什么"条目数的 1/3
2. 对每个"做什么"追问："如果资源只够做 3 件事，这个还在列表里吗？"
3. 对"不做什么"要区分"战略性放弃"和"暂时不做"：
   - 战略性放弃：与定位方向不符，即使有资源也不做
   - 暂时不做：方向对但时机/资源不到位，需要明确启动条件

**修复方法**：强制产出不做清单。对每个做什么的项目，写出"如果不做这个，会怎样？"——如果答案是"没什么影响"，就移出做什么清单。

### 陷阱 3：脱离现实

**表现**：
- 定位要求的能力公司目前不具备
- 差异化依赖于尚未验证的技术
- 忽视资源约束

**根因**：定位设计时没有考虑"我们凭什么能做到"。

**防范措施**：
1. 每个差异化点追问："现在就能做到，还是需要建设？需要多久？"
2. 价值主张三重检验中"真实性"必须严格把关
3. 列出支撑每个价值点所需的核心能力和资源
4. 对"需要建设的能力"，评估建设成本和时间

**修复方法**：对不现实的部分做两步处理——
- 能力已有但需要增强 → 保留，标注建设计划
- 能力完全缺失 → 移出核心定位，考虑替代方案

### 陷阱 4：自说自话

**表现**：
- 分析逻辑自洽但与用户数据矛盾
- 假设用户需要某个功能但没有数据支持
- 忽视 Phase 1 输出中的关键发现

**根因**：从"我想做什么"出发而非"用户需要什么"出发。

**防范措施**：
1. 每个核心判断标注数据来源（Phase 1 报告的具体章节）
2. 如果找不到数据支持 → 标注为"假设"并列入验证清单
3. 邀请 Phase 1 专家审查（用户研究专家审查客群定义、市场评估专家审查竞争判断）

**修复方法**：重新遍历 Phase 1 输出，标记每个定位判断的数据依据。没有依据的 → 降级为假设或重新调研。

### 陷阱 5：假大空叙事

**表现**：
- "我们致力于构建教育的未来"
- "以技术驱动教育变革"
- 战略叙事读起来像品牌广告

**根因**：用"态度"代替"逻辑"，用"愿景"代替"推理"。

**防范措施**：
1. 战略叙事必须用"因为…所以…"因果链条格式
2. 叙事中每个断言都能在前文分析中找到对应依据
3. 做"删除测试"：把所有形容词和副词删掉，如果剩下的内容没有实质信息 → 重写
4. 禁用词检查：如果出现以下词汇，替换为具体描述：
   - "赋能" → 具体说明提供什么能力
   - "引领" → 具体说明在什么维度上领先
   - "创新" → 具体说明创新了什么

**修复方法**：按因果链条模板重写。每一句从 Phase 1 数据中找到支撑。找不到支撑的句子删掉。
