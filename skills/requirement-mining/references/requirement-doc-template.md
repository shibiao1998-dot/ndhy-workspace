# 需求文档输出模板

收敛确认后，使用此模板输出结构化需求文档。

---

## 模板

```markdown

[用 2-3 句话描述：谁提出了这个需求？当前遇到了什么问题？]

### 目标
[这个需求要达成什么效果？解决什么问题？注意写"问题"不是"方案"]

### 商业价值
[做了这个能带来什么商业价值？降本？提效？增收？用户满意度？]

---
```

## 使用说明

1. **不是每个字段都必须填**：根据需求复杂度灵活调整。简单需求可以精简
2. **验收标准必须可检验**："好用"不是验收标准，"首屏加载 ≤ 3 秒"才是
3. **优先级标注必须有**：P0（必须做）/ P1（应该做）/ P2（锦上添花）
4. **"明确不做"比"要做什么"更重要**：画清边界可以避免后期 80% 的扯皮
5. **遗留问题必须显式列出**：不确定的点标出来，比藏着掖着好 100 倍
6. **简洁需求可压缩格式**：不必拘泥于完整模板，保留核心信息即可（目标 + 需求列表 + 验收标准 + 约束 + 遗留问题）
