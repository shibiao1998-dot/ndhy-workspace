# 教育行业用户研究特别指南

## 核心认知：教育用户是一个利益相关者网络

教育产品不是"一个用户群"，而是多层角色交织的利益网络。**使用者和决策者往往不是同一个人。**

### 多层用户结构

| 角色层 | 典型角色 | 与产品的关系 | 研究重点 |
|--------|----------|-------------|----------|
| **直接使用者** | 学生���教师 | 日常使用产品 | 使用体验、功能需求、行为模式 |
| **决策者** | 学校管理者、教育局 | 决定是否采购/推广 | 采购标准、政策合规、管理需求 |
| **付费者** | 家长、学校、政府 | 支付费用 | 价格敏感度、价值感知、决策过程 |
| **影响者** | 教研员、同行教师、家长社群 | 影响采购和使用决策 | 口碑传播路径、专业评价标准 |

### 研究操作要点

1. **必须分别画像**：每层角色独立构建画像，不混为一谈
2. **识别决策链**：谁发起 → 谁评估 → 谁拍板 → 谁付款 → 谁使用
3. **冲突识别**：不同层角色的需求可能冲突（如：教师要灵活性 vs 管理者要标准化）
4. **影响力图谱**：画出各角色间的影响关系，识别关键影响节点

## 教育场景四大特殊性

### 1. 效果滞后性

**核心问题：用户满意度 ≠ 学习效果**

- 学生喜欢的产品不一定教学效果好
- 教学效果需要长期（一学期/一学年）才能显现
- 短期用户反馈可能误导产品方向

**研究对策：**
- 区分"满意度指标"和"效果指标"
- 不以短期好评替代长期价值判断
- 如引用效果数据，需标注观测周期

### 2. 强政策属性

**核心问题：政策变动直接改变用户需求**

- 教育政策（双减、新课标、AI 教育政策等）会瞬间改变市场格局
- 不同地区政策执行力度不同
- 政策合规是采购决策的硬性前提

**研究对策：**
- 每次研究纳入政策环境分析
- 标注政策发布时间和影响范围
- 区分"政策驱动的需求"和"自发需求"
- 关注政策趋势，预判需求变化

### 3. 强周期性

**核心问题：教育有固定节奏，用户行为随周期变化**

| 周期节点 | 用户行为特征 |
|---------|-------------|
| 开学季（2月/9月） | 集中采购、新用户涌入 |
| 期中/期末 | 使用频率高峰、评测需求爆发 |
| 寒暑假 | 使用模式变化、培训/备课需求 |
| 招生季 | 决策者关注升学率、成果展示需求 |
| 教师培训期 | 新功能接受度高、培训需求 |

**研究对策：**
- 标注研究时间所处的周期阶段
- 用户行为数据需考虑周期因素
- 不同周期阶段的研究侧重不同
- 避免在特殊周期得出"普遍"结论

### 4. 强地域差异

**核心问题：一线城市和县城的需求可能截然不同**

| 维度 | 一线/新一线城市 | 三四线城市/县城 |
|------|----------------|----------------|
| 技术素养 | 普遍较高 | 差异大，部分较低 |
| 设备条件 | 良好 | 可能受限 |
| ��学理念 | 偏创新、个性化 | 偏应试、标准化 |
| 信息渠道 | 多元、线上为主 | 线下口碑、行政推荐 |
| 付费能力 | 较强 | 价格敏感 |
| 政策执行 | 紧跟最新政策 | 可能滞后 |

**研究对策：**
- 画像中必须标注地域维度
- 不用一线城市数据代表全国
- 分群时考虑地域作为独立维度
- 关注下沉市场的特殊需求和约束

## 教育项目研究附加检查项

在标准 9 项自检清单之外，教育项目额外检查：

| # | 附加检查项 | 通过标准 |
|---|-----------|----------|
| E1 | **多层用户识别** | 是否识别并分别研究了直接使用者、决策者、付费者、影响者？ |
| E2 | **决策链完整** | 是否描述了从发起到使用的完整决策链？ |
| E3 | **政策纳入** | 是否分析了相关教育政策及其对需求的影响？ |
| E4 | **周期标注** | 是否标注了研究时间所处的教育周期阶段？ |
| E5 | **地域覆盖** | 是否声明了研究覆盖的地域范围？是否避免以偏概全？ |
| E6 | **效果/满意度区分** | 是否区分了用户满意度和教育效果？ |
