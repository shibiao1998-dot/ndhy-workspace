# 标准输出物模板

## §1 用户画像文档

```markdown
# 用户画像：[画像名称，如"一线城市初中数学教师"]

## 画像概要
- 用户类型：[核心用户 / 次要用户 / 边缘用户 / 影响者]
- 数据支撑强度：[强-多源验证 / 中-有限数据 / 弱-主要靠推演]

## 基础属性
- 典型年龄段：
- 职业角色：
- 教育背景：
- 技术素养：[高/中/低]，具体表现为___
- 所处环境：

## 核心目标
1. [最根本的目标]
2. [次要目标]

## 行为特征
- 日常工作/生活模式：
- 获取信息的方式：
- 做决策的方式：
- 使用工具的习惯：

## 痛点清单（按严重程度排序）
| 排序 | 痛点描述 | 严重程度 | 发生频率 | 数据来源 |
|-----|---------|---------|---------|---------|
| 1   |         | 高/中/低 | 高/中/低 |         |
| 2   |         |          |          |         |
| 3   |         |          |          |         |

## 需求清单（按优先级排序）
| 优先级 | 需求描述 | 对应痛点 | 当前替代方案 |
|-------|---------|---------|------------|
| P0    |         |         |            |
| P1    |         |         |            |
| P2    |         |         |            |

## 关键引述（如有真实用户声音）
> "引述内容" —— 来源

## 标注
- 假设部分：[哪些结论是推演的，需后续验证]
- 数据时效：[最新数据的日期]
- 局限性：[本画像未能覆盖的方面]
```
