# 领域角色池：教育/培训类

> 根据方案需要挑选，也可创造池子里没有的新角色。每个角色选用后必须按 7 维度定义。

## 角色清单

### 1. 教学设计

- **典型定位**：设计教学策略和方法，确保学习目标可达成
- **典型输入**：学习目标、目标学员画像、教学约束（时长、形式）
- **典型输出**：教学设计方案（含教学策略、互动方式、评估方式）
- **下游角色**：课程结构设计、知识点拆解
- **验收参考**：教学目标可衡量；教学策略匹配学员水平；评估方式与目标对齐

### 2. 课程结构设计

- **典型定位**：设计课程的整体结构（模块、章节、课时）
- **典型输入**：教学设计方案、内容范围
- **典型输出**：课程结构图（模块→章节→课时，含时长分配）
- **下游角色**：知识点拆解、教学素材开发
- **验收参考**：结构逻辑清晰；难度梯度合理；总时长符合约束

### 3. 知识点拆解

- **典型定位**：将课程内容拆解为最小可教学的知识点
- **典型输入**：课程结构、领域知识
- **典型输出**：知识点清单（含知识点间的前置依赖关系）
- **下游角色**：题目/练习设计、教学素材开发
- **验收参考**：知识点粒度合适（可在单次教学中讲清）；前置依赖无循环

### 4. 题目/练习设计

- **典型定位**：设计练习题和考核题目
- **典型输入**：知识点清单、评估目标（理解/应用/分析）
- **典型输出**：题库（含答案、解析、难度标注）
- **下游角色**：评估方案设计、课程质量审核
- **验收参考**：覆盖所有核心知识点；难度梯度合理；答案准确

### 5. 学习路径规划

- **典型定位**：为不同学员群体设计个性化学习路径
- **典型输入**：课程结构、知识点依赖图、学员画像
- **典型输出**：学习路径图（含分支、跳过条件、推荐顺序）
- **下游角色**：课程质量审核
- **验收参考**：路径无死胡同；考虑了不同基础的学员；有明确的里程碑

### 6. 教学素材开发

- **典型定位**：开发具体的教学内容素材（讲义、PPT、示例代码、案例等）
- **典型输入**：课程结构、知识点清单、教学设计方案
- **典型输出**：教学素材包
- **下游角色**：课程质量审核
- **验收参考**：素材覆盖所有知识点；风格统一；示例可运行/可验证
- **说明**：素材的具体类型取决于课程形式（视频课需配合多媒体角色；文字课需配合文档角色）

### 7. 评估方案设计

- **典型定位**：设计课程的考核评估体系
- **典型输入**：学习目标、题库、评估形式约束
- **典型输出**：评估方案（含评估时机、形式、评分标准、通过条件）
- **下游角色**：课程质量审核
- **验收参考**：评估覆盖核心学习目标；评分标准客观可执行

### 8. 课程质量审核

- **典型定位**：从学习体验角度审核课程整体质量
- **典型输入**：完整课程包（结构+素材+题目+评估方案）
- **典型输出**：质量审核报告（通过/修改意见）
- **下游角色**：对应制作角色（修改）、项目管理（交付确认）
- **验收参考**：学习目标可达成；内容准确无误；体验流畅

---

## 典型协作模式

```
教学设计 → 课程结构设计 → 知识点拆解 → [题目/练习设计 || 学习路径规划 || 教学素材开发]
                                                        ↓
                                                 评估方案设计
                                                        ↓
                                                 课程质量审核
```

## 教育类项目的特殊注意事项

- **跨领域协作频繁**：视频课程需要多媒体角色配合；在线课程需要软件开发角色配合
- **迭代周期长**：教学效果需要实际验证，常需多轮调整
- **外部依赖**：面对面教学、线下考试等属于物理世界操作，标注为"外部依赖"
