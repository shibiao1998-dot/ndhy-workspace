# 领域角色池：软件开发类

> 根据方案需要挑选，也可创造池子里没有的新角色。每个角色选用后必须按 7 维度定义。

## 角色清单

### 1. 数据库设计

- **典型定位**：设计数据存储结构，定义表/集合的字段、索引、关系
- **典型输入**：PRD 中的数据需求、业务实体关系
- **典型输出**：ER 图、DDL 脚本、数据字典
- **下游角色**：后端开发、API 设计
- **验收参考**：表结构覆盖所有业务实体；索引覆盖高频查询；无冗余字段

### 2. API 设计

- **典型定位**：定义系统间/模块间的接口契约
- **典型输入**：PRD 功能需求、数据库设计输出
- **典型输出**：API 文档（OpenAPI/Swagger 格式）、接口契约
- **下游角色**：前端开发、后端开发、移动端开发
- **验收参考**：接口覆盖所有功能需求；请求/响应格式明确；错误码完整

### 3. 后端开发

- **典型定位**：实现服务端业务逻辑和数据处理
- **典型输入**：API 文档、数据库设计、技术方案
- **典型输出**：可运行的后端服务代码、单元测试
- **下游角色**：联调集成、自动化测试
- **验收参考**：API 全部可调通；单元测试覆盖核心逻辑；代码通过 lint

### 4. 前端开发

- **典型定位**：实现用户界面和交互逻辑
- **典型输入**：UI 设计稿、API 文档、交互规范
- **典型输出**：可运行的前端页面/组件、响应式适配
- **下游角色**：联调集成、自动化测试
- **验收参考**：页面还原设计稿；交互流程完整；响应式适配通过

### 5. 移动端开发

- **典型定位**：实现 iOS/Android 原生应用
- **典型输入**：UI 设计稿、API 文档、平台规范
- **典型输出**：可安装的 App 包、平台适配
- **下游角色**：联调集成、自动化测试
- **验收参考**：核心流程可跑通；平台规范合规；性能达标

### 6. 小程序开发

- **典型定位**：实现微信/支付宝等平台小程序
- **典型输入**：UI 设计稿、API 文档、平台规范
- **典型输出**：可预览的小程序包
- **下游角色**：联调集成、自动化测试
- **验收参考**：平台审核规范合规；核心流程可跑通

### 7. 联调集成

- **典型定位**：将各模块组装并验证端到端流程
- **典型输入**：前端代码、后端服务、API 文档
- **典型输出**：联调通过的完整系统、联调问题清单
- **下游角色**：自动化测试、部署运维
- **验收参考**：核心业务流程端到端跑通；跨模块数据一致

### 8. 代码审查

- **典型定位**：审查代码质量、安全性和规范性
- **典型输入**：待审查代码、编码规范
- **典型输出**：审查意见（含严重级别）、修改建议
- **下游角色**：对应开发角色（修复后回到审查）
- **验收参考**：无高危问题遗留；核心逻辑有审查记录

### 9. 自动化测试

- **典型定位**：编写和执行自动化测试用例
- **典型输入**：API 文档、功能需求、联调通过的系统
- **典型输出**：测试用例集、测试报告、缺陷列表
- **下游角色**：对应开发角色（修复缺陷）、部署运维
- **验收参考**：用例覆盖核心流程；自动化覆盖率达标；无阻塞级缺陷

### 10. 性能测试

- **典型定位**：验证系统在负载下的表现
- **典型输入**：性能指标要求、联调通过的系统
- **典型输出**：性能测试报告、瓶颈分析、优化建议
- **下游角色**：后端开发（优化）、部署运维（调参）
- **验收参考**：核心指标达标（响应时间、吞吐量、并发数）

### 11. 安全测试

- **典型定位**：发现系统安全漏洞
- **典型输入**：安全需求、联调通过的系统
- **典型输出**：安全测试报告、漏洞清单及修复建议
- **下游角色**：对应开发角色（修复漏洞）
- **验收参考**：无高危漏洞遗留；OWASP Top 10 已覆盖

### 12. 部署运维

- **典型定位**：将系统部署到目标环境并保障运行
- **典型输入**：测试通过的代码包、部署配置需求
- **典型输出**：部署脚本/配置、监控告警配置、运维文档
- **下游角色**：项目管理（交付确认）
- **验收参考**：部署脚本可重复执行；监控覆盖核心指标；回滚方案就绪

### 13. 技术文档编写

- **典型定位**：编写技术文档保障可维护性
- **典型输入**：代码、API 文档、架构设计
- **典型输出**：开发文档、部署文档、使用手册
- **下游角色**：项目管理（交付确认）、后续维护团队
- **验收参考**：新人按文档可独立搭建环境；核心模块有架构说明

---

## 典型协作模式

```
数据库设计 → API设计 → [后端开发 || 前端开发 || 移动端开发] → 联调集成 → [自动化测试 || 性能测试 || 安全测试] → 代码审查 → 部署运维
                                                                                                              ↕
                                                                                                        技术文档编写
```

说明：`||` 表示可并行，`→` 表示依赖。代码审查可穿插在开发阶段。
