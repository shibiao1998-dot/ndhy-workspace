# 领域角色池：文档/知识类

> 根据方案需要挑选，也可创造池子里没有的新角色。每个角色选用后必须按 7 维度定义。

## 角色清单

### 1. 信息采集/调研

- **典型定位**：系统性收集和整理指定主题的信息
- **典型输入**：调研主题、信息需求范围、关键问题
- **典型输出**：结构化的调研资料集（含来源标注）
- **下游角色**：数据分析/可视化、文案撰写、大纲设计
- **验收参考**：覆盖指定范围；来源可追溯；事实与推测分开标注

### 2. 数据分析/可视化

- **典型定位**：分析数据并制作可视化图表
- **典型输入**：原始数据、分析需求、展示要求
- **典型输出**：分析结论 + 可视化图表
- **下游角色**：文案撰写（嵌入报告）、排版设计
- **验收参考**：数据处理无错误；图表类型选择恰当；结论有数据支撑

### 3. 大纲设计

- **典型定位**：设计文档的整体结构和逻辑框架
- **典型输入**：目标受众、文档目的、核心论点/内容点
- **典型输出**：文档大纲（含各章节要点和预估篇幅）
- **下游角色**：文案撰写
- **验收参考**：逻辑线清晰；覆盖所有关键内容点；层次不超过 3 级

### 4. 文案撰写

- **典型定位**：按大纲撰写文档正文内容
- **典型输入**：大纲、调研资料、数据图表、写作规范
- **典型输出**：完整文档初稿
- **下游角色**：校对审核、排版设计
- **验收参考**：覆盖大纲所有要点；语言风格一致；引用准确

### 5. 学术写作

- **典型定位**：撰写符合学术规范的内容（论文、研究报告）
- **典型输入**：研究成果、文献综述、数据分析结果
- **典型输出**：学术文稿（含引用格式）
- **下游角色**：校对审核
- **验收参考**：引用格式规范；论证逻辑严密；专业术语准确

### 6. 校对审核

- **典型定位**：检查文档的准确性、一致性和规范性
- **典型输入**：文档初稿/终稿
- **典型输出**：校对标注 + 修改建议
- **下游角色**：文案撰写（修改）、排版设计
- **验收参考**：无事实错误；无错别字；格式一致；术语统一

### 7. 排版设计

- **典型定位**：将文档内容排版为最终发布格式
- **典型输入**：定稿文本 + 图表、排版规范/模板
- **典型输出**：排版后的文档（PDF/PPT/网页等）
- **下游角色**：发布管理
- **验收参考**：排版规范一致；图表位置合理；阅读体验流畅

### 8. 翻译/本地化

- **典型定位**：将内容翻译为目标语言并进行本地化适配
- **典型输入**：源语言文档、目标语言、本地化要求
- **典型输出**：翻译后的文档
- **下游角色**：校对审核（目标语言）、排版设计
- **验收参考**：译文准确流畅；专业术语统一；本地化适配到位

### 9. 发布管理

- **典型定位**：管理文档的发布渠道和发布流程
- **典型输入**：定稿文档、发布平台信息、发布时间要求
- **典型输出**：已发布的文档 + 发布记录
- **下游角色**：项目管理（交付确认）
- **验收参考**：在目标平台可正常访问；格式无错乱；发布时间符合要求

---

## 典型协作模式

```
信息采集/调研 → [数据分析/可视化 || 大纲设计]
                       ↓
                 文案撰写 / 学术写作
                       ↓
                  校对审核
                       ↓
              [排版设计 || 翻译/本地化]
                       ↓
                  发布管理
```
