# 质量测试 — 反模式速查

| # | ❌ 错误做法 | ✅ 正确做法 |
|---|-----------|-----------|
| 1 | 只测 happy path，不测边界和异常 | 主路径 + 边界 + 异常 + 兼容性全覆盖 |
| 2 | 缺陷描述模糊："这里有问题" | 复现步骤 + 期望结果 + 实际结果 + 环境 + 截图/日志 |
| 3 | 缺陷不分级，平铺列出 | 按 P0-P3 分级，P0/P1 标红，带处理要求 |
| 4 | 回归范围失控，盲目全量回归 | 按影响范围 + 修复关联 + 核心路径设计回归清单 |
| 5 | 把联调通过当成测试通过 | 联调验证集成链路，测试验证功能/边界/异常/回归 |
| 6 | 把代码 review 通过当成测试通过 | 代码审查验证结构质量，测试验证运行时行为 |
| 7 | 发现问题不跟踪，"提了就算" | 跟踪到修复 → 回归验证 → 关闭，形成闭环 |
| 8 | 为赶进度跳过关键验证不标注风险 | 跳过必须在报告中明确标注为"未验证风险" |
| 9 | 用例数量堆砌却不覆盖关键风险 | 先覆盖高风险场景，再补充常规场景 |

-> 详细防范与案例 [checklists.md](checklists.md) 反模式详细防范
