# 教育行业测试特别指南

## 概述

教育行业产品在测试中有其特殊性，需要在标准测试流程基础上额外关注以下维度。本指南作为标准测试流程的补充，不替代通用流程。

---

## 一、多角色验收

### 教育产品典型角色

| 角色 | 特点 | 测试关注点 |
|------|------|-----------|
| **学生** | 主要使用者，操作能力参差不齐 | 操作容错性、引导清晰度、学习体验 |
| **教师** | 内容创建/管理者，需要效率 | 批量操作、数据管理、班级管理 |
| **家长** | 查看者，关注结果 | 信息准确性、隐私保护、通知及时 |
| **管理员** | 系统管理者 | 权限控制、数据统计、配置管理 |
| **超级管理员** | 平台级管理 | 多租户隔离、全局配置 |

### 多角色测试策略

1. **权限矩阵验证**：每个角色只能看到/操作权限范围内的内容
2. **数据隔离验证**：A 班学生不能看到 B 班数据
3. **角色切换验证**：同一账号多角色切换时数据和权限正确
4. **跨角色流程验证**：教师发布 → 学生提交 → 教师批改 → 家长查看

### 权限验证清单

| 验证项 | 验证方法 |
|--------|---------|
| 角色 A 不能访问角色 B 的页面 | 直接访问 URL |
| 角色 A 不能调用角色 B 的接口 | 直接调用 API |
| 数据列表按角色过滤正确 | 对比不同角色看到的数据 |
| 操作按钮按权限显示/隐藏 | 检查界面元素 |
| 多租户数据完全隔离 | 跨租户数据访问验证 |

## 二、内容安全

### 教育内容安全测试维度

| 维度 | 测试要点 | 为什么重要 |
|------|---------|-----------|
| **内容合规** | 用户生成内容是否经过审核 | 教育环境中不当内容影响严重 |
| **敏感词过滤** | 输入敏感词是否被正确拦截 | 保护未成年人 |
| **内容版权** | 引用内容是否标注来源 | 避免版权纠纷 |
| **答案安全** | 试题答案是否可被学生端获取 | 考试公平性 |
| **成绩安全** | 成绩数据是否可被篡改 | 数据可信度 |

### 内容安全测试用例

1. **UGC 内容审核**：
   - 提交含敏感词的作业/讨论 → 验证是否被拦截或进入审核
   - 上传不当图片 → 验证是否被检测
   - 提交恶意链接 → 验证是否被过滤

2. **答案防泄露**：
   - 学生端网络请求中是否包含答案数据
   - 教师端答案是否正确加密/隐藏
   - 考试未提交时答案是否不可见

3. **成绩防篡改**：
   - 直接修改请求中的成绩数据 → 验证后端是否校验
   - 学生端是否只能查看不能修改成绩
   - 成绩修改是否有操作日志

## 三、高并发场景

### 教育产品特有的高并发场景

| 场景 | 并发特点 | 风险 |
|------|---------|------|
| **考试开始** | 全班/全年级同时开始 | 服务器过载、试卷加载失败 |
| **考试提交** | 大量学生同时提交 | 数据丢失、超时、重复提交 |
| **成绩发布** | 家长集中查看 | 页面响应慢、数据不一致 |
| **选课抢课** | 学生集中选课 | 超选、数据冲突 |
| **直播课堂** | 大量学生同时进入 | 连接失败、视频卡顿 |

### 高并发测试策略

1. **模拟并发**：使用工具模拟多用户同时操作
2. **关注数据一致性**：并发操作后数据是否正确
3. **关注边界行为**：名额限制/时间限制在并发下是否正确
4. **关注用户体验**：高负载下的响应时间和错误提示

### 并发测试清单

- [ ] 多用户同时提交相同操作 → 数据不冲突、不丢失
- [ ] 名额限制操作并发 → 不超额、不重复
- [ ] 定时任务触发并发 → 考试开始/结束的并发处理
- [ ] 大量数据查询并发 → 成绩/报告的并发查询性能
- [ ] 写操作并发 → 幂等处理、乐观锁验证

## 四、隐私合规

### 教育数据隐私要求

教育产品涉及未成年人数据，隐私合规要求更高。

| 数据类型 | 隐私等级 | 测试要求 |
|---------|---------|---------|
| **学生个人信息** | 高 | 最小必要收集、加密存储、授权访问 |
| **学习行为数据** | 中 | 脱敏展示、限制导出范围 |
| **成绩数据** | 高 | 仅授权角色可见、不公开排名(视政策) |
| **家长联系方式** | 高 | 加密存储、限制使用场景 |
| **人脸/生物数据** | 极高 | 单独授权、加密、过期删除 |

### 隐私合规测试清单

1. **数据收集**：
   - [ ] 是否只收集业务必要数据
   - [ ] 是否在收集前获得适当授权/告知
   - [ ] 隐私政策是否完整

2. **数据存储**：
   - [ ] 敏感数据是否加密存储
   - [ ] 数据是否有过期/删除机制
   - [ ] 日志中是否脱敏处理个人信息

3. **数据访问**：
   - [ ] 是否按最小权限原则授权
   - [ ] 数据导出是否有审计日志
   - [ ] API 是否返回了不必要的敏感字段

4. **数据展示**：
   - [ ] 手机号/身份证是否脱敏展示
   - [ ] 其他学生的成绩是否不可见
   - [ ] 批量查询是否限制范围

5. **数据删除**：
   - [ ] 用户注销后数据是否可清除
   - [ ] 过期数据是否自动归档/删除
   - [ ] 删除操作是否彻底（包括备份/缓存）

## 五、教育行业常见缺陷模式

| 缺陷模式 | 描述 | 影响 |
|---------|------|------|
| **考试时间不同步** | 服务器时间和客户端时间不一致 | 考试提前结束/延迟开始 |
| **成绩计算精度** | 浮点数精度丢失 | 总分不等于各题分数之和 |
| **班级数据交叉** | A 班学生看到 B 班数据 | 数据泄露 |
| **权限继承错误** | 角色变更后旧权限残留 | 权限越权 |
| **并发选课超额** | 并发下名额控制失效 | 超出计划名额 |
| **内容状态不一致** | 教师端修改未同步到学生端 | 学生看到旧内容 |
| **离线数据丢失** | 弱网环境下学生作业未保存 | 数据丢失 |

## 六、教育项目测试附加检查清单

在标准检查清单基础上，教育项目额外检查：

| # | 检查项 | 通过条件 |
|---|--------|---------|
| E1 | 多角色权限 | 权限矩阵已验证，数据隔离正确 |
| E2 | 内容安全 | 敏感词过滤有效，答案不泄露 |
| E3 | 并发场景 | 关键并发场景（考试/选课）已验证 |
| E4 | 隐私合规 | 敏感数据加密、脱敏、最小授权 |
| E5 | 时间一致性 | 定时任务和时间相关功能跨时区正确 |
| E6 | 数据精度 | 成绩、统计数值计算精度正确 |
| E7 | 弱网容错 | 弱网环境下数据不丢失、有合理提示 |
