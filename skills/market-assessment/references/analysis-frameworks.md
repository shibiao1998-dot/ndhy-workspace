# 市场评估分析框架详解

## §1 市场规模与结构分析框架（TAM/SAM/SOM）

### TAM/SAM/SOM 三层估算

```
TAM（Total Addressable Market）— 市场总量
  │  这个领域整体有多大？
  │  方法：行业报告、政策规划、统计年鉴、上市公司财报
  │  ⚠️ TAM 不是"我们的机会"，只是"整个盘子"
  │
  ├── SAM（Serviceable Addressable Market）— 可触达市场
  │     以我们的能力和渠道，能够到的范围有多大？
  │     裁剪维度：
  │     · 公司定位（B端/C端/G端？哪个细分？）
  │     · 渠道能力（能触达哪些区域/客户类型？）
  │     · 产品形态（SaaS/硬件/服务？适配哪些场景？）
  │     · 地域限制（全国/区域/城市？）
  │
  └── SOM（Serviceable Obtainable Market）— 可争取市场
        现实来看，我们能拿到多少？
        估算依据：
        · 竞争格局（市场份额分布、新进入者通常能拿多少）
        · 资源投入（我们打算投入多少人力/资金）
        · 差异化程度（我们的独特价值有多强）
        · 时间框架（1年/3年/5年的目标不同）
```

### 市场结构拆解维度

按需选择适用维度：
- **用户类型**：个人/企业/政府；大企业/中小企业
- **地域**：一线/二三线/下沉；国内/海外
- **价格带**：高端/中端/低端/免费
- **应用场景**：哪些使用场景，各占多大比例
- **产品形态**：SaaS/本地化部署/混合
- **价值链环节**：上游/中游/下游

### 关键原则

1. 不追求精确数字，追求数量级正确 → "约 50 亿"比"52.3 亿"更诚实
2. 标注数据来源和推算方法 → 让看的人知道数字怎么来的
3. 比起绝对数字，更关注趋势方向 → 增长还是萎缩？加速还是放缓？
4. 不同来源数据不一致时，取保守值或标注范围

## §2 竞争格局分析框架

### 三层竞品识别

```
竞争格局全景
│
├── 直接竞品（做同样的事，服务同样的人）
│   ├── 头部玩家：谁？市场份额？核心优势？
│   ├── 腰部玩家：谁？差异化策略？
│   └── 新进入者：最近谁入场？带来什么变化？
│
├── 间接竞品（不同方式解决同一个问题）
│   └── 用户可能用什么替代方案？
│
└── 潜在竞品（目前没做但有能力做的）
    └── 哪些巨头可能随时切入？
```

### 五维度分析

| 维度 | 分析要点 |
|------|----------|
| **产品/功能** | 谁做了什么，做到什么程度。功能覆盖度、深度、易用性 |
| **价格** | 各家怎么收费，价格带分布，性价比对比 |
| **渠道** | 各家怎么获客，依赖什么渠道，渠道成本 |
| **品牌/口碑** | 用户怎么评价各家，NPS、应用商店评分、社区口碑 |
| **技术/资源** | 各家的技术壁垒、团队规模、融资情况、资源储备 |

### 竞品信息采集矩阵

| 采集对象 | 采集渠道 | 获取什么 |
|---------|---------|---------|
| 竞品产品 | 官网、应用商店、试用体验、产品截图 | 功能范围、设计理念、技术路线 |
| 竞品口碑 | 应用商店评论、社交媒体、行业论坛、知乎 | 用户真实评价、优势与槽点 |
| 竞品商业 | 财报、融资新闻、招聘信息、合作公告 | 营收规模、战略方向、团队规模 |
| 竞品动态 | 新闻、发布会、版本更新日志 | 最新动向、未来规划 |
| 行业评测 | 第三方评测报告、行业媒体对比 | 客观对比视角 |

## §3 趋势与时机四层模型

```
趋势分析四层模型
│
├── 宏观趋势（3-5年）
│   ├── 政策方向：国家和地方政策走向
│   ├── 技术演进：关键技术（如 AI、大模型）成熟度
│   ├── 社会变化：人口结构、消费习惯、观念变迁
│   └── 经济环境：相关支出趋势、预算变化
│
├── 行业趋势（1-3年）
│   ├── 行业热点：当前业内最关注什么
│   ├── 模式创新：哪些新商业模式/产品形态出现
│   ├── 标杆事件：哪些事件在重塑行业认知
│   └── 资本流向：钱在往哪流（融资/并购/IPO）
│
├── 市场节奏（6-12个月）
│   ├── 采购周期：客户什么时候买
│   ├── 政策窗口：什么政策即将落地
│   └── 竞品节奏：竞品在什么阶段（发布期/稳定期/转型期）
│
└── 时机判断（决策依据）
    ├── 做早了的风险是什么？（市场未成熟、教育成本高）
    ├── 做晚了的代价是什么？（先发优势丧失、市场被占）
    └── 当前是进入/加速/观望/退出的时机？
```

### 时机判断输出格式

- 市场阶段：[早期 / 成长期 / 成熟期 / 衰退期]
- 进入时机：[偏早 / 合适 / 偏晚]
- 窗口期估计：[X 个月到 X 年]
- 关键时间节点：[政策落地日期、竞品发布节点等]

## §4 机会与风险评估框架

### 机会评估：四象限模型

```
              自身适配度
              高              低
市场   高   ★核心机会        值得争取
吸引力      全力投入          但要补短板
       低   防守型业务        放弃
            维持但不加码
```

**市场吸引力** = 规模 × 增速 × 利润率
**自身适配度** = 能力 × 资源 × 渠道

### 风险评估：7 类风险详表

| 风险类型 | 考察要素 | 评估标准 |
|---------|---------|---------|
| **市场风险** | 市场是否真实存在？需求是否被验证？ | 致命：市场不存在 / 重大：需求未充分验证 / 可控：细分需求待验证 |
| **竞争风险** | 巨头是否会进入？差异化是否可持续？ | 致命：巨头已在布局 / 重大：差异化易被模仿 / 可控：有一定壁垒 |
| **政策风险** | 是否依赖特定政策？政策变动可能性？ | 致命：政策可能禁止 / 重大：强依赖特定政策 / 可控：政策方向一致 |
| **技术风险** | 技术是否成熟？是否有技术壁垒？ | 致命：核心技术不可行 / 重大：技术成熟度不足 / 可控：技术可迭代 |
| **渠道风险** | 获客渠道是否畅通？渠道成本趋势？ | 致命：无有效渠道 / 重大：渠道成本高且不可控 / 可控：渠道可建设 |
| **执行风险** | 团队是否有能力交付？资源是否足够？ | 致命：核心能力缺失 / 重大：需要大量补充 / 可控：可逐步建设 |
| **时机风险** | 是否太早或太晚？窗口期有多长？ | 致命：窗口已关闭 / 重大：窗口极窄 / 可控：窗口充足 |

## §5 差异化定位三步法

### Step 1：绘制竞争地图

1. 从五维度分析中选择 **2 个最重要的竞争维度** 作为坐标轴
   - 选择标准：用户决策时最看重的维度
   - 常见组合：功能深度 × 价格、专业性 × 易用性、全面性 × 垂直度
2. 把所有玩家（直接 + 间接竞品）标注在地图上
3. 找到 **空白区域**（没有或很少玩家占据的位置）

### Step 2：评估空白区域价值

对每个空白区域回答三个问题：
- **有需求吗？** → 联动用户研究专家验证。没有用户需求的空白是假空白
- **有能力吗？** → 我们的团队/技术/资源能支撑吗？
- **可持续吗？** → 竞品是否容易模仿？是否有壁垒保护？

三个都是"是" → 目标定位区域

### Step 3：输出定位建议

- **一句话定位**：我们是___，为___提供___，不同于___
- **核心差异点**：1-3 个关键差异（越少越聚焦）
- **差异化支撑**：为什么我们能做到（能力、资源、技术、洞察）
- **定位风险**：这个定位的潜在挑战

## §6 商业可行性检验

### 商业模式草图 7 要素

| 要素 | 检验问题 |
|------|----------|
| **价值主张** | 提供什么独特价值？用户为什么选我们不选竞品？ |
| **客户群体** | 卖给谁？与用户画像对齐 |
| **收入模式** | 怎么收钱？订阅/买断/免费增值/项目制/广告？ |
| **成本结构** | 主要成本是什么？粗估量级（研发/运营/获客/内容） |
| **渠道路径** | 怎么触达客户？直销/代理/线上/线下？ |
| **竞争壁垒** | 凭什么别人做不了/做不好？技术/数据/品牌/网络效应？ |
| **单位经济** | 算得过账吗？（见下方） |

### 单位经济初判

```
获客成本（CAC）≈ [渠道成本 + 销售成本] / 新增客户数
  → 大概在什么范围？百元级/千元级/万元级？

客户终身价值（LTV）≈ 客单价 × 购买频次 × 客户生命周期
  → 大概在什么范围？

关键判断：
  · LTV > 3×CAC → 健康
  · LTV > CAC 但 < 3×CAC → 可行但紧张
  · LTV < CAC → 商业模式有问题

⚠️ 这不是精确的财务模型，是量级层面的合理性检验。
用于判断"大方向算不算得过账"，不用于精确的投资决策。
```

### 商业可行性红旗信号

出现以下任一信号，需在报告中重点标注：
- 找不到清晰的收费模式
- CAC 可能远高于 LTV
- 收入依赖单一大客户或政府项目
- 竞品大量免费提供类似服务
- 市场增长完全依赖政策补贴
