# 与用户研究专家的联合工作模式

## §1 协作定位

市场评估专家和用户研究专家是 Phase 0 的两个并行专家，服务于同一目标：在项目正式需求分析前，提供完整的外部环境判断。

```
用户研究专家                    市场评估专家
───────────────                ──────────────
关注的是"人"                    关注的是"局"
这些用户是谁？                  这个市场有多大？
他们有什么痛点？                谁已经在做了？
他们需要什么？                  我们做有什么优势？
他们会怎么使用？                这个时机合不合适？

→ 微观视角，从用户出发           → 宏观视角，从市场出发
→ 回答"需不需要"                → 回答"做不做得成"
```

**两者结合才能做出完整判断**：用户研究告诉你"用户确实需要这个"，市场评估告诉你"但这个市场已经被巨头占满了 / 时机还没到 / 我们没有渠道优势"。

---

## §2 并行工作 + 交叉校验

```
项目启动
│
├── 用户研究专家                 ├── 市场评估专家
│   │                          │   │
│   ├ 构建用户画像               │   ├ 分析市场规模
│   ├ 识别用户痛点               │   ├ 分析竞争格局
│   ├ 绘制用户旅程               │   ├ 判断趋势时机
│   └ 推演用户需求               │   └ 评估机会风险
│                               │
│←────────── 交叉校验 ──────────→│
│                               │
│  · 用户画像与市场分层是否一致？  │
│  · 用户需求与市场机会是否匹配？  │
│  · 痛点排序与竞品空白是否对应？  │
│                               │
└────────── 联合输出 ──────────→ Phase 1：需求分析
```

## §3 双向信息传递

### 用户研究专家 → 市场评估专家

| 传递内容 | 对市场评估的价值 |
|---------|----------------|
| 用户真实需求 | 验证市场机会是否有真实需求支撑 |
| 用户痛点深度 | 评估差异化方向是否命中真实痛点 |
| 用户付费意愿线索 | 输入到商业可行性初判 |
| 用户群体分布 | 校准 SAM/SOM 中的用户基数 |

### 市场评估专家 → 用户研究专家

| 传递内容 | 对用户研究的价值 |
|---------|----------------|
| 市场规模数据 | 为用户群体规模提供宏观参照 |
| 竞品如何满足同类需求 | 理解用户的替代方案和迁移成本 |
| 行业趋势对用户的影响 | 预判用户需求的变化方向 |
| 竞品用户口碑数据 | 补充竞品视角的用户声音 |

---

## §4 典型协作场景

| 场景 | 触发方 | 协作动作 |
|------|--------|----------|
| 用户研究发现用户有强烈需求 | 用户研究 → | 市场评估检验这个需求在市场上是否已被满足 |
| 市场评估发现一个空白机会 | ← 市场评估 | 用户研究验证用户是否真的需要这个 |
| 用户研究提供用户群体画像 | 用户研究 → | 市场评估评估这个群体的市场规模和商业价值 |
| 市场评估发现竞品口碑两极分化 | ← 市场评估 | 用户研究深入分析差评背后的真实痛点 |
| 差异化定位需要用户验证 | ← 市场评估 | 用户研究验证目标用户是否认可定位方向 |

---

## §5 交叉校验清单

联合输出前，双方核对以下一致性：

- [ ] 用户画像中的用户分群 与 市场结构拆解 是否对齐？
- [ ] 用户需求优先级 与 市场机会评估 是否一致？
- [ ] 用户痛点排序 与 竞品空白区域 是否对应？
- [ ] 用户付费意愿判断 与 商业可行性初判 是否匹配？
- [ ] 存在矛盾的地方是否已标注并说明原因？

---

## §6 联合输出物模板：项目背景研究综合报告

```markdown
# [项目名称] 背景研究综合报告

## Part A：用户洞察（来自用户研究专家）
[用户画像 + 需求洞察精华版]
- 核心用户群画像（精简版）
- Top 3 用户痛点
- 关键需求列表（按优先级排序）
- 用户付费意愿判断

## Part B：市场评估（来自市场评估专家）
[市场评估报告精华版]
- 市场规模（TAM/SAM/SOM）
- 竞争格局概览
- 时机判断
- 关键风险

## Part C：交叉验证结论
- 用户需求 × 市场机会 的匹配度：[高/中/低]
- 一致的地方：[需求和机会吻合的点]
- 矛盾或存疑的地方：[需求存在但市场不支持 / 市场有空白但用户无需求]

## Part D：综合建议
- 这个项目是否应该做：[明确判断 + 理由]
- 如果做，关键成功因素是什么：
- 如果做，最大的风险是什么：
- 给需求分析阶段的核心输入：
  · 必须满足的用户需求
  · 必须考虑的市场约束
  · 建议的差异化方向
```
