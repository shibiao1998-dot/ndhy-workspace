# 技术调研 — 5步标准工作流

## 总览

```
Step 1: 明确调研目标
  |  收到任务 → 理解"帮谁做什么技术决策"
  |  -> 明确: 调研类型(选型/可行性/方案/竞品)
  |  -> 明确: 决策者是谁、决策标准是什么
  |  -> 明确: 约束条件(技术栈/预算/时间/团队能力)
  |  -> 判断深度级别(L1/L2/L3)
  |
Step 2: 广域扫描候选方案
  |  -> 快速扫描所有可能的候选方案
  |  -> 信息源分级采集(详见 source-validation.md)
  |  -> 初步筛选: 排除明显不符合约束的方案
  |  -> 产出: 候选方案清单(3-5个)
  |
Step 3: 深度评估与对比
  |  -> 确定评估维度(详见 evaluation-framework.md)
  |  -> 逐方案深度调研: 技术成熟度/社区活跃度/学习成本/性能/安全
  |  -> 填充对比矩阵: 维度 × 方案 × 评分
  |  -> 量化数据采集: benchmark/Star数/Issue响应/发版频率
  |
Step 4: 形成结论与推荐
  |  -> 综合评分排序
  |  -> 推荐方案 + 推荐理由(3条以上)
  |  -> 替代方案 + 何时该选替代方案
  |  -> 风险清单 + 缓解措施
  |  -> 局限性声明
  |
Step 5: 输出调研报告
     -> 按模板组织(详见 templates.md)
     -> 过质量检查清单(详见 checklists.md)
     -> 标注所有信息来源和获取日期
```

## 4大调研类型

### 1. 技术选型评估

**目标**: 在多个候选技术方案中选出最适合当前场景的方案

**关键动作**:
- 明确选型维度（功能满足度、性能、学习成本、社区生态、维护成本、许可证）
- 构建对比矩阵，每个维度打分（1-5分）并加权
- 做原型验证（L3深度时）
- 输出推荐 + 理由 + 风险

**典型场景**: 前端框架选型、数据库选型、消息队列选型、云服务选型

### 2. 可行性分析

**目标**: 判断某个技术方案是否可行，评估成本和风险

**关键动作**:
- 技术可行性: 能不能做？有没有先例？技术壁垒在哪？
- 成本评估: 开发人力、基础设施、第三方服务费用
- 风险识别: 技术风险、依赖风险、维护风险
- 时间评估: 从调研到落地的时间线

**典型场景**: 新技术引入评估、架构迁移可行性、第三方服务接入评估

### 3. 技术方案调研

**目标**: 深度调研某项技术的实现路径、最佳实践、生态状况

**关键动作**:
- 官方文档精读（架构设计、核心概念、API）
- 社区生态评估（插件生态、第三方集成、社区活跃度）
- 最佳实践收集（官方推荐、社区共识、踩坑记录）
- 实现路径设计（从当前到目标的步骤）

**典型场景**: 深度理解某个框架/工具、技术预研

### 4. 竞品技术分析

**目标**: 分析竞品的技术栈、架构方案，为自身决策提供参考

**关键动作**:
- 技术栈识别（前端框架、后端语言、数据库、基础设施）
- 架构推断（单体/微服务、部署方式、扩展策略）
- 性能表现评估（响应速度、可用性、并发能力）
- 技术团队分析（招聘信息、技术博客、开源贡献）

**典型场景**: 了解竞品技术实力、学习竞品架构经验

## 工作模式决策树

```
收到技术调研任务
  |- 单技术点确认 / 快速可行性判断
  |   -> L1 快速扫描，单独完成
  |
  |- 模块级选型 / 方案对比（明确边界）
  |   -> L2 标准调研，作为 subagent 执行
  |
  +- 新产品技术栈 / 大规模架构决策
      -> L3 深度研究
      |- 涉及架构决策 → 产出交技术架构专家决策
      |- 涉及产品方向 → 产出交产品定义专家参考
      +- 涉及成本预算 → 产出交 Leader 审批
```

## 协作接口

### 上游输入

| 来源 | 接收内容 | 使用方式 |
|------|---------|---------|
| Leader / 项目管理 | 调研目标、约束条件、决策标准 | 明确调研范围和深度 |
| 技术架构专家 | 架构约束、技术栈限制 | 在约束内评估候选方案 |
| 产品定义专家 | 功能需求、场景描述 | 评估技术方案与需求适配度 |

### 下游输出

| 接收方 | 传递内容 | 约束关系 |
|--------|---------|---------|
| 技术架构专家 | 选型报告、对比矩阵 | 作为架构决策的输入 |
| 后端/前端开发专家 | 技术方案、最佳实践 | 作为实现参考 |
| Leader | 可行性结论、成本评估 | 作为决策依据 |
