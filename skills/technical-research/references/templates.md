# 技术调研 — 交付模板

## 模板 1：技术选型报告

```markdown
# 技术选型报告：[选型主题]

> 调研日期：YYYY-MM-DD | 深度级别：L1/L2/L3 | 决策者：[谁]

## 1. 背景与目标

- **决策问题**: 为[什么场景]选择[什么类型的技术方案]
- **约束条件**: [技术栈/预算/时间/团队能力]
- **决策标准**: [列出关键决策标准及其优先级]

## 2. 候选方案总览

| 方案 | 简述 | 初筛结果 |
|------|------|---------|
| A | ... | ✅ 进入深度评估 |
| B | ... | ✅ 进入深度评估 |
| C | ... | ❌ 排除（原因：...） |

## 3. 深度评估 — 对比矩阵

| 评估维度 | 权重 | 方案A | 方案B |
|---------|------|-------|-------|
| ... | ...% | x/5 | x/5 |
| **加权总分** | 100% | **x.xx** | **x.xx** |

## 4. 各方案详细分析

### 方案A：[名称]
- **核心优势**: ...
- **主要劣势**: ...
- **适用场景**: ...
- **关键数据**: [benchmark/社区指标/案例]

### 方案B：[名称]
（同上结构）

## 5. 推荐结论

- **推荐方案**: [方案X]
- **推荐理由**:
  1. ...
  2. ...
  3. ...
- **替代方案**: [方案Y]，当[什么条件下]应选替代方案
- **风险提示**: ...

## 6. 参考来源

| # | 来源 | 类型 | 获取日期 |
|---|------|------|---------|
| 1 | [URL] | Tier 1 | YYYY-MM-DD |
| 2 | [URL] | Tier 2 | YYYY-MM-DD |

## 7. 局限性声明

- 本次调研覆盖范围: ...
- 未覆盖/未充分验证: ...
```

## 模板 2：可行性评估报告

```markdown
# 可行性评估报告：[评估主题]

> 调研日期：YYYY-MM-DD | 深度级别：L1/L2/L3

## 1. 评估目标

- **问题**: [要解决什么问题]
- **拟采用方案**: [计划评估的技术方案]
- **成功标准**: [什么条件算可行]

## 2. 技术可行性

| 维度 | 评估 | 说明 |
|------|------|------|
| 技术壁垒 | 高/中/低 | ... |
| 先例存在性 | 有/无 | ... |
| 依赖成熟度 | 成熟/发展中/早期 | ... |
| 团队能力匹配 | 匹配/需学习/缺口大 | ... |

## 3. 成本评估

| 成本项 | 一次性 | 持续性(月) | 说明 |
|--------|--------|-----------|------|
| 开发人力 | x人天 | - | ... |
| 基础设施 | ¥xxx | ¥xxx/月 | ... |
| 第三方服务 | - | ¥xxx/月 | ... |
| **合计** | **¥xxx** | **¥xxx/月** | |

## 4. 风险清单

| # | 风险 | 等级 | 缓解措施 |
|---|------|------|---------|
| 1 | ... | 高/中/低 | ... |

## 5. 结论

- **可行性判断**: ✅ 可行 / ⚠️ 有条件可行 / ❌ 不可行
- **关键条件**: [如果有条件可行，列出条件]
- **建议下一步**: ...

## 6. 参考来源 & 局限性

（同选型报告结构）
```
