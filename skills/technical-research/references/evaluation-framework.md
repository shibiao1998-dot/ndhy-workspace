# 技术调研 — 对比矩阵与评估框架

## 对比矩阵模板

### 基础矩阵结构

| 评估维度 | 权重 | 方案A | 方案B | 方案C |
|---------|------|-------|-------|-------|
| 功能满足度 | 30% | 4/5 | 5/5 | 3/5 |
| 性能表现 | 20% | 3/5 | 4/5 | 5/5 |
| 学习成本 | 15% | 5/5 | 3/5 | 2/5 |
| 社区生态 | 15% | 4/5 | 5/5 | 3/5 |
| 维护成本 | 10% | 4/5 | 3/5 | 4/5 |
| 许可证友好度 | 10% | 5/5 | 5/5 | 4/5 |
| **加权总分** | 100% | **4.05** | **4.25** | **3.35** |

### 评分标准

| 分数 | 含义 |
|------|------|
| 5 | 优秀，完全满足甚至超出预期 |
| 4 | 良好，基本满足需求，有小不足 |
| 3 | 一般，能用但有明显局限 |
| 2 | 较差，需要较多补充工作 |
| 1 | 不可用，存在根本性缺陷 |

## 评估维度库

根据调研类型选择合适的维度组合：

### 技术选型通用维度

| 维度 | 评估要点 | 量化指标 |
|------|---------|---------|
| **功能满足度** | 核心功能覆盖率、扩展能力 | 需求覆盖率(%) |
| **性能表现** | 响应时间、吞吐量、资源占用 | benchmark数据 |
| **学习成本** | 文档质量、学习曲线、示例丰富度 | 上手时间估算 |
| **社区生态** | Star数、贡献者、插件/集成数量 | GitHub指标 |
| **维护成本** | 发版频率、Issue响应时间、破坏性变更频率 | 近6月数据 |
| **成熟度** | 版本号、生产案例、已知问题数 | 生产部署案例数 |
| **许可证** | 开源协议、商用限制 | MIT/Apache/GPL等 |
| **安全性** | CVE记录、安全更新响应速度 | 近1年CVE数量 |

### 可行性分析维度

| 维度 | 评估要点 | 量化指标 |
|------|---------|---------|
| **技术可行性** | 有无先例、技术壁垒、依赖成熟度 | 先例数量、壁垒等级 |
| **开发成本** | 人力投入、时间周期 | 人天估算 |
| **基础设施成本** | 服务器、第三方服务、许可费 | 月度/年度费用 |
| **风险等级** | 技术风险、依赖风险、人员风险 | 高/中/低 |
| **ROI预估** | 投入产出比、回收周期 | 预计回收月数 |

### 竞品技术分析维度

| 维度 | 评估要点 | 信息来源 |
|------|---------|---------|
| **技术栈** | 前后端框架、数据库、中间件 | 招聘信息、Wappalyzer、开源仓库 |
| **架构模式** | 单体/微服务、同步/异步 | 技术博客、公开分享 |
| **部署方式** | 云服务商、容器化、CDN | DNS分析、HTTP头 |
| **性能表现** | 页面加载速度、API响应时间 | Lighthouse、实际测试 |
| **团队能力** | 团队规模、技术水平 | 招聘信息、技术博客 |

## 权重分配指南

| 场景 | 高权重维度 | 原因 |
|------|-----------|------|
| 初创产品/快速验证 | 学习成本、开发效率 | 速度优先 |
| 高并发系统 | 性能、可扩展性 | 性能瓶颈致命 |
| 长期产品 | 社区生态、维护成本、成熟度 | 长期可维护性 |
| 安全敏感场景 | 安全性、成熟度 | 安全无妥协 |
| 成本敏感场景 | 许可证、基础设施成本 | 预算约束 |

## 决策辅助规则

1. **加权总分差距 < 0.5**: 方案接近，需要进一步深挖关键差异维度或做原型验证
2. **某维度评分 ≤ 2**: 标记为红旗，即使总分高也需特别说明风险
3. **最高分方案和次高分方案**: 都要给出推荐理由和适用条件
4. **权重分配必须在调研开始前确认**: 不允许看到结果后调权重
