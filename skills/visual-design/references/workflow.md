# 视觉设计 — 标准工作流程

## 8 步标准流程

```
Step 1：理解输入
  │  消化体验设计文档，确认视觉设计范围
  │  → 提取：信息架构、交互模式、组件清单、体验语言、响应式策略
  │  → 确认品牌现状：有无现有品牌规范？是延伸还是从零开始？
  │  → 确认技术约束：目标平台、前端框架、暗色模式需求
  │  → 信息不足时，有权要求上游补充
  │
Step 2：色彩体系设计
  │  → 定义品牌色（主色 + 品牌辅助色）
  │  → 定义语义色（成功/警告/错误/信息）
  │  → 定义中性色阶梯（灰阶 50~900）
  │  → 定义色彩比例规范（60-30-10 或自定义）
  │  → 暗色模式色彩映射（如需要）
  │  → 产出【色彩 Token 表 + 使用场景对照】
  │  → 详细方法 → color-system.md
  │
Step 3：字体排印规范
  │  → 选择字体家族（中文 + 英文 + 等宽）
  │  → 定义字体层级（Display/H1~H4/Body/Caption/Overline）
  │  → 定义行高、字距、段间距规则
  │  → 中英文混排处理规则
  │  → 产出【字体 Token 表 + 排印规范】
  │  → 详细方法 → typography.md
  │
Step 4：空间系统与网格
  │  → 定义基础间距单元（Base Unit，通常 4px 或 8px）
  │  → 构建间距阶梯（xs/sm/md/lg/xl/2xl...）
  │  → 定义网格系统（栏数、槽宽、边距）
  │  → 定义组件内部间距规则
  │  → 产出【间距 Token 表 + 网格规范】
  │  → 详细方法 → spacing-grid.md
  │
Step 5：UI 组件视觉规范
  │  → 逐组件定义视觉标准（按钮/卡片/表单/导航/弹窗/表格...）
  │  → 每个组件：尺寸变体 + 状态变体 + 色彩应用 + 圆角 + 阴影
  │  → 组件间视觉关系（层级、亲密性、对比）
  │  → 产出【组件视觉规范表 + 状态矩阵】
  │  → 详细方法 → component-visuals.md
  │
Step 6：图标、插画与动效
  │  → 图标风格定义（线性/面性/双色、网格、笔触、圆角）
  │  → 插画风格定义（如适用：风格、色彩、人物比例）
  │  → 动效语言定义（过渡类型、缓动函数、时长阶梯）
  │  → 产出【图标规范 + 动效 Token 表】
  │  → 详细方法 → icon-illustration-motion.md
  │
Step 7：设计 Token 输出
  │  → 汇总所有视觉决策为结构化 Token
  │  → 分层组织：Global Token → Alias Token → Component Token
  │  → 输出格式：JSON / CSS Custom Properties / Tailwind Config
  │  → 暗色模式 Token 映射
  │  → 产出【完整 Token 文件 + Token 文档】
  │  → 详细方法 → design-tokens.md
  │
Step 8：响应式视觉适配与质量检查
     → 定义断点策略和视觉适配规则
     → 字体/间距/网格在各断点的变化规则
     → 视觉一致性走查（跨页面/跨组件/跨状态）
     → 可达性检查（对比度/触控/非色彩依赖）
     → Token 完整性检查（无遗漏/无冗余/命名一致）
     → 走查清单 → checklists.md
```

## 设计深度速查

| 深度 | 适用场景 | 覆盖范围 | 输出物 |
|------|----------|----------|--------|
| **L1 快速设计** | 已有视觉体系，补充组件或微调 | 复用现有 Token + 组件视觉 | 增量组件规范 + Token 补充 |
| **L2 标准设计** | 新模块或品牌局部刷新 | 完整 8 步流程 | 视觉规范文档（标准版） |
| **L3 深度设计** | 全新品牌视觉体系从 0 到 1 | 完整流程 + Token 全套 + 暗色模式 + 响应式全覆盖 | 视觉规范文档（完整版）+ Token 文件 |

判断不清时向上确认。

## 触发时首要动作

无论如何触发，第一步永远是：

> "体验设计告诉了我们什么？组件清单是什么、交互模式是什么、有无现有品牌规范、技术约束在哪？"

→ 判断设计深度 → 消化体验设计输入 → 按 8 步流程执行
