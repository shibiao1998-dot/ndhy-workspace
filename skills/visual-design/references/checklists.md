## 视觉一致性检查清单

### 色彩一致性

| # | 检查项 | ✅ 标准 |
|---|--------|--------|
| 1 | 所有颜色来自 Token | 零硬编码色值 |
| 2 | 品牌色使用场景正确 | 主色仅用于核心交互和强调 |
| 3 | 语义色使用正确 | 成功/警告/错误/信息不混用 |
| 4 | 同级元素色彩一致 | 同样功能在不同页面同样颜色 |
| 5 | 色彩比例合理 | 大面积中性色 + 少量品牌色强调 |
| 6 | 暗色模式映射完整 | 每个亮色 Token 都有暗色对应 |

### 字体一致性

| # | 检查项 | ✅ 标准 |
|---|--------|--------|
| 1 | 字体层级严格遵循 | 每个文字元素都对应一个字体 Token |
| 2 | 无字体层级外的字号 | 不出现 15px、17px 等非标字号 |
| 3 | 字重使用规范 | 只用 Regular/Medium/Semibold/Bold |
| 4 | 行高一致 | 同层级文字行高一致 |
| 5 | 中英文混排规则已定义 | 间距、标点、数字字体有明确规则 |

### 间距一致性

| # | 检查项 | ✅ 标准 |
|---|--------|--------|
| 1 | 所有间距来自 Token | 零魔法数字 |
| 2 | 同类场景同间距 | 卡片 padding 全局统一 |
| 3 | 亲密性原则符合 | 相关元素间距 < 不相关元素间距 |
| 4 | 层级感清晰 | 间距阶梯从组件内→组件间→区块间递增 |

### 组件一致性

| # | 检查项 | ✅ 标准 |
|---|--------|--------|
| 1 | 同类组件外观统一 | 所有按钮同圆角同阴影 |
| 2 | 状态变体完整 | 每个交互组件 6 态（default~loading）全定义 |
| 3 | 尺寸变体规律 | Large→Medium→Small 有规律递减 |
| 4 | 圆角/阴影全局统一 | 使用 radius-*/shadow-* Token，不自定义 |

## 可达性检查清单

| # | 检查项 | ✅ 标准 | 检测方式 |
|---|--------|--------|---------|
| 1 | **正文对比度** | ≥ 4.5:1 (WCAG AA) | 对比度计算工具 |
| 2 | **大字对比度** | ≥ 3:1（≥18px Bold 或 ≥24px） | 同上 |
| 3 | **UI 组件对比度** | 边框/图标与背景 ≥ 3:1 | 同上 |
| 4 | **非色彩依赖** | 颜色 + 图标/文字双重编码 | 色盲模拟 |
| 5 | **触控目标** | 可点击区域 ≥ 44×44px | 测量 |
| 6 | **触控间距** | 可点击元素间距 ≥ 8px | 测量 |
| 7 | **Focus 可见** | 所有可交互元素有清晰 focus ring | Tab 键测试 |
| 8 | **文字可缩放** | 使用相对单位(rem)，支持 200% 放大 | 浏览器放大测试 |
| 9 | **动效可关闭** | 尊重 prefers-reduced-motion | 系统设置测试 |

## Token 完整性检查清单

| # | 检查项 | ✅ 标准 |
|---|--------|--------|
| 1 | 颜色 Token 完整 | 品牌色阶 + 语义色 + 中性色阶 + Alias 全覆盖 |
| 2 | 间距 Token 完整 | 从 space-0 到 space-20 无缺失 |
| 3 | 字体 Token 完整 | family + size + weight + line-height + letter-spacing |
| 4 | 形状 Token 完整 | radius + shadow + border-width |
| 5 | 动效 Token 完整 | easing + duration |
| 6 | 三层架构清晰 | Global → Alias → Component 无越级引用 |
| 7 | 暗色模式 Token | Alias 层有暗色映射 |
| 8 | 命名一致 | kebab-case，语义化，无原始值 |
| 9 | 无冗余 Token | 无同值不同名的重复 Token |
| 10 | 输出格式就绪 | JSON / CSS / Tailwind 至少一种可用 |

## 组件视觉质量检查

每个组件交付前过此检查：

| # | 检查项 | ✅ 标准 |
|---|--------|--------|
| 1 | 尺寸变体齐全 | L/M/S/Mini 至少有 M/S 两种 |
| 2 | 状态变体齐全 | Default/Hover/Active/Focus/Disabled 全定义 |
| 3 | 色彩变体齐全 | Primary/Secondary/Danger 至少三种 |
| 4 | 所有数值来自 Token | 圆角/阴影/颜色/间距/字号无硬编码 |
| 5 | 对齐方式正确 | 垂直居中、水平对齐规则明确 |
| 6 | 可达性达标 | 对比度、触控目标、focus态 |
| 7 | 响应式已考虑 | 小屏下的表现已定义 |

## 8 个视觉质量问题防范

| # | 问题 | 表现 | ❌ 错误 | ✅ 正确 | 防范 |
|---|------|------|--------|--------|------|
| 1 | **魔法数字** | 间距/字号/颜色用硬编码值 | padding: 13px; color: #567890 | padding: var(--space-3); color: var(--color-text-primary) | 全局搜索非 Token 值 |
| 2 | **色彩污染** | 使用色板外颜色 | 设计师随手取色 | 所有颜色必须从 Token 系统中选取 | 审查所有色值是否在 Token 表中 |
| 3 | **层级混乱** | 视觉重量分配不当 | 次要按钮比主要按钮更醒目 | 主>次>辅，色彩/大小递减 | 眯眼测试（模糊看页面） |
| 4 | **圆角不一致** | 同类组件圆角不同 | 按钮 6px，卡片 10px，弹窗 8px | 统一使用 radius-md Token | 审查所有圆角值 |
| 5 | **阴影滥用** | 到处加阴影缺少层级 | 每个元素都有阴影 | 阴影仅用于浮层，≤3 级 | 数页面上的阴影层级数 |
| 6 | **对比度不足** | 文字在背景上看不清 | 浅灰文字 + 白色背景 | 正文 ≥ 4.5:1，大字 ≥ 3:1 | 对比度工具逐项检查 |
| 7 | **动效过度** | 满屏动画干扰使用 | 每个元素都有入场动画 | 动效服务功能，≤ 300ms | prefers-reduced-motion 测试 |
| 8 | **Token 越级** | Component 直接引用 Global | button-bg: {color-blue-500} | button-bg: {color-interactive-primary} | 检查 Token 引用链 |

### 质量问题速查

| 信号 | 可能的问题 | 检查行动 |
|------|-----------|---------|
| "这个颜色和那边不一样" | 色彩污染/不一致 | 审查色值是否来自 Token |
| "这个间距感觉不对" | 魔法数字 | 审查间距是否来自 Token |
| "看不清文字" | 对比度不足 | 对比度工具检测 |
| "太花了" | 阴影滥用/动效过度 | 减少装饰，回归功能 |
| "暗色模式下出问题" | Token 映射不完整 | 检查 Alias 层暗色覆盖 |
| "手机上点不到" | 触控目标过小 | 检查点击区域 ≥ 44px |
