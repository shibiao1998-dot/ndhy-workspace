# 视觉设计 — 空间系统与网格

## 基础间距单元（Base Unit）

**推荐：4px 作为 Base Unit。**

所有间距必须是 Base Unit 的整数倍。禁止出现 3px、5px、7px 等非倍数值。

### 间距阶梯

| Token | 值 | 用途示例 |
|-------|----|---------|
| **space-0** | 0px | 无间距 |
| **space-1** | 4px | 图标与文字间距、紧凑组内间距 |
| **space-2** | 8px | 组件内部紧凑间距、相关元素间距 |
| **space-3** | 12px | 表单字段间距、列表项间距 |
| **space-4** | 16px | 标准组件内部间距、卡片 padding |
| **space-5** | 20px | 区块间距 |
| **space-6** | 24px | 大区块间距、section padding |
| **space-8** | 32px | 页面级区块间距 |
| **space-10** | 40px | 大段落间距 |
| **space-12** | 48px | 页面顶部/底部间距 |
| **space-16** | 64px | 页面级大留白 |
| **space-20** | 80px | 营销页段落间距 |

### 间距使用原则

| 原则 | 说明 |
|------|------|
| **亲密性** | 相关元素间距小，不相关元素间距大 |
| **层级感** | 同级元素间距一致，不同级间距递增 |
| **呼吸感** | 越重要的内容周围留白越多 |
| **一致性** | 同场景同间距，全产品统一 |

### 组件内部间距规则

| 组件类型 | padding | 子元素 gap |
|---------|---------|-----------|
| **按钮** | 水平 space-4(16px)，垂直 space-2(8px) | 图标与文字 space-2(8px) |
| **卡片** | space-4(16px) ~ space-6(24px) | 内容区 space-3(12px) |
| **表单字段** | 输入框 space-3(12px) | 字段间 space-4(16px)~space-5(20px) |
| **对话框** | space-6(24px) | 内容区 space-4(16px) |
| **列表项** | 垂直 space-3(12px)，水平 space-4(16px) | — |

## 网格系统

### 响应式网格定义

| 断点 | 屏幕宽度 | 栏数 | 槽宽(Gutter) | 边距(Margin) | 内容最大宽 |
|------|---------|------|-------------|-------------|-----------|
| **Mobile** | < 768px | 4 栏 | 16px | 16px | 100% |
| **Tablet** | 768~1023px | 8 栏 | 24px | 24px | 100% |
| **Desktop** | 1024~1439px | 12 栏 | 24px | 32px~auto | 1200px |
| **Wide** | ≥ 1440px | 12 栏 | 32px | auto | 1280~1440px |

### 网格使用规则

1. **内容居中**：Desktop 及以上，内容区域居中，两侧 auto margin
2. **栏数弹性**：组件宽度用栏数表达（如"占 4 栏"），不用固定像素
3. **打破网格**：全宽背景色/图片可以打破网格，但内容仍需对齐
4. **嵌套网格**：组件内部可用更细的子网格，但间距仍用 Token

## 视觉重量分配

### 层级构建手段

| 手段 | 权重 | 使用方式 |
|------|------|---------|
| **字号差异** | ★★★★★ | 最有效的层级手段，标题 vs 正文 vs 辅助文字 |
| **字重差异** | ★★★★ | Bold/Semibold 用于标题，Regular 用于正文 |
| **色彩对比** | ★★★★ | 深色=重要，浅色=次要，品牌色=交互/强调 |
| **空间分配** | ★★★ | 更多留白=更重要（呼吸空间传达重要性） |
| **尺寸差异** | ★★★ | 大元素更引人注目 |
| **位置** | ★★ | 左上最先被看到（F型/Z型阅读模式） |
| **装饰** | ★ | 阴影/边框/背景色区分层级，谨慎使用 |

### 圆角体系

| Token | 值 | 用途 |
|-------|----|------|
| **radius-none** | 0px | 分割线、全宽元素 |
| **radius-sm** | 4px | 小组件（Tag、Badge） |
| **radius-md** | 8px | 标准组件（Button、Input、Card） |
| **radius-lg** | 12px | 大组件（Dialog、大卡片） |
| **radius-xl** | 16px | 特大组件（Sheet、抽屉） |
| **radius-full** | 9999px | 圆形（Avatar、Pill Badge） |

### 阴影体系

| Token | 值 | 用途 |
|-------|----|------|
| **shadow-xs** | 0 1px 2px rgba(0,0,0,0.05) | 微弱浮起（按钮、输入框） |
| **shadow-sm** | 0 2px 4px rgba(0,0,0,0.06), 0 1px 2px rgba(0,0,0,0.04) | 卡片默认 |
| **shadow-md** | 0 4px 8px rgba(0,0,0,0.08), 0 2px 4px rgba(0,0,0,0.04) | 下拉菜单、悬停卡片 |
| **shadow-lg** | 0 8px 16px rgba(0,0,0,0.10), 0 4px 8px rgba(0,0,0,0.04) | 弹窗、对话框 |
| **shadow-xl** | 0 16px 32px rgba(0,0,0,0.12), 0 8px 16px rgba(0,0,0,0.06) | 模态框、Toast |

**阴影原则**：浮层越高阴影越大。同一页面阴影层级不超过 3 级。

## 响应式视觉适配

### 断点策略

```
Mobile First: 从最小屏幕开始设计，逐步增强。

@media (min-width: 768px)  { /* Tablet */  }
@media (min-width: 1024px) { /* Desktop */ }
@media (min-width: 1440px) { /* Wide */    }
```

### 跨断点适配规则

| 元素 | Mobile → Desktop 变化 |
|------|----------------------|
| **字号** | Body 保持 14px；H1 从 24px → 32px；Display 从 32px → 48px |
| **间距** | 按比例缩小，但不低于 space-2(8px) |
| **网格** | 4栏 → 8栏 → 12栏 |
| **卡片排列** | 1列 → 2列 → 3~4列 |
| **导航** | 底部Tab/汉堡菜单 → 侧边栏/顶部导航 |
| **表格** | 转为卡片列表或横向滚动 |

### 响应式 Token

```
// 间距随断点变化（使用 CSS clamp 或媒体查询）
space-page-x: clamp(16px, 4vw, 32px)
space-section-y: clamp(32px, 6vw, 64px)

// 字号随断点变化
font-size-h1: clamp(24px, 3vw, 32px)
font-size-display: clamp(32px, 5vw, 48px)
```
