# 教学设计原则

> 聚焦：教学设计框架在教育产品学习流程设计中的直接应用。

## 1. ADDIE 模型

**五阶段及产品设计对应**：

| 阶段 | 核心任务 | 产品设计对应 |
|------|---------|-------------|
| Analysis（分析） | 学习者特征、需求、环境 | 用户研究、学情分析功能 |
| Design（设计） | 目标、策略、评估、序列 | 学习路径设计、内容架构 |
| Development（开发） | 制作学习材料 | 内容生产、资源制作 |
| Implementation（实施） | 交付、辅导 | 功能发布、教师培训 |
| Evaluation（评估） | 效果评估、迭代改进 | 数据分析、A/B 测试 |

**产品设计中的常见误区**：
- ❌ 跳过 Analysis，直接做功能
- ❌ Evaluation 只看使用数据，不评估学习效果
- ✅ 分析阶段必须回答：**谁在学？学什么？在什么条件下学？**

## 2. 逆向设计（UbD / Understanding by Design）

**三阶段（从终点倒推）**：

```
Stage 1: 确定预期结果 → Stage 2: 确定评估证据 → Stage 3: 设计学习体验
   我要学生理解什么？      怎么证明他们理解了？       什么活动能达成理解？
```

**六个理解维度**：

| 维度 | 说明 | 产品中的评估方式 |
|------|------|-----------------|
| 解释 | 能用自己的话说明 | 开放式简答 |
| 诠释 | 能赋予个人意义 | 联系生活的写作/讨论 |
| 应用 | 能在新情境中使用 | 变式题、真实问题解决 |
| 洞察 | 能看到不同视角 | 辩论、多角度分析任务 |
| 同理 | 能理解他人感受 | 角色扮演、情境模拟 |
| 自知 | 知道自己知道和不知道什么 | 自评、反思日志 |

**对产品设计的核心价值**：
- 先定义"学会了是什么样"，再设计学习过程——避免"做了很多活动但学生没学到"
- 评估不是最后一步，而是设计的第二步

## 3. 梅里尔首要教学原理（Merrill's First Principles）

**五条原则及产品设计实现**：

| 原则 | 说明 | 产品实现 |
|------|------|---------|
| 问题中心 | 学习围绕真实问题展开 | 以真实场景/任务开篇，非知识点列表 |
| 激活旧知 | 激活已有相关经验 | 学前预测、前概念探测、经验回顾 |
| 示范新知 | 展示而非仅仅讲述 | 动画演示、操作示范、案例展示 |
| 应用新知 | 给机会练习并获反馈 | 即时练习、操作实验、反馈矫正 |
| 融会贯通 | 迁移到真实情境 | 综合项目、开放题、真实问题解决 |

**设计检查**：一个完整的学习单元应包含全部五步。缺失任何一步都会降低学习效果。

## 4. 加涅九大教学事件

**九事件及产品功能对应**：

| 事件 | 目的 | 产品功能 |
|------|------|---------|
| 1. 引起注意 | 激发好奇 | 悬念导入、反直觉现象展示 |
| 2. 告知目标 | 建立期望 | 学习目标卡片 |
| 3. 唤起先备知识 | 连接旧知 | 前置回顾、知识关联提示 |
| 4. 呈现新内容 | 信息传递 | 多媒体讲解、3D 演示 |
| 5. 提供学习指导 | 深层加工 | 提示、类比、关键点标注 |
| 6. 引发表现 | 主动练习 | 互动练习、实验操作 |
| 7. 提供反馈 | 矫正强化 | 即时反馈、错误解析 |
| 8. 评估表现 | 检验达标 | 单元测试、实验报告评估 |
| 9. 促进迁移 | 延伸应用 | 拓展题、跨学科连接、真实问题 |

## 5. 学习内容序列设计

**三种序列策略**：

| 策略 | 适用场景 | 说明 |
|------|---------|------|
| 简单到复杂 | 技能类学习 | 先基础操作，再复合操作 |
| 已知到未知 | 概念类学习 | 从学生已有经验出发，逐步扩展 |
| 整体到局部 | 系统类学习 | 先给全景，再深入细节（精细加工理论） |

**Reigeluth 精细加工理论（Elaboration Theory）**：
- 先呈现**最简化的整体框架**（如化学实验的大致流程）
- 再逐层**精细化**每个步骤的细节
- 每层精细化后**回到整体框架**重新定位
- 适合复杂知识体系的教学设计

## 6. 微学习设计原则

**关键参数**：
- 单个学习单元：**5-15 分钟**
- 每个单元**一个核心知识点**
- 包含**练习/检测环节**（测试效应）
- 单元间有**逻辑连接**（不是碎片堆砌）

**适用场景**：
- ✅ 知识点复习、技能训练、碎片化学习
- ❌ 不适合深层概念理解、复杂问题解决（这类需要长时学习）

**与认知负荷理论的关系**：微学习本质上是**分割原则**的应用——降低内在认知负荷。
