# 学科教学法

> 聚焦：各学科特有的教学方法论在教育产品内容设计中的应用。

## 1. 化学实验教学

### 核心方法论

**概念转变教学（Conceptual Change）**：
- 化学中大量**反直觉概念**（如质量守恒、微观粒子运动）
- 教学策略：引出前概念 → 制造认知冲突 → 建构新概念 → 巩固应用
- 产品设计：先让学生**预测**实验结果 → 观察实际结果 → 引导分析差异原因

**宏观-微观-符号三重表征**：

| 表征层面 | 说明 | 产品实现 |
|----------|------|---------|
| 宏观 | 可观察的实验现象 | 实验视频/3D 动画中的颜色、气泡、沉淀 |
| 微观 | 分子/原子层面的变化 | 3D 粒子动画、分子模型操作 |
| 符号 | 化学方程式、分子式 | 方程式编辑器、配平练习 |

**关键设计**：三重表征必须**同步呈现**或**可切换**，帮助学生建立联系。

### 实验教学流程

```
预测 → 观察 → 解释 → 应用（POE + A）
```

1. **预测（Predict）**：基于已有知识预测实验结果
2. **观察（Observe）**：操作实验，记录现象
3. **解释（Explain）**：分析预测与观察的差异
4. **应用（Apply）**：迁移到新情境

### 化学实验安全教学

**虚拟实验的独特价值**：
- 允许"犯错"——真实实验中不允许的危险操作可以在虚拟环境中体验后果
- 提供**放大视角**——微观反应过程可视化
- **不能完全替代**真实实验——触感、气味、意外状况的处理经验不可替代

## 2. 数学概念教学

### 核心方法论

**CPA 教学法（Concrete → Pictorial → Abstract）**：

| 阶段 | 说明 | 产品实现 |
|------|------|---------|
| 具体（Concrete） | 操作实物/虚拟操作物 | 可拖拽的计数棒、几何积木 |
| 图示（Pictorial） | 用图像表示 | 数形结合的可视化 |
| 抽象（Abstract） | 符号运算 | 公式推导、计算练习 |

**关键**：三个阶段不是线性的，学生可能需要在抽象遇困时**退回**到具体层面。

### 数学思维培养

| 思维类型 | 教学策略 | 产品功能 |
|----------|---------|---------|
| 推理能力 | 要求解释"为什么"而非只答"是什么" | 解题过程录入、推理链展示 |
| 建模能力 | 真实情境中的数学应用 | 情境化问题、数据分析项目 |
| 抽象能力 | 从具体案例归纳一般规律 | 模式发现任务、归纳练习 |
| 直觉培养 | 估算、数感训练 | 估算游戏、数量感知训练 |

### 常见教学误区
- ❌ 过早进入抽象符号运算（跳过 CPA 的前两步）
- ❌ 只关注计算准确率，不关注数学理解
- ❌ 练习全是程序性题目，缺少概念理解题

## 3. 物理探究教学

### 核心方法论

**科学探究七步骤**：
1. 提出问题
2. 猜想与假设
3. 设计实验方案
4. 进行实验（操作、观察、记录）
5. 分析数据
6. 得出结论
7. 交流与评估

**变量控制法（控制变量法）**：
- 物理实验的核心方法——每次只改变一个变量
- 产品设计：提供变量控制面板，让学生选择自变量/因变量/控制变量
- **常见困难**：学生不理解"为什么要控制变量"——需要通过反例演示

### 物理概念教学

**从直觉到科学概念的转变路径**：

| 阶段 | 学生状态 | 教学策略 |
|------|---------|---------|
| 前概念暴露 | 持有日常经验形成的直觉 | 预测任务、概念探测题 |
| 认知冲突 | 发现直觉与实验结果矛盾 | 反直觉实验演示 |
| 概念建构 | 开始理解科学解释 | 多重表征、类比推理 |
| 概念巩固 | 在新情境中应用 | 变式练习、迁移任务 |

**高频前概念（物理）**：
- "力是维持运动的原因"（牛顿第一定律前概念）
- "重的物体下落更快"（自由落体前概念）
- "电流从正极出来会被消耗"（电路前概念）
- 产品设计中需要**有针对性地设计概念转变活动**

## 4. 跨学科教学法（STEM/STEAM）

### 整合层次

| 层次 | 说明 | 示例 |
|------|------|------|
| 学科内 | 单学科内知识整合 | 化学实验综合运用多个知识点 |
| 多学科 | 多学科围绕同一主题但独立教学 | 水污染：化学检测 + 生物影响 |
| 跨学科 | 学科边界模糊，以问题为中心 | 设计净水装置（化学 + 物理 + 工程） |
| 超学科 | 连接真实世界问题 | 社区环保项目 |

### 产品设计启示
- 教育产品可以支持**学科内+多学科**整合，跨学科和超学科需要更复杂的项目式学习支撑
- 内容标签系统应支持**多学科关联**——一个实验可以关联多个学科知识点
