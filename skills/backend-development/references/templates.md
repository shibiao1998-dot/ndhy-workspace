# 后端实现交付包模板

## 概览

后端实现交付包包含 6 个标准文档 + 实现代码：

```
{project}/backend/
├── service-implementation/         # 实现代码
├── backend-impl-notes.md           # 实现说明
├── known-limitations.md            # 已知限制
├── test-notes.md                   # 测试说明
├── integration-handoff.md          # 联调交接说明
└── refactor-summary.md             # 重构说明（仅重构任务）
```

## 6. 代码结构模板

### 典型项目目录结构

```
src/
├── handler/              # Handler/Controller 层
│   ├── dto/              # 请求/响应 DTO
│   │   ├── request.go    # 请求 DTO
│   │   └── response.go   # 响应 DTO
│   ├── middleware/        # 中间件（认证/日志/限流）
│   └── {module}_handler.go
│
├── service/              # Service/Use Case 层
│   ├── {module}_service.go      # 服务接口
│   └── {module}_service_impl.go # 服务实现
│
├── repository/           # Repository/DAO 层
│   ├── {module}_repository.go      # 仓储接口
│   └── {module}_repository_impl.go # 仓储实现
│
├── domain/               # 领域模型
│   ├── entity/           # 实体
│   ├── valueobject/      # 值对象
│   └── event/            # 领域事件
│
├── infrastructure/       # 基础设施
│   ├── config/           # 配置
│   ├── database/         # 数据库连接
│   ├── cache/            # 缓存
│   └── external/         # 外部服务客户端
│
├── pkg/                  # 公共包
│   ├── errors/           # 错误定义与错误码
│   ├── validator/        # 校验工具
│   └── logger/           # 日志工具
│
└── test/                 # 测试
    ├── unit/             # 单元测试
    ├── integration/      # 集成测试
    └── fixture/          # 测试数据和 mock
```

### 命名规范

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| Handler 方法 | 动词 + 资源 | `CreateUser`, `GetUserById` |
| Service 方法 | 业务动词 + 名词 | `RegisterUser`, `CalculateScore` |
| Repository 方法 | CRUD 动词 + 实体 | `Save`, `FindById`, `DeleteById` |
| DTO | 资源 + Request/Response | `CreateUserRequest`, `UserResponse` |
| 错误码 | 模块前缀 + 序号 | `USER_001`, `AUTH_002` |
| 配置键 | 模块.子模块.属性 | `app.database.max-connections` |
