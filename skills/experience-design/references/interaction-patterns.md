## 交互模式库

### 表单填写类

#### 单步表单 vs 分步表单

| 维度 | 单步表单 | 分步表单 |
|------|---------|---------|
| 适用 | 字段 ≤ 7 个 | 字段 > 7 个，或有逻辑分组 |
| 优势 | 一目了然，快速完成 | 降低认知负担，支持中断恢复 |
| 劣势 | 字段多时压迫感强 | 不知道还要填多少 |
| 关键设计 | 合理分组，视觉分割 | 步骤指示器 + 可回退 + 自动保存 |

**分步表单设计要点：**
- 步骤指示器始终可见（显示当前步骤 / 总步骤数）
- 每步 ≤ 5 个字段
- 可自由回退修改已填内容
- 每步切换时自动保存
- 最后一步提供全部信息的预览/确认

#### 表单验证策略

| 策略 | 触发时机 | 适用场景 |
|------|---------|---------|
| **即时验证** | 字段失焦时 | 格式类校验（邮箱、手机号） |
| **提交后验证** | 点击提交时 | 业务规则校验（需要调用后端） |
| **混合验证** | 简单规则即时 + 复杂规则提交时 | ✅ 推荐默认方案 |

**验证反馈设计：**
- 错误信息紧贴字段下方，红色文字
- 说明错在哪 + 怎么改（"手机号应为 11 位数字"）
- 提交时有错：滚动到第一个错误字段 + 聚焦
- 通过验证的字段：绿色勾 ✓（可选）

#### 自动保存 vs 手动保存

| 策略 | 适用场景 | 设计要点 |
|------|---------|---------|
| **自动保存** | 长内容编辑（文档、教案） | 显示"已自动保存"时间戳，支持版本回溯 |
| **手动保存** | 短表单、配置页 | 明确的"保存"按钮，离开时提示未保存 |
| **混合** | 草稿自动保存 + 发布手动 | ✅ 推荐：编辑时自动存草稿，发布/提交需手动确认 |

### 内容浏览类

#### 列表 vs 卡片 vs 瀑布流

| 模式 | 适用场景 | 信息密度 | 视觉重点 |
|------|---------|---------|---------|
| **列表** | 结构化数据、比较、批量操作 | 高 | 文字信息 |
| **卡片** | 多维信息、含图片、独立操作 | 中 | 视觉预览 |
| **瀑布流** | 图片/创意内容、探索浏览 | 低 | 图片为主 |

**选择决策树：**
```
内容有图片/预览图？
  ├── 是 → 图片是核心信息？
  │        ├── 是 → 瀑布流 / 卡片网格
  │        └── 否 → 卡片列表
  └── 否 → 需要比较/批量操作？
           ├── 是 → 表格列表
           └── 否 → 简单列表
```

#### 筛选 + 排序 + 搜索

| 功能 | 触发条件 | 位置 | 设计要点 |
|------|---------|------|---------|
| **搜索** | 内容 > 20 条 | 列表顶部 | 即时搜索 / 回车搜索 |
| **筛选** | 有明确分类维度 | 搜索下方或侧边 | 常用筛选外露，更多筛选折叠 |
| **排序** | 有多种有意义的排序方式 | 列表右上角 | 默认最有用的排序 + 切换按钮 |

**筛选设计要点：**
- 已选筛选条件以"标签"形式显示，支持单个移除
- 提供"清除全部"
- 筛选结果为空时：友好提示 + 清除筛选引导
- 筛选条件数 ≤ 5 个常驻外露，其余折叠到"更多筛选"

#### 分页 vs 无限滚动

| 模式 | 适用场景 | 优势 | 劣势 |
|------|---------|------|------|
| **分页** | 管理后台、需要精确定位 | 可定位、可分享URL | 操作中断感 |
| **无限滚动** | 信息流、社交内容 | 沉浸体验 | 难以定位、页脚不可达 |
| **加载更多** | 折中方案 | 用户主动控制 | ✅ 推荐多数场景 |

### 操作反馈类

#### 反馈三层模型

| 层级 | 触发条件 | 形式 | 时长 | 示例 |
|------|---------|------|------|------|
| **即时反馈** | 所有用户操作 | 视觉变化 | 即时 | 按钮按下态变化、表单校验 |
| **过程反馈** | 操作耗时 > 1 秒 | 进度条/骨架屏/加载动画 | 操作期间 | "正在保存...（60%）" |
| **结果反馈** | 重要操作完成 | Toast/结果页/弹窗 | 2-5 秒或手动关闭 | "教案已保存成功" |

#### 反馈形式选择

| 形式 | 适用场景 | 设计要点 |
|------|---------|---------|
| **Toast** | 轻量操作成功/失败 | 2-3 秒自动消失，不阻断操作 |
| **内联提示** | 表单验证、字段级反馈 | 紧贴相关元素，不消失直到修复 |
| **Snackbar** | 操作成功 + 可撤销 | 底部浮出，带"撤销"按钮，5 秒后消失 |
| **弹窗（Modal）** | 重要确认、复杂结果 | 阻断操作，需用户主动关闭/确认 |
| **结果页** | 重要流程完成 | 独立页面，展示结果 + 下一步引导 |
| **全局通知** | 系统级消息 | 顶部横幅或通知中心 |

#### 加载状态设计

| 场景 | 推荐方案 | 说明 |
|------|---------|------|
| 页面首次加载 | 骨架屏 | 展示内容结构轮廓，减少焦虑 |
| 列表加载更多 | 底部 loading 指示器 | 小圆圈 / 文字提示 |
| 操作后等待 | 按钮 loading 态 | 按钮变灰 + 旋转图标 + 禁止重复点击 |
| 长时间处理 | 进度条 + 百分比 | 让用户知道还需等多久 |
| 后台处理 | 通知回调 | "处理中，完成后会通知你" |

### 引导类

#### 新手引导

| 模式 | 适用场景 | 侵入度 | 设计要点 |
|------|---------|--------|---------|
| **蒙层引导** | 首次���用，关键功能 ≤ 5 步 | 高 | 可跳过、步骤清晰、突出目标区域 |
| **气泡提示** | 新功能上线 | 中 | 点击一次即消失，不反复出现 |
| **空状态引导** | 页面无内容时 | 低 | ✅ 推荐优先使用：告诉用户该做什么 |
| **任务清单** | 初始化设置 | 中 | 进度可见，完成可跳过 |
| **交互式教程** | 操作复杂的核心功能 | 高 | 用户跟着做，不只是看 |

**新手引导铁律：**
- 永远可以跳过
- 不要超过 5 步
- 引导的是行动不是概念（"点击这里创建"而非"这个功能可以创建"）
- 用户已经会了就不再出现

#### 空状态引导

**空状态三要素：**
1. **说明**：这个地方应该有什么
2. **引导**：怎么创建/添加第一条
3. **激励**（可选）：创建后能得到什么

```
┌────────────────────────────────────────┐
│                                        │
│           [插图/图标]                   │
│                                        │
│     还没有创建教案                      │
│     点击下方按钮，开始你的第一次备课     │
│                                        │
│          [+ 新建教案]                   │
```

## 页面布局方法

### 页面结构设计原则

1. **一个页面一个核心目标**：用户来这个页面是为了做什么？只服务这一个目标
2. **F 型视觉动线**：重要信息放左上，操作按钮放视觉终点
3. **信息优先级分区**：核心信息固定可见，次要信息可滚动
4. **留白是设计**：不要填满每一寸空间

### 通用页面骨架

```
┌────────────────────────────────────────┐
│ [全局导航]                              │
├────────────────────────────────────────┤
│ [页面标题]              [主操作按钮]     │
├──────────┬─────────────────────────────┤
│          │                             │
│ [侧边栏]  │     [主内容区]              │
│ (可选)   │                             │
│          │                             │
├──────────┴─────────────────────────────┤
│ [页脚/辅助信息]                         │
└────────────────────────────────────────┘
```

### 布局选型

| 布局 | 适用场景 | 示例 |
|------|---------|------|
| **单栏** | 内容阅读、表单填写 | 文章详情页、注册页 |
| **双栏（侧边+内容）** | 导航+内容、列表+详情 | 邮件客户端、管理后台 |
| **三栏** | 复杂工作台 | IDE、设计工具 |
| **卡片网格** | 多个同类项目展示 | 课程列表、模板选择 |
| **仪表板** | 数据概览、多模块聚合 | 首页工作台 |

### 关键按钮位置

| 按钮类型 | 推荐位置 | 说明 |
|---------|---------|------|
| 页面主操作 | 页面标题右侧 / 内容区右上 | 醒目突出，用主色 |
| 表单提交 | 表单底部右侧 | 视觉动线终点 |
| 二步确认 | 弹窗底部右侧 | 确认在右，取消在左 |
| 返回/取消 | 页面左上 / 表单底部左侧 | 位置一致 |
| 危险操作 | 远离常规按钮，用警示色 | 防误触 |

## 响应式策略

### 策略制定步骤

1. **确定核心设备**：产品主要在什么设备上使用？
2. **选择设计策略**：桌面优先 or 移动优先？
3. **定义断点**：通常 3 档足够
4. **决定内容取舍**：移动端保留什么、简化什么、移除什么

### 通用断点

| 档位 | 宽度 | 设备 |
|------|------|------|
| 移动端 | < 768px | 手机 |
| 平板 | 768px - 1024px | 平板横竖屏 |
| 桌面端 | > 1024px | 笔记本、显示器 |

### 响应式变化策略

| 变化类型 | 做法 | 示例 |
|---------|------|------|
| **布局变化** | 多栏 → 单栏 | 桌面双�� → 移动单栏堆叠 |
| **导航变化** | 顶部/侧边 → 底部 Tab / 汉堡 | 侧边导航 → 底部 Tab |
| **内容简化** | 隐藏次要信息 | 桌面端显示描述，移动端仅标题 |
| **交互适配** | 悬停 → 长按 / 滑动 | hover 效果改为 tap 展开 |
| **功能取舍** | 移除复杂功能 | 移动端不支持批量操作 |

### 移动端特殊考虑

| 考虑点 | 标准 |
|--------|------|
| 触控目标大小 | ≥ 44px × 44px |
| 触控间距 | ≥ 8px |
| 最小字号 | ≥ 14px |
| 单手操作 | 核心操作在拇指可达区域（底部 2/3） |
| 横屏处理 | 至少不崩溃，关键页面考虑横屏布局 |
| 键盘弹出 | 输入框不被键盘遮挡 |
