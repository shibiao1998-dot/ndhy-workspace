# 体验设计 — 标准工作流程

## 8 步标准流程

```
Step 1：理解输入
  │  消化产品定义文档，确认核心场景和任务
  │  → 提取：产品概念、核心场景、功能范围、产品规则、成功标准
  │  → 信息不足时，有权要求上游补充
  │
Step 2：信息架构设计
  │  → 内容组织（按用户心理模型，非技术架构）
  │  → 导航体系（主导航 + 辅助导航 + 位置导航）
  │  → 层级控制（≤ 3 层深度，每层 ≤ 7 选项）
  │  → 产出【信息架构图 + 导航结构表】
  │  → 过信息架构四检验（见 checklists.md）
  │  → 详细方法 → information-architecture.md
  │
Step 3：任务流程设计
  │  → 绘制任务流（最优路径、最少步骤）
  │  → 绘制用户流（加入分支：决策/异常/中断/放弃）
  │  → 标注关键节点（📌决策 ⏳等待 ⚠️风险 🎯价值）
  │  → 遵守任务流四铁律（见 checklists.md）
  │  → 详细方法 → task-flows.md
  │
Step 4：界面布局与交互模式
  │  → 每个关键页面定义框架结构（区块 + 优先级 + 固定/滚动）
  │  → 选择交互模式（表单/浏览/反馈/引导，用成熟模式）
  │  → 处理所有页面状态（加载/有内容/空/错误）
  │  → 定义响应式策略（核心设备 + 断点 + 移动端取舍）
  │  → 交互模式库 → interaction-patterns.md
  │
Step 5：体验语言定义
  │  → 定义产品人设（类型 + 称呼 + 句式 + 情感温度）
  │  → 设计关键文案（按钮/提示/错误/空状态/加载）
  │  → AI 对话语言设计（如适用）
  │  → 产出【语言风格指南 + 文案模板 + 禁用词清单】
  │  → 详细方法 → experience-language.md
  │
Step 6：体验地图绘制
  │  → 横轴：认知 → 首次使用 → 核心使用 → 持续使用 → 推荐/流失
  │  → 纵轴：行为 + 接触点 + 情感曲线 + 机会点
  │  → 标注：⭐ 峰值 / ⚠️ 谷底 / 🚪 转化点
  │  → 应用峰终定律：刻意制造正面峰值 + 消除负面谷底 + 确保终值正面
  │  → 绘制指南 → checklists.md §体验地图绘制指南
  │
Step 7：可用性走查
  │  → 逐页检查四个层面（信息/操作/状态/效率）
  │  → 问题记录 + 修复方案
  │  → 可达性检查（色觉/字号/触控/弱网）
  │  → 走查清单 → checklists.md §可用性走查清单
  │
Step 8：规范输出
     → 组件行为规范（弹窗/Toast/下拉/卡片使用场景）
     → 状态转换规范（按钮状态链 + 页面状态链）
     → 反馈规范（即时/过程/结果三层）
     → 动效规范（场景 + 时长 + 节制原则）
     → 按 7 章节模板组织文档 → templates.md
```

## 设计深度速查

| 深度 | 适用场景 | 覆盖范围 | 输出物 |
|------|----------|----------|--------|
| **快速设计** | 小功能迭代、已有设计语言延伸 | 任务流程 + 界面布局，复用现有规范 | 体验设计文档（轻量版） |
| **标准设计** | 新模块或重要改版 | 完整 8 步流程 | 体验设计文档（标准版） |
| **深度设计** | 全新产品从 0 到 1 | 完整流程 + 体验地图 + 可用性测试设计 + 全套规范 | 体验设计文档（完整版） |

判断不清时向上确认。

## 触发时首要动作

无论如何触发，第一步永远是：

> "产品定义告诉了我们什么？核心场景是什么、核心任务是什么、用户是谁、约束在哪？"

→ 判断设计深度 → 消化产品定义输入 → 按 8 步流程执行
