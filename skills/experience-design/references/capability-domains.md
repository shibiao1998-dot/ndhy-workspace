# 体验设计 — 7 大能力域速查

## 域 1：信息架构设计

**核心问题**：产品里的信息怎么组织、怎么分类、怎么找到？

三个核心任务：
- **内容组织**：按用户心理模型分类，非技术架构或业务部门
- **导航体系**：用户在任何位置能回答"我在哪？能去哪？怎么回去？"
- **层级控制**：深度 ≤ 3 层 / 每层 ≤ 7 选项 / 重要信息在最浅层

→ 详细方法 [information-architecture.md](information-architecture.md)

## 域 2：任务流程设计

**核心问题**：用户完成每个任务的具体步骤路径是什么？

设计方法：
1. 绘制**任务流**（最少步骤、无分支）
2. 绘制**用户流**（加入决策/异常/中断/放弃/纠错分支）
3. 标注**关键节点**（📌决策 ⏳等待 ⚠️风险 🎯价值）

→ 详细方法 [task-flows.md](task-flows.md)

## 域 3：界面布局与交互模式

**核心问题**：每个页面的结构和交互行为是什么？

三个层面：
- **页面结构**：区块组成、优先级排序、固定/可滚动
- **交互模式**：表单（单步/分步）、浏览（列表/卡片）、反馈（Toast/结果页）、引导（新手/空状态）
- **响应式策略**：核心设备、断点、移动端取舍

→ 交互模式库 [interaction-patterns.md](interaction-patterns.md)

## 域 4：体验语言定义

**核心问题**：产品和用户"说话"的方式是什么？

三个维度：
- **语言风格**：产品人设 + 称呼 + 句式 + 情感温度 + 禁用词
- **关键文案**：按钮（动词优先）/ 提示（说人话+给例子）/ 错误（问题+解法）/ 空状态（该有什么+怎么创建）
- **AI 对话语言**：角色定位 + 不比用户更权威 + 表达不确定性

→ 详细方法 [experience-language.md](experience-language.md)

## 域 5：体验地图绘制

**核心问题**：用户从认知到推荐的全过程感受如何？

关键标注：
- ⭐ **峰值时刻**：刻意设计（超预期/成就感/惊喜/效率飞跃），不靠运气
- ⚠️ **谷底时刻**：识别并设计应对措施
- 🚪 **关键转化点**：从"试试看"到"真的用"的转折

**峰终定律**：人对体验的记忆取决于峰值和终值 → 刻意制造正面峰值 + 确保结束体验正面

→ 绘制指南 [checklists.md](checklists.md) §体验地图绘制指南

## 域 6：可用性保障

**核心问题**：产品对所有目标用户都"用得了、用得懂、用得顺"吗？

五项原则：可见性 / 一致性 / 容错性 / 效率 / 可达性

走查四层面：信息层面 / 操作层面 / 状态层面 / 效率层面

→ 走查清单 [checklists.md](checklists.md) §可用性走查清单

## 域 7：设计规范输出

**核心问题**：为视觉设计和开发落地输出什么规范？

四类规范：
- **组件行为**：弹窗/Toast 使用场景、下拉 vs 平铺选用标准
- **状态转换**：按钮状态链（默认→悬停→按下→禁用→加载中）、页面状态链（加载→有内容→空→错误）
- **反馈规范**：即时（所有操作）/ 过程（>1秒）/ 结果（重要操作）
- **动效规范**：节制原则——能不加就不加
