# 迭代优化 — 五阶段迭代循环 & 标准流程

```
┌──→ ① 审视现状 ──→ ② 确定方向 ──→ ③ 设计方案 ─┐
│                                                │
│    ⑤ 复盘学习 ←── ④ 执行验证 ←─────────────────┘
│         │
└─────────┘（进入下一轮）
```

| 阶段 | 核心问题 | 关键动作 | 输出 |
|------|---------|---------|------|
| ① 审视现状 | 我们在哪？ | 回顾上轮数据、收集反馈、扫描环境变化、定位瓶颈 | 现状诊断报告 |
| ② 确定方向 | 我们去哪？ | 锁定1个核心目标、定义成功标准、明确排除项 | 迭代目标声明 |
| ③ 设计方案 | 怎么去？ | 方案对比、MVP拆解、定验收标准、拆成≤1周的交付块 | 迭代方案 |
| ④ 执行验证 | 走起来 | 按优先级逐个交付、小块验证、及时止损、记录过程 | 交付物+执行记录 |
| ⑤ 复盘学习 | 学到了什么？ | 目标达成度、继续/停止/开始清单、经验沉淀 | 复盘记录 |

→ 每阶段详细输入/输出/关键动作 references/iteration-cycle.md

## 标准交付物

```
{project}/iteration/
  iteration-goal.md         # 迭代目标声明
  iteration-plan.md         # 迭代方案（含需求清单和验收标准）
  retrospective.md          # 迭代复盘记录
  release-decision.md       # 发布决策记录（如需要）
```

## 协作接口

### 上游输入

| 来源 | 接收内容 | 使用方式 |
|------|---------|---------|
| 产品定义专家 | 产品路线图、功能范围、成功标准 | 框定迭代的战略方向 |
| 数据分析 | 用户行为数据、漏斗转化、留存曲线 | 驱动信号捕捉（阶段①） |
| 用户反馈 | 客服汇总、调研报告、社群声音 | 驱动信号捕捉（阶段①） |
| 上轮复盘 | 继续/停止/开始清单、改进项 | 直接输入本轮审视现状 |

### 下游输出

| 接收方 | 传递内容 | 约束关系 |
|--------|---------|---------|
| 项目管理 | 迭代目标+方案+需求清单 | 在迭代范围内排期调度 |
| 开发团队 | 优先级排序、验收标准 | 按优先级逐个交付 |
| 测试团队 | 验收标准、核心场景 | 在验收标准内设计用例 |
| 发布策略 | 发布决策、灰度建议 | 在迭代方案内执行发布 |
