## 反模式详解

### 1. 只看接口通不通，不看完整流程

- **表现**：用 Postman 调通接口就宣布联调完成
- **危害**：前端消费、展示、反馈链路未验证，上线后才发现问题
- **正确做法**：按端到端链路验证——请求→处理→响应→展示→用户反馈

### 2. 前后端互推，联调专家不归因

- **表现**：前端说"后端返回有问题"，后端说"前端调用方式不对"，联调专家转发了事
- **危害**：问题在团队间来回弹球，无人解决
- **正确做法**：联调专家按归因决策树定位根因层，附证据回流

### 3. 问题分类混乱

- **表现**：把环境问题、契约问题、实现问题、测试问题混在一起
- **危害**：问题无法高效分派，修复方向不清
- **正确做法**：严格按六层模型分类——环境/契约/后端/前端/数据/时序

### 4. 无复现路径就报问题

- **表现**：报告"登录有时候不行"但没有具体步骤
- **危害**：开发无法复现和修复，浪费排查时间
- **正确做法**：每个问题必须附——复现步骤 + 实际结果 + 期望结果 + 截图/日志

### 5. 修完不回归

- **表现**：开发说"改好了"，联调专家标记"已解决"
- **危害**：修复可能不完整或引入新问题
- **正确做法**：每个修复必须走回归验证流程，验证通过才标记已解决

### 6. 跳过异常路径

- **表现**：正常路径跑通后就宣布联调完成，跳过错误处理和边界场景
- **危害**：上线后遇到异常场景就崩溃
- **正确做法**：异常路径（错误/权限/超时/降级）是联调核心，不是附加项

### 7. mock 通过 = 真实通过

- **表现**：用 mock 数据跑通流程，在报告中未标记 mock 状态
- **危害**：真实对接时才发现 mock 和真实行为不一致
- **正确做法**：明确区分 mock 环节和真实对接环节，报告中分别标记

### 8. "差不多通了"替代闭环

- **表现**：大部分流程通了，剩下的"应该没问题"
- **危害**：未验证的部分恰恰可能是问题所在
- **正确做法**：按就绪判断标准逐项评估，给出明确结论

## 回归验证

### 回归流程

```
Step 6 产出问题清单
  |
  |- 生成回归清单
  |   -> 每个问题 -> 对应回归验证项
  |   -> 每项包含: 原问题编号 + 验证方式 + 通过条件
  |
  |- 等待修复
  |   -> 跟踪修复进度
  |   -> 修复到位后触发回归
  |
  |- 执行回归验证
  |   -> 逐项验证
  |   -> 记录: 通过 / 未通过 / 新发现
  |
  |- 处理回归结果
  |   |- 全部通过 -> 更新联调状态
  |   |- 部分未通过 -> 重新归因 -> 再次回流
  |   +- 发现新问题 -> 归因 -> 加入下轮回归
  |
  +- 回归轮次管理
      -> 每轮回归独立编号
      -> 记录轮次结果
      -> 超过 3 轮仍有 P0 -> 升级项目管理专家
```

### 回归验证原则

| 原则 | 说明 |
|------|------|
| **逐项验证** | 每个修复对应明确的回归项，不批量模糊确认 |
| **原路径复现** | 用原来的复现路径验证，不换路径绕过 |
| **检查副作用** | 验证修复是否引入新问题（影响相邻功能） |
| **证据留存** | 回归结果附截图/日志，不只标记通过 |
| **不信口头确认** | "改好了"不等于通过，必须实际验证 |

## 就绪判断标准（详版）

### 三级就绪结论

| 结论 | 条件 | 后续动作 |
|------|------|---------|
| **✅ 就绪** | 7 项就绪条件全部满足，无 P0/P1 残留 | 进入测试阶段 |
| **⚠️ 有条件就绪** | 关键路径通过，残留 P1 ≤ 3 且有修复计划 | 进入测试，P1 并行修复 |
| **❌ 未就绪** | 存在 P0 残留 / 关键路径未跑通 / 环境阻塞 | 继续联调，不进入下一阶段 |

### 就绪七条件

1. **关键流程全部跑通**：关键流程清单中所有正常路径验证通过
2. **契约消费一致**：前后端实际请求/响应与契约定义一致
3. **状态与错误反馈一致**：状态流转正确，错误码/消息/降级行为对齐
4. **环境前提满足**：无 mock 冒充真实、无临时绕过、配置正确
5. **核心问题已归因并回流**：无 P0 残留，P1 已有明确修复计划
6. **回归验证通过**：所有已修复问题的回归验证通过
7. **无阻断性残余风险**：无已知但未处理的阻断性问题

### 不可宣称"联调完成"的红线

- ❌ 仍有 P0 问题未修复
- ❌ 关键流程正常路径未全部跑通
- ❌ mock 跑通被当作真实联调通过（且未标记）
- ❌ 异常路径和错误反馈完全未验证
- ❌ 回归验证未执行
- ❌ 环境存在已知但未解决的临时绕过
