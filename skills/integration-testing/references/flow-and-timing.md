# 端到端流程验证与时序诊断

覆盖关键流程的端到端跑通验证，以及时序、异步、并发相关问题的系统性诊断。

> 接口能调通 ≠ 流程能跑通。必须按用户真实操作路径验证完整链路。

---

## 端到端验证方法

### 验证范围定义

```
确定验证范围:
  1. 从上游获取关键流程清单（通常来自 PRD 或项目管理）
  2. 对每个流程标注:
     - 涉及的接口（按调用顺序）
     - 涉及的前端页面/组件
     - 涉及的后端服务/模块
     - 前置条件（数据状态/用户角色/环境要求）
  3. 区分验证路径:
     - 正常路径（Happy Path）: 用户正常操作的主流程
     - 异常路径（Error Path）: 参数错误/权限不足/资源不存在/服务异常
     - 边界路径（Boundary Path）: 空数据/首次使用/大数据量/并发操作
```

### 正常路径验证

对每条关键流程的正常路径：

| 步骤 | 执行内容 | 记录内容 |
|------|---------|---------|
| 1. 前置准备 | 确保前置数据和状态就位 | 初始状态截图/数据快照 |
| 2. 用户操作 | 在前端按预期流程操作 | 每步操作和页面响应 |
| 3. 请求捕获 | 捕获前端发出的请求 | URL/Method/Headers/Body |
| 4. 后端处理 | 确认后端正确处理请求 | 日志/数据变更/��作用 |
| 5. 响应验证 | 验证后端响应符合契约 | 状态码/响应体/响应头 |
| 6. 前端展示 | 验证前端正确展示结果 | UI 状态/数据展示/交互反馈 |
| 7. 状态确认 | 确认最终状态一致 | 数据库状态/前端状态/缓存状态 |

### 异常路径验证

| 异常类别 | 测试场景 | 验证要点 |
|---------|---------|---------|
| **参数错误** | 缺少必填参数/类型错误/超长输入/非法格式 | 后端返回正确错误码 + 前端展示友好提示 |
| **权限不足** | 未认证/角色不足/资源越权 | 后端返回 401/403 + 前端引导登录/提示无权 |
| **资源不存在** | 访问已删除/不存在的资源 | 后端返回 404 + 前端展示"不存在"状态 |
| **业务规则违反** | 违反业务约束（如余额不足/名额已满） | 后端返回业务错误码 + 前端展示业务错误消息 |
| **服务异常** | 后端超时/后端500/依赖服务不可用 | 前端展示降级状态 + 重试提示 |
| **并发冲突** | 多人同时操作同一资源 | 后端返回冲突错误 + 前端提示刷新 |

### 边界路径验证

| 边界类别 | 测试场景 | 验证要点 |
|---------|---------|---------|
| **空数据** | 列表为空/详情字段为 null | 前端展示空状态（非崩溃） |
| **首次使用** | 新用户/空数据库/初始化状态 | 引导流程正常/默认值正确 |
| **大数据量** | 列表 100+ 条/分页边界/超长文本 | 分页正常/性能可接受/展示不变形 |
| **快速操作** | 快速连续点击/重复提交 | 防重提交生效/数据不重复 |
| **浏览器刷新** | 操作中途刷新页面 | 状态恢复正常/不丢失关键数据 |

## 时序与异步诊断

### 常见时序问题分类

| 问题类型 | 表现 | 根因 | 诊断方法 |
|---------|------|------|---------|
| **请求顺序依赖** | 操作A的结果还没返回就发起了操作B | 前端未等待前置请求完成 | 检查前端异步调用链，确认 await/then 顺序 |
| **异步回调丢失** | 后端处理完成但前端未收到通知 | Webhook/WebSocket/轮询机制失效 | 检查回调注册/连接状态/轮询间隔 |
| **轮询时机不当** | 前端获取到过时状态 | 轮询间隔过长/未在关键时机触发刷新 | 检查轮询频率和触发条件 |
| **竞态条件** | 两个请求的响应顺序不确定导致显示错误 | 后返回的响应覆盖了先返回的正确结果 | 检查前端对并发响应的处理逻辑 |
| **乐观更新冲突** | 前端乐观更新后发现后端失败 | 乐观更新的回滚逻辑缺失 | 检查乐观更新的失败回滚路径 |
| **事件顺序不一致** | 事件消费顺序与生产顺序不一致 | 消息队列无序消费/多消费者竞争 | 检查消息队列配置和消费逻辑 |

### 时序问题诊断流程

```
发现疑似时序问题
  |
  |- 1. 确认复现路径
  |   -> 记录精确的操作步骤和时间点
  |   -> 标记哪一步出现异常
  |
  |- 2. 分析请求时序
  |   -> 捕获网络请求的发出时间和响应时间
  |   -> 绘制时序图：操作→请求→处理→响应→展示
  |   -> 标记异常点
  |
  |- 3. 定位问题层
  |   |- 前端发出顺序有误？-> 前端异步控制问题
  |   |- 后端处理顺序有误？-> 后端并发/事务问题
  |   |- 回调/通知未到达？-> 通信机制问题
  |   +- 响应处理顺序有误？-> 前端响应管理问题
  |
  |- 4. 归因并记录
  |   -> 归因到具体层和责任方
  |   -> 附时序图和复现步骤
  |
  +- 5. 建议修复方向
      -> 不替代开发专家写实现
      -> 给出明确的修复方向和约束
```

### 异步流程验证要点

| 异步模式 | 验证要点 | 常见问题 |
|---------|---------|---------|
| **轮询（Polling）** | 轮询间隔/停止条件/超时处理 | 间隔过长/不停止/无超时 |
| **WebSocket** | 连接建立/消息格式/重连机制/断线处理 | 不重连/消息丢失/格式不匹配 |
| **Webhook** | 回调注册/签名验证/重试机制/幂等处理 | 未注册/签名错误/重复处理 |
| **消息队列** | 消息发布/消费确认/死信处理/顺序保障 | 消息丢失/重复消费/乱序 |
| **长任务** | 任务提交/状态查询/结果获取/取消机制 | 状态不更新/结果丢失/无法取消 |

## 流程优先级排序

当流程数量较多时，按以下优先级排序验证：

| 优先级 | 流程特征 | 示例 |
|--------|---------|------|
| **P0 必验** | 核心业务主流程 | 注册/登录/下单/支付 |
| **P1 重要** | 频繁使用的辅助流程 | 搜索/筛选/编辑/删除 |
| **P2 一般** | 低频但必须可用的流程 | 导出/批量操作/设置 |
| **P3 补充** | 边缘场景和极端路径 | 并发操作/大数据量/特殊字符 |

快速验证模式只覆盖 P0；标准联调覆盖 P0+P1；深度联调覆盖全部。
