# 教育行业联调特别指南

教育行业产品的联调具有独特的复杂性：多角色链路、时序敏感流程、学期/考试等时间窗口、以及频繁的外部系统对接。本指南补充教育项目联调时的特别关注点。

---

## 多角色链路验证

教育产品通常涉及多种用户角色，且角色间的操作存在严格的依赖关系。

### 教育场景典型角色链

| 场景 | 角色链路 | 联调重点 |
|------|---------|---------|
| **课程发布** | 教师创建课程 → 管理员审核 → 学生可见 | 审核状态流转 + 可见性联动 |
| **作业提交** | 教师布置 → 学生提交 → 教师批改 → 学生查看 | 全链路状态 + 截止时间 |
| **考试流程** | 管理员创建 → 教师出题 → 学生答题 → 自动/人工批改 → 成绩发布 | 时间窗口 + 状态锁定 + 成绩计算 |
| **班级管理** | 管理员建班 → 分配教师 → 学生加入 → 关联课程 | 多实体关联 + 权限传递 |
| **请假审批** | 学生/家长申请 → 班主任审批 → 教务确认 → 状态同步 | 审批链路 + 状态联动 |

### 多角色验证要点

| 要点 | 说明 |
|------|------|
| **角色切换验证** | 同一流程用不同角色账号按顺序验证，不只用管理员走完 |
| **权限隔离** | 验证学生不能做教师操作、教师不能做管理员操作 |
| **数据可见性** | A 班学生看不到 B 班的作业和成绩 |
| **状态联动** | 教师发布课程后，学生侧是否实时可见 |
| **角色特有视图** | 同一个资源不同角色看到不同的字段和操作按钮 |

## 时序敏感流程

教育场景中有大量时间敏感的业务流程。

### 时间窗口类流程

| 流程 | 时间约束 | 联调要点 |
|------|---------|---------|
| **考试作答** | 开始时间~结束时间 | 未到时间不可进入/到时间自动提交/超时处理 |
| **作业提交** | 截止时间前可提交 | 截止后不可提交/迟交标记/宽限期处理 |
| **选课** | 选课时间窗口 | 窗口外不可选/名额满处理/并发选课 |
| **成绩查询** | 发布时间后可见 | 发布前不可见/发布动作触发可见性 |
| **学期切换** | 学期起止时间 | 旧学期数据归档/新学期初始化/跨学期数据隔离 |

### 时序验证方法

```
对时间敏感流程:
  1. 在时间窗口内验证正常操作
  2. 在时间窗口外验证拒绝/提示行为
  3. 验证临界点行为（恰好到时间的那一刻）
  4. 验证时区处理（客户端时区 vs 服务端时区）
  5. 验证自动触发行为（如考试结束自动提交）
  6. 验证并发场景（多人同时在截止时间提交）
```

## 外部系统对接

教育产品经常需要对接外部系统，增加联调复杂性。

### 常见外部对接

| 对接系统 | 对接内容 | 联调关注点 |
|---------|---------|-----------|
| **学信网/教育平台** | 学籍验证/成绩上报 | 数据格式/编码规范/接口可用性/超时处理 |
| **支付系统** | 学费/课程付费 | 支付回调/对账/退款/超时/重复支付 |
| **消息推送** | 短信/邮件/App推送 | 消息模板/发送时机/失败重试/频率限制 |
| **SSO/统一认证** | 单点登录/统一身份 | Token 交换/会话同步/登出联动/过期处理 |
| **视频/直播平台** | 在线课堂/录播回放 | 房间创建/推拉流/回放生成/并发量 |
| **AI 服务** | 智能批改/推荐/分析 | 接口延迟/结果格式/降级策略/异步回调 |

### 外部系统联调原则

| 原则 | 说明 |
|------|------|
| **mock 优先对接** | 先用 mock 跑通链路，再切换真实对接 |
| **明确标记** | mock 和真实对接的接口在报告中明确区分 |
| **超时处理** | 外部系统延迟不可控，验证超时和降级行为 |
| **幂等验证** | 外部回调可能重复，验证幂等处理 |
| **数据映射** | 内外系统字段映射/编码转换需逐字段验证 |
| **沙箱环境** | 优先使用外部系统提供的沙箱/测试环境 |

## 教育行业特有检查项

在标准联调检查清单基础上，教育项目额外检查：

| # | 检查项 | 说明 |
|---|--------|------|
| E1 | **多角色链路完整** | 关键流程用所有涉及角色逐一走完 |
| E2 | **权限隔离正确** | 不同角色只能看到/操作自己权限范围内的数据 |
| E3 | **时间窗口正确** | 时间敏感流程在窗口内外行为正确 |
| E4 | **数据隔离正确** | 班级/年级/学校之间数据正确隔离 |
| E5 | **外部对接标记清晰** | mock 和真实对接环节已明确标记 |
| E6 | **并发场景验证** | 选课/提交等高并发场景已验证 |
| E7 | **成绩计算正确** | 分数计算/统计/排名逻辑正确 |
| E8 | **学期边界处理** | 跨学期数据隔离/归档/初始化正确 |

## 教育场景常见集成问题

| 问题 | 根因 | 归因 |
|------|------|------|
| 教师发布课程后学生看不到 | 状态联动未实现/缓存未刷新 | 后端（状态联动）或前端（缓存刷新） |
| 学生提交作业后教师端无显示 | WebSocket/轮询未实现/数据查询条件错误 | 后端（通知机制）或前端（刷新逻辑） |
| 考试结束未自动提交 | 定时任务未触发/前端倒计时不准 | 后端（定时任务）或前端（倒计时逻辑） |
| 选课并发导致超额 | 并发控制缺失 | 后端（并发控制/锁机制） |
| 成绩计算不准确 | 浮点精度/计算公式/权重配置 | 后端（计算逻辑）或数据（权重配置） |
| A 班学生看到 B 班作业 | 数据隔离查询条件错误 | 后端（查询条件/权限过滤） |
| 跨时区时间显示错误 | 服务端/客户端时区处理不一致 | 前端（时区转换）或后端（时间存储格式） |
