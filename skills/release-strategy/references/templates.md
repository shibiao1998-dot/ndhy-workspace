### 典型需要记录的决策

- 是否按计划发布 vs 延期
- 是否继续灰度 vs 暂停/回退
- 试点学校的选择理由
- 某个功能是否纳入本次发布
- 发现问题后是"带病发布+后续修复"还是"延期修复后再发"

## 三、发布评审检查清单

发布策略制定完成后，用此清单做自查：

### 策略完整性

- [ ] 发布类型和规模已明确定义
- [ ] 发布窗口已���认（避开教育行业敏感时段）
- [ ] 灰度/试点方案已制定（非轻量发布必须有）
- [ ] 各阶段的通过标准已量化定义

### 用户准备

- [ ] 培训方案已制定且内容已准备
- [ ] 培训覆盖了���有目标用户角色
- [ ] 帮助文档/FAQ已更新
- [ ] 种子教师已识别和培养（大范围发布时）

### 沟通到位

- [ ] 内部所有相关团队已对齐
- [ ] 客服团队已培训且FAQ已到位
- [ ] 外部通知已准备且经过审核
- [ ] 更新日志已撰写（用户能看懂的语言）

### 风险可控

- [ ] 回退方案已制定且验证过
- [ ] 回退触发条件已明确
- [ ] 应急联系人已确认
- [ ] 数据迁移方案已充分测试（如涉及）

### 效果可量化

- [ ] 发布成功的标准已用数字定义
- [ ] 监控指标已确定且埋点已部署
- [ ] 复盘时间已排进日历
- [ ] 经验沉淀的方式已明确
