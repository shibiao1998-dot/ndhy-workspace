# 用户准备与培训

> 用户用不好，99%不是用户笨，是你没教好。教育产品的用户培训尤其重要——教师是最挑剔也最忙的用户群体。

## 一、培训策略设计

### 核心原则

1. **教师时间极其宝贵** — 培训越短越好、越直接越好
2. **不要假设信息化水平** — 从最基础的开始
3. **不要一次塞太多内容** — 分次学、分步学
4. **"能用起来"比"全部学会"重要** — 先教核心功能，其他用到再学
5. **最好的培训是"不需要培训"** — 产品本身设计得好，很多东西不需要教

### 培训对象分层

| 角色 | 培训重点 | 深度要求 |
|------|---------|---------|
| **学校管理员（信息化老师）** | 系统配置、权限管理、数据管理 | **最深入** — 他们是校内"第一道客服" |
| **学校领导** | 数据看板、管理功能 | **最短最直接** — 只需知道"怎么看结果" |
| **教师** | 日常教学相关功能 | **核心对象** — "对教学有什么用"比"功能怎么操作"重要 |
| **学生** | 学生端使用 | **通常不需要专门培训** — 产品本身要够简单，由教师在课堂引导 |

## 二、培训内容"三层蛋糕"

### 第一层：快速上手（必须的）

```
时长：5-10分钟
目标：只教"最核心的一件事"
形式：短视频（2-3分钟）+ 一页图文指南

设计要点：
  → 让教师在最短时间内获得"成就感"——"原来这么方便"
  → 聚焦一个完整操作闭环，从打开到完成
  → 例：如何用AI生成一份教案——从打开到生成到保存

评判标准：
  → 教师看完能独立完成核心操作
  → 不需要额外求助
```

### 第二层：日常使用（重要的）

```
时长：20-30分钟
目标：覆盖日常工作中会用到的主要功能
形式：录播课程 + 实操练习 + 常见问题FAQ

设计要点：
  → 按真实工作场景组织，不要按功能菜单组织
  → ✗ "AI功能怎么用"
  → ✓ "备课时怎么用AI帮忙"
  → 每个场景包含：场景描述 → 操作步骤 → 预期结果 → 常见问题

评判标准：
  → 覆盖教师80%的日常使用场景
  → 教师能独立处理常见问题
```

### 第三层：进阶技巧（锦上添花的）

```
时长：按需
目标：高级功能、效率技巧、个性化设置
形式：定期推送"小贴士"、进阶使用指南

设计要点：
  → 用了一段时间后再推送
  → 不强制要求学习
  → 基于用户的实际使用数据，推荐最相关的进阶内容
  → 例：用户已经熟练使用基础功能 → 推送"3个提高效率的技巧"

触发时机：
  → 用户使用核心功能达到一定频次后
  → 用户主动搜索帮助时
  → 产品迭代新增功能时
```

## 三、6种培训形式

| 形式 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **现场培训** | 效果最好，手把手教 | 成本最高，无法大规模 | 试点学校、大版本首发 |
| **线上直播培训** | 可以大规模，成本适中 | 互动性较弱，需组织同时在线 | 大范围推广阶段 |
| **录播视频** | 可反复观看，随时学习 | 没有互动，完成率低 | 所有阶段（作为补充） |
| **图文指南** | 制作快，查阅方便 | 不够直观 | 简单功能说明 |
| **产品内引导** | 在使用中学习，效果最自然 | 开发成本高 | 新功能引导 |
| **种子教师带动** | 教师教教师，说服力最强 | 需要找到合适的种子教师 | 全面推广阶段 |

### 推荐组合

| 场景 | 推荐形式组合 |
|------|-------------|
| **核心功能** | 产品内引导 + 短视频 |
| **试点阶段** | 现场培训 + 图文指南 |
| **全面推广** | 线上直播 + 录播视频 + 种子教师带动 |
| **新功能上线** | 产品内引导 + 社群通知 + 短视频 |

## 四、种子教师策略

> 这是教育产品推广中最有效的策略，没有之一。

### 为什么有效

- 教师对"同行"的信任远大于对"厂商"的信任
- "李老师说这个工具帮她每天省半小时"比任何广告都管用
- 种子教师能用教师听得懂的语言解释产品价值

### 种子教师选择标准

| 标准 | 说明 |
|------|------|
| **对新事物有好奇心** | 不一定技术能力强，但愿意试 |
| **在学校有影响力** | 其他教师愿意听他/她的 |
| **有表达能力** | 能把使用体验说清楚 |
| **有时间精力** | 不能选已经忙得不可开交的教师 |

### 种子教师培养方式

1. **提供更早、更深入的培训** — 比普通教师提前1-2周接触新功能
2. **建立专属社群** — 提供优先支持，问题24小时内响应
3. **给予身份认同** — "产品体验官""教育创新先锋"等称号
4. **给予实质回馈** — 优先试用新功能、教学资源赠送
5. **鼓励分享经验** — 校内教研活动、社群分享会

### 种子教师的数量建议

| 规模 | 建议数量 | 说明 |
|------|---------|------|
| 单个学校 | 3-5位 | 覆盖不同学科 |
| 区域推广 | 每校1-2位核心 | 形成区域种子网络 |
| 全面推广 | 按学校总数的5-10% | 确保每个学校至少1位 |

## 五、培训效果评估

培训做了不等于培训有效，要量化评估。

### 四层评估模型（柯氏评估模型简化版）

| 层次 | 问题 | 衡量指标 | 目标 | 评估时间 |
|------|------|---------|------|---------|
| **第一层：学了吗？** | 参与情况 | 培训参与率、完成率 | 参与率 > 80% | 培训当天 |
| **第二层：会了吗？** | 掌握程度 | 实操测试通过率/培训后一周内使用核心功能的比例 | > 70% | 培训后1周 |
| **第三层：用了吗？** | 实际使用 | 月活跃使用率、核心功能使用频次 | 月活 > 50% | 培训后1个月 |
| **第四层：受益了吗？** | 实际效果 | 工作效率提升、教学效果改善 | 定性+定量综合判断 | 培训后1-3个月 |

### 评估数据采集方式

| 层次 | 数据来源 |
|------|---------|
| 第一层 | 培训签到/在线平台播放数据 |
| 第二层 | 产品使用数据/简单测试 |
| 第三层 | 产品后台活跃数据 |
| 第四层 | 问卷调查 + 用户访谈 + 产品数据综合 |

### 评估结果的应用

```
参与率低 → 调整培训时间、形式、通知方式
掌握度低 → 优化培训内容，降低难度，增加实操
使用率低 → 检查产品本身是否有问题，优化使用引导
受益度低 → 反馈给产品团队，可能是功能本身需要优化
```
