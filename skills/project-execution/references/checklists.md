# 决策树与质量检查清单

> 本文件包含调度决策树（模块 3）和质量检查清单（模块 4）。

---

## §1 调度决策树

### 1.1 节点就绪判断

逐条检查，全部通过才可调度：

```
1. 所有依赖节点是否已完成？
   ├── 否 → ⏸️ 跳过，处理其他可执行节点
   └── 是 ↓

2. 上游产出物是否已通过质量检查？
   ├── 否 → ⏸️ 等待上游修正（记录阻塞原因）
   └── 是 ↓

3. 执行角色的设定（SOUL.md + STANDARDS.md）是否就绪？
   ├── 否 → 先获取/确认角色设定
   └── 是 ↓

4. 指令包是否已组装完成？
   ├── 否 → 按模板组装
   └── 是 ↓

5. 上下文总量是否可控？（指令包 + 角色设定 + 上游产出 < 上下文窗口 60%）
   ├── 否 → 执行上下文压缩后重新评估
   └── 是 → ✅ 就绪，可调度
```

### 1.2 串行 vs 并行决策

当有多个就绪节点时，全部满足以下条件才可并行：

```
1. 输入独立性：各节点依赖不同的上游产出，或依赖同一上游但只读不写？
   ├── 否 → 串行
   └── 是 ↓

2. 输出独立性：各节点的产出文件/目录完全不重叠？
   ├── 否 → 串行
   └── 是 ↓

3. 无实时交互：各节点不需要在执行过程中互相传递中间结果？
   ├── 否 → 串行
   └── 是 ↓

4. 失败隔离：一个节点失败不影响另一个节点的产出有效性？
   ├── 否 → 串行（或串行关键路径，并行非关键路径）
   └── 是 → ✅ 可并行调度
```

### 1.3 并行调度 4 要点

| # | 要点 | 操作 |
|---|------|------|
| 1 | **同时分发** | 并行节点的指令包在同一批次组装并发出 |
| 2 | **异步收取** | 先完成的先检查，不阻塞后完成的 |
| 3 | **产出一致性校验** | 全部完成后检查产出间是否存在矛盾（命名冲突、接口不匹配） |
| 4 | **单点失败不丢另一方** | 一个失败时，另一个的成功产出保留。失败节点单独重试 |

### 1.4 依赖冲突处理

| 场景 | 判断 | 处理 |
|------|------|------|
| 上游节点失败，下游等待中 | 检查失败类型 | 可重试 → 重试上游；根本性 → 暂停下游，上报 |
| 两个并行节点产出矛盾 | 一致性校验不通过 | 分析矛盾原因，选择修正成本更低的一方重做 |
| 循环依赖 | 排期阶段检测 | 流程设计缺陷 → 上报，不自行解决 |
| 可选依赖 | 上游标注为可选 | 上游完成 → 注入；上游失败 → 跳过，指令包注明"无此输入，用默认方案" |

## §2 质量检查清单

> 每个节点产出返回后，依序执行两层检查。第一层必做，第二层按需。

### 2.1 第一层：形式检查（必做）

| # | 检查项 | 判断标准 | 不通过处理 |
|---|--------|----------|------------|
| 1 | **文件存在** | 产出文件在指定路径？ | 要求补交 |
| 2 | **格式正确** | 文件格式符合要求？能正常解析？ | 打回重做 |
| 3 | **完整性** | 所有要求的产出项齐全？无缺失章节/模块？ | 列出缺失项，要求补全 |
| 4 | **量级合理** | 长度/规模在合理范围？ | 评估是否偷工减料 |
| 5 | **编码正常** | 无乱码、无异常字符、中英文标点一致 | 标注问题位置，要求修正 |
| 6 | **命名规范** | 文件名、目录名符合项目约定？ | 要求重命名 |

> **第一层全部通过后，才进入第二层。**

### 2.2 第二层：内容检查（按需深度不同）

| # | 检查项 | 判断标准 | 不通过处理 |
|---|--------|----------|------------|
| 1 | **目标一致** | 产出内容回答了指令包"本节点任务说明"的核心问题？ | 方向性问题 → 重新调度 |
| 2 | **呼应上游** | 正确引用/基于了上游产出物？无曲解？ | 标注曲解点，附修正指引重做 |
| 3 | **内部无矛盾** | 产出物内部各部分自洽？ | 标注矛盾点，要求修正 |
| 4 | **满足下游** | 能直接作为下游节点的输入？格式/内容够用？ | 评估缺什么，要求补充 |
| 5 | **验收标准逐条** | 逐条对照验收标准，每条是否达标？ | 列出未达标项，附差距说明 |
| 6 | **无越界** | 未超出指令包边界？未替下游做不该做的事？ | 删除越界部分，保留有效部分 |

### 2.3 质量判定与处理

| 判定 | 标准 | 处理 |
|------|------|------|
| ✅ **合格** | 两层检查全部通过 | 更新产出记录 → 生成摘要 → 继续下一节点 |
| 🟡 **小问题** | 形式通过，内容有 1-2 处小瑕疵不影响方向 | 自行修正并标注修改点，不重新调度 |
| 🔴 **方向性问题** | 内容检查中目标不一致或严重曲解上游 | 重新调度 + 修正指引（问题描述 + 正确方向）。最多 2 次 |
| ⛔ **根本性问题** | 流程设计缺陷、需求本身有歧义 | 暂停该节点，上报用户决策 |
