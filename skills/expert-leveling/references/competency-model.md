# 通用胜任力模型 — 5 维度 × 13 级速查

> 来源：D2 通用胜任力模型

## 维度总览

| # | 维度 | 权重 | 定义 |
|---|------|------|------|
| C1 | 产出完整度 | 20% | 交付物覆盖了多少该覆盖的内容 |
| C2 | 复杂度承接 | 15% | 能独立处理多复杂的任务 |
| C3 | 决策质量 | 15% | 自主选择是否合理、有据、可追溯 |
| C4 | 可消费性 | 10% | 下游能否直接使用 |
| C5 | 影响力半径 | 10% | 产出的价值辐射范围 |

---

## C1 产出完整度（20%）

| 等级 | 行为描述 |
|------|---------|
| P0 | 仅覆盖指令显式提及的主路径，覆盖度≤60%，≥2处关键遗漏需补全 |
| P1 | 覆盖指令全部要求，覆盖度≥80%，指令未提及的边界未处理 |
| P2 | 覆盖指令+常见边界（空值、异常输入、权限缺失），无需审查即可直接使用 |
| P3 | 附自检清单，覆盖度≥90%（主路径+边界+异常），附下游消费说明 |
| P4 | 模糊任务下自主拆解决定覆盖范围，与明确指令下差异≤20% |
| P5 | 显式说明"为什么覆盖这些、不覆盖那些"，边界判断有据 |
| P6 | 预判并覆盖下一步扩展点≥1个，预判被后续验证有效 |
| P7 | 包含可复用的完整度检查标准/模板，被≥2次任务采用 |
| P8 | 完整度标准覆盖≥2个领域交叉地带 |
| P9 | 定义端到端完整度体系，每环节有检查点，经≥1个完整项目验证 |
| P10 | 体系被全组织采纳，遗漏率下降≥30% |
| P11 | 方法论具备行业通用性 |
| P12 | 提出全新完整度范式，行业之前无同类方法 |

## C2 复杂度承接（15%）

| 等级 | 行为描述 |
|------|---------|
| P0 | 仅能处理结构化指令（输入/输出/步骤均明确），遇歧义停滞 |
| P1 | 处理结构化指令稳定，小幅歧义能正确推断意图 |
| P2 | 处理含1-2个未说明边界条件的任务，自主补全常见场景 |
| P3 | 处理标准复杂度任务（主路径+边界+异常均自主覆盖），产出稳定 |
| P4 | 处理模糊任务——仅给目标和约束，自主拆解+规划+交付 |
| P5 | 处理含多决策点的复杂任务，每个决策点有替代方案对比 |
| P6 | 处理需前瞻性判断的任务，识别并处理潜在未来需求 |
| P7 | 处理需制定标准的任务，从实践中抽象可复用规则 |
| P8 | 处理跨≥2个领域的交叉任务，识别矛盾并产出整合方案 |
| P9 | 处理端到端体系构建任务，≥3个协调子系统覆盖完整链路 |
| P10 | 处理组织级变革任务，可测量效率/质量提升≥30% |
| P11 | 处理行业级命题，产出可被其他组织复用 |
| P12 | 处理开创性命题，定义新范式 |

## C3 决策质量（15%）

| 等级 | 行为描述 |
|------|---------|
| P0 | 未做自主决策，遇选择跳过或随机选择，无说明 |
| P1 | 简单二选一做出合理选择，无理由说明 |
| P2 | 常见场景决策合理，符合行业惯例 |
| P3 | 所有决策附一句话理由，可追溯到指令要求或行业惯例 |
| P4 | 关键决策点列出≥2选项并选择其一，附依据 |
| P5 | 关键决策附完整替代方案对比（≥2方案优劣），理由引用数据/标准/先例 |
| P6 | 决策含风险预判——潜在风险+缓解措施，标注重新评估条件 |
| P7 | 从多次决策中提炼可复用决策框架，被≥2次任务采用 |
| P8 | 决策覆盖跨域权衡，≥2个领域约束均显式纳入 |
| P9 | 构建系统性决策体系，覆盖完整链路各环节 |
| P10 | 决策体系被全组织采纳，决策趋同率≥80% |
| P11 | 决策方法论具备行业通用性 |
| P12 | 提出全新决策范式 |

## C4 可消费性（10%）

| 等级 | 行为描述 |
|------|---------|
| P0 | 需下游≥2处补充/修正，格式不符预期 |
| P1 | 格式正确结构完整，部分内容需下游推断补全 |

| P4 | 结构适配下游消费习惯（开发看代码示例，老板看摘要） |
| P5 | 标注下游可能疑问并预先解答，关键决策附上下文 |
| P6 | 预留下游扩展接口，续接无需重构 |
| P7 | 包含面向下游的标准化接口规范/模板，被≥2个下游采用 |
| P8 | 同时适配≥2类不同下游，各方无需二次加工 |
| P9 | 构建端到端产出交接体系，全链路零损耗 |
| P10 | 交接体系被全组织采纳，返工率下降≥30% |
| P11 | 方法论具备行业通用性 |
| P12 | 提出全新交接范式 |

| 等级 | 行为描述 |
|------|---------|

## 打分规则

### 稳定水平判定

| 记录数 | 规则 |
|--------|------|
| ≥5 次 | 最近 5 次中≥3 次达到 = 稳定达到 |
| 3-4 次 | ≥60% 达到 = 稳定达到 |
| 1-2 次 | 按实际打分，标注"数据不足" |

### 综合分数计算

```
综合分数 = C1×0.20 + C2×0.15 + C3×0.15 + C4×0.10 + C5×0.10 + 领域加权分
```

领域加权分：
- 1 个专属维度：D1×0.30
- 2 个专属维度：D1×权重 + D2×权重（和=0.30）
- 无专属标准：各维度分数 ÷ 0.70

### 等级判定

- 等级 = floor(综合分数)
- **地板规则**：min(全部维度) < 等级-2 → 等级 = min+2

### 短板处理

| 最高与最低差距 | 处理 |
|--------------|------|
| ≤2 级 | 正常 |
| 3 级 | 标注"短板警告：[维度]偏低" |
| ≥4 级 | 触发地板规则，标注"短板限制：[维度]拖低定级" |
