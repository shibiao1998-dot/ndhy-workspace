## 单次评估记录模板

存放路径：`projects/expert-leveling/eval-records/[专家英文ID]/[YYYY-MM-DD]-[任务简称].md`

```markdown
## 评估记录

- 专家：[emoji + 角色名]
- 任务：[任务简述，≤30 字]
- 日期：[YYYY-MM-DD]
- 项目：[项目名]

### 通用维度评分

| 维度 | 本次表现等级 | 判定依据（≤50 字） |
|------|------------|-------------------|
| C1 产出完整度 | P? | [具体证据] |
| C2 复杂度承接 | P? | [具体证据] |
| C3 决策质量 | P? | [具体证据] |
| C4 可消费性 | P? | [具体证据] |
| C5 影响力半径 | P? | [具体证据] |

### 领域维度评分

| 维度 | 本次表现等级 | 判定依据（≤50 字） |
|------|------------|-------------------|
| D1 [维度名] | P? | [具体证据] |
| D2 [维度名]（如有） | P? | [具体证据] |

### 短板标记
- 维度间差距 ≥3 级：[是/否]，涉及维度：[...]
- 地板规则触发：[是/否]

### 备注
[任何影响本次评分的特殊情况]
```

**填写规则**：
- 单条记录 ≤500 字
- 判定依据用事实不用形容词 — 写"覆盖主路径+3个边界+异常处理"，不写"完整度较高"
- 无领域标准卡片时删除"领域维度评分"整节

## 晋升提案模板

存放路径：晋升批准后存入 `projects/expert-leveling/eval-records/[专家英文ID]/promotions/[YYYY-MM-DD]-P?toP?.md`

```markdown
## 晋升提案

### 基本信息
- 专家：[emoji + 角色名]
- 当前等级：P? → 提议等级：P?
- 是否跨段：[是（[段名]→[段名]）/ 否]
- 评估周期：[起止日期]
- 评估记录数：[N 次]

### 晋升理由（≤200 字）
[用老板能理解的语言概括：这个专家做了什么、达到了什么水平、为什么该升级]

### 关键证据（≤3 条）
1. [最能说明等级提升的产出事实，附项目名和任务名]
2. [第二条证据]
3. [第三条证据]（可选）

### 综合评分摘要
| 维度 | 稳定水平 |
|------|---------|
| C1 产出完整度 | P? |
| C2 复杂度承接 | P? |
| C3 决策质量 | P? |
| C4 可消费性 | P? |
| C5 影响力半径 | P? |
| D1 [领域维度名] | P? |
| D2 [领域维度名]（如有） | P? |
| **综合分数** | **X.XX → P?** |

### 晋升后变化
- SOUL.md：等级标识更新为 [P?]，[身份描述/信条/性格/风格 的具体变化]
- STANDARDS.md：[新增/提升/合并的标准条目说明]

### Leader 建议
[通过 / 暂缓（附理由）]
```

**关键要求**：
- 整个提案 ≤500 字
- "晋升理由"用非专业语言，不写"C2维度稳定P5"，写"连续3次独立拆解模糊任务并高质量交付"
- "关键证据"指向具体项目和任务，不写"多次表现优异"
