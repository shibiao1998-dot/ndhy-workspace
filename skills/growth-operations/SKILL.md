# growth-operations -- 增长运营执行工具箱

将产品定义转化为可持续的用户增长——获取、激活、留存、活跃、传播，形成增长飞轮。

> 增长不是"拉人头"，而是"让更多人真正用起来并持续获得价值"。100个每天都在用的教师，比10000个注册了没打开过的账号有价值得多。

## 6 步标准流程

```
Step 1: 诊断现状
  |  当前增长瓶颈在哪？（获取？激活？留存？活跃？）
  |  -> 关键指标现状和趋势
  |  -> 用户反馈中的高频问题
  |  -> 漏斗各环节转化率
  |  -> 信息不足 -> 要求补充数据（不猜测）
  |
Step 2: 定位机会
  |  -> 漏斗中转化率最低的环节 = 最大的增长机会
  |  -> "如果只能做一件事来提升增长，应该做什么？"
  |  -> 投入产出比最高的方向
  |  -> 详细方法 -> references/metrics-model.md
  |
Step 3: 制定策略
  |  -> 明确目标（量化 + 有时间限制）
  |  -> 确定策略方向和具体打法
  |  -> 制定资源需求和时间计划
  |  -> 获取策略 -> references/acquisition-activation.md
  |  -> 留存策略 -> references/retention-engagement.md
  |  -> 区域策略 -> references/activities-content.md 区域化增长
  |
Step 4: 设计执行方案
  |  -> 拆分为可执行的具体动作
  |  -> 明确每个动作的负责人和交付时间
  |  -> 设定过程指标（不只看最终结果）
  |  -> 实验方案 -> references/experimentation.md
  |  -> 活动方案 -> references/activities-content.md
  |
Step 5: 执行与迭代
  |  -> 快速执行，小步快跑
  |  -> 每周回顾数据，判断方向是否正确
  |  -> 有效的加大投入，无效的果断停止
  |  -> 教育行业节奏 -> references/education-growth.md
  |
Step 6: 复盘与沉淀
     -> 什么有效、什么无效、为什么
     -> 有效策略标准化、可复制化
     -> 更新增长策略知识库
     -> 按模板输出 -> references/templates.md
```

## 五条铁律速查

| # | 铁律 | 一句话 |
|---|------|--------|
| 1 | **留存优先于获取** | 留不住就不要拉——拉来了流失，还败坏口碑。获取成本越来越高，留存才是复利 |
| 2 | **真实使用优先于数字** | 100 个每天使用的教师 > 10000 个注册账号。指标好看但用户没在用 = 指标选错了 |
| 3 | **教师时间是稀缺资源** | 一切运营动作都要问"占教师多少时间？值得吗？""省时间"的比"花时间"的有效 100 倍 |
| 4 | **口碑是核心引擎** | 一个真实好评 > 100 篇营销软文。口碑建立靠时间，毁灭只需一次。保护口碑 = 保护增长根基 |
| 5 | **增长要有耐心** | 教育产品是阶梯式增长——每攻下一个区域上一个台阶。急功近利在教育行业必然反噬 |

## 教育增长飞轮速查

```
教师用得好 → 教学出效果 → 学校愿意推 → 同行来学习 → 更多教师用
     ↑                                                    |
     └────────────────────────────────────────────────────┘

驱动力 = 示范效应（不是网络效应）
增长节点 = 学校/区域（不是个人）
增长曲线 = 阶梯式（不是指数级）
增长刹车 = "用不起来"（不是竞品）
```

**启动条件**：至少 1 个教师真实用好的案例 + 效果可量化 + 有传播渠道

-> 详细模型 [references/metrics-model.md](references/metrics-model.md)

## AARRR 速查

| 环节 | 核心指标 | 教育产品重点 |
|------|---------|-------------|
| **获取** Acquisition | 新学校接入数、新教师注册数 | 10 所有需求的学校 > 100 所凑数的 |
| **激活** Activation | 注册→核心操作转化率、啊哈时刻到达率 | 注册后 7 天未激活 = 大概率流失 |
| **留存** Retention | 周留存率、学期留存率 | 看周不看日，跨学期还在用才算留住 |
| **活跃** Engagement | WAU、核心功能使用率、功能深度 | 区分"登录"和"真正使用" |
| **传播** Referral | 校际推荐率、校内传播率、NPS | 口碑推荐 = 教育行业最有效的获取渠道 |
| **变现** Revenue | 续费率、LTV/CAC | 续费比新签重要得多 |

**北极星指标选择**：探索期→种子用户周活跃留存率 | 成长期→WAU | 规模期→活跃学校数×校均活跃教师占比

-> 详细指标体系 [references/metrics-model.md](references/metrics-model.md)

## 用户分层速查

| 层级 | 定义 | 运营策略 | 目标 |
|------|------|---------|------|
| **深度用户** | 周使用 3+ 次 + 高级功能 | 荣誉、展示、收集反馈、发展 KOL | 维护 + 杠杆 |
| **常规用户** | 周使用 1-2 次 + 基础功能 | 推荐进阶功能、使用技巧 | → 深度 |
| **浅度用户** | 月使用几次 | 了解原因、针对性干预、一对一帮助 | → 常规 |
| **沉睡用户** | 30 天+ 未使用 | 召回策略（产品更新/同伴成果/学期节点） | 尝试召回 |
| **流失用户** | 60 天+ 未使用 | 流失原因分析，无重大更新不强拉 | 分析原因 |

**按角色**：校长→数据报表 | 教研组长→协作共享 | 骨干教师→高级功能 | 普通教师→稳住基础

-> 详细分层运营 [references/retention-engagement.md](references/retention-engagement.md)

## 反模式速查

| # | 反模式 | 错误做法 | 正确做法 |
|---|--------|---------|---------|
| 1 | **虚假繁荣** | 注册量大但活跃率极低 | 关注激活率和留存率，不只看注册量 |
| 2 | **重获取轻留存** | 不断拉新，用户不断流失 | 留存率未达标之前不加大获取投入 |
| 3 | **一刀切运营** | 对所有用户用同样策略 | 用户分层，差异化运营 |
| 4 | **活动自嗨** | 活动很多但指标没影响 | 每个活动必须有明确的指标目标 |
| 5 | **忽略学期节奏** | 在不合适的时间做运营 | 运营日历和教学日历对齐 |
| 6 | **依赖单一渠道** | 全靠一个渠道获客 | 建立多渠道获取体系 |
| 7 | **数据不看** | 凭感觉做决策 | 建立数据看板，每周看数据 |

## 合格增长策略检查清单

| # | 检验项 | 通过条件 |
|---|--------|---------|
| 1 | **瓶颈已识别** | 当前增长瓶颈有数据支撑，不是猜测 |
| 2 | **北极星指标已选** | 选择了反映真实价值的北极星指标，有选择理由 |
| 3 | **AARRR 已建模** | 各环节指标已定义，漏斗转化率已测算 |
| 4 | **策略聚焦** | 一个阶段聚焦 1-2 个核心问题，不是全面撒网 |
| 5 | **目标可量化** | 有明确的数字目标和时间限制 |
| 6 | **执行可落地** | 拆分为具体动作，有负责人和交付时间 |
| 7 | **实验有设计** | 关键假设有对应的实验方案 |
| 8 | **用户已分层** | 有明确的用户分层和差异化运营策略 |
| 9 | **学期节奏已考虑** | 策略和教学日历对齐（教育产品必检） |
| 10 | **复盘有机制** | 有定期的数据回顾和策略调整节奏 |

## 协作接口

### 上游输入

| 来源 | 接收内容 | 使用方式 |
|------|---------|---------|
| 产品定义（产品经理） | 产品定义、功能范围、目标用户 | 制定增长策略的基础 |
| 发布策略（发布专家） | 发布节奏、渠道规划 | 协调获取时机和渠道 |

| 接收方 | 传递内容 | 约束关系 |
|--------|---------|---------|

## 工作模式决策树

```
收到增长运营任务
  |- 小范围任务（单指标诊断/单活动策划/数据分析）
  |   -> 单独执行
  |
  |- 明确边界的专项任务（获取方案/留存优化/实验设计）
  |   -> 作为 subagent 被调用（默认模式）
  |
  +- 大 feature / 全面增长体系搭建
      |- 需要产品改动/技术支撑/市场资源？
      |   -> Agent Team（拉产品/技术/市场专家协作）
      +- 否 -> subagent 模式 + 输出后交相关专家执行
```

## 触发时首要动作

无论如何触发，第一步永远是：

> 1. 确认增长阶段——探索期？成长期？规模期？策略完全不同
> 2. 确认当前瓶颈——获取？激活？留存？活跃？先聚焦一个点
> 3. 确认数据基线——现状是什么？目标是什么？差距多大？
> 4. 确认可用资源——人、钱、时间、渠道，有什么牌可以打？
> 5. 根据以上判断 → 选择工作深度 → 按 6 步流程执行

**教育行业项目**：额外读取 [references/education-growth.md](references/education-growth.md) 了解行业特殊约束。
