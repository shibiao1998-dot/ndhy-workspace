# 增长实验与数据驱动

## 一、增长实验框架

### 实验循环

```
提出假设 → 设计实验 → 执行 → 分析结果 → 决策
    ↑                                      |
    └──── 新假设 ←── 学到了什么 ←──────────┘
```

### 假设的标准格式

```
"如果我们做了 [X 改变]，
 目标用户的 [Y 行为] 会提升 [Z%]，
 因为 [原因]。"
```

**示例**：
> "如果我们在注册后立刻推荐一个备课模板，新用户的首日激活率会提升 20%，因为模板降低了'不知道从哪开始'的门槛。"

### 实验四铁律

| # | 铁律 | 说明 |
|---|------|------|
| 1 | **一次只改一个变量** | 否则不知道是什么起了作用 |
| 2 | **样本量要够** | 教育产品用户量不大，注意统计显著性 |
| 3 | **给够时间** | 至少跑一个完整的教学周 |
| 4 | **正面结果要复验，负面结果也要分析原因** | 不因单次结果过早下结论 |

### 实验优先级排序框架（ICE）

| 维度 | 含义 | 评分 1-10 |
|------|------|----------|
| **I**mpact（影响力） | 这个实验如果成功，对核心指标的影响有多大？ | |
| **C**onfidence（信心度） | 基于已有数据和认知，你对假设成立有多大把握？ | |
| **E**ase（容易度） | 实现和执行这个实验需要多少资源？ | |

**ICE 得分 = (I + C + E) / 3**，优先做得分最高的实验。

## 二、关键增长实验方向

### 激活优化类

| 实验方向 | 假设示例 | 核心指标 |
|---------|---------|---------|
| 注册流程简化 | 减少注册字段 → 注册完成率 ↑ | 注册完成率 |
| 新手引导 A/B 测试 | 任务式引导 vs 视频引导 → 激活率 ↑ | 首日激活率 |
| 首个任务推荐策略 | 推荐备课模板 vs 推荐AI功能 → 啊哈到达率 ↑ | 啊哈时刻到达率 |
| 欢迎消息优化 | 不同内容/时机 → 首日操作率 ↑ | 首日核心操作率 |

### 留存优化类

| 实验方向 | 假设示例 | 核心指标 |
|---------|---------|---------|
| 使用提醒频率 | 每日提醒 vs 每周提醒 → 周活跃 ↑ | 周活跃率 |
| 进阶功能推荐时机 | 第2周推荐 vs 第4周推荐 → 功能采用 ↑ | 进阶功能使用率 |
| 周报/月报内容 | 数据型报告 vs 成就型报告 → 打开率 ↑ | 报告打开率 |
| 社交功能引入 | 协作功能 → 团队使用率 ↑ | 校内扩散率 |

### 传播优化类

| 实验方向 | 假设示例 | 核心指标 |
|---------|---------|---------|
| 分享功能设计 | 一键分享案例 → 分享率 ↑ | 内容分享率 |
| 邀请奖励机制 | 邀请奖励 → 校内扩散 ↑ | 邀请转化率 |
| 案例包装角度 | 效率角度 vs 教学质量角度 → 传播 ↑ | 案例阅读/转化率 |
| 校内推广方案 | 骨干教师带动 vs 行政推动 → 渗透 ↑ | 校均活跃教师数 |

## 三、A/B 测试在教育产品中的难点

### 挑战与应对

| 挑战 | 原因 | 应对策略 |
|------|------|---------|
| **用户量小** | 教育产品用户规模有限 | 延长实验周期、降低显著性要求（p < 0.1）、使用贝叶斯方法 |
| **周期长** | 教学效果需要时间验证 | 用过程指标（如激活率）替代结果指标（如学期留存） |
| **干扰因素多** | 学期节奏、考试、假期影响 | 控制时间窗口，避开特殊节点 |
| **分组困难** | B 端以学校为单位 | 按学校分组而非个人分组 |
| **伦理顾虑** | 教育产品影响教学 | 确保对照组不受负面影响，实验后推广到全部用户 |

### 教育产品 A/B 测试的最佳实践
1. **以学校为分组单位**：避免同校教师体验不一致
2. **实验周期 ≥ 1 个教学周**：匹配教学节奏
3. **设置护栏指标**：确保实验不影响核心使用体验
4. **记录外部因素**：标注实验期间的学期节点、政策变化等
5. **小范围验证→大范围推广**：先在 2-3 所学校验证

## 四、数据分析方法

### 同期群分析（Cohort Analysis）

**用途**：判断产品改进是否真的改善了留存。

```
做法：
1. 把同一时期进来的用户分为一组（如"3月第1周注册的用户"）
2. 追踪每组用户在后续各周/各月的留存和活跃
3. 对比不同时期的群组，看趋势是否在改善

关键看点：
→ 新群组的留存曲线是否比老群组更好？（产品改进有效）
→ 留存曲线是否趋于平稳？（找到了稳定用户群）
→ 不同渠道来的群组，留存差异有多大？（渠道质量评估）
```

### 漏斗分析

**用途**：找到转化率最低的环节，集中资源优化。

```
做法：
1. 把用户行为路径拆成步骤
2. 计算每个步骤的转化率
3. 找到流失最严重的环节

典型教育产品漏斗：
注册 → 首次登录 → 进入核心场景 → 完成首个任务 → 第二次使用 → 形成习惯
 100%    85%        60%            35%            20%          10%
                     ↑
               这里流失最多 = 最大优化机会
```

### 用户路径分析

**用途**：了解用户实际行为 vs 预期行为的差异。

```
关键问题：
→ 用户在产品里实际的行为路径是什么？
→ 和你设计的"理想路径"有什么差异？
→ 用户自发探索出了你没想到的用法吗？
→ 最常见的"退出点"在哪里？
```

### 相关性分析

**用途**：找到影响长期留存的关键早期行为。

```
方法：
→ 哪些早期行为和长期留存强相关？
→ 例："注册后7天内完成3次备课的用户，月留存率高出2倍"
→ 这个"3次备课"就是激活的关键目标

注意：
→ 相关性 ≠ 因果性
→ 需要通过实验验证因果关系
→ 但相关性分析能帮你找到实验方向
```

### 数据分析的输出标准

每次数据分析都应输出：
1. **发现了什么**（数据事实）
2. **意味着什么**（业务解读）
3. **建议做什么**（可执行的下一步）
4. **怎么验证**（如何确认建议有效）
