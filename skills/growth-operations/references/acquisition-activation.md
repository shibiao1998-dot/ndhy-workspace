# 用户获取与激活

## 一、四梯队获取渠道矩阵

### 第一梯队：口碑与转介绍（最有效、成本最低）

| 策略 | 具体动作 | 关键成功因素 |
|------|---------|-------------|
| 成功案例传播 | 把"用得好的学校"包装成可传播的案例 | 数据真实、效果可量化 |
| 教师社群分享 | 在教师群中做经验分享 | 自然分享，不是硬广 |
| 教研活动露出 | 在教研活动中展示产品使用场景 | 嵌入教学场景，不是单独推销 |
| 核心动作 | 把"用得好的学校"变成"会说话的案例" | 持续产出、定期更新 |

### 第二梯队：教育系统内渠道

| 策略 | 具体动作 | 关键成功因素 |
|------|---------|-------------|
| 行政推荐 | 教育局/教研院推荐或项目合作 | 和教育行政体系建立信任关系 |
| 培训植入 | 教师培训项目中的产品植入 | 作为培训工具，不是推销对象 |
| 展会论坛 | 教育信息化展会/论坛参展 | 重互动体验，轻纸面宣传 |

### 第三梯队：内容获取

| 策略 | 具体动作 | 关键成功因素 |
|------|---------|-------------|
| 资源分享 | 教学资源/模板免费分享 | 先给价值，再引导到产品 |
| 干货传播 | 教育干货内容传播 | 内容本身有价值，不是软文 |
| 成长内容 | 教师成长类内容 | 长期持续，建立信任 |

### 第四梯队：商务拓展

| 策略 | 具体动作 | 关键成功因素 |
|------|---------|-------------|
| 区域代理 | 发展区域代理商 | 选对合作伙伴，管好服务质量 |
| 系统集成 | 与系统集成商合作 | 产品接口标准化 |
| 硬件捆绑 | 与硬件厂商捆绑 | 确保用户体验不降级 |

### 不推荐的渠道
- ❌ **大规模线上广告投放**：教育产品决策链复杂，广告 ROI 通常很差
- ❌ **低价促销拉新**：吸引来的是"贪便宜的"而不是"有需求的"

## 二、"以校为单位"的获取策略

### 核心认知
> 教育产品的获取不是 C 端"一个个拉用户"，而是 B 端"一所所拓学校"。

### 校级获取关键路径

```
Step 1: 找到切入点人物
  → 校长（决策权）
  → 信息中心主任（执行权）
  → 骨干教师（影响力）

Step 2: 给试用理由
  → 不是"产品功能多强大"
  → 而是"能解决你的某个具体问题"

Step 3: 降低试用门槛
  → 免费试用期
  → 不需要复杂部署
  → 有人帮着上手

Step 4: 校内扩展
  → 一个人用 → 一群人用 → 全校用
```

### 校际扩展路径
```
一所学校 → 同区域学校（校长圈子互相推荐）
    ↓
一个区域 → 教育局关注 → 区域推广
    ↓
一个区域成功 → 复制到其他区域
```

### 校级获取的常见误区
1. ❌ 只搞定校长，不管教师 → 签约了但用不起来
2. ❌ 功能演示代替场景解决 → 校长听不懂技术语言
3. ❌ 急于全校铺开 → 应该先做好一个试点组

## 三、种子用户获取

### 种子用户画像
| 特征 | 说明 |
|------|------|
| 信息化热情 | 不是被迫用的，主动拥抱新工具 |
| 教学追求 | 想把课上得更好，追求教学创新 |
| 容忍度高 | 愿意尝试不完美的产品 |
| 有影响力 | 能带动周围人，在学校有话语权 |

### 寻找渠道
- 教育信息化社群和论坛
- 参加过信息化培训的骨干教师
- 教育创新比赛的参与者
- 已有合作学校中的活跃教师

### 留住种子用户的三板斧
1. **特权感**：优先体验新功能、专属沟通渠道
2. **参与感**：认真听反馈，让他们觉得在"参与打造产品"
3. **荣誉感**：给展示机会和荣誉——不是给钱

### 种子用户的杠杆价值
```
种子教师用得好
  → 产出教学案例
  → 在教研活动中分享
  → 影响同校/同区域教师
  → 飞轮启动
```

## 四、定义"啊哈时刻"

### 什么是啊哈时刻
> "啊哈时刻"= 用户第一次真正感受到产品价值的瞬间

### 教育产品典型啊哈时刻
| 场景 | 啊哈瞬间 | 用户内心独白 |
|------|---------|-------------|
| 备课 | 用产品完成一次备课 | "真的比以前快！" |
| 作业 | 用产品布置一次作业 | "数据自动就出来了！" |
| AI 教案 | 用 AI 生成一份教案 | "比自己写的还有启发！" |
| 数据 | 看到一张数据报表 | "以前要收一周的数据，现在实时看！" |

### 找到啊哈时刻的方法

| 方法 | 具体做法 |
|------|---------|
| 数据分析 | 对比留存用户和流失用户的行为差异 |
| 用户访谈 | 问留存用户"什么时候决定继续用的？" |
| 行为规律 | 完成"完整任务闭环"的用户，留存率显著更高 |

### 啊哈时刻的验证标准
- 到达啊哈时刻的用户，留存率比未到达的高 **2 倍以上**
- 啊哈时刻可以在 **首次使用中** 到达（不需要多次使用）
- 啊哈时刻对 **不同角色** 可能不同（教师 vs 管理者）

## 五、激活路径设计

### 核心原则
> 最短路径到达啊哈时刻。新用户的前 30 分钟决定生死。

### 激活路径设计要点

| 环节 | 原则 | 做法 |
|------|------|------|
| 注册 | 能多简单就多简单 | 不要在注册时问一堆问题 |
| 首页 | 不给空白页面 | 给一个"你可以先试试这个" |
| 引导 | "做"而不是"看" | 让用户动手，不是看教程 |
| 首个任务 | 小、快、有成就感 | 降低完成门槛 |
| 反馈 | 即时正向反馈 | "你已完成第一个XX，比80%新用户都快" |

### 激活漏斗
```
注册 → 完善基本信息 → 进入核心场景 → 完成首个任务 → 看到效果
 ↓        ↓               ↓             ↓           ↓
每个环节都要测转化率，找到流失最严重的环节重点优化
```

### 新手引导策略对照

| ❌ 不要做 | ✅ 要做 |
|----------|--------|
| 一打开就弹出操作指引蒙层 | 任务式引导："完成这3个步骤就上手了" |
| 强制用户看完引导才能使用 | 模板/示例："不知道怎么开始？试试这个" |
| 纯文字帮助文档 | 渐进式：先教最核心的一个操作 |
| — | 同伴引导："李老师是这样用的..." |

## 六、72 小时法则

### 核心规律
> 用户注册后 72 小时内没有完成激活，后续激活概率断崖下降。

### 黄金窗口触达计划

| 时间节点 | 触达动作 | 目标 |
|---------|---------|------|
| 注册后即刻 | 欢迎 + 引导完成第一个操作 | 立即进入核心场景 |
| 注册后 24h（未激活） | 提醒 + 提供帮助 | 消除使用障碍 |
| 注册后 48h（仍未激活） | 推送成功案例 + 提供一对一帮助 | 用同伴案例激励 |
| 注册后 72h（仍未激活） | 最后一次主动触达 | 最后机会 |
| 72h 后 | 转入低频触达 | 不骚扰，保留可能 |

### B 端学校用户的调整
- 窗口可延长到"首周"，但逻辑一样
- 校级用户需要考虑学校的部署和培训周期
- 关键是"首次真正使用"而非"首次登录"

### 触达内容设计原则
1. **有用**：每次触达都给用户带来价值，不是纯提醒
2. **递进**：从自助到一对一，逐步加重服务力度
3. **个性化**：根据用户角色和行为定制内容
4. **不骚扰**：超过 72h 未激活，降低频率而非增加
