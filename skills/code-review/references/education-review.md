# 教育行业代码审查特别关注

教育行业项目在代码审查中需要额外关注以下领域。本文件作为标准审查流程的补充，在教育行业项目中额外加载。

---

## 一、隐私合规

### 学生数据保护

| 检查项 | 具体审查内容 | 默认分级 |
|--------|-------------|---------|
| 未成年人数据 | 涉及未成年学生的个人信息是否有额外保护措施 | 🔴 P0 |
| 家长同意 | 收集学生信息前是否有家长/监护人同意机制 | 🔴 P0 |
| 数据最小化 | 是否只收集必要的学生信息，无过度采集 | 🟠 P1 |
| 数据保留 | 学生数据是否有明确的保留期限和删除策略 | 🟠 P1 |
| 数据导出 | 是否支持学生/家长的数据导出请求 | 🟡 P2 |

### 教师与员工数据

| 检查项 | 具体审查内容 | 默认分级 |
|--------|-------------|---------|
| 教师信息保护 | 教师个人信息（身份证/薪资等）是否脱敏存储 | 🟠 P1 |
| 行为数据 | 教师的操作日志是否只用于系统运维，不用于绩效评价 | 🟡 P2 |

### 合规要求

| 检查项 | 具体审查内容 | 默认分级 |
|--------|-------------|---------|
| 个保法合规 | 处理个人信息是否符合《个人信息保护法》要求 | 🔴 P0 |
| 教育数据规范 | 是否符合教育部关于教育数据管理的相关规定 | 🟠 P1 |
| 跨境传输 | 学生数据是否存在跨境传输风险（云服务选择） | 🔴 P0 |
| 隐私协议 | 隐私协议中是否明确说明数据使用目的和范围 | 🟠 P1 |

---

## 二、内容安全

### 用户生成内容（UGC）

| 检查项 | 具体审查内容 | 默认分级 |
|--------|-------------|---------|
| 内容过滤 | 学生提交的作业/讨论是否有内容安全过滤 | 🟠 P1 |
| 敏感词检测 | 是否有敏感词/违禁词检测机制 | 🟠 P1 |
| 图片/文件安全 | 上传的图片/文件是否有安全扫描 | 🟡 P2 |
| 举报机制 | 是否有不良内容的举报和处理流程 | 🟡 P2 |

### AI 生成内容

| 检查项 | 具体审查内容 | 默认分级 |
|--------|-------------|---------|
| AI 内容审核 | AI 生成的教学内容是否经过安全过滤 | 🟠 P1 |
| 幻觉防护 | AI 回答是否有事实性校验或置信度标注 | 🟠 P1 |
| 年龄适配 | AI 输出内容是否适合目标年龄段 | 🟠 P1 |
| 标注声明 | AI 生成内容是否有明确标注 | 🟡 P2 |

### 教学内容

| 检查项 | 具体审查内容 | 默认分级 |
|--------|-------------|---------|
| 内容准确性 | 教学内容呈现是否有防篡改机制 | 🟠 P1 |
| 版权保护 | 付费课程内容是否有防盗取保护 | 🟡 P2 |
| 内容版本 | 教学内容更新是否有版本管理 | 🟡 P2 |

---

## 三、权限模型

### 多角色权限

教育系统典型角色层级：平台管理员 > 学校管理员 > 教师 > 学生 > 家长

| 检查项 | 具体审查内容 | 默认分级 |
|--------|-------------|---------|
| 角色隔离 | 不同角色是否只能访问其权限范围内的数据 | 🔴 P0 |
| 学校隔离 | 跨学校的数据是否严格隔离（多租户） | 🔴 P0 |
| 班级隔离 | 教师是否只能看到自己班级的学生数据 | 🔴 P0 |
| 家长-学生关联 | 家长是否只能查看自己孩子的数据 | 🔴 P0 |
| 成绩可见性 | 学生成绩是否只对相关教师、本人和家长可见 | 🔴 P0 |

### 操作权限

| 检查项 | 具体审查内容 | 默认分级 |
|--------|-------------|---------|
| 成绩修改 | 成绩录入/修改是否有权限控制和审计日志 | 🔴 P0 |
| 学籍操作 | 学籍变动（转学/休学等）是否有审批流程 | 🟠 P1 |
| 课程管理 | 课程创建/修改/删除是否有权限控制 | 🟠 P1 |
| 批量操作 | 批量导入/导出/删除是否有额外的权限校验 | 🟠 P1 |

## 四、并发与数据一致性

### 教育场景特有的并发挑战

| 场景 | 并发风险 | 审查重点 | 默认分级 |
|------|---------|---------|---------|
| 考试开始 | 大量学生同时加载试卷 | 是否有预加载/缓存策略 | 🟠 P1 |
| 考试提交 | 大量学生同时提交答案 | 是否支持断点续传，提交是否幂等 | 🔴 P0 |
| 选课抢课 | 大量学生同时选课 | 课程名额是否有并发保护（乐观锁/队列） | 🔴 P0 |
| 成绩发布 | 家长/学生同时查询成绩 | 是否有缓存策略，查询是否优化 | 🟠 P1 |
| 直播上课 | 大量学生同时进入直播间 | 是否有限流和弹性扩容 | 🟠 P1 |
| 作业批改 | 多位教师同时批改同一份作业 | 是否有冲突检测和锁机制 | 🟠 P1 |

### 数据一致性

| 检查项 | 具体审查内容 | 默认分级 |
|--------|-------------|---------|
| 成绩一致性 | 成绩录入→统计→报表的数据一致性 | 🔴 P0 |
| 考勤一致性 | 考勤记录→统计→通知的数据一致性 | 🟠 P1 |
| 课程-学生关系 | 选课/退课操作的事务完整性 | 🟠 P1 |
| 学期切换 | 学期结束→数据归档→新学期初始化的完整性 | 🟠 P1 |

## 五、教育行业审查检查清单补充

在标准审查检查清单（references/checklists.md）基础上，教育项目额外检查：

| # | 检查项 | 通过条件 |
|---|--------|---------|
| EDU-1 | 学生数据保护 | 未成年人信息有额外保护，符合个保法 |
| EDU-2 | 多角色权限隔离 | 学校/班级/师生/家长数据隔离到位 |
| EDU-3 | 成绩数据安全 | 成绩操作有权限控制 + 审计日志 |
| EDU-4 | 内容安全 | UGC 和 AI 生成内容有过滤机制 |
| EDU-5 | 考试并发安全 | 考试提交幂等，选课有并发保护 |
| EDU-6 | 跨租户隔离 | 学校间数据严格隔离，无泄露路径 |

---

## 使用方式

1. 在标准审查流程的 Step 2-6 中，每步额外参照本文件的对应章节
2. 在 Step 7 分级时，教育特有的问题使用本文件建议的默认分级
3. 在审查报告中增加"教育行业合规"专项章节
4. 教育相关的 P0 问题不可降级
