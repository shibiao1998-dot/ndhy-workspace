# 真实世界代码审查发现模式

> **来源**：ai-org-showcase v2 产品官网代码审查报告
> **提炼日期**：2026-03-16
> **性质**：实际审查中发现的模式，非理论清单

---

## 模式 1：定时器启动无停止（资源泄漏）

**实际案例**：Pipeline 动画用 `setInterval(fn, 400)` 启动，`intervalId` 被赋值但从未 `clearInterval`。IntersectionObserver 只 disconnect 了自身，interval 永远运行。

**审查动作**：
- 搜索 `setInterval` → 检查同作用域内是否有 `clearInterval`
- 搜索 `setTimeout` → 检查是否需要清理（尤其在组件卸载/页面离开时）
- 搜索 `addEventListener` → 检查是否有对应的 `removeEventListener`
- 搜索 `IntersectionObserver` / `MutationObserver` → 检查 `disconnect()` 时机

**检查问句**：「这个资源的生命周期是什么？谁负责清理它？」

## 模式 2：O(n²) 算法无设备分层保护

**实际案例**：Canvas 粒子动画 80 个粒子的双层循环连线计算（~3160 次/帧），在桌面端无感，在低端移动设备可能掉帧。

**审查动作**：
- 发现嵌套循环时，估算数据量 × 执行频率
- 如果是每帧（requestAnimationFrame）或高频执行（scroll/resize），标注性能风险
- 检查是否有移动端降级策略（设备检测 / 粒子数减少 / 直接跳过）
- 检查是否尊重 `prefers-reduced-motion` 用户偏好

**检查问句**：「这段代码在最弱的目标设备上能跑多快？」

---

## 模式 3：配置一刀切（缓存/安全头未差异化）

**实际案例**：
- Caddyfile `@static` matcher 包含 `*.html`，所有文件 `max-age=3600`——HTML 更新后用户 1 小时内看到旧版
- `X-XSS-Protection: 1; mode=block` 照搬旧模板，实际已被 Chrome v78+ 废弃

**审查动作**：
- 检查缓存配置时，确认 HTML 和静态资源是否分开策略
- 检查安全头时，逐个验证当前浏览器支持状态
- 搜索"模板化配置"——来自旧项目的配置是否适用于当前场景

**检查问句**：「这个配置是针对当前项目定制的，还是从别处复制来的？」

---

## 模式 4：架构选型的安全代价未标注

**实际案例**：单文件 HTML 架构（46KB，CSS/JS 全内联）→ CSP 必须使用 `unsafe-inline` → 削弱 XSS 防护。这是架构选型的固有代价，不是 bug，但需要被意识到。

**审查动作**：
- 发现 `unsafe-inline` / `unsafe-eval` 时，追问是技术必须还是可避免
- 如果是技术必须，确认是否有缓解措施（无 eval、无 innerHTML 拼接用户输入）
- 在审查报告中标注为"已知权衡"，而非"待修复问题"

**检查问句**：「这是架构决策的必然代价，还是实现偷懒？」

---

## 模式 5：内容与契约偏离（展示优化 vs 文案定义）

**实际案例**：content-v2.md 定义三个分组（组织基建 3 人 + 知识管理 1 人 + 项目管理 1 人），HTML 为视觉紧凑合并为一个"组织基建"（5 人）。开发自行判断"合并更好"，未确认文案契约约束力。

**审查动作**：
- 代码审查时对照内容定义文档，逐条核查一致性
- 发现偏差时，区分"错误"和"有意优化"
- 如果是有意优化，追问是否获得文案/产品负责人确认

**检查问句**：「这里和文案定义不一致——是故意的还是疏忽？如果是故意的，谁确认的？」

---

## 模式 6：过时技术知识残留

**实际案例**：`X-XSS-Protection` 头在 2019 年后已被主流浏览器废弃，但仍出现在 2026 年的配置中。保留无害，但反映出配置基于过时知识。

**审查动作**：
- 安全相关配置不能只看"是否存在"，还要验证"是否仍有效"
- 对照 MDN / caniuse 确认浏览器支持状态
- 标注"无害但过时"和"有害且过时"的区别

**检查问句**：「这个配置在 2026 年还有意义吗？」

---

## 模式 7：特殊值未差异化处理

**实际案例**：数字计数器统一从 0 动画到 target 值。当 target=0 时（"0 行代码"），动画从 0 到 0，无视觉效果。其他数字（1、26）都有明显的滚动效果。

**审查动作**：
- 通用组件 / 统一逻辑遇到特殊值（0、空、null、极大值、极小值）时检查表现
- 动画/过渡效果对"不变"状态的处理
- 列表/计数/分页在 0 条数据时的表现

**检查问句**：「如果输入值是 0 / 空 / 极端值，这段代码会怎样？」

## 审查亮点记录（值得学习的做法）

以下是同一份审查报告中发现的**值得复用的好实践**：

1. **零外部依赖**：46KB 单文件站不依赖任何 CDN/外部库，完全自包含。消除了外部资源不可用风险。
2. **Docker 安全配置标杆**：`read_only: true` + `no-new-privileges` + 精准 tmpfs + 资源限制 + 健康检查——静态站的安全配置模板。
3. **IntersectionObserver 驱动滚动动画**：比 scroll 事件监听性能好得多，`rootMargin` 偏移确保触发时机自然。
4. **CSS 变量 + clamp() 响应式方案**：无需多套媒体查询，流式适配。
5. **移动端适配完整度**：汉堡菜单、grid 单列回退、表格移动端重排（`::before` 伪元素标签）、`scroll-padding-top` 适配固定导航栏。
