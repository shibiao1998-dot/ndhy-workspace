# 技术架构设计 — 10 大能力域速查

## 域 1：系统架构设计
**核心问题**：当前阶段该用什么架构模式？系统怎么分层？

关键判断：
- 团队 < 5 人 + 业务未验证 → 模块化单体
- 明确需独立扩展的子域 → 微服务（仅该子域）
- 事件驱动需求明确 → 引入消息队列，不改整体模式

→ 详细方法 [architecture-patterns.md](architecture-patterns.md)

## 域 2：技术选型
**核心问题**：用什么技术、为什么、退出成本多大？

选型铁律：
- 团队会用的 > 理论最优的
- 有退出方案的 > 锁定最深的
- 社区活跃的 > 功能最全的

→ 详细方法 [tech-selection.md](tech-selection.md)

## 域 3：数据架构协同
**核心问题**：数据怎么流动、存在哪、一致性怎么保证？

职责边界：架构给方向和约束（存储类型选择、一致性策略、分片方向），细化交数据库专家。

## 域 4：接口协同设计
**核心问题**：模块间、系统间怎么通信？

职责边界：架构定义通信风格（REST/gRPC/事件）和调用拓扑，细化交 API 专家。

## 域 5：性能与扩展性
**核心问题**：能扛住多大量？瓶颈在哪？怎么扩？

关键交付：性能目标量化 → 瓶颈识别 → 扩展策略（垂直/水平/缓存/CDN/异步化）

→ 详细方法 [nfr-design.md](nfr-design.md) §性能与扩展

## 域 6：安全架构
**核心问题**：怎么防？防什么？合规要求是什么？

关键交付：认证授权方案 → 数据加密策略 → 威胁模型 → 合规清单

→ 详细方法 [nfr-design.md](nfr-design.md) §安全架构

## 域 7：AI 能力集成
**核心问题**：AI 在系统里扮演什么角色？怎么集成、怎么控成本、怎么降级？

关键交付：AI 中间层设计 → 成本控制 → 降级方案 → Prompt 管理

→ 详细方法 [ai-integration.md](ai-integration.md)

## 域 8：可靠性与容灾
**核心问题**：挂了怎么办？多久恢复？数据丢不丢？

关键交付：SLA 定义 → 故障域划分 → 降级策略 → 监控告警 → 回滚方案

→ 详细方法 [nfr-design.md](nfr-design.md) §可靠性与容灾

## 域 9：DevOps 与交付
**核心问题**：怎么构建、怎么测试、怎么发布、怎么回滚？

关键交付：CI/CD 流水线 → 环境规划 → 发布策略 → 回滚机制

→ 详细方法 [devops-delivery.md](devops-delivery.md)

## 域 10：架构决策记录（ADR）
**核心问题**：为什么选 A 不选 B？当时考虑了什么？

每个重要决策一条 ADR，格式见 [templates.md](templates.md) §ADR 模板。
