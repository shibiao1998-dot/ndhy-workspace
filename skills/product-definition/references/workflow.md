# 产品定义 — 标准工作流程

## 8 步标准流程

```
Step 1：消化输入
  │  完整理解战略定位文档，回溯 Phase 1 关键洞察
  │  → 提取：方向、客群、差异化、边界
  │  → 信息不足时，有权要求上游补充
  │
Step 2：概念定义
  │  → 回答五个核心问题：是什么？给谁？什么场景？干什么？得到什么？
  │  → 产出【产品概念卡】
  │  → 通过"出租车司机测试"：普通人听完能复述
  │
Step 3：场景拆解
  │  → 识别 3-5 个核心场景
  │  → 每个场景展开三层：核心场景 → 场景故事 → 场景边界
  │  → 遵守场景三铁律（见 checklists.md）
  │  → 详细方法 → analysis-frameworks.md §1
  │
Step 4：功能范围界定
  │  → 从场景推导功能，不凭空想象
  │  → 按三层模型分级：核心层 / 重要层 / 扩展层
  │  → 每个功能过四检验（见 checklists.md）
  │  → 核心层不超过 5 个功能
  │  → 详细方法 → analysis-frameworks.md §2
  │
Step 5：规则设计
  │  → 定义四类核心规则：准入 / 流转 / 限制 / 异常
  │  → 规则要少、透明、不矛盾
  │  → 每条规则考虑"被违反了怎么办"
  │  → 详细方法 → analysis-frameworks.md §3
  │
Step 6：成功标准
  │  → 定义两层标准：可用标准（及格线）+ 成功标准（优秀线）
  │  → 分短/中/长期，必须可量化或可观测
  │  → 区分虚荣指标和真实指标
  │  → 详细方法 → analysis-frameworks.md §4
  │
Step 7：边界确认
  │  → 明确三组关系：与现有产品 / 与外部系统 / 与未来演进
  │  → 详细方法 → analysis-frameworks.md §5
  │
Step 8：输出产品定义文档
     → 按模板组织 6 章节文档 → templates.md
     → 交付前跑质量自检清单 → checklists.md
     → 教育类产品额外读 → education-product.md
```

## 定义深度速查

| 深度 | 适用场景 | 覆盖范围 | 输出物 |
|------|----------|----------|--------|
| **轻量定义** | 已有产品的功能迭代 | 概念确认 + 功能范围 + 成功标准 | 产品定义文档（轻量版） |
| **标准定义** | 新产品模块或重要改版 | 完整 8 步流程 | 产品定义文档（标准版） |
| **深度定义** | 全新产品从 0 到 1 | 完整流程 + 多轮场景验证 + 边界深度展开 | 产品定义文档（完整版） |

判断不清时向上确认。

## 触发时首要动作

无论如何触发，第一步永远是：

> "战略定位告诉了我们什么？方向是什么、服务谁、差异化在哪、边界在哪？"

→ 判断定义深度 → 消化战略定位输入 → 按 8 步流程执行
