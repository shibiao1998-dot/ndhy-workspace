# 原型质量自检清单

> 交付前逐项检查，全部 ✅ 才能提交评审。

## Design Token 使用检查

| # | 检查项 | 标准 |
|---|--------|------|
| T1 | 颜色 Token | 所有颜色使用 `var(--color-xxx)`，无 `#hex` 硬编码 |
| T2 | 字号 Token | 所有字号使用 `var(--font-size-xxx)`，无 `14px` 硬编码 |
| T3 | 间距 Token | 所有间距使用 `var(--spacing-xxx)`，无 `8px` 硬编码 |
| T4 | 圆角 Token | 所有圆角使用 `var(--radius-xxx)` |
| T5 | 阴影 Token | 所有阴影使用 `var(--shadow-xxx)` |
| T6 | 过渡 Token | 所有动画使用 `var(--transition-xxx)` |

## HTML 语义化检查

| # | 检查项 | 标准 |
|---|--------|------|
| H1 | 页面结构 | 使用 `<header>/<nav>/<main>/<aside>/<footer>` |
| H2 | 标题层级 | `<h1>` 到 `<h6>` 不跳级，每页一个 `<h1>` |
| H3 | 按钮标签 | 使用 `<button>`，不是 `<div onclick="">` |
| H4 | 链接标签 | 使用 `<a href="">`，不是 `<span onclick="">` |
| H5 | 列表标签 | 列表用 `<ul>/<ol>/<li>` |
| H6 | 表格标签 | 表格用 `<table>/<thead>/<tbody>/<th>/<td>` |
| H7 | 图片替代 | 所有 `<img>` 有 `alt` 属性 |
| H8 | 语言声明 | `<html lang="zh-CN">` |
| H9 | viewport | `<meta name="viewport" content="width=device-width, initial-scale=1.0">` |

## 响应式检查

| # | 检查项 | 标准 |
|---|--------|------|
| R1 | 320px | 在 320px 宽度下内容不溢出、不截断、可阅读 |
| R2 | 375px | iPhone SE/Mini 正常显示 |
| R3 | 768px | 平板断点布局切换正确 |
| R4 | 1024px | 桌面断点布局切换正确 |
| R5 | 1440px | 设计稿理想宽度正常显示 |
| R6 | 2560px | 大屏不过度拉伸，有 max-width |

## 可访问性检查

| # | 检查项 | 标准 |
|---|--------|------|
| A1 | 文本对比度 | 普通文本 ≥ 4.5:1（WCAG AA） |
| A2 | 焦点可见 | Tab 键可遍历所有交互元素，焦点环清晰 |
| A3 | 焦点顺序 | Tab 顺序与视觉阅读顺序一致 |
| A4 | 触摸目标 | 可点击元素 ≥ 44×44px（移动端） |
| A5 | 不依赖颜色 | 信息传达不仅靠颜色（错误状态有图标+文字） |

## 代码质量检查

| # | 检查项 | 标准 |
|---|--------|------|
| C1 | 可运行 | 浏览器直接打开无报错（或 `npm run dev` 可启动） |
| C2 | 无 console 错误 | 打开 DevTools Console 无报错 |

## 反模式速查

| ❌ 反模式 | 后果 | ✅ 正确做法 |
|----------|------|-----------|
| 颜色/字号硬编码 | 换主题要全部修改，与设计系统脱节 | 使用 CSS 变量 |
| 只做 default 状态 | PM 看不到异常状态，上线后才发现问题 | 全状态覆盖 + 控制面板切换 |
| 不做响应式 | 移动端不可用 | Mobile First + media queries |
| 用 `<div>` 做按钮 | 键盘不可操作、屏幕阅读器无法识别 | 语义化标签 |
| 移除焦点环 | 键盘用户无法看到焦点位置 | 保留 `:focus-visible` |
| 假数据用 test/xxx | PM 无法感受真实使用效果 | 使用有业务含义的数据 |
| 原型不可运行 | 无法在浏览器预览 | 确保可直接打开/启动 |
| 原型当生产代码交付 | 生产代码需要完整功能和工程化 | 明确标注"原型参考" |
