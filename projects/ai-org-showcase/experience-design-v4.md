## 1. 信息架构升级（7+1 区块）

### 1.1 区块结构与叙事映射

| # | 区块 | 叙事角色 | 情绪目标 | 视口高度 | 信息密度 |
|---|------|---------|---------|---------|---------|
| ① | 悬念开场 | 认知冲突制造器 | 好奇 → 悬念 | 100vh | 极低 |
| ② | 揭幕震撼 | 悬念兑现 + 规模震撼 | **✦ 第一峰值：震撼** | 120vh | 中（拓扑图） |
| ③ | 运转揭秘 | 自证可信度 | 沉浸 → 信服 | 130vh（含展开） | 中高（时间线） |
| 🫁 | 呼吸点 | 情绪重置 | 停顿 → 准备 | 60vh | 极低 |
| ④ | 认知落差 | 对比优越感 | 清醒 → 不安 | 110vh | 中 |
| ⑤ | 深层架构 | 理性信服（一屏一概念） | 理解 → 认同 | 3×80vh=240vh | 高（分散） |
| ⑥ | 更大图景 | 行业宣言 | **✦ 第二峰值：共鸣** | 120vh | 低（纯文字力量） |
| ⑦ | 收束 | 叙事闭环 | 余韵 → 行动意愿 | 60vh | 极低 |

**总页面高度**：约 940vh（~10 个全屏），滚动时长预期 3–5 分钟。

### 1.2 层级控制

- **L1 主叙事线**：区块标题 + 核心文案，占视觉权重 70%，所有用户必经
- **L2 展开细节**：步骤展开（③）、tooltip（②）、标签说明（④），用户主动触发
- **L3 技术深度**：GitHub 入口（⑦），面向技术受众的可选深入路径

信息层级铁律：每个区块只有**一个核心认知任务**，L2/L3 内容不干扰 L1 的扫描式阅读。

## 2. 滚动驱动叙事设计

### 2.1 全局 IntersectionObserver 配置

```javascript
// 桌面端：6 阶梯阈值，精准控制入场时机
const desktopIO = new IntersectionObserver(callback, {
  threshold: [0, 0.15, 0.3, 0.5, 0.7, 1.0],
  rootMargin: '0px 0px -8% 0px'  // 提前 8% 预触发
});

// 移动端：3 阶梯阈值，减少计算开销
const mobileIO = new IntersectionObserver(callback, {
  threshold: [0, 0.25, 1.0],
  rootMargin: '0px 0px -5% 0px'
});

// 规则：每个区块根元素注册 IO；子元素用 CSS class 切换触发
// 动画完成后立即 unobserve
```

### 2.2 逐区块滚动触发规则

| 区块 | 触发条件 | 滚动行为 | 动效启动 |
|------|---------|---------|---------|
| ① 悬念 | 页面加载 | 首屏自动播放，**不锁定滚动**（V3 的 scroll-lock 已移除，改用视觉引导留住用户） | 立即 |
| ② 拓扑图 | 可见度 ≥ 15% | 自由滚动 + GSAP ScrollTrigger pin（拓扑图区域固定，滚动驱动亮起序列） | 连续映射 |
| ③ 时间线 | 可见度 ≥ 25% | 标准滚动，各步骤独立 IO | 逐步触发 |
| 🫁 呼吸点 | 可见度 ≥ 30% | 标准滚动 | 淡入后静止 |
| ④ 对比 | 可见度 ≥ 20% | 标准滚动 | 左右错位进入 |
| ⑤ 剖面 A/B/C | 各剖面独立 IO ≥ 25% | 标准滚动，一屏一概念 | 独立触发 |
| ⑥ 宣言 | 可见度 ≥ 15% | **GSAP ScrollTrigger scrub**：滚动距离线性映射文字淡入进度 | 连续映射 |
| ⑦ 收束 | 可见度 ≥ 30% | 标准滚动 | 淡入 |

### 2.3 双峰情绪曲线控制

```
情绪强度
  高 │         ②✦                      ⑥✦
     │        ╱  ╲                    ╱   ╲
  中高│      ╱    ╲    ⑤A  ⑤B  ⑤C  ╱      ╲
     │    ╱  ③    ╲  ╱  ╲╱  ╲╱  ╱        ⑦收束
  中 │  ╱    沉浸   ╲╱              
  中低│╱  🫁呼吸  ④落差
  低 │①好奇
     └─────────────────────────────────→ 滚动进度
     0%    15%   30%  40%  50% 60% 70%  85%  100%
```

**峰值控制策略**：

- **第一峰值（②震撼）**：拓扑图滚动驱动亮起 → 全亮后触发"高潮爆发"（整体 glow 扩散 + scale-up + 底文弹出）。动效密度最高，是全站视觉能量顶点。
- **第二峰值（⑥共鸣）**：纯文字力量。"答案是——组织。" 全屏独立展示，字号放大 2×，滚动驱动淡入。零装饰，零图形——用空间和文字重量创造情绪高峰。

## 3. 动效规范

### 3.1 全局动效变量

```css
:root {
  /* 时长 */
  --dur-micro: 120ms;     /* 微交互：hover/focus */
  --dur-fast: 200ms;      /* 快反馈：tooltip/toggle */
  --dur-normal: 400ms;    /* 标准过渡：淡入/滑入 */
  --dur-slow: 800ms;      /* 慢展开：剖面/大区块入场 */
  --dur-dramatic: 1200ms; /* 戏剧性：高潮爆发 */

  /* 缓动 */
  --ease-out: cubic-bezier(0.22, 1, 0.36, 1);         /* 快进慢出：入场 */
  --ease-in-out: cubic-bezier(0.45, 0, 0.55, 1);       /* 对称：循环动画 */
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);    /* 弹性：高潮/hover */
  --ease-dramatic: cubic-bezier(0.16, 1, 0.3, 1);      /* 戏剧性：爆发 */

  /* 位移量 */
  --offset-subtle: 24px;   /* 文字淡入 */
  --offset-medium: 48px;   /* 卡片/区块入场 */
  --offset-large: 80px;    /* 滑入/剖面展开 */
}
```

### 3.2 入场动画（按元素类型）

| 元素类型 | 动画方式 | 参数 | 说明 |
|---------|---------|------|------|
| **文字** | 淡入 + 上移 | `opacity: 0→1`, `translateY(24px→0)`, `--dur-normal`, `--ease-out` | 区块标题、正文段落 |
| **强调文字** | 淡入 + 上移 + 模糊清晰 | `opacity: 0→1`, `translateY(32px→0)`, `filter: blur(8px)→blur(0)`, `--dur-slow`, `--ease-out` | 区块⑥关键金句、底部陈述 |
| **卡片/面板** | 淡入 + 缩放 | `opacity: 0→1`, `scale(0.92)→scale(1)`, `--dur-normal`, `--ease-spring` | 步骤卡片、对比面板 |
| **图表/剖面** | 裁切展开 | `clip-path: inset(50% 0)→inset(0% 0)`, `--dur-slow`, `--ease-dramatic` | 区块⑤每个剖面 |
| **拓扑图节点** | 发光亮起 | `opacity: 0.05→1`, `filter: drop-shadow(0 0 0px)→drop-shadow(0 0 8px var(--accent))`, 间隔 60ms/节点 | 区块② |
| **连线** | 路径描边 | `stroke-dashoffset: length→0`, `--dur-dramatic`, `--ease-out` | 区块②节点间连线 |
| **数据行** | 逐行滑入 | `opacity: 0→1`, `translateX(-20px)→0`, 间隔 60ms/行, `--dur-fast`, `--ease-out` | 区块⑤层级数据 |

### 3.3 高潮时刻动效（区块②）

**触发条件**：拓扑图所有节点 + 连线动画完成后 300ms。

```
序列：
T+0ms    整体 glow 扩散：SVG 容器 filter: drop-shadow(0 0 0px #818cf8) 
         → drop-shadow(0 0 40px rgba(129,140,248,0.4))
         duration: 800ms, ease: --ease-dramatic

T+200ms  整体 scale-up：SVG 容器 scale(1) → scale(1.03) → scale(1)
         duration: 1000ms, ease: --ease-spring

T+400ms  👤 老板节点从顶部淡入（最后出现 = 人类是特殊的）
         opacity: 0→1, translateY(-30px)→0, duration: 600ms

T+800ms  底部文案弹出：
         "一个人类。31 个 AI 专家。一个完整的产品组织。"
         opacity: 0→1, translateY(16px)→0, scale(0.96)→scale(1)
         duration: 500ms, ease: --ease-spring

T+1300ms 第二句淡入：
         "不是工具。不是框架……是一支有灵魂、有纪律、有记忆、会进化的 AI 组织。"
         opacity: 0→1, duration: 600ms, ease: --ease-out
```

### 3.4 滚动联动动效

| 区块 | 联动方式 | 参数 |
|------|---------|------|
| ② 拓扑图 | **GSAP ScrollTrigger pin + scrub**：滚动进度 0–100% 线性映射节点亮起序列 | `scrub: 1`, `pin: true`, 滚动距离 = 120vh |
| ⑥ 宣言 | **scrub 文字淡入**：每段文字的 opacity/translateY 与滚动位置绑定 | `scrub: 0.5`, 每段占 15% 滚动区间 |
| 全局 | **Lenis 平滑滚动**：覆盖原生滚动，统一缓动 | `lerp: 0.1`, `duration: 1.2`, `smoothWheel: true` |

### 3.5 微交互

| 交互 | 触发 | 效果 | 参数 |
|------|------|------|------|
| 节点 Hover（桌面） | mouseenter, 80ms 延迟 | `scale(1.15)` + tooltip 弹出 + 相邻节点微缩 `scale(0.95) opacity(0.6)` | `--dur-micro`, `--ease-spring` |
| 节点 Tap（移动端） | touchstart | 同 Hover 视觉 + tooltip；tap-outside 关闭 | 同上 |
| 步骤展开（③） | click/tap | 箭头旋转 90° + 内容区 `grid-rows: 0fr→1fr` 展开 | `--dur-normal`, `--ease-out` |
| 步骤收起（③） | click/tap 再次 | 箭头旋转回 0° + 内容区 `grid-rows: 1fr→0fr` | `--dur-normal`, `--ease-out` |
| 对比行联动（④） | hover 左栏某行 | 右栏对应行同步高亮（`background: rgba(129,140,248,0.08)`） | `--dur-micro` |
| CTA hover（⑦） | mouseenter | 轻微 glow 扩散 `box-shadow: 0 0 0→20px rgba(129,140,248,0.2)` | `--dur-fast`, `--ease-out` |
| 下滚指示（①） | 持续 | `translateY(0→8px→0)`, 循环, 2s | `--ease-in-out`, infinite |
| Tooltip 出现 | hover/tap | `opacity: 0→1`, `translateY(6px→0)` | `--dur-micro`, `--ease-out` |
| Tooltip 消失 | mouseleave（150ms 延迟防闪烁） | `opacity: 1→0` | 100ms |

### 3.6 prefers-reduced-motion 降级

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
  /* 拓扑图：跳过亮起序列，直接显示完成态 */
  /* 区块⑥：跳过 scrub，直接显示全部文字 */
  /* Lenis 平滑滚动禁用 */
}
```

降级后保留：布局、颜色、内容完整性、交互响应（hover/click 仍有状态变化，只去掉过渡动画）。

## 4. 移动端独立方案

> V3 致命问题：移动端是桌面缩放版。V4 移动端是**独立设计**的体验。

### 4.1 移动端拓扑图方案（区块②）

**≤ 768px：同心环简化版**

不再缩放桌面 SVG。移动端使用独立的**垂直分层布局**：

```
┌─────────────────────┐
│    👤 老板（人类）     │  ← 顶部金色标识
├─────────────────────┤
│    🧧 Leader         │  ← 组织核心
├─────────────────────┤
│    🏗️ 项目管理专家    │  ← 调度层
├─────────────────────┤
│  ┌─────┐ ┌─────┐    │
│  │🎯需求│ │🔍用研│    │  ← 专家网格
│  └─────┘ └─────┘    │     2 列 × 15 行
│  ┌─────┐ ┌─────┐    │     每格：emoji + 2字角色名
│  │📊市场│ │🔭竞品│    │     Tap 展开完整名+定位
│  └─────┘ └─────┘    │
│  ...                 │
└─────────────────────┘
```

- 节点以 **2列网格** 排列，按职能域分组，域名作小标题
- 每个节点 Tap → 展开卡片显示 emoji + 角色全名 + 一句话定位
- 亮起动画保留：从上到下逐行亮起，间隔 80ms/行
- 连线省略（移动端无法有效传达网络关系，用分组替代）
- 触控目标：每个网格格子最小 48×48px

**≤ 375px（小屏手机）**：网格缩为 2 列但格子高度增大至 56px，保证可触达。

### 4.2 移动端区块适配

| 区块 | 桌面行为 | 移动端独立方案 |
|------|---------|-------------|
| ① 悬念 | 大字号居中，打字机 40ms/字 | 字号缩至 `clamp(1.5rem, 5vw, 2.5rem)`；打字机加速至 30ms/字（移动端用户耐心更短） |
| ② 拓扑 | SVG 全尺寸 + Hover tooltip | **垂直分层网格**（见 4.1），Tap 展开 |
| ③ 时间线 | 水平时间线 + 右侧面板展开 | **纵向时间线**，全宽步骤卡片，Tap 展开手风琴 |
| 🫁 呼吸 | 全屏留白 + 文字居中 | 同桌面，高度减至 50vh |
| ④ 对比 | 左右双栏并列 | **上下堆叠**，先灰（散装 AI）后亮（AI 组织），中间加"vs"分隔 |
| ⑤ 剖面 | 每剖面 80vh 独立展示 | 每剖面高度自适应（min-height: 70vh），层级数据改为卡片式纵向排列 |
| ⑥ 宣言 | scrub 驱动文字淡入 | 改为标准 IO 触发（移动端 scrub 体感差），每段独立淡入 |
| ⑦ 收束 | 按钮并排 | 按钮纵向堆叠，全宽，min-height 48px |

### 4.3 触控交互规范

| 元素 | 最小触控目标 | 触控行为 |
|------|------------|---------|
| 拓扑节点 | 48×48px | Tap 展开，Tap-outside 关闭，同时只展开一个 |
| 时间线步骤 | 48px 高度 | Tap 展开/收起，手风琴模式 |
| 下滚箭头 | 48×48px（padding 扩展） | Tap 平滑滚动至区块② |
| CTA 按钮 | 全宽, min-height 48px | Tap 触发 mailto / 跳转 |
| 对比标签 | 自然尺寸, min-height 44px | 纯展示，无交互 |

**移动端手势**：
- 禁止任何方向的 scroll-lock（V3 教训）
- 不使用横向滑动手势（避免与系统返回冲突）
- 所有展开/收起用纵向展开，不用横向滑入

## 6. 逐区块详细交互规格

### 区块① 悬念开场

| 元素 | 交互 | 规格 |
|------|------|------|
| "2026 年 3 月 10 日" | 淡入 | delay 0.3s, `--dur-normal`, `--ease-out`, `translateY(24px→0)` |
| "一个人启动了一个项目。" | 逐字打字机 | 40ms/字（移动端 30ms），光标闪烁 530ms 周期 |
| "6 天后…" | 淡入 | delay 2.8s, `--dur-normal` |
| "他没有招过一个人。" | 淡入 + 亮度提升 | delay 4.2s, `color: #f0f0f0`（比正文 `#a0a0a0` 更亮） |
| "AI Organization" 品类提示 | 常驻 | `opacity: 0.15`, 字号 0.75rem, 无衬线, 不参与动画序列 |
| 下滚箭头 | 呼吸循环 | `translateY(0→8px→0)`, 2s, `--ease-in-out`, infinite |

**V3 改进**：移除 scroll-lock。改用视觉引导（打字机序列 + 下滚箭头呼吸）自然留住用户。用户可随时滚动。

### 区块② 揭幕震撼

**桌面端**（≥1024px）：

- SVG `viewBox="0 0 800 600"`，居中
- **滚动驱动 pin**：区块②滚入时 pin 住，滚动距离 120vh 内完成完整动画序列
- 节点亮起序列映射到滚动进度 0–60%
- 连线描边映射到滚动进度 60–80%
- 高潮爆发映射到滚动进度 80–90%
- 底文出现映射到滚动进度 90–100%

**节点状态**：

| 状态 | 视觉 |
|------|------|
| 未亮 | `opacity: 0.05`, `fill: #222` |
| 亮起中 | `opacity: 0→1`, `drop-shadow(0 0 8px #818cf8)` |
| 常态 | `opacity: 1`, `fill: #818cf8` |
| Hover | `scale(1.15)`, 相邻节点 `scale(0.95) opacity(0.6)`, tooltip 弹出 |

**Tooltip 规格**：
- 位置：节点正上方 12px，超出视口时翻转
- 样式：`bg: #1a1a1a`, `border: 1px solid #333`, `radius: 8px`, `padding: 8px 14px`
- 进入：`--dur-micro`, `--ease-out`, `translateY(6px→0) + opacity 0→1`
- 退出延迟：150ms（防止鼠标移向 tooltip 时闪烁）

### 区块③ 运转揭秘

| 元素 | 交互 | 规格 |
|------|------|------|
| 标题 | 淡入 + 上移 | IO ≥ 25%, `--dur-normal`, `translateY(24px→0)` |
| 时间线节点 | 滚动驱动逐个亮起 | 每个节点在其自身位置 IO ≥ 30% 时亮起，`opacity 0→1`, `scale(0.8→1)`, `--dur-fast` |
| 步骤卡片 | Tap/Click 展开 | **手风琴模式**（同时只展开一个）|
| 展开动画 | `grid-template-rows: 0fr→1fr` | `--dur-normal`, `--ease-out`（替代 V3 的 max-height 反模式）|
| 展开箭头 | 旋转 | `rotate(0→90deg)`, `--dur-fast` |
| 展开内容 | 淡入 | delay 100ms after grid 展开, `opacity 0→1`, `--dur-fast` |
| "6 天。10 位专家…" | 淡入 + 缩放 | 最后出现, `scale(0.95→1)`, `--dur-normal`, `--ease-spring` |

### 区块④ 认知落差

| 元素 | 交互 | 规格 |
|------|------|------|
| 标题 | 淡入 | IO ≥ 20%, 标准文字淡入 |

### 区块⑤ 深层架构

**三剖面独立一屏**（V3 审查要求：一屏一概念）：

| 剖面 | 入场方式 | 参数 |
|------|---------|------|
| A 权力结构 | 裁切展开 | `clip-path: inset(50% 0)→inset(0% 0)`, `--dur-slow`, `--ease-dramatic` |
| B 记忆体系 | 裁切展开 | 同 A，独立 IO 触发 |
| C 持续进化 | 裁切展开 | 同 A，独立 IO 触发 |
| 层级数据行 | 逐行滑入 | 剖面展开完成后 150ms, `translateX(-20px)→0`, 60ms 间隔/行 |
| 副标题 | 淡入 + 模糊清晰 | `blur(6px)→blur(0)`, `--dur-normal` |
| 底文 | 淡入 | 该剖面所有内容完成后 200ms |

## 7. 性能约束

| 指标 | 预算 |
|------|------|
| 首次可交互（TTI） | < 1.5s |
| 最大内容绘制（LCP） | < 2.0s |
| 累积布局偏移（CLS） | < 0.05 |
| 同屏并行动画 | ≤ 3 个（仅 transform/opacity/filter） |
| JS 主线程单帧 | < 8ms |
| GSAP + Lenis 总体积 | < 25KB gzip |

**原则**：CSS 动画优先 > GSAP（仅用于 ScrollTrigger scrub）> JS requestAnimationFrame（仅打字机）。
