# 🎨 世界级水平审查 — 交互体验维度

> **审查人**：体验设计专家  
> **审查对象**：`index.html`（NDHY AI Team 产品官网）  
> **对标标杆**：Stripe.com（滚动叙事）· linear.app（动效精品）· vercel.com（开发者体验）  
> **审查日期**：2026-03-16  

---

## 总评

**综合得分：6.4 / 10**

这是一个**有叙事野心、有架构意识、但在执行精度上距离世界级还有明显差距**的作品。它做到了"结构完整"和"概念表达清楚"，但在滚动节奏的电影感、动画的精致度、交互反馈的一致性、移动端的打磨程度上，距离 Stripe/Linear/Vercel 的标准还有 3-4 分的提升空间。

好消息是：**骨架是对的，问题在肌肉和皮肤**。信息架构的逻辑递进是成立的，叙事的情绪设计是有想法的。但世界级不是"想法对了就行"——是每一帧动画、每一个 hover、每一次滚动停留都经过精密调校。

## 维度 1：信息架构

**得分：7.5 / 10**

### 做对了什么

7 个区块的逻辑递进是成立的，且有叙事思维：

```
悬念（Hook）→ 震撼（Reveal）→ 证据（Process）→ 对比（Contrast）→ 深度（Architecture）→ 愿景（Vision）→ 行动（CTA）
```

这不是"关于我们 / 产品功能 / 联系我们"的平铺直叙，而是**故事弧线**。从"一个人启动了一个项目"到"他没有招过一个人"，用户 10 秒内确实能理解"这是什么"——至少能理解"这是一个很不一样的东西"。

区块之间的信息密度递增设计合理：Hook 极简（一句话），Reveal 开始丰富（拓扑图 + 总结语），Process 进入中密度（时间线），Architecture 高密度（三张剖面图）。这符合用户认知负荷的渐进曲线。

### 差距在哪

1. **Block 5（Architecture）信息密度过高，与 Block 4（Contrast）形成瓶颈**。三张剖面图连续呈现，每张内部又有多行文字，用户很容易在这里"滑过去"。Stripe 的做法是**一屏一个概念**，绝不在同一个滚动区间塞三个并列概念。
   
2. **Block 3（Process）的 8 个时间线步骤缺乏信息分层**。8 步同等视觉权重，用户无法快速抓住"哪几步最关键"。应有 2-3 个**高光步骤**（视觉权重更大）和 5-6 个**支撑步骤**（可折叠/弱化）。

3. **缺少全局导航锚点**。Stripe 和 Vercel 都有极简导航条（固定或渐显），用户在 Block 5 时知道自己在页面的什么位置，也能快速跳转。当前页面完全依赖线性滚动，超过 3 屏后用户容易迷失。

4. **"[待回填]"占位符虽然被 JS 隐藏了，但语义上暴露了未完成状态**。如果隐藏失败（JS 错误、CSP 限制），用户看到的就是一个半成品。

### 对标差距

| 维度 | 当前 | Stripe 水平 |
|------|------|------------|
| 10 秒理解 | ✅ 做到了 | ✅ |
| 区块逻辑递进 | ✅ 清晰 | ✅ |
| 信息密度节奏 | 🟡 Block 5 过载 | 一屏一概念 |
| 全局导航感 | ❌ 无 | 极简固定导航 |

## 维度 2：滚动叙事

**得分：6.0 / 10**

### 做对了什么

叙事结构有电影意识。Block 1 的悬念 → Block 2 的揭幕 → Block 6 的哲学升华，构成了一个完整的**三幕结构**。"散兵赢不了战争"这句文案的力量是存在的。

Block 1 → Block 2 的过渡是全页最好的：打字机结束 → 下滚箭头呼吸 → 滚入拓扑图逐个亮起。这 30 秒的体验是有节奏感的。

### 差距在哪

1. **滚动没有"粘性"（sticky）编排**。Stripe 的标志性手法是用 `position: sticky` + 滚动进度映射来实现"一段滚动触发一系列变化"——用户的滚动速度被内容节奏"接管"。当前页面的所有动画都是 IntersectionObserver 触发的 `.visible` 切换——本质上是"滚到这里就淡入"，没有滚动进度与动画进度的绑定。这是**触发式动画**，不是**滚动驱动动画**。

   Stripe 水平：滚动 100px，拓扑图旋转 15°，文字渐变 30%，背景色过渡 10%——一切与滚动距离成线性/缓动映射。
   
   当前水平：滚到 threshold → 触发 opacity 0→1。二值化的，没有中间态。

2. **Block 3 → Block 4 → Block 5 节奏塌陷**。连续三个"信息展示"区块，没有节奏变化——都是"标题 + 内容区淡入"。缺少呼吸空间、缺少视觉节奏变化、缺少情绪转折的间隔。Stripe 会在信息密集区之间插入**全屏留白 + 一句话**作为节奏呼吸。

3. **Block 6（Vision）的文字墙问题**。这是全页最重要的哲学表达，但呈现方式是"一段接一段的文字逐个淡入"。没有视觉辅助、没有空间张力。Stripe 的愿景表达会用**极大字号 + 极少文字 + 极深呼吸空间**。当前的 `1.1rem` 字号和密集段落排布，把最震撼的内容做成了最平的阅读体验。

4. **缺少视差（parallax）层次感**。整个页面是平面的——所有内容在同一个 z-depth 上滚动。Stripe/Linear 会用至少 2-3 层速度差来创造深度感：背景慢滚、内容正常滚、装饰元素快滚。

5. **区块间过渡太硬**。`::after` 伪元素画了一条 120px 渐变线作为分隔，但这只是装饰，不是过渡。世界级的过渡是**背景色渐变 + 内容预告**——在当前区块的末尾，用户已经隐约看到下一区块的开头，形成视觉连续性。

### 对标差距

| 维度 | 当前 | Stripe 水平 |
|------|------|------------|
| 叙事三幕结构 | ✅ 有 | ✅ |
| 滚动驱动动画 | ❌ 仅 IntersectionObserver | ScrollTimeline / sticky 编排 |
| 节奏呼吸 | ❌ 中段塌陷 | 信息区间穿插呼吸空间 |
| 视差深度 | ❌ 平面 | 2-3 层视差 |
| 区块过渡 | 🟡 装饰线 | 背景渐变 + 内容预告 |

## 维度 3：动画质量

**得分：6.5 / 10**

### 做对了什么

1. **打字机动画（Block 1）**：实现干净，速度适中（40ms/字），光标闪烁自然。这是全页最好的动画。
2. **拓扑图逐个亮起（Block 2）**：域分组 + 间隔递增的设计有仪式感，glow filter 的运用恰当。
3. **fade-in 参数选择合理**：`800ms` + `cubic-bezier(0.22,1,0.36,1)` 是经过考量的，不是默认的 `ease`。
4. **动画克制**：没有弹跳（bounce）、没有旋转、没有粒子。暗色调 + 克制动效的风格定位是对的。

### 差距在哪

1. **动画类型单一**。全站只有两种动画：(a) opacity 0→1 + translateY 16→0 的淡入，(b) Block 2 的 glow 亮起。Linear.app 用了至少 6 种动画类型：scale、blur→clear、路径描边、颜色过渡、布局动画、滚动进度映射。单一动画模式在 7 个区块重复 30+ 次后会产生**动画疲劳**。

2. **translateY(16px) 位移量太保守**。16px 的位移在 1920px 屏幕上几乎不可感知，淡入效果退化为"突然出现"。Vercel 的淡入位移是 40-80px，配合更长的 duration，创造出明确的"从下方升起"的空间感。

3. **Block 2 拓扑图动画缺少"高潮时刻"**。节点逐个亮起是好的，但缺少一个**统一亮起的爆发瞬间**（比如所有连线同时闪亮 + 整体 glow 放大）。当前的结束是"连线淡入 → 老板亮起 → 底部文字出现"——平淡收尾，缺少高潮。Linear 的组织图展示在最后一刻会有一个统一的"呼吸"效果。

4. **Block 5 的 cs-row 逐行动画太慢**。`150ms * (i+1)` 的间隔导致 4 行数据需要 600ms 才能全部出现，用户已经在等了。信息展示型动画应该更快（50-80ms 间隔），让用户感觉是"快速展开"而非"缓慢出现"。

5. **`.hook-emphasis::after` 下划线动画缺少精密编排**。下划线与文字同时出现（都在 `.visible` 时触发），应该**延迟 200-400ms**，先看到文字再看到强调线，形成"二次确认"的认知节奏。

6. **scroll-locked 解除的时机可能导致卡顿**。`body.scroll-locked` 在 Hook 动画完成后才移除，但如果用户在动画过程中尝试滚动，`overflow: hidden` 会吞掉滚动意图，解除后不会补偿——用户必须重新滚动。这不是流畅，是打断。

### 对标差距

| 维度 | 当前 | Linear.app 水平 |
|------|------|---------------|
| 动画类型多样性 | ❌ 仅 2 种 | 6+ 种 |
| 位移空间感 | 🟡 16px 太保守 | 40-80px |
| 高潮时刻 | ❌ 平淡收尾 | 爆发瞬间 |
| 动画节奏变化 | ❌ 匀速序列 | 快-慢-快 |
| easing 精度 | ✅ 自定义贝塞尔 | ✅ spring physics |

## 维度 4：交互反馈

**得分：5.5 / 10**

### 做对了什么

1. **按钮 hover/active 状态完整**：`.btn-primary` 有 hover 提亮 + active 缩放，`.btn-ghost` 有 hover 背景填充。
2. **时间线步骤 hover 有边框 + 阴影反馈**。
3. **拓扑节点 hover 有 tooltip**，且桌面端用 mouseenter/mouseleave，移动端用 tap toggle。
4. **`:focus-visible` 已实现**，键盘用户有可见焦点环。

### 差距在哪

1. **时间线步骤的点击反馈不清晰**。点击步骤会展开详情（accordion），但展开行为**没有即时视觉反馈**——没有步骤的 active 状态（按下变暗），没有展开方向的暗示（箭头旋转），没有展开过程中的进度感。用户点了之后要等 `max-height` 过渡完成才能确认"我的操作生效了"。Stripe 的 accordion 在点击瞬间就有**箭头旋转 + 背景色微变**。

2. **展开的 `max-height` 过渡有明显跳跃**。`max-height: 0 → 400px` 配合 `transition: max-height 300ms` 是一个已知的 CSS 反模式——如果实际高度是 150px，过渡会在 150px 处突然停止（因为到达了真实高度），看起来像"弹了一下"。应该用 `grid-template-rows: 0fr → 1fr` 或 JS 测量真实高度。

3. **拓扑 tooltip 的 200ms 延迟太长**。鼠标滑过节点时 200ms 后才出现 tooltip，在快速浏览时用户已经移走了。Linear 的 tooltip 延迟是 50-80ms。同时，tooltip 消失没有延迟（mouseleave 立即隐藏），这导致鼠标从节点移到 tooltip 的过程中 tooltip 会闪烁。

4. **下滚箭头 `↓` 的点击目标太小**。字符 `↓` 的实际可点击面积约 24×24px，远小于 44px 的触控最小标准。虽然它是一个 `<button>`，但没有 padding 扩展点击区域。

5. **Block 4（Contrast）的面板缺乏交互**。两个面板是纯静态展示，没有 hover、没有点击、没有差异高亮。这是全页信息量最大的对比区，用户期望能**hover 高亮对应项**（比如"没有记忆" hover 时同时高亮"分层记忆"）。Vercel 的对比表格就有行高亮联动。

6. **按钮的 transition duration 不一致**。`.btn-primary` 的 hover 是 `150ms`，timeline step 的 hover 是 `300ms`，tooltip 是 `150ms`。150ms 和 300ms 混用导致交互节奏不统一——有些地方反应快，有些地方反应慢。世界级标准是所有微交互统一到 `100-200ms`。

7. **CTA 按钮缺少 hover 时的动效惊喜**。Stripe 的 CTA 按钮 hover 时有渐变流光效果，Linear 的按钮有微妙的 glow 扩散。当前的按钮 hover 只是颜色变浅 `#9ba3fb`——太平了。

### 对标差距

| 维度 | 当前 | Vercel 水平 |
|------|------|------------|
| 点击即时反馈 | 🟡 部分有 | 每个可交互元素都有 |
| Accordion 动画 | ❌ max-height 反模式 | smooth height / layout animation |
| Tooltip 响应 | 🟡 200ms 偏慢 | 50-80ms + 移出延迟 |
| 交互一致性 | ❌ duration 混用 | 统一微交互时长 |
| CTA hover 惊喜 | ❌ 仅变色 | 渐变流光/glow |
| 对比面板联动 | ❌ 纯静态 | 行高亮联动 |

## 维度 5：移动端体验

**得分：5.5 / 10**

### 做对了什么

1. **有响应式断点**：600px 和 1023px 两个断点，字号、间距、布局都有调整。
2. **对比面板在移动端改为单列堆叠**（`grid-template-columns: 1fr`）。
3. **CTA 按钮在移动端改为全宽竖排**。
4. **拓扑图没有被隐藏**——SVG 通过 `width: 100%` + `viewBox` 自适应缩放，保留了核心视觉。
5. **移动端有触摸跳过 Hook 动画**（touchstart 监听）。
6. **移动端拓扑节点增加了 22px 最小触控区域**。

### 差距在哪

1. **拓扑图在 375px 屏幕上的可读性严重不足**。`viewBox="0 0 960 980"` 在 375px 宽度下，每个节点的 emoji 只有约 7px 实际渲染大小，文字标签约 4px——**完全不可读**。虽然保留了视觉震撼感，但失去了信息传达功能。应该在 ≤600px 时切换为简化版拓扑（只显示核心 5-8 个节点 + Leader），或者改用横向滚动的节点列表。

2. **触控目标不达标的元素**：
   - 下滚箭头 `↓`：约 24×24px（标准要求 44×44px）
   - 时间线步骤的 `.step-content`：高度约 60px 但可点击区域不明确（整个 `.timeline-step` 是点击目标，包含空白区域，容易误触）
   - 拓扑节点虽然有 22px 的隐形触控区，但 22px 仍然低于 44px 标准

3. **`scroll-locked` 在移动端更具破坏性**。移动端用户对滚动锁定更敏感（"为什么划不动？"），且 touchstart 跳过逻辑要求用户**在 Hook 区域内触摸**——如果用户试图从屏幕边缘滑动（iOS 返回手势区域），不会触发 Hook 区域的 touchstart。

4. **Block 5（Architecture）三张剖面图在移动端占据 3+ 屏高度**，且每张图内部的 `cs-row` 使用等宽字体（monospace），在窄屏上可能溢出或换行不优雅。

5. **Block 6（Vision）的长段落在移动端阅读体验差**。段落间距 `margin-bottom: 24px` 在桌面端合理，但移动端需要更大的间距来适应拇指滚动的颗粒度。

6. **没有横竖屏切换处理**。iPad 横屏 → 竖屏时，拓扑图的布局不会重新计算 tooltip 位置（tooltip 使用 `position: fixed` + 基于 `getBoundingClientRect` 的计算，旋转后可能偏移）。

7. **缺少 `prefers-reduced-motion` 支持**。部分用户（尤其是有前庭障碍的用户）需要减少动画。当前所有动画无条件播放。

### 对标差距

| 维度 | 当前 | 世界级标准 |
|------|------|----------|
| 拓扑图移动端可读性 | ❌ 7px emoji | 简化/替代方案 |
| 触控目标 ≥ 44px | 🟡 部分达标 | 全部达标 |
| scroll-lock 体验 | ❌ 移动端破坏性 | 无锁或智能解锁 |
| 横竖屏适配 | ❌ 未处理 | 布局重计算 |
| reduced-motion | ❌ 未支持 | 尊重系统偏好 |

## 维度 6：情绪曲线

**得分：7.0 / 10**

### 做对了什么

情绪设计有**明确意图**，7 个区块对应的情绪是：

```
好奇（Hook）→ 震撼（Reveal）→ 信服（Process）→ 优越感（Contrast）→ 理解（Architecture）→ 共鸣（Vision）→ 行动（CTA）
```

这是一条经过设计的弧线，不是随机堆砌。特别好的几个点：

1. **Hook 的悬念设计是有效的**。"一个人启动了一个项目" → （停顿）→ "6 天后，他拥有了一个 28 人的产品团队" → （停顿）→ "他没有招过一个人"。三段式悬念揭示，每段之间有呼吸空间。这个序列的情绪张力是到位的。

2. **Contrast（Block 4）的情绪设计是全页最聪明的**。不是"我们很好"，而是"看看没有我们的世界有多痛苦"。dim 面板的 dashed border + 低 opacity + "凑合能用了" 的文案，vs bright 面板的 solid border + glow + 结构化流程——视觉对比直接转化为情绪对比。

3. **Vision（Block 6）的"散兵赢不了战争"是全页最好的文案**。简短、有力、不装。

### 差距在哪

1. **震撼峰值（Block 2）的情绪冲击不够强**。拓扑图的逐个亮起是"缓慢展开"而非"震撼揭幕"。真正的震撼需要一个**爆发瞬间**——所有节点亮起后有一个整体的 scale-up + glow 扩散 + 底部文字弹出。当前的结束太平了，像"陈列品"而非"揭幕时刻"。

2. **Block 3 → Block 4 → Block 5 的情绪平坦区太长**。这三个区块都是"理性证据"，连续 3 屏的理性输入会导致情绪疲劳。应该在 Block 4 和 Block 5 之间插入一个**情绪呼吸点**——一句话 + 大留白，比如"那么问题来了——它为什么能运转？"。

3. **Block 7（CTA）的情绪号召力弱**。"想看看这个组织是怎么运转的？"是一个理性问题，不是情绪号召。对比 Stripe 的 CTA："Start building with Stripe" + 一个会发光的按钮。情绪号召需要**动词 + 视觉惊喜**。当前的 CTA 像一个表单页面的结尾，不像一个史诗叙事的终章。

4. **结尾的 footer 反高潮**。从 Vision 的哲学高度 → CTA 的行动号召 → footer 的 "Built by NDHY-AI-TEAM"——情绪直接从 80% 跌到 20%。footer 本身不是问题，问题是 CTA 和 footer 之间没有缓冲。"包括你正在看的这个网站。" 这句 footer-meta 是一个 callback（回扣），但因为视觉权重太低（xs 字号 + 斜体），被淹没了。这句话值得更大的视觉权重。

5. **情绪弧线缺少"意外"**。从 Hook 到 CTA，用户能预期每一步——"接下来大概是介绍团队 / 展示流程 / 说愿景 / 让我注册"。世界级的叙事会有**至少一个意外转折**——比如 Stripe 在展示了一堆产品功能后，突然来一屏"Even if you're not a developer…"，打破目标受众预期。

### 情绪曲线图

```
情绪强度
  10 │
```

## 维度 7：无障碍

**得分：6.0 / 10**

### 做对了什么

1. **`<a class="skip-link">` 存在**，且样式正确（`:focus` 时从 `top: -40px` 移到 `top: 0`）。
2. **`:focus-visible` 全局规则**，焦点环使用 accent 色 + 2px offset。
3. **所有 `<section>` 有 `aria-label`**，屏幕阅读器能识别区块名称。
4. **拓扑 SVG 有 `role="img"` + `aria-label`**。
5. **打字机元素有 `aria-live="polite"`**，内容变化会被朗读。
6. **tooltip 有 `role="tooltip"` + `aria-hidden` 动态切换**。
7. **按钮有 `aria-label`**（下滚箭头、CTA 按钮）。

### 差距在哪

1. **拓扑 SVG 内部的 30 个节点对屏幕阅读器不可见**。SVG 内的 `<text>` 元素虽然有文字内容，但没有 `role` 或 `aria-label`。整个拓扑图在屏幕阅读器中只读出一句 aria-label 描述——28 个专家的具体信息全部丢失。应该为每个 `.node` 组添加 `role="listitem"` + `aria-label`，或在 SVG 旁提供一个 visually-hidden 的文本列表。

2. **时间线的 accordion 缺少 ARIA 扩展属性**。`.timeline-step` 缺少 `aria-expanded="true/false"`，`.step-detail` 缺少 `aria-controls` / `id` 配对。屏幕阅读器用户无法知道"这个步骤是否已展开"。

3. **键盘无法操作拓扑节点**。所有 `.node` 都是 `<g>` 元素，没有 `tabindex`，键盘用户无法 Tab 到任何节点来触发 tooltip。应该添加 `tabindex="0"` + `role="button"` + 键盘事件。

4. **Hook 区域的 `scroll-locked` 对键盘用户有影响**。`overflow: hidden` 会阻止键盘 Tab 焦点自然滚动到视口外的元素。如果用户在 Hook 动画期间尝试 Tab 到 Block 2 的内容，焦点会被困在 Block 1。

5. **对比度问题**：
   - `.dim .flow-step` 的 `color: #999` 在 `background: rgba(10,10,15,0.8)` 上的对比度约 **4.2:1**，刚好低于 WCAG AA 要求的 4.5:1 for normal text。
   - `.footer-stats` 的 `color: #666` 在 `#0d0d14` 上的对比度约 **3.5:1**，不达标。
   - 拓扑图未亮起时的 label `fill: rgba(232,232,232,0.12)` 几乎不可见（对比度 < 1.5:1），虽然这是设计意图（等待动画亮起），但对低视力用户来说是完全不可感知的。

6. **缺少 `lang` 属性的子区域标注**。页面整体是 `lang="zh-CN"`，但有英文内容（"Built by NDHY-AI-TEAM"、"Leader"、"Controller/Service/Repository"），缺少 `lang="en"` 标注，屏幕阅读器可能用中文语音引擎读英文。

7. **动画无 `prefers-reduced-motion` 支持**（维度 5 已提到，此处为无障碍影响）。前庭障碍用户可能会因持续的淡入动画、呼吸动画感到不适。

这个网站最大的资产不是代码，是**叙事结构**。7 个区块的逻辑递进、"散兵赢不了战争"的核心论点、Hook 区的三段式悬念——这些是对的，且不容易做到的。
