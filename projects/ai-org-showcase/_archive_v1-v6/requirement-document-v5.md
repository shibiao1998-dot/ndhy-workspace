# 官网 V5 — 迭代需求文档

> 🎯 需求分析专家 · 2026-03-16
> 基于 V4 批准原型 + 33 项修复清单 + 3.16 组织进化成果的增量迭代

---

## 一、迭代目标

**V5 = V4 批准原型的完整实现 + 组织进化内容更新**

V4 确立了视觉方向（深色渐变 + 粒子特效 + 科技字体 + 专家悬停卡片），老板已批准。V5 的核心任务不是重新设计，而是：

| 维度 | V4 现状 | V5 增量 |
|------|---------|---------|
| 视觉 | 方向已定，静态原型获批 | 动效精度补齐（33项修复） |
| 内容 | 基于 3.12-3.15 组织状态 | 更新为 3.16 最新组织能力 |
| 技术 | 单文件 HTML 原型 | 修复全部阻断项，提升健壮性 |

**一句话目标**：把批准的原型从"看起来对"变成"真正能跑"，同时让内容反映组织最新进化

V4 原型的视觉框架已获批，V5 不改方向，只补精度：

| # | 迭代项 | 说明 | 优先级 |

## 七、约束与注意事项

1. **不改视觉方向**：V4 原型已获批，V5 是精度补齐不是重新设计
2. **ACP 铁律**：高保真原型必须通过 Claude Code 产出（今日确立的规则）
3. **单文件保持**：V4 原型是单 HTML 文件，V5 保持同一形态（不拆框架），降低交付复杂度
4. **修复优先于新增**：先把 33 项修好，再考虑新功能
5. **Token 规范一致性**：所有修改必须使用已定义的 Design Token，禁止新增裸值
