## 一、V3 七幕叙事结构评价

### 1.1 每个区块在干什么

先用大白话说清楚这 7 个区块各自的"叙事工作"：

| 区块 | 叙事功能 | 大白话 |
|------|---------|--------|
| ① 悬念开场 | **认知冲突制造器** | 用一个"不可能"的事实勾住注意力——"一个人怎么可能 6 天有 28 人团队？" |
| ② 揭幕震撼 | **悬念兑现 + 规模震撼** | 回答"怎么做到的"——不是招人，是 AI 组织。28 个节点亮起来的视觉直接打规模感 |
| ③ 运转揭秘 | **自证可信度** | "你正在看的这个网站，就是这帮 AI 做出来的"——用产品自身当证据，这是全站最硬的一招 |
| ④ 认知落差 | **制造对比优越感** | "散装 AI vs AI 组织"——让访客看到差距，产生"原来还可以这样"的认知 |
| ⑤ 深层架构 | **建立理性信服** | 用三张剖面图（权力/记忆/进化）解释"为什么它能运转"，从"看到"变成"理解" |
| ⑥ 更大图景 | **提升到行业宣言** | 从"我们做了什么"上升到"AI 行业该怎么走"，给访客一个更大的认同框架 |
| ⑦ 行动号召 | **推动行动** | 给一个出口：联系我们 / 看 GitHub |

### 1.2 整体情绪曲线

V3 的情绪设计是这样走的：

```
情绪强度
  高 │     ②震撼          ⑥共鸣
     │    ╱  ╲          ╱    ╲
  中 │  ╱  ③沉浸╲    ╱  ⑤理解  ╲
     │╱          ╲  ╱            ╲⑦行动
  低 │①好奇      ④落差
     └──────────────────────────────→ 滚动进度
```

这是一条**双峰曲线**——第一个高峰在区块②（视觉震撼），第二个高峰在区块⑥（认知共鸣），中间有一个刻意的情绪低谷（区块④对比制造的"不安感"）。这种双峰设计在认知心理学里有一个概念叫**峰终定律**（Peak-End Rule）：人们记住的是体验中的峰值和结尾。V3 让峰值落在"AI 组织亮起"的视觉震撼和"散兵赢不了战争"的认知共鸣上——这两个点是对的。

### 1.3 哪些是世界级的

**① 区块一的悬念开场：9/10**

为什么好：三段式递进悬念（"一个人启动" → "6 天 28 人" → "没招过一个人"），每一段之间有停顿，让受众的大脑自动填补认知空缺。这用的是**认知缺口理论**（George Loewenstein, 1994）——好奇心产生于"知道一点但不知道全部"的状态。V3 精准地制造了这个状态：你知道"发生了一件不可能的事"，但不知道"怎么做到的"，所以你会往下滚。

更厉害的是**刻意不出现任何术语**——不说 AI、不说 Agent、不说组织。纯用事实说话。这比 Cognition.ai 首页的"a collaborative AI teammate"更有悬念感，因为 Cognition 一开口就告诉你答案了。

**② 区块三的自证策略：10/10**

为什么好："你正在看的这个网站，是这样被做出来的。真实过程，0 行人类代码。"——这是全站叙事策略的**核武器**。

所有产品官网都在"告诉"访客自己有多强。NDHY 是在"展示"。这叫**show, don't tell**——文学叙事的第一法则。Stripe 的做法是在首页嵌入一段真实可运行的 API 代码来演示支付能力，这已经是行业标杆了。但 NDHY 的自证比 Stripe 更彻底——Stripe 展示的是产品能力的**片段**（一段代码），NDHY 展示的是产品能力的**完整闭环**（整个网站从需求到上线的全过程）。

这个策略之所以是 10 分，因为它做到了一件极难的事：**让受众无法反驳**。你可以质疑一段文案是不是吹牛，但你没法质疑"你正在看的网站就是证据"。

**③ 区块四的对比结构：8.5/10**

为什么好：不是说"我们好"，而是说"看看没有我们的世界"。这用的是**损失厌恶**（Loss Aversion, Kahneman & Tversky）——人对"失去"的恐惧比"得到"的渴望强 2 倍。左栏灰暗的"散装 AI"流程让访客想到自己用 ChatGPT 凑合搞东西的经历，右栏明亮的"AI 组织"流程立刻产生对比。四组 ❌/✅ 标签（记忆/标准/协作/进化）把差异打成了清单，扫描式阅读极其友好。

**④ 区块六的行业宣言：8.5/10**

"AI 不缺能力……但它们是散兵。散兵赢不了战争。"——这是全站最好的金句。它做了一件叙事策略中很难做到的事：**重新框定问题**。访客来之前可能觉得"AI 的问题是模型不够强"，看完这段他会觉得"AI 的问题是没有组织"。这种认知转换是叙事的终极目标——不是灌输你的结论，而是让受众觉得是自己想明白的。

### 1.4 哪些有改进空间

**① 区块三和区块五之间的节奏塌陷：7/10**

问题：区块③（运转揭秘）、④（认知落差）、⑤（深层架构）连续三个区块都在做同一件事——**证明**。只是维度不同：③是过程证明，④是对比证明，⑤是结构证明。连续三屏"证据型"内容会导致**认知疲劳**——受众的情绪从③的"哦，原来如此"开始走平，到⑤已经有点"还在讲啊"。

根据认知心理学的**注意力恢复理论**（Attention Restoration Theory, Kaplan 1995），高密度信息输入之间需要"微恢复"时间。V3 在③和⑤之间缺少一个情绪呼吸点。

**② 区块七 CTA 的收尾偏弱：5/10**

问题："想看看这个组织是怎么运转的？"是一个理性问题，不是情绪号召。全站从区块一到区块六建立了一个从好奇到震撼到信服到共鸣的完整情绪弧线——然后在最后一步，情绪突然冷了下来，变成了一个"有空联系一下"的语气。

这就好比一部电影，前 90 分钟把你的情绪推到了高峰，最后一幕突然切到一个"欲知后事如何，请拨打以下电话"。峰终定律告诉我们：**结尾的体验会被重点记忆**。当前的 CTA 不配当这部"电影"的结尾。

**③ 首屏价值主张的缺失：风险可控但需要关注**

悬念策略的代价是：如果访客不往下滚，他们带走的认知为零。Stripe 和 Linear 的首屏即使被秒退，访客也能带走一个关键词——"支付基础设施"或"产品开发系统"。NDHY 的首屏只留下"有个人搞了个团队"。

这不一定要改——悬念策略的高回报就是以高风险为代价的。但建议在不破坏悬念的前提下，留一个极轻的品类提示。

## 二、对标分析

### 2.1 Stripe.com 的叙事策略

**当前首页结构**（2026.03 实测）：
Stripe 现在的首页叙事已经从纯粹的"金融基础设施"定义升级为**数据驱动的信任叙事**。开场直接用宏大数据锚定——"$1.9T in payments volume processed in 2025"、"135+ currencies"、"99.999% uptime"。然后通过客户案例（Mindbody、Jobber、Substack、Lightspeed 等真实引言）建立社会认同，最后用产品矩阵和开发者入口完成转化。

Stripe 的核心叙事策略是：**用数据建立信任，用客户验证可靠性，用产品深度展示能力**。它是"基础设施"叙事——不讲故事，讲事实。

**与 NDHY 的对比**：

| 维度 | Stripe | NDHY |
|------|--------|------|
| 开场策略 | 数据锚定（$1.9T） | 悬念制造（认知冲突） |
| 信任建立 | 客户引言 + 社会认同 | 自证（网站本身是产品） |
| 情感类型 | 理性信任（安全感） | 情感震撼（惊叹感） |
| 首屏价值 | 3 秒内知道"它是什么" | 3 秒内只有"怎么可能" |
| CTA 策略 | 多层转化入口贯穿全页 | 仅最底部一个 CTA |

**Stripe 做了什么我们没做的**：
1. **社会证明层**——客户 logo、真人引言、使用数据。NDHY 全站零社会证明。
2. **多层 CTA**——Stripe 首屏有 CTA、中段有 CTA、尾部有 CTA。NDHY 只有尾部。
3. **数据锚定**——用 $1.9T 这种大数字建立"我们已经是行业标准"的认知。

**NDHY 做了什么 Stripe 没做的**：
1. **故事性**——Stripe 没有故事，只有信息。NDHY 有完整的叙事弧线。
2. **自证策略**——Stripe 用代码演示能力片段，NDHY 用整个网站演示能力闭环。
3. **情绪设计**——Stripe 的情绪是平的（信任感贯穿全程），NDHY 有波动、有高潮、有节奏。

### 2.2 Linear.app 的叙事策略

**当前首页结构**（2026.03 实测）：
Linear 最近的首页定位为"The product development system for teams and agents"，采用了**功能叙事 + AI 融合**策略。结构是：品牌宣言（"A new species of product tool"）→ 三大核心价值（Purpose-built / AI agents / Speed）→ 四段产品流程演示（Intake → Plan → Build → Diffs），每段都有真实的 UI 截图和交互动画。

Linear 的叙事策略本质是**产品即叙事**——它不讲"为什么你需要我们"的故事，它直接展示"用我的感觉有多爽"。那些精致的 UI 动画和产品截图本身就是最有说服力的论据。

**与 NDHY 的对比**：

| 维度 | Linear | NDHY |
|------|--------|------|
| 叙事类型 | 产品演示叙事 | 故事/宣言叙事 |
| 动效水平 | 6+ 种动画类型，spring physics | 2 种动画类型（淡入+亮起） |
| 信息节奏 | 一屏一概念，节奏分明 | 中段三屏连续证据，节奏略平 |
| 品类定义 | 直接说"product development system" | 通过故事引导理解"AI 组织" |

**Linear 做了什么我们没做的**：
1. **动效精度**——Linear 的每一个微交互都经过数十次调校，是行业动效标杆。V3 的动效在类型和精度上差距明显。
2. **一屏一概念**——Linear 严格做到每屏只传达一个信息，NDHY 的区块五塞了三个剖面。
3. **产品可感知性**——Linear 让你"看到"产品，NDHY 让你"听到"故事。

**NDHY 做了什么 Linear 没做的**：
1. **品类创造**——Linear 在已有品类（项目管理工具）里竞争，NDHY 在定义一个新品类（"AI 组织"）。定义新品类需要更多的叙事教育，这是 V3 用 7 个区块循序渐进的根本原因。
2. **行业宣言**——Linear 不做行业预判，NDHY 区块六敢说"答案是——组织"。
3. **创始人叙事**——"一个人启动了一个项目"的叙事天然自带人格化魅力，Linear 没有这层。

### 2.3 Cognition.ai（Devin）的叙事策略

**当前首页结构**（2026.03 实测）：
Cognition.ai 首页极度简洁——品牌定义（"An Agent Lab building the future of software engineering"）+ Devin 产品描述（"a collaborative AI teammate"）+ 博客/更新流。它的叙事策略是**权威型**——不讲故事，直接用"第一个 AI 软件工程师"的品类定义 + 持续的技术博客输出建立技术权威。

最近的更新（SWE-1.6 Preview、Devin 2.2、Cognition for Government）表明 Cognition 在用"How Cognition Uses Devin to Build Devin"这种自证叙事——和 NDHY 的"网站是 AI 做的"异曲同工。

**与 NDHY 的对比**：

| 维度 | Cognition | NDHY |
|------|-----------|------|
| 首页策略 | 极简品牌定义 + 博客流 | 7 幕电影式叙事 |

**Cognition 做了什么我们没做的**：
1. **技术深度内容**——Cognition 通过大量技术博客（Agent Trace、SWE benchmarks）建立技术权威。NDHY 缺少技术深度展示层。
2. **产品 demo 入口**——Cognition 让你可以直接试用 Devin。NDHY 没有可交互的产品体验。

**NDHY 做了什么 Cognition 没做的**：
1. **叙事完整度**——Cognition 首页信息极其有限，NDHY 首页是一部完整的"电影"。
2. **叙事层次**——NDHY 从事实（区块①②③）到对比（④）到原理（⑤）到宣言（⑥），有完整的认知递进。Cognition 只有一层：品牌定义。
3. **情绪设计**——Cognition 的首页情绪是平的，NDHY 有完整的情绪曲线。

## 三、建议：保留核心叙事，精准调整结构

### 3.1 明确结论：**调整**，不重做

理由：

1. **V3 的七幕叙事骨架是对的。** 悬念→震撼→自证→对比→原理→宣言→行动，这个认知递进路径符合 AIDA 模型（注意→兴趣→欲望→行动）的底层逻辑，且在自证策略上做了远超标杆的创新。重做意味着放弃这个已被验证有效的结构。

2. **问题出在执行精度和结构节奏，不在叙事策略。** 产品审查给叙事结构打了 9/10，UX 审查给情绪曲线打了 7/10——差的 2 分是动效执行和中段节奏，不是叙事逻辑。

3. **品类教育需求决定了必须保留足够的叙事深度。** "AI 组织"是一个全新概念。不像 Stripe（支付大家都懂）或 Linear（项目管理大家都懂），NDHY 需要教育受众"什么是 AI 组织"。7 幕结构恰好提供了从"不知道"到"理解并认同"所需的认知台阶。砍掉任何一个台阶都会让认知路径断裂。

### 3.2 具体调整清单

| 区块 | 动作 | 调整内容 | 理由 |
|------|------|---------|------|
| ① 悬念开场 | **保留，微调** | 在首屏底部（滚动箭头上方）加一行极淡的品类提示词"AI 组织" | 给秒退用户留一个关键词种子，不破坏悬念 |
| ② 揭幕震撼 | **保留，强化** | 在节点全亮后增加一个"爆发时刻"（整体 glow 扩散 + scale-up） | 用峰值体验强化震撼感，让这个高峰更高 |
| ③ 运转揭秘 | **保留，精简** | 从 8 步精简为 4-5 个高光步骤，其余折叠。步骤叙述差异化 | 减少信息密度，避免模式化重复 |
| ③→④ 之间 | **新增呼吸点** | 插入一个全屏留白 + 一句话过渡，如"这不是个例。这是一种新范式。" | 打破中段证据型内容的连续性，让情绪有呼吸 |
| ④ 认知落差 | **保留** | 结构不变，视觉执行升级（联动高亮等） | 叙事功能完美，只需动效层面升级 |
| ⑤ 深层架构 | **保留，重构展示** | 三个剖面从"连续展示"改为"一屏一概念"，每个剖面独立一屏 | 遵循 Linear 的一屏一概念法则，降低认知负荷 |
| ⑥ 更大图景 | **保留，强化** | 关键句（"答案是——组织"）字号放大到 2-3 倍，配合更大呼吸间距 | 这是全站情绪高峰，视觉权重必须匹配内容权重 |
| ⑦ 行动号召 | **重做** | 从理性提问改为情绪号召。增加中段 CTA，替换 mailto 为专业联系方式 | 当前 CTA 是全站最弱环节，严重拉低整体体验 |

### 3.3 论证总结

把 V3 的叙事比作一部电影：

- **剧本（叙事策略）**：9/10。故事怎么讲、按什么顺序讲、制造什么情绪——都是对的。
- **表演（内容文案）**：8/10。文案克制有力，几处金句世界级。
- **摄影和剪辑（动效和节奏）**：6/10。镜头运动单一，中段节奏拖沓。
- **结尾（CTA）**：5/10。一部好电影配了一个烂结尾。

**V4 要做的不是重新写剧本，而是请更好的摄影师、重剪中段节奏、重拍结尾。**

## 四、V4 叙事策略建议

### 4.1 叙事策略定位：从"讲故事"到"创造体验"

V3 的叙事是**被动观看型**——访客滚动页面，看一帧帧的信息，就像看电影。这已经比 95% 的产品官网更好了。

V4 要做的是升级到**沉浸体验型**——不只是"告诉你这个组织有多强"，而是"让你身临其境地感受这个组织在运转"。

**核心叙事升级**：
- V3：你在看一个**关于** AI 组织的故事
- V4：你在**经历**一个 AI 组织的运转

这个升级不需要改变七幕结构，需要改变的是每个区块的**体验方式**——从"读信息"变成"感受场景"。

### 4.2 V4 情绪曲线设计

```
情绪强度
  高 │         ②✦              ⑥✦
     │        ╱  ╲            ╱   ╲
  中高│      ╱    ╲    ⑤    ╱      ╲✦
     │    ╱  ③    ╲  ╱ ╲ ╱    ⑦  收束
  中 │  ╱          ╲╱          
  中低│╱     [呼吸]  ④
  低 │①
     └──────────────────────────────→ 滚动进度
     
✦ = 峰值时刻（需要动效"爆发"配合）
```

| 区块 | 情绪目标 | 情绪关键词 | 目标停留 |
|------|---------|-----------|---------|
| ① 悬念开场 | 从平静到好奇 | "不可能……" | 4-5s |
| ② 揭幕震撼 | 从好奇到震撼 **✦ 第一峰值** | "天哪，28 个 AI……" | 10-15s |
| ③ 运转揭秘 | 从震撼到沉浸 | "原来是这样做出来的" | 15-20s |
| [呼吸点] | 微停顿，准备下一波 | "嗯，这确实不一样" | 3-5s |
| ④ 认知落差 | 从沉浸到清醒 | "我之前的方式好 low" | 8-12s |
| ⑤ 深层架构 | 从清醒到理解 | "我懂了，它为什么能运转" | 15-20s |
| ⑥ 更大图景 | 从理解到共鸣 **✦ 第二峰值** | "他说的对，答案就是组织" | 10-15s |
| ⑦ 行动号召 | 从共鸣到行动意愿 | "我想了解更多" | 5-8s |

**相比 V3 的变化**：
1. ③和④之间插入了呼吸点，打破中段平坦区
2. ⑤从连续三剖面改为独立展示，情绪有起伏不再一条直线
3. ⑦从理性提问升级为情绪收束，和①形成更强的首尾呼应

### 4.3 受众心理路径

V3 的受众定义是"所有人，不分优先级"。这意味着叙事结构必须是**主干统一、分支满足**——主叙事线对所有人有效，特定受众在特定区块得到更深的满足。

| 区块 | 技术人心理 | 产品人心理 | 投资人心理 | 创业者心理 |
|------|-----------|-----------|-----------|-----------|
| ① 悬念 | "技术上怎么实现的？" | "这个团队模型太有意思了" | "6 天 28 人=极低边际成本" | "如果我也能这样……" |
| ② 揭幕 | "28 个 AI 角色，各司其职——架构设计是对的" | "完整的产品流水线，覆盖全链路" | "不需要招人=人力成本趋近于零" | "这个组织架构我能复制吗？" |
| ③ 自证 | ★ **高光时刻**：看到真实的技术链路 | ★ **高光时刻**：看到需求→交付全流程 | "它能做出完整产品，不是概念" | "这比我用 ChatGPT 强太多了" |
| ④ 对比 | "对，散装 AI 确实没有记忆和标准" | "对，一个模型扮演所有角色确实不行" | "组织化 vs 工具化，赛道级差异" | ★ **高光时刻**：强烈代入感 |
| ⑤ 架构 | ★ **高光时刻**：权力分层和记忆体系的技术细节 | "这个治理结构比我见过的 AI 框架都成熟" | ★ **高光时刻**：进化机制=复利效应 | "原来需要权力分层和记忆" |
| ⑥ 宣言 | "说的有道理，组织化确实是正确方向" | "他在定义一个新品类" | "如果这是对的，这是一个巨大的市场" | ★ **高光时刻**："一个人+AI=公司" |
| ⑦ CTA | "给我看代码/文档" | "能不能合作？" | "怎么联系？" | "我想试试" |

**关键洞察**：每类受众在不同区块到达自己的"高光时刻"——技术人在③和⑤被打动，产品人在③被打动，投资人在⑤和⑥被打动，创业者在④和⑥被打动。这意味着**七幕结构的完整性是必要的**——砍掉任何一个区块，都会让某类受众失去他们的高光时刻。

### 4.4 关于"超越所有网站"

老板说要"比现在任何一个前端网页都要牛逼"。基于对标分析，我认为**叙事策略层面 V3 已经做到了**。NDHY 的叙事在结构完整度、自证策略、差异化定位上已经超过了 Stripe、Linear 和 Cognition。

V4 需要"超越"的部分不在叙事策略，而在：

1. **动效执行精度**——从 V3 的 2 种动画类型升级到 6+ 种，从触发式动画升级到滚动驱动动画。这是 Linear 的地盘，V4 需要在这里追平甚至超越。
2. **CTA 和转化设计**——从 mailto 个人 Gmail 升级到专业联系方式 + 多层转化入口。这是 Stripe 的地盘，V4 需要在这里补齐。
3. **社会证明**——哪怕只是 GitHub star 数、项目完成数、组织运转天数这样的自有数据，也比零社会证明强。
4. **技术深度入口**——为技术受众提供一个"Deep Dive"链接，指向更详细的技术文档或 GitHub 仓库。这是 Cognition 的地盘。

**叙事策略层面，V4 唯一需要"超越"V3 的是**：把"观看型叙事"升级为"沉浸体验型叙事"——让受众不只是看到故事，而是感受到 AI 组织的运转。这需要动效、交互和内容三方面的联合升级，是后续体验设计和视觉设计专家的核心任务。

## 五、给后续专家的叙事指引

基于本报告的分析结论，以下是给下游专家的关键约束：

1. **七幕结构不可删减**——每个区块服务不同受众的高光时刻，缺一不可
2. **区块③和④之间必须插入呼吸点**——这是解决中段节奏塌陷的核心修改
3. **区块⑤必须从"三剖面连续展示"改为"一屏一概念"**——降低认知负荷
4. **区块⑥的关键句必须获得最大视觉权重**——它是第二峰值的锚点
5. **区块⑦必须重做**——从理性问题改为情绪收束，与①形成首尾呼应
6. **自证策略（区块③）是全站核武器**——数据回填是前提，视觉呈现必须让人信服
7. **情绪曲线的双峰设计（②和⑥）必须保持**——体验设计和动效设计围绕这两个峰值来打
