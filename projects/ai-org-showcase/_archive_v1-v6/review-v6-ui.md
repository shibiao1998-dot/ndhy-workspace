# 🖌️ UI 设计走查报告 — V6 官网 S3-S7

> 🖌️ UI 设计专家产出 · 2026-03-18
>
> 走查基准：`design-v6-visual.md`（四层深度合成 + Token 体系 + 色彩弧线）
> 老板反馈："感觉有点奇怪，并且很生硬，不够丝滑和流畅"

---

## 总评

| 维度 | 总体评分 | 说明 |
|------|---------|------|
| 视觉层次 | ⭐⭐☆☆☆ 2/5 | 四层深度合成大面积缺失，大多数 Section 仅实现了 1-2 层 |
| Token 一致性 | ⭐⭐⭐⭐☆ 4/5 | 整体良好，少量裸色值在 GSAP 内联中（受限于技术限制），可接受 |
| 内容密度 | ⭐⭐⭐☆☆ 3/5 | S5 偏空（min-height: 80vh 强制撑高），S7 偏空；S3/S4/S6 适当 |
| 排版层级 | ⭐⭐⭐☆☆ 3/5 | 基本建立但层次感不够鲜明，标题辉光效果偏弱 |
| 色彩叙事 | ⭐⭐⭐☆☆ 3/5 | Token 已定义并应用于各 Section，但视觉冲击力不够——缺少背景特效层的色彩渲染 |
| 间距节奏 | ⭐⭐⭐☆☆ 3/5 | S5 内部留白过大（三面板各 80vh），S7 底部 margin-bottom: space-16 过大 |

**老板反馈根因分析**："生硬、不够丝滑"的核心原因：

1. **四层深度合成严重缺失** — 设计要求纹理→背景特效→内容→前景动效四层，实际大多只有内容层 + 一个顶部辉光（glow-top）。缺少深层纹理和区块专属粒子系统，页面看起来"平"。
2. **Section 间过渡生硬** — 使用 `scroll-snap` 强制全屏对齐 + GSAP pin，Section 之间是硬切而非流动过渡。缺少设计规范中要求的"区块光效过渡"（section::before 的渐变光晕从上一区块色调渐入当前色调）。
3. **背景特效层（Layer 2）几乎为空** — 代码雨仅 S3 有，S4 的双色场背景有但初始 opacity: 0 需 GSAP 触发，S5 蓝图网格同样初始隐藏，S6/S7 的光晕效果太微弱。
4. **前景动效层（Layer 4）完全缺失** — 设计要求的交互粒子（鼠标跟随散开/聚拢）在 S3-S7 全部没有实现。
5. **scroll-snap + height:100vh 导致内容被截断** — `section--snap` 强制 `height: 100vh; overflow: hidden`，当内容超出一屏时不可见，用户体验极差。

## 逐 Section 走查

### S3 时间线（TimelineSection）— ⭐⭐⭐☆☆ 3/5

**四层深度合成检查**：

| 层 | 设计要求 | 实际状态 | 判定 |
|----|---------|---------|------|
| L1 纹理层 | 全局微粒子 + 网格 + 扫描线 | ❌ 无（依赖全局层，但 S3 本身无补充） | 🟡 |
| L2 背景特效 | 代码雨 Canvas（40 列桌面/15 移动） | ✅ 已实现，参数匹配 | 🟢 |
| L3 内容层 | 标题发光 + 卡片毛玻璃 + 时间线轨道 | ✅ 基本实现 | 🟢 |
| L4 前景动效 | 交互粒子 | ❌ 完全缺失 | 🔴 |

**具体问题**：

- 🔴 **P0 — `section--snap` 未添加但 S3 有 GSAP pin** — S3 的 `#timeline` 没有 `section--snap` class，但 GSAP 使用 `pin: true` + `end: +=200%`。这是正确的——S3 是唯一不用 snap 的 Section。但时间线的 10 张卡片在 pin 阶段展示时，滚动距离 200% 可能不够所有卡片在视口内展示完毕（取决于卡片高度）。
- 🟡 **P1 — 时间线轨道缺少 GSAP 光速推进感** — 设计要求轨道从顶部"光速"向下延伸并有 trackHead 光点跟随。实际 GSAP 实现了 height 0% → 100% 和 trackHead 的 top 跟随，但 `scrub: 0.8` 让动效显得缓慢。设计规范说 "光速延伸"，建议 scrub 值降到 0.3-0.5 并增加一段 ease-in 加速感。
- 🟡 **P1 — 卡片入场缺少"能量展开"伴随效果** — 设计要求卡片入场时有 `box-shadow` 能量场闪烁，GSAP 中只有一个从 `S3_GLOW` → `S3_GLOW_DIM` 的过渡，缺少设计规范中的 `border-color` 闪烁变化。
- 🟡 **P2 — 代码雨 Canvas 使用 `monospace` 而非 `var(--font-mono)`** — Canvas 无法消费 CSS 变量（技术限制），但可以在 JS 中硬编码 `'SF Mono', 'Fira Code', Consolas, monospace'` 匹配 Token。
- 🟢 **通过** — 标题渐变使用 `var(--color-s3-accent)` + `var(--color-cyan)`，符合青绿色调。
- 🟢 **通过** — 卡片使用 `var(--color-glass-bg)` + `backdrop-filter: blur(12px)` 毛玻璃效果。
- 🟢 **通过** — 手风琴展开使用 `grid-template-rows: 0fr/1fr` 过渡，流畅。

### S4 对比（ContrastSection）— ⭐⭐⭐☆☆ 3/5

**四层深度合成检查**：

| 层 | 设计要求 | 实际状态 | 判定 |
|----|---------|---------|------|
| L1 纹理层 | 全局层 | ❌ 无额外纹理 | 🟡 |
| L2 背景特效 | 双色场径向渐变（红左蓝右） | ✅ 已实现，初始 opacity:0 由 GSAP 驱动 | 🟢 |
| L3 内容层 | 两列卡片 + VS 能量球 + 标签 | ✅ 已实现 | 🟢 |
| L4 前景动效 | 碰撞火花粒子（40 个从中线向两侧爆裂） | ❌ 完全缺失 | 🔴 |

**具体问题**：

- 🔴 **P0 — `section--snap` + `height: 100vh` + `overflow: hidden` 截断内容** — S4 使用 `section--snap` 类，强制 `height: 100vh; overflow: hidden`。在移动端（flex-direction: column），两列 + VS + tags 内容极有可能超出 100vh。用户将看不到底部内容。这是导致"生硬感"的核心原因之一。
- 🔴 **P0 — 碰撞火花粒子缺失** — 设计规范明确要求 "碰撞火花 40 个，小圆 + 拖尾，从中线向两侧爆裂"。完全未实现。这是 S4 "双色场碰撞"视觉冲击力的关键元素。
- 🟡 **P1 — VS 文字辉光使用了错误的颜色** — VS 文字使用 `var(--color-s4-clash)` + 三层 `text-shadow` 引用 `oklch(0.80 0.20 280 / ...)` 匹配了设计规范中的 `--glow-text-burst` 参数，但设计要求 VS 使用 `--glow-text-burst` Token 而非手写 shadow 值。当前写法硬编码了等效值，不是严格的 Token 引用。
- 🟡 **P1 — 中线 clash-line 桌面才显示** — `display: none` 在非桌面端，移动端没有替代方案。设计缺少移动端碰撞视觉的降级方案。
- 🟡 **P2 — 扫描线揭示效果缺失** — 设计要求 `clip-path: inset(100% 0 0 0)` 驱动逐行揭示 + 跟随扫描线。GSAP 中只实现了列的 slide-in（translateX），没有 clip-path 揭示效果。
- 🟢 **通过** — 红蓝双色场渐变正确使用了 `var(--color-s4-red-glow)` 和 `var(--color-s4-blue-glow)`。
- 🟢 **通过** — VS 环形脉冲动画 `vsRingPulse` 与设计规范一致。

### S5 架构（ArchSection）— ⭐⭐☆☆☆ 2/5

**四层深度合成检查**：

| 层 | 设计要求 | 实际状态 | 判定 |
|----|---------|---------|------|
| L1 纹理层 | 蓝图网格底纹（粗线80px + 细线20px） | ✅ CSS background 已实现，初始 opacity:0 | 🟢 |
| L2 背景特效 | X光扫描线 + 蓝图线条 | ⚠️ X光线存在但效果弱（无明确的"扫描后才显现"状态变化） | 🟡 |
| L3 内容层 | 三面板（权力/记忆/进化） | ✅ 已实现 | 🟢 |
| L4 前景动效 | 层级光纤连接动画 | ❌ 完全缺失 | 🔴 |

**具体问题**：

- 🔴 **P0 — 内容超出 100vh 严重** — S5 有三个面板，每个 `min-height: 80vh`（移动端 70vh），加上 intro 和间距，总高度 ≈ 280vh+。但 `section--snap` 强制 `height: 100vh; overflow: hidden`，绝大部分内容不可见。这是最严重的 UI 问题。用户只能看到 intro 部分。
- 🔴 **P0 — 光纤连接完全缺失** — 设计规范明确要求"架构层级之间的连接线，模拟光纤数据传输"（SVG stroke-dasharray + 流光动画）。三面板之间没有任何视觉连接。
- 🟡 **P1 — X光扫描揭示效果不完整** — 设计要求：初始面板只有轮廓（蓝图线条，transparent 背景），扫描线经过后才 "填充内容"（background → surface，border → accent）。实际实现：GSAP 只做了 fadeUp + borderColor 过渡，缺少扫描线同步经过的视觉效果。xrayLine 的 top 是从 0% 到 100%，但没有与面板的 "被扫描后变色" 联动。
- 🟡 **P1 — 面板间距过大** — 三面板各 min-height: 80vh + margin-bottom: space-16(64px)。在桌面端每个面板高度 80vh，看起来内容在巨大的空间里"漂浮"。建议去掉 min-height，改为 padding 撑高。
- 🟡 **P2 — 进化面板（Panel C）的卡片布局** — 四张卡片在桌面端使用 `grid-template-columns: repeat(2, 1fr)`，但每张卡片内容不多（一行标题 + 一行描述），密度偏低。可以考虑三列或减少 padding。
- 🟢 **通过** — 蓝图网格 CSS 与设计规范 background-size 一致（80px 粗线 + 20px 细线）。
- 🟢 **通过** — 所有颜色使用 `var(--color-s5-*)` Token。

### S6 愿景（VisionSection）— ⭐⭐⭐⭐☆ 4/5

**四层深度合成检查**：

| 层 | 设计要求 | 实际状态 | 判定 |
|----|---------|---------|------|
| L1 纹理层 | 全局层 | 🟡 无额外纹理 | 🟡 |
| L2 背景特效 | 光晕脉冲（呼吸感） | ✅ `auraBreathe` 动画 + 径向渐变，匹配设计 | 🟢 |
| L3 内容层 | 文字墨水扩散凝聚 + 高潮句爆发 | ✅ GSAP scrub 精确控制 + flash + shockwave | 🟢 |
| L4 前景动效 | 光晕粒子（50 个柔和模糊圆，多彩渐变） | ❌ 缺失 | 🟡 |

**具体问题**：

- 🟡 **P1 — 高潮句 `climaxGlow` 使用了错误的辉光颜色** — `climaxGlow` 关键帧使用了 `oklch(0.78 0.15 195 / ...)` 即 `--color-cyan` 的色相，但设计规范要求高潮句使用 `--color-s6-burst` (oklch(0.90 0.10 85)) 即白金色。当前效果是青色辉光而非金色爆发，与色彩叙事弧线不符——S6 的高潮应该是"白金色爆发"。
- 🟡 **P1 — 文字墨水扩散效果在 GSAP 中简化了** — 设计要求 `scale(1.05)` → `scale(1)` 的凝聚效果，GSAP 中用了 `scale: 1.05` 但偏偏为各段文字用了不同参数，高潮句用了 `scale: 0.8` → `scale: 1`（放大入场），非高潮句用了 `scale: 1.05` → `scale: 1`（缩小凝聚），方向不一致。
- 🟡 **P2 — 光晕粒子缺失** — 设计要求"50 个柔和模糊圆，多彩渐变，缓慢扩散 + 脉冲"。未实现。S6 只靠 CSS 光晕呼吸（bg-aura），缺少 Canvas 粒子增加层次感。
- 🟡 **P2 — `section--snap` 不适用于 S6** — S6 的 `min-height: 120vh` 与 `section--snap`（强制 100vh）冲突。虽然 GSAP pin 可能覆盖，但 CSS 级别的 `height: 100vh; overflow: hidden` 仍是隐患。
- 🟢 **通过** — GSAP ScrollTrigger 的 scrub 叙事节奏感很好（250% 的滚动距离给文字逐行展示足够空间）。
- 🟢 **通过** — flash overlay 白闪 + 冲击波 shockwave 在高潮处实现了情感冲击力。
- 🟢 **通过** — 移动端降级（IO fallback）实现完整。

### S7 CTA（CtaSection）— ⭐⭐☆☆☆ 2/5

**四层深度合成检查**：

| 层 | 设计要求 | 实际状态 | 判定 |
|----|---------|---------|------|
| L1 纹理层 | 全局层 | 🟡 无额外纹理 | 🟡 |
| L2 背景特效 | 能量场收束（径向渐变 + converge 动画） | ✅ `fieldConverge` 已实现 | 🟢 |
| L3 内容层 | 品牌名发光 + 统计数字 + 邮件/源码按钮 | ⚠️ 有但简陋（见下） | 🟡 |
| L4 前景动效 | 收束粒子 80 个（Canvas 螺旋从边缘向中心） | ❌ 完全缺失 | 🔴 |

**具体问题**：

- 🔴 **P0 — 收束粒子完全缺失** — 设计规范明确要求 "80 个粒子从容器边缘生成，向中心汇聚，螺旋路径，到达中心后重新从边缘生成"。这是 S7 "能量收束" 的核心视觉，当前完全缺失。
- 🔴 **P0 — CTA 缺少 GSAP ScrollTrigger** — S7 没有 GSAP 触发动画（section-triggers.ts 中没有 `initS7ScrollTrigger`）。所有元素用的是 IO + CSS transition，与其他 Section 的 GSAP scrub 体验不一致。在其他 Section 都是精确的 scrub 叙事后，突然变成简单的 IO fade-up，体验断层明显。
- 🟡 **P1 — 品牌名发光效果弱化** — 设计要求使用 `--glow-text-strong` Token 并有 `brightness(1.15)` 增强。实际 `brandGlowEnhance` 使用了 filter drop-shadow（而非 text-shadow），且参数与设计规范的 `--glow-text-medium/strong` 不对应。品牌名应该是整个 CTA 的视觉焦点，当前发光太弱。
- 🟡 **P1 — 按钮浮动效果过于微弱** — `buttonFloat` 只有 `translateY(-4px)`，设计规范虽然也是 -4px，但配合 `box-shadow` 变化（`shadow-glow-sm → shadow-glow-md`）。实际只有 translateY，没有 shadow 联动，浮动感不足。
- 🟡 **P2 — 内容密度低** — S7 内容就品牌名 + 两行统计 + 一句回调 + 两个按钮。整个区域 min-height: 80vh，空洞感明显。建议增加团队亮点数据或视觉装饰元素。
- 🟡 **P2 — footer 与 CTA 断层** — footer 使用 `border-top: 1px solid oklch(1 0 0 / 0.05)` 分隔，太突兀。CTA 的能量场应该自然过渡到 footer。
- 🟢 **通过** — 按钮底部光圈 `::after` 效果正确。
- 🟢 **通过** — 邮件和源码链接的 hover/active/focus-visible 状态完整。

## 结论

**核心矛盾**：设计规范 (design-v6-visual.md) 定义了一个极其丰富的四层视觉体系（45+ Design Token，6 种区块专属粒子，交互粒子，光效系统），但实际实现只完成了约 47%。主要缺失的是 Layer 1（纹理层几乎靠全局 body::before/after 撑着）和 Layer 4（前景动效层完全空白），以及大量的 Layer 2 区块专属粒子。

**老板说的"生硬、不丝滑"**本质是：
1. 缺少粒子系统 → 没有"活"的感觉
2. Section 间硬切 → 没有过渡流动感
3. overflow: hidden 截断内容 → 信息展示不完整
4. 背景特效太微弱 → 页面"平"，缺乏深度

**建议执行顺序**：先修 P0-1（overflow 截断，这是功能性 bug），再实现 P0-2（全局交互粒子，提升整体活力），然后按优先级逐项推进。
