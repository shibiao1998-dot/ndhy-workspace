# OpenClaw AI Org 产品官网 — 深度设计方案（V2）

## 0. 先回答最根本的问题

### 上一版方案为什么不够好？

三个致命问题：

**第一，套模板。** 多页面、SaaS 三栏定价、独立开发者/创业者/企业三套解决方案——这是 2020 年的 SaaS 官网公式。把 "OpenClaw" 换成任何一个 AI 产品，这个方案都能用。**没有灵魂。**

**第二，贪多。** 8+ 个页面、博客、文档中心、定价、多语言……一个还在"证明自己"阶段的产品，不需要一个 Stripe 级别的完整商业网站。你需要的是 **一把尖刀**，而不是一个百货商场。

**第三，视角错误。** 上一版的核心叙事是"我们有 26 个专家、45 个技能、完整流水线"——这依然是 **卖方视角**。只不过从"看看我们多牛"变成了"看看我们能帮你做什么"。但真正打动人的网站，不是在"卖"，而是在 **创造一种认知震撼**。

------

## 1. 这个网站的真正使命

先搞清楚一件事：**OpenClaw AI Org 在 2026 年 3 月的竞争态势中到底是什么？**

看看竞品：

- **Devin**：40 亿美金估值，Goldman Sachs 12000 人试点，定位是"AI 软件工程师"——一个超级个体
- **MetaGPT/Atoms**：蚂蚁集团投资，月访问 120 万，模拟软件公司但走的是自动化路线
- **CrewAI**：450M+ 月运行量，60% Fortune 500，但本质是个框架/平台
- **Cursor**：Jensen Huang 说 4 万 NVIDIA 工程师在用，但它是 IDE 增强工具
- **Manus**：被 Meta 收购，走通用 AI Agent 路线

**OpenClaw AI Org 跟它们全都不一样。** 那些都是 **工具或框架**——你用它来帮你干活。而 OpenClaw AI Org 是一个 **真正在运转的 AI 组织**——它不是工具，它是一个有 27 个角色、有灵魂文件、有记忆、有纪律、有纠正学习机制的 **活的组织体**。

人类不是在"使用"它，而是在 **和它一起工作**。人类是老板，发发是 Leader，27 个专家各司其职。这不是技术 Demo，这是一种 **全新的人机协作范式**。

**所以这个网站的使命不是"卖产品"，而是：**

> **让来访者在 60 秒内意识到：AI 组织化，是一场正在发生的范式转移。而 OpenClaw 是第一个真正做到的。**

------

## 2. 核心叙事策略：不卖，证明

研究了 Stripe、Vercel、Linear、Devin、Cursor、Anthropic 这些世界级网站后，我发现一个规律：

| 级别     | 策略                       | 例子                                                         |
| -------- | -------------------------- | ------------------------------------------------------------ |
| **C 级** | 罗列功能特性               | 大部分 SaaS 官网                                             |
| **B 级** | 用户痛点 → 我的方案        | 我上一版方案                                                 |
| **A 级** | 让产品自己说话             | Cursor（首屏就是 IDE 动态 Demo）                             |
| **S 级** | 创造认知震撼，重新定义品类 | Stripe（"金融基础设施"重新定义支付）、Anthropic（"安全前沿"重新定义 AI） |

OpenClaw 应该打 **S 级** 策略：

**不解释"我们是什么"，而是让访客自己得出结论。**

怎么做？

**这个网站本身就是最强证据。** 当前 v2 版已经在这么做了（"0 行人类代码"），但做得太含蓄了——只在底部提了一句。应该把这个概念贯穿全站，变成一种 **沉浸式体验**。

## 3. 信息架构：单页叙事，但不是长滚动罗列

### 放弃多页面，回归单页——但做法完全不同

**理由：**

1. 多页面适合成熟商业产品（有明确的产品线、定价、文档）。OpenClaw 当前需要的是 **一个有力的声明**，不是百货商场
2. 世界级的品牌声明网站几乎都是单页：Anthropic 首页、Cognition.ai、Apple 产品发布页
3. 单页的叙事控制力远强于多页——你控制用户的阅读节奏和情绪曲线

### 但跟 v2 的单页完全不同

v2 的问题是"平铺罗列"：Hero → 26 专家卡片 → 5 大创新 → 对比表格 → 证据。信息密度高但 **没有叙事节奏**——像一份技术文档，不像一个故事。

新架构是 **电影式叙事**——有起承转合、有节奏张弛、有情绪曲线：



```
情绪曲线设计：

高 ┃        ②              ⑤          ⑦
   ┃      ╱  ╲          ╱  ╲       ╱
   ┃    ╱      ╲      ╱      ╲   ╱
   ┃  ╱    ③    ╲  ╱    ⑥    ╲╱
   ┃╱    沉浸     ╲╱    沉思
低 ┃①                ④
   ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   悬念  震撼  理解  对比  高潮  共鸣  行动

① 悬念开场（不急着亮牌）
② 核心震撼（亮出组织全貌）
③ 沉浸理解（怎么运转的）
④ 认知落差（对比传统方式）
⑤ 证据高潮（真实交付案例）
⑥ 深度共鸣（为什么这很重要）
⑦ 行动召唤（加入这场变革）
```

## 4. 七个叙事区块详细设计

### 区块一：悬念开场（The Hook）

**设计意图**：不直接说"我们是 AI 组织"。先制造悬念，让访客主动想知道答案。

**视觉**：纯黑背景，只有文字。极致简洁。参考 Cognition.ai 的首页——几乎什么都没有，但你记住了。



```
┌─────────────────────────────────────────────────────────┐
│                                                          │
│                        （黑屏）                           │
│                                                          │
│                                                          │
│     2026 年 3 月 10 日                                    │
│     一个人启动了一个项目。                                  │
│                                                          │
│     6 天后，                                              │
│     他拥有了一个 27 人的产品团队。                          │
│                                                          │
│     他没有招过一个人。                                     │
│                                                          │
│                                                          │
│                    ↓                                      │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**交互**：

- 页面加载后 0.5 秒，"2026 年 3 月 10 日"淡入
- 1.5 秒后，"一个人启动了一个项目。"逐字打出
- 3 秒后，"6 天后，"淡入，0.5 秒后 "他拥有了一个 27 人的产品团队。"淡入
- 4.5 秒后，"他没有招过一个人。"淡入——这句用稍亮的白色，制造重点
- 下滚箭头轻微呼吸动画

**为什么这么设计**：

- 不说"AI"、不说"Agent"、不说"组织"——故意留白，制造好奇心
- 真实的时间线（确实是 03-10 启动、03-15 完成 27 专家阵列）——不是虚构，是事实
- "没有招过一个人"——直觉上不可能，但事实如此。这个认知冲突是最强的钩子

**交互**：

**核心创意**：不用传统的左右对比表格（太老套），用 **两个平行宇宙** 的概念。

### 区块五：深层架构（The Architecture）

**设计意图**：给真正感兴趣的深度读者看"为什么这个组织能运转"。这是 v2 的"五大创新"的升级版，但做减法——不需要五个都讲，挑最震撼的三个。

**核心创意**：三张"剖面图"——像 CT 扫描一样切开这个 AI 组织，让你看到它的内部结构。

**剖面 1：权力结构**（不是所有 AI 都平等）



```
L0 ━━ 老板 ━━━━━━━━━━━━━━━━ 做不做
L1 ━━ Leader ━━━━━━━━━━━━━━ 做成什么样
L2 ━━ 项目管理专家 ━━━━━━━━━ 怎么拆、谁先做
L3 ━━ 各领域专家 ━━━━━━━━━━ 自己领域内怎么做最好

不是 AI 替代人类决策。
是人类授权 AI 在专业范围内自主决策。
```

**剖面 2：记忆体系**（AI 组织为什么越用越强）



```
🔥 HOT  ━━ 价值观、经验法则 ━━ 永远记住
🟡 WARM ━━ 今天发生了什么 ━━ 每日归档
📁 PROJ ━━ 这个项目的上下文 ━━ 项目生命周期
🆘 应急 ━━ 上下文溢出备份 ━━ 按需调用

每一次纠正 → 最高优先级信号 → 立即记录 → 永不重犯
同一模式出现 3 次 → 自动提炼为规则
```

**剖面 3：数字人格**（为什么不是"一个模型装 27 个帽子"）



```
每个 AI 专家都有：
├── SOUL.md    ━━ 我是谁、我相信什么、我的思维方式
├── STANDARDS.md ━━ 我的专业标准、产出规范、质量要求
└── Skills/    ━━ 我会的专业技能包，按需加载

代码审查专家 ≠ 用 ChatGPT 审查代码
代码审查专家 = 一个有质量信仰、有审查标准、有审查流程、
               看过所有前人纠正记录的专职质量守门人。
```

## 5. 视觉设计哲学

### 不要"科技感"，要"克制感"

v2 的问题：粒子动画 + 渐变色 + 发光文字 + 卡片网格——这是 2024 年"AI 官网模板"的标准配置。你在任何一个 AI 创业公司的官网上都能看到相同的视觉语言。

**新方向：对标 Cognition.ai + Anthropic + Linear 的设计气质——极致克制，信息密度高，每个视觉元素都有意义。**

| 维度 | v2（丢弃）                   | v3（采用）                                     |
| ---- | ---------------------------- | ---------------------------------------------- |
| 色彩 | 渐变 + 多彩 + 发光           | 接近纯黑白 + 单一强调色（靛蓝 #818cf8）        |
| 动效 | 粒子网络 + 计数器 + 交错淡入 | 极致克制的文字淡入 + 节点亮起                  |
| 排版 | 信息密度低，大间距           | 信息密度高，紧凑排版，参考 Linear              |
| 图形 | 装饰性粒子                   | 功能性图形（组织拓扑图、流水线图、架构剖面图） |
| 字体 | 系统字体                     | 系统字体但更讲究字号/字重/行高的层次           |

### 视觉关键词

- **Stark（鲜明）**：黑底白字为主，信息像刀刻出来的
- **Structural（结构化）**：每个图形都在传达组织结构，不是装饰
- **Confident（自信）**：不用感叹号，不用"最好"/"颠覆"这类词，让事实自己说话
- **Spare（节制）**：如果一个元素不帮助叙事，删掉它

## 6. 技术方案

### 为什么这次依然选纯 HTML/CSS/JS

**理由逆转上一版的 Next.js 建议**：

1. **"0 行人类代码"的 meta-statement 要成立**——如果引入 Next.js 构建工具链，这个声明的纯粹性就被稀释了。纯 HTML/CSS/JS 的极致简洁本身就是一种声明
2. **单页不需要框架**——没有路由、没有状态管理、没有数据获取。框架只会增加复杂度
3. **性能天花板**——零构建、零依赖、一个文件加载。任何框架都达不到这个极致
4. **这是 AI 组织做的**——越朴素的技术栈，越能证明"不是靠工具，是靠组织"

### 但有升级

| 维度       | v2                | v3                                                           |
| ---------- | ----------------- | ------------------------------------------------------------ |
| 动画       | 手写 Canvas + CSS | 手写 Canvas（大幅简化） + CSS `@keyframes` + `IntersectionObserver` |
| 组织拓扑图 | 无                | SVG + CSS 动画实现可交互节点网络                             |
| 时间线     | 无                | 纯 CSS Grid + 伪元素                                         |
| 代码展示   | 无                | `<pre><code>` + 语法高亮（纯 CSS）                           |
| 性能       | 好                | 追求 Lighthouse 100/100                                      |
