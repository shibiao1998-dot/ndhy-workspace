# 体验重设计方案：提示词工程系统

> 推翻现有前端，解决"像开发者调试工具"的致命问题。后端 API 不变。

---

## 一、整体布局：步骤引导流

从"平铺所有功能"改为**分步流程感**：

```
┌─ Header：品牌区（保留现有）─────────────────────────┐
├─ Step 1：描述任务 ─────────────────────────────────┤
│  首屏：预设场景卡片 + 输入框 + 核心数据               │
├─ Step 2：确认维度 ─────────────────────────────────┤
│  路由结果 + 类别手风琴面板                            │
├─ Step 3：选择引擎 ─────────────────────────────────┤
│  引擎选择 + 生成按钮                                 │
├─ Step 4：查看结果 ─────────────────────────────────┤
│  提示词输出 + 覆盖报告                               │
└──────────────────────────────────────────────────┘
```

**进度指示**：左侧或顶部显示步骤条（Steps 组件），当前步骤高亮，已完成步骤带 ✓。步骤条随用户操作自动推进，不需要手动点"下一步"——输入完成自动进入 Step 2，路由成功自动展开维度面板。

**视觉分隔**：每个 Step 是一个独立 Card，Card 间距 16px，步骤标题用 `H2 + 步骤编号`。

### 3.1 置信度文案化

取消百分比数字和红黄绿圆点，改为三级文案：

| 置信度范围 | 显示文案 | 样式 |

## 四、维度面板重设计（Step 2 主体）

### 4.1 按类别归组的手风琴

**核心变化**：从"按优先级三列"改为"按 A-L 类别手风琴"。

每个类别是一个可展开的折叠面板（Collapse），显示：

```
┌─────────────────────────────────────────────┐
│  📂 A · 战略与价值观         3/8 已选    ▼   │
└─────────────────────────────────────────────┘
```

- 左侧：类别字母 + 类别名称
- 右侧：`已选数/总数` + 展开箭头
- 路由命中的类别默认展开（含必须维度的类别），其他折叠
- 类别无任何维度被路由命中时，整个类别折叠且灰色显示

### 4.2 维度展示：Tag/Chip 形式

展开后，该类别下的维度用 **Tag 标签流**（flex wrap）排列，而非列表：

**已选维度（实心标签）**：
- 背景：品牌蓝浅色 `#E8F1FA`，文字：品牌蓝 `#1B65A9`，边框：`#A3C8E8`
- 点击可取消选择

**未选维度（描边标签）**：
- 背景：透明，文字：`#999`，边框：`#D9D9D9` 虚线
- 点击可选中

### 4.3 优先级视觉（微妙差异）

不用红黄绿圆点。用**标签左侧竖线粗细**区分：

| 优先级 | 左侧竖线 | 背景深浅 |
|--------|---------|---------|
| 必须（一级） | 3px 品牌蓝实线 | 标准 `#E8F1FA` |
| 建议（二级） | 2px 青色实线 | 稍浅 `#F0F7FF` |
| 可选（三级） | 无竖线 | 最浅/透明 |

取消必须维度时弹出确认 Popconfirm。

### 4.4 隐藏字符数

字符数不再显示在维度标签上。仅在 hover tooltip 中显示维度简介（summary 前100字）。

### 4.5 优先级调整

点击已选标签上的小箭头图标 ↕ 可循环切换优先级（视觉上看到竖线粗细和背景色变化）。
