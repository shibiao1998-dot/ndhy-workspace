## 一、信息架构

### 1.1 页面结构

单页应用（SPA），垂直三区布局，从上到下：**输入区 → 配置区 → 输出区**。

页面无路由跳转，无弹窗流程，所有操作在同一个滚动页面内完成。

```
┌──────────────────────────────────────────────────────────┐
│  页头（固定）                                              │
│  产品名称 + 一句话说明                                      │
├──────────────────────────────────────────────────────────┤
│  【A 区：输入区】                                          │
│  任务描述输入 → 匹配结果 → 引擎选择 → 预估长度              │
├──────────────────────────────────────────────────────────┤
│  【B 区：维度配置区】                                       │
│  三级分组维度列表 → 勾选/优先级调整                          │
│  生成按钮                                                  │
├──────────────────────────────────────────────────────────┤
│  【C 区：输出区】                                          │
│  提示词结果 → 复制按钮 → 覆盖报告                           │
└──────────────────────────────────────────────────────────┘
```

### 1.2 信息层级

| 层级 | 内容 | 视觉权重 |
|------|------|----------|
| L1 最高 | 任务输入框、生成按钮、提示词结果 | 最大尺寸、最强对比 |
| L2 高 | 匹配结果、引擎选择、复制按钮 | 中等尺寸、突出色 |
| L3 中 | 维度列表（三列分组） | 标准尺寸、可折叠 |
| L4 低 | 预估长度、覆盖报告、页头说明 | 小尺寸、辅助色 |

### 1.3 导航策略

无导航菜单。单页内容按操作顺序从上到下排列，用户自然滚动即完成全流程。生成后自动滚动到输出区。

## 二、核心任务流程

### 2.1 主路径：从打开页面到拿到提示词

| 步骤 | 用户行为 | 系统响应 | 交互方式 | 预期耗时 |
|------|----------|----------|----------|----------|
| 1 | 打开页面 | 加载维度索引和引擎列表 | 页面自动加载 | 1-2s |
| 2 | 在输入框输入任务描述 | — | 键盘输入，textarea | 10-30s |
| 3 | 点击输入框外或按 Enter | 发起路由请求，显示匹配结果 | 失焦/按键触发 | 0.5-1s |
| 4 | 查看匹配结果，确认或切换类型 | 若切换：重新路由，更新维度列表 | 下拉选择（可选） | 0-5s |
| 5 | 查看维度列表，可选调整 | 维度列表按三级分组显示 | checkbox + 级别切换（可选） | 0-30s |
| 6 | 确认引擎（默认 Claude） | 引擎已预选，可切换 | 下拉选择（可选） | 0-3s |
| 7 | 点击"生成提示词" | 发起生成请求，显示 loading | 按钮点击 | 1-3s |
| 8 | 查看生成结果 | 页面滚动到输出区，展示提示词 | 自动滚动 | — |
| 9 | 点击"复制" | 复制到剪贴板，按钮变"✓ 已复制" | 按钮点击 | 即时 |

**最短路径（3步）**：输入任务 → 点击生成 → 复制。不调整任何配置时，默认值全部可用。

### 2.2 分支路径

#### 分支 A：手动切换任务类型（对应场景 4）
- 触发：用户认为自动匹配不准确
- 步骤：点击任务类型下拉 → 选择正确类型 → 维度列表自动刷新
- 约束：切换类型不清空任务描述输入框

#### 分支 B：手动调整维度（对应场景 4、5）
- 触发：用户想增删维度或调整优先级
- 步骤：在维度列表中勾选/取消维度 或 切换优先级标签
- 约束：取消必须维度时，checkbox 旁弹出行内警告文案"⚠️ 此为该任务的必须维度"（不阻止操作）

#### 分支 C：切换引擎（对应场景 2）
- 触发：用户需要不同引擎的提示词
- 步骤：切换引擎下拉 → 预估长度指示器实时更新 → 点击重新生成
- 约束：切换引擎不清空已有维度配置；若已有生成结果，切换引擎后输出区显示"引擎已切换，请重新生成"提示

#### 分支 D：维度预览
- 触发：用户想了解某维度的具体内容
- 步骤：点击维度名称 → 该维度行下方展开摘要区域（前 200 字）→ 再次点击收起
- 约束：同一时间只展开一个维度摘要（手风琴模式）

### 2.3 失败路径与恢复

| 失败场景 | 系统响应 | 恢复方式 |
|----------|----------|----------|
| 路由请求失败（网络/服务错误） | 输入区下方红色提示"匹配失败，请检查网络后重试" | 用户可重新触发（再次失焦输入框或手动选类型） |
| 生成请求失败 | 输出区显示错误卡片"生成失败：[原因]"，生成按钮恢复可点击 | 用户点击重新生成 |
| 复制失败（浏览器不支持 Clipboard API） | 自动选中全部文本，toast 提示"请手动 Ctrl+C 复制" | 用户手动复制 |
| 维度索引加载失败 | 页面显示全屏错误"系统初始化失败，请刷新页面" + 刷新按钮 | 用户刷新 |

## 三、页面布局方案

### 3.1 页头

| 元素 | 控件 | 说明 |
|------|------|------|
| 产品名称 | 文本 `<h1>` | "提示词工程系统" |
| 副标题 | 文本 `<p>` | "输入任务场景，自动生成适配目标引擎的高质量提示词" |

固定高度 64px，不吸顶（60分版本不需要）。

### 3.2 A 区：输入区

整体为一个卡片容器，内含以下元素：

#### 3.2.1 任务描述输入

| 属性 | 值 |
|------|-----|
| 控件 | `<textarea>` |
| 尺寸 | 宽度 100%，最小高度 80px，最大高度 200px，自动撑高 |
| placeholder | "描述你的任务，例如：设计一个初中化学虚拟实验的交互方案" |
| 触发路由 | 失焦（blur）时触发路由请求 |
| 验证 | 无字数限制；空内容时生成按钮禁用（disabled） |

#### 3.2.2 匹配结果行

在输入框下方，单行水平排列：

```
[匹配结果图标] 匹配类型：原型设计  置信度 85%    [切换类型 ▼]
```

| 元素 | 控件 | 说明 |
|------|------|------|
| 匹配类型 | 文本标签 | 显示自动匹配的任务类型名称 |
| 置信度 | 带色文本 | ≥70% 绿色，40%-69% 黄色，<40% 红色 |
| 切换类型 | `<select>` 下拉 | 8 个任务类型选项；选中后置信度显示 100% 并标注"手动选择" |

**初始状态**（未输入时）：匹配结果行隐藏，仅显示输入框。

#### 3.2.3 引擎与预估行

匹配结果下方，单行水平排列：

```
目标引擎：[Claude ▼]     预估长度：45,000 / 180,000 字符  [████████░░░░] 25%
```

| 元素 | 控件 | 说明 |
|------|------|------|
| 引擎选择 | `<select>` 下拉 | 选项：Claude / GPT-4 / DeepSeek / Midjourney / DALL-E / Suno；默认 Claude |
| 预估长度 | 文本 + 进度条 | 分子=已选维度总字符数，分母=引擎最大字符数 |
| 进度条颜色 | 视觉指示 | ≤60% 绿，61-85% 黄，>85% 红 |

**引擎切换行为**：切换引擎后，预估长度分母更新，进度条颜色重算。若已有生成结果，输出区上方显示黄色 banner"引擎已切换为 [引擎名]，点击「重新生成」查看新结果"。

### 3.3 B 区：维度配置区

整体为一个卡片容器。

#### 3.3.1 维度列表

三列布局（桌面端），每列对应一个优先级级别：

```
┌──────────────────┬──────────────────┬──────────────────┐
│ 🔴 必须维度 (8)   │ 🟡 建议维度 (12)  │ 🟢 可选维度 (6)   │
├──────────────────┼──────────────────┼──────────────────┤
│ ☑ A01 战略定位    │ ☑ C01 竞品分析    │ ☐ L01 备选池管理  │
│   4,200 字        │   3,800 字        │   2,100 字        │
│ ☑ A02 红线约束    │ ☑ E01 设计方法    │ ☐ L02 排期规划    │
│   1,500 字        │   5,200 字        │   1,800 字        │
│ ...              │ ...              │ ...              │
├──────────────────┴──────────────────┴──────────────────┤
│                  [ 🚀 生成提示词 ]                       │
└────────────────────────────────────────────────────────┘
```

**单个维度行的结构**：

```
[☑] [优先级标签 🔴/🟡/🟢] A01 战略定位    4,200字    [▼ 展开]
```

| 元素 | 控件 | 行为 |
|------|------|------|
| checkbox | `<input type="checkbox">` | 勾选=包含该维度，取消=排除。默认：必须和建议维度勾选，可选维度不勾选 |
| 优先级标签 | 三态切换按钮 | 点击在 🔴→🟡→🟢→🔴 之间循环切换。切换后维度移动到对应列 |
| 维度名称 | 可点击文本 | 点击展开/收起维度摘要（前 200 字） |
| 字符数 | 辅助文本 | 灰色小字，标注该维度的内容字符数 |
| 展开按钮 | 图标按钮 | ▼ 展开 / ▲ 收起维度摘要 |

**维度摘要展开区**：
- 在该维度行下方插入一个浅灰背景区块
- 显示维度前 200 字 + "..."
- 手风琴模式：展开新维度时自动收起上一个

**待填充维度**（status = incomplete）：
- 维度名称后显示灰色标签"内容未完成"
- checkbox 可勾选但勾选后在维度名称旁显示 ⚠️ 图标

**取消必须维度**：
- 取消 checkbox 后，该行下方出现行内黄色警告："⚠️ 此为该任务类型的必须维度，取消可能影响提示词质量"
- 不阻止操作

#### 3.3.2 列头统计

每列标题旁显示数量统计，格式："🔴 必须维度 (8)"。维度增删或级别切换时实时更新数量。

#### 3.3.3 生成按钮

| 属性 | 值 |
|------|-----|
| 控件 | `<button>` |
| 位置 | 维度列表下方，居中 |
| 尺寸 | 宽度 240px，高度 48px |
| 文案 | "🚀 生成提示词" |
| 样式 | 主色（蓝色系）实心按钮，最大视觉权重 |
| 禁用条件 | 任务输入为空 **或** 无勾选维度 **或** 正在生成中 |
| 禁用样式 | 灰色，cursor: not-allowed |
| loading 状态 | 按钮文案变为"⏳ 生成中..."，显示旋转图标，按钮禁用 |

### 3.4 C 区：输出区

整体为一个卡片容器。初始状态隐藏（未生成时不显示）。

#### 3.4.1 提示词结果

| 属性 | 值 |
|------|-----|
| 控件 | `<div>` 只读文本区域（非 textarea，用于更好的排版） |
| 字体 | 等宽字体（`monospace`），14px |
| 背景 | 浅灰 `#f5f5f5` |
| 最大高度 | 600px，超出内部滚动（`overflow-y: auto`） |
| 内容格式 | 保留 Markdown 格式渲染（标题、列表、代码块、粗体） |
| 复制按钮 | 右上角固定定位（相对于结果容器），不随内容滚动 |

#### 3.4.2 复制按钮

| 属性 | 值 |
|------|-----|
| 控件 | `<button>` |
| 位置 | 结果区域右上角 |
| 默认文案 | "📋 复制全文" |
| 点击行为 | 调用 `navigator.clipboard.writeText()`，将**原始文本**（非渲染后 HTML）复制到剪贴板 |
| 成功反馈 | 按钮文案变为"✅ 已复制"，2 秒后恢复 |
| 失败反馈 | 选中结果区全部文本 + toast 提示"复制失败，请手动 Ctrl+C" |

#### 3.4.3 覆盖报告

结果下方，统计卡片：

```
┌────────────────────────────────────────────────────────┐
│  📊 维度覆盖报告                                        │
│                                                        │
│  已使用维度：23 / 89    覆盖率：87%（按权重）             │
│                                                        │
│  🔴 一级：8 个    🟡 二级：12 个    🟢 三级：3 个         │
│                                                        │
│  ⚠️ 缺失必须维度：无                                    │
│  （或：⚠️ 缺失必须维度：A03 品牌定位、B05 用户画像）      │
└────────────────────────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| 已使用维度 | 分子=生成时实际包含的维度数，分母=该任务类型的全部相关维度数 |
| 覆盖率 | 后端返回的按权重计算覆盖率 |
| 级别分布 | 一级/二级/三级各多少个 |
| 缺失必须维度 | 列出被用户取消勾选或因长度限制被截断的必须维度。无缺失时显示"无" |
| 缺失维度样式 | 红色文字 + ⚠️ 图标 |

## 四、交互模式

### 4.1 输入交互

#### 4.1.1 任务描述输入

| 交互 | 细节 |
|------|------|
| 聚焦 | 输入框边框高亮（蓝色），placeholder 文字保持可见直到开始输入 |
| 输入中 | 无实时反馈（不做输入时即时路由，避免频繁请求） |
| 失焦触发路由 | 输入框失焦时，若内容非空且与上次路由输入不同，发起 `POST /api/route` 请求 |
| Enter 触发 | 按 Shift+Enter 换行；按 Enter（无 Shift）触发路由并失焦 |
| 清空输入 | 输入框右侧显示 × 清除按钮（内容非空时可见）；清空后：匹配结果行隐藏、维度列表清空、输出区隐藏、生成按钮禁用 |

#### 4.1.2 路由触发时机防抖

- 失焦触发前检查：内容是否为空、内容是否与上次相同
- 均满足则不发请求
- 防抖延迟：300ms（避免切换 tab 再回来时重复触发）

### 4.2 选择交互

#### 4.2.1 任务类型切换

| 交互 | 细节 |
|------|------|
| 触发 | 点击类型下拉，展开 8 个选项 |
| 选中 | 选中后立即发起路由请求（带 `task_type` 参数），维度列表整体刷新 |
| 已有维度调整 | 切换类型后，用户之前对维度的手动调整（勾选、优先级）全部重置为新类型的默认配置 |
| 视觉反馈 | 置信度显示更新为"100% · 手动选择" |

#### 4.2.2 引擎切换

| 交互 | 细节 |
|------|------|
| 触发 | 点击引擎下拉，展开 6 个选项 |
| 选中 | 预估长度分母立即更新；进度条颜色重新计算 |
| 已有结果 | 若输出区已有生成结果，显示黄色 banner 提示需重新生成 |
| 不触发重新路由 | 引擎切换不影响维度选择，只影响组装适配 |

#### 4.2.3 维度 checkbox

| 交互 | 细节 |
|------|------|
| 勾选 | 维度纳入生成范围，预估长度分子增加 |
| 取消 | 维度移出生成范围，预估长度分子减少 |
| 取消必须维度 | 显示行内警告，不阻止 |
| 视觉 | 勾选=正常色，未勾选=文字灰色 + 删除线 |

#### 4.2.4 优先级切换

| 交互 | 细节 |
|------|------|
| 控件 | 维度行上的三态标签按钮 |
| 点击行为 | 🔴→🟡→🟢→🔴 循环切换 |
| 切换后 | 维度移动到新级别对应的列（带 200ms 滑动动画）；列头数量更新 |
| 不触发请求 | 优先级变更仅在前端状态管理，生成时一起发送 |

### 4.3 生成交互

| 阶段 | 行为 |
|------|------|
| 点击生成 | 按钮进入 loading 态；收集当前状态：task + engine + dimensions[] + priorities{} |
| 请求中 | 按钮显示"⏳ 生成中..."；输出区显示骨架屏（灰色占位区块） |
| 成功 | 输出区渲染提示词内容 + 覆盖报告；页面平滑滚动到输出区顶部；按钮恢复 |
| 失败 | 输出区显示错误卡片；按钮恢复可点击 |
| 重新生成 | 新结果替换旧结果（无历史对比）；覆盖报告同步更新 |

### 4.4 复制交互

| 阶段 | 行为 |
|------|------|
| 点击复制 | 调用 Clipboard API 复制原始文本（Markdown 源码，非渲染 HTML） |
| 成功 | 按钮文案变"✅ 已复制"，2s 后恢复"📋 复制全文" |
| 失败（API 不可用） | 选中结果区全文 + 显示 toast"请手动 Ctrl+C 复制"，3s 后 toast 消失 |

## 五、全态定义

### 5.1 页面级状态

| 状态 | 触发条件 | 界面表现 |
|------|----------|----------|
| **初始化 loading** | 页面首次加载，`GET /api/dimensions` + `GET /api/engines` 请求中 | 页面中央显示旋转加载图标 + "系统初始化中..." |
| **初始化成功（空态）** | 维度和引擎数据加载完毕，用户未输入 | 显示输入框（聚焦态）+ 引擎默认选中 Claude；匹配行隐藏、维度列表隐藏、输出区隐藏 |
| **初始化失败** | 维度或引擎接口请求失败 | 全屏错误："系统初始化失败，请检查网络连接后刷新页面" + 🔄 刷新按钮 |

### 5.2 输入区状态

| 状态 | 条件 | 界面表现 |
|------|------|----------|
| **空态** | 输入框无内容 | placeholder 可见，生成按钮禁用，匹配结果行隐藏 |
| **输入中** | 用户正在输入 | 输入框高亮，无其他变化 |
| **路由中（loading）** | 失焦触发路由请求 | 匹配结果区域显示内联 spinner + "正在匹配任务类型..." |
| **路由成功** | 路由返回结果 | 显示匹配类型+置信度，维度列表填充并展开 |
| **路由失败（error）** | 路由请求报错 | 匹配结果区域显示红色文字"匹配失败，请重试"；维度列表不变（若有旧数据保留） |

### 5.3 维度配置区状态

| 状态 | 条件 | 界面表现 |
|------|------|----------|
| **隐藏** | 未触发过路由 | 整个 B 区不可见 |
| **正常展示** | 路由成功后 | 三列维度列表完整显示，生成按钮可用 |
| **空维度** | 路由返回 0 个维度（理论上不会出现） | 维度区域显示"未找到相关维度，请尝试更换任务描述或手动选择任务类型" |
| **全部取消勾选** | 用户取消了所有维度 | 生成按钮禁用，按钮下方红色提示"请至少选择一个维度" |

### 5.4 输出区状态

| 状态 | 条件 | 界面表现 |
|------|------|----------|
| **隐藏** | 从未生成过 | 整个 C 区不可见 |
| **生成中（loading）** | `POST /api/generate` 请求中 | C 区可见，内部显示骨架屏（3 行灰色占位块，模拟文本布局） |
| **生成成功（success）** | 生成返回结果 | 提示词文本渲染 + 复制按钮 + 覆盖报告 |
| **生成失败（error）** | 生成请求报错 | 红色错误卡片："生成失败：[后端返回的错误信息]。请检查配置后重试。" |
| **引擎已切换（stale）** | 有旧结果但引擎已切换 | 旧结果仍显示，但结果区上方叠加黄色 banner："引擎已切换为 [新引擎]，点击「重新生成」查看新结果" |

### 5.5 状态转换图

```
页面加载 ──→ 初始化loading ──→ 初始化成功(空态)
                              │
                        初始化失败 ──→ 用户刷新 ──→ 页面加载
                              
初始化成功 ──→ 用户输入 ──→ 失焦触发 ──→ 路由loading ──→ 路由成功 ──→ 配置区可见
                                                      │
                                                路由失败 ──→ 用户重试
                                                      
配置区可见 ──→ 点击生成 ──→ 生成loading ──→ 生成成功 ──→ 输出区可见
                                           │
                                      生成失败 ──→ 用户重试

输出区可见 ──→ 切换引擎 ──→ stale状态 ──→ 重新生成 ──→ 生成loading
           ──→ 切换类型 ──→ 维度刷新 ──→ 可重新生成
           ──→ 修改输入 ──→ 重新路由 ──→ 维度刷新
```

## 九、前端消费数据需求清单
