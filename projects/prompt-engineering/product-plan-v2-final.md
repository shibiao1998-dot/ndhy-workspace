## 一、产品定位

**一句话**：为华渔 AI 设计师提供信息对称的提示词工程服务——用户描述设计任务，系统自动补全所有维度信息，调用 AI 直接产出设计方案。

**DJ 原话定义**：

> "提示词工程就是信息对等，只做信息对等……为AI提供最全面的信息，保证它能够把后续工作做好的信息。"

**DJ 两条公理**（系统灵魂，必须在 UI 中体现）：

| # | 公理 | DJ 原话 |
|---|------|---------|
| 1 | **信息对称** | "为AI提供最全面的信息，保证它能够把后续工作做好" |
| 2 | **相信 AI** | "结果不好都是我们的问题，都是信息不对称的问题" — 不是"当做"，是"相信" |

**不是什么**：

- 不是"提示词生成器"（用户手动勾选维度）
- 不是"设计方法论10步工具"（那是 design.101.com 的事）
- 不是中长期规划平台（DJ 要快速迭代）
- 不是"自动迭代系统"（DJ 明确说短期不可能）

## 二、用户与场景

| 用户 | 场景 | 核心需求 |
|------|------|----------|
| **设计师/策划** | 接到设计任务，需要 AI 辅助产出方案 | 输入任务 → 拿到高质量设计方案 |
| **DJ/高级设计师** | 审视提示词工程体系，验证信息全面性 | 看到"引擎盖"：用了哪些维度、覆盖率、组装的提示词、为什么选这些 |
| **高级人员** | 维护维度知识库，确保信息质量 | 审核/编辑维度描述，追溯数据来源 |

**主角关系**（DJ 核心纠正）：

| 旧模型 | 新模型（DJ 定义） |
|--------|-----------------|
| 设计师做设计 → AI 辅助 | **AI 做设计** → 设计师帮 AI 补充信息 |
| 设计师是使用者 | 设计师是 **AI 的信息供应商 + 质量审查员** |
| 提示词是"对 AI 的指令" | 提示词是"**给 AI 的全面信息包**" |

> DJ 原话："是由AI提供全面的信息，我们来查缺补漏……而不是反过来了。"

```
    ↓
```

#### 对话区（左侧）

- 类 ChatGPT 对话界面
- 支持多轮对话，AI 带着完整维度上下文
- AI 首轮会主动追问补充信息（动态追问，非固定模板）
- 每轮 AI 回复中如涉及约束禁忌，单独标记为"约束与禁忌"段
- 对话历史保留在当前会话中

#### 引擎盖面板（右侧，可折叠）

DJ 验收时的核心展示区域，包含 4 个模块：

**模块 3：提示词原文**
- 展示组装后发送给 AI 的完整 system prompt
- 分为两部分：
  - **正向信息段**：维度内容按优先级排列

- 引擎列表见下方"引擎配置"章节

#### AI 动态追问机制

系统通过 system prompt 指导 AI 进行智能追问：

```
追问策略：
1. AI 接收用户任务描述后，分析当前已知信息覆盖了哪些维度
2. 识别必须维度中缺失的关键信息（如用户画像、使用场景、技术约束等）
3. 生成 2-4 个针对性追问，优先覆盖必须维度的信息缺口
4. 用户回答后，AI 将答案整合到维度上下文中
5. 如果仍有关键缺口，可继续追问（但不超过 2 轮追问）
6. 追问完成后，直接产出设计方案
```

追问示例：
- "这个设计面向哪个年龄段的用户？他们的设备环境是什么？"（覆盖 B 类用户维度）
- "是否有竞品参考？你希望和他们有什么差异化？"（覆盖 C 类竞品维度）
- "DJ 或公司对这类项目有什么特别的价值观要求或红线？"（覆盖 A 类战略维度）

## 五、正向/反向信息分离

### 设计原理

DJ 会议中杜建强明确提出，DJ 肯定：

> 杜建强："还有一个是反向，有些时候你告诉它这个不能做"
> DJ："很好，这确实也是一个"

### 实现方式

#### 1. 维度层面标记

每个维度在管理页可标记信息方向：
- 🟢 **正向**：描述"要什么"（如 A01 战略愿景、B01 目标用户画像）
- 🔴 **反向**：描述"不要什么"（如 A02 红线与禁区）
- 🔵 **混合**：同时包含正向和反向内容

#### 2. 提示词组装时分离

```
组装后的提示词结构：

═══ 正向信息段 ═══
## 一级信息（必须）
### A01 战略愿景
[维度正向内容]
### B01 目标用户画像
[维度正向内容]
...

## 二级信息（建议）
### C01 竞品格局
[维度正向内容]
...

═══ 约束与禁忌段 ═══
### 红线与禁区
- ❌ 不做大模型研发相关内容
- ❌ 不涉及政治敏感话题
- ❌ 不使用未经授权的第三方IP
[从所有维度中提取的 ❌、"不要"、"禁止"、"红线" 标记内容]

### 排除条件
- 本次设计不面向幼儿园用户
- 不考虑 VR 头显场景
[用户在追问中明确排除的内容]
```

#### 3. AI 输出中的体现

AI 产出的设计方案中应包含独立的"约束与禁忌"章节：
```
## 4. 约束与禁忌
本方案遵循以下约束：
- ...
```

#### 4. 引擎盖中的展示

引擎盖面板中，提示词原文分色展示：
- 正向信息段：默认色
- 约束与禁忌段：红色背景高亮

### System Prompt 组装逻辑

```python
system_prompt = f"""
你是华渔教育的 AI 设计师。你的任务是基于以下完整的信息上下文，
```

### 数据库设计（SQLite）

```sql
-- 维度表
CREATE TABLE dimensions (
  id TEXT PRIMARY KEY,           -- A01, B01...
  name TEXT NOT NULL,            -- 维度名称
  category TEXT NOT NULL,        -- A/B/C...
  category_name TEXT,            -- 战略与价值观
  description TEXT,              -- 当前生效的维度描述
  pending_description TEXT,      -- 待审核的新描述
  data_source TEXT,              -- 数据来源（会议/文档/调研）
  update_frequency TEXT,         -- 更新频率
  direction TEXT DEFAULT 'positive', -- positive/negative/mixed
  priority INTEGER DEFAULT 2,    -- 默认优先级 1/2/3
  last_updated_at TIMESTAMP,
  review_status TEXT DEFAULT 'approved', -- approved/pending/rejected
  reviewer TEXT,
  reviewed_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 对话表
CREATE TABLE conversations (
  id TEXT PRIMARY KEY,
  title TEXT,                    -- 对话标题（从首条消息提取）
  task_type TEXT,                -- 匹配的任务类型
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 消息表
CREATE TABLE messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  conversation_id TEXT REFERENCES conversations(id),
  role TEXT NOT NULL,             -- user/assistant/system
  content TEXT NOT NULL,
  dimensions_used TEXT,           -- JSON: 本次用到的维度 ID 列表
  prompt_snapshot TEXT,           -- 本轮组装后的完整提示词快照
  coverage_stats TEXT,            -- JSON: 覆盖率统计
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 维度审核日志
CREATE TABLE dimension_reviews (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  dimension_id TEXT REFERENCES dimensions(id),
  old_description TEXT,
  new_description TEXT,
  action TEXT NOT NULL,           -- approve/reject/edit
  reviewer TEXT,
  comment TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
