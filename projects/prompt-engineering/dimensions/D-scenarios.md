## D01 需求来源类型库

> **质量作用**：每个需求必须有明确来源，来源不同，可信度和验证方法不同。

### 合法需求来源分类

| 来源类型 | 可信度 | 适用场景 | 验证方法 | 华渔实例 |
|----------|--------|----------|----------|----------|
| **用户调研（一手）** | ⭐⭐⭐⭐⭐ | 功能定义、场景设计、痛点识别 | 原声可溯源（访谈编号+原话） | 101 PPT 将教师分为"依赖资源型/修改整合型/自主设计型"三类，基于真实教师使用行为观察 |
| **用户行为数据** | ⭐⭐⭐⭐⭐ | 功能优先级、流失分析、使用路径优化 | 数据源+样本量+时间范围可追溯 | 应用市场差评分析：200条差评中35%抱怨"题目重复"→ 推导AI推荐需求 |
| **战略目标下放** | ⭐⭐⭐⭐ | 方向性需求、新品类探索 | 可追溯至DJ原话或正式战略文件 | DJ提出"3E教育"（Effective, Efficient, Engaging）→ 产品必须体现三E |
| **竞品参考** | ⭐⭐⭐ | 功能补齐、体验对标、差异化发现 | 需同时附用户验证（用户是否真正需要） | 竞品科大讯飞AI语音技术 → 101 PPT增加智能语音识别（但基于用户备课场景真实需求，非盲目跟随） |
| **专家/顾问建议** | ⭐⭐⭐ | 专业领域判断（教育理论、技术可行性） | 需标注专家背景+是否有实证支持 | 北师大智慧学习研究院建构主义教学理论 → 影响3D虚拟实验室"探究式学习"设计 |
| **客服/运营反馈** | ⭐⭐⭐ | 已上线功能的改进、Bug修复 | 需统计频率+影响范围，单条反馈可信度低 | 客服记录高频问题："3D课件加载慢"→ 技术优化需求 |
| **DJ/高管直接指令** | ⭐⭐⭐⭐ | 战略级判断、价值观指引 | 原话可追溯，但需区分"战略指引"和"具体方案"（后者需验证） | DJ："万物皆可设计" → 设计方法论产品化需求；DJ："不做大模型研发" → 战略红线 |
| **行业趋势/政策** | ⭐⭐⭐ | 合规需求、长期布局 | 需标注政策文件编号+生效时间 | 教育部《生成式AI服务管理暂行办法》→ AI课件必须有内容审查机制 |
| **技术能力驱动** | ⭐⭐ | 技术探索、能力展示 | 最低优先级，需反向验证用户是否需要 | "我们有3D引擎能力" ≠ "用户需要3D功能"，需回到场景验证 |

### 来源质量规则
1. **单一来源不决策**：任何需求至少需要2个不同类型来源交叉验证
2. **数据优先于观点**：行为数据 > 用户原话 > 专家推断 > 团队感觉
3. **来源必须标注**：进入需求池的需求必须标注来源类型+具体来源+可信度评级
4. **DJ指令特殊处理**：DJ的战略判断直接采信，但具体方案仍需场景验证

## D02 伪需求识别库

> **质量作用**：自动识别伪需求特征，避免资源浪费。**DJ特别关注此维度。**
> DJ原话："先搞清楚提示词工程到底是什么，再谈怎么做。"——这个态度本身就是反伪需求的。

### 伪需求模式清单

#### 模式 1：领导拍脑袋型
| 维度 | 内容 |
|------|------|
| **识别特征** | 需求来源只有"XX说的"，无用户证据、无数据支持、无场景描述 |
| **典型话术** | "DJ说过要做XX"（但实际是截取了某次会议中的探索性讨论，非正式决策） |
| **华渔案例** | ⚠️ DJ在会议中探索性提到"思维链可以作为提示词的一部分"——这是认可的观点，不是"必须做思维链产品"的需求。需区分DJ的"认同"和"指令" |
| **验证方法** | 追溯原始会议记录全文，确认是探索讨论还是明确决策；如果是决策，找到决策场景和约束条件 |
| **防御规则** | 所有引用DJ观点的需求，必须附原话+上下文，不接受二手转述 |

#### 模式 2：竞品驱动型（"别人有我们也要"）
| 维度 | 内容 |
|------|------|
| **识别特征** | 需求描述直接引用竞品功能，无独立的用户场景分析 |
| **典型话术** | "科大讯飞有AI批改作文，我们也要做"、"希沃的互动白板有XX功能" |
| **华渔案例** | 竞品Google Classroom有AI学习报告 → 如果直接做AI学习报告而不分析华渔用户（教师备课场景）是否需要，就是伪需求 |
| **验证方法** | 问三个问题：①我们的用户也有这个痛点吗？（用户证据）②我们做这个有独特优势吗？（资源匹配）③这个对齐我们的战略吗？（战略对齐） |
| **防御规则** | 竞品功能只能作为"灵感来源"，不能直接转化为需求。必须走需求验证全流程 |

#### 模式 3：设计师/策划自嗨型
| 维度 | 内容 |
|------|------|
| **识别特征** | 需求描述充满技术细节或交互花样，但说不出用户在什么场景下为什么需要 |
| **典型话术** | "加一个酷炫的3D动画""做一个AI生成的动态知识图谱" |
| **华渔案例** | "在化学实验中加入元宇宙社交功能"——听起来很前沿，但中学生做化学实验的核心场景是"理解实验原理、观察实验现象"，社交功能是干扰，不是需求 |
| **验证方法** | 场景审判：能否用一句话说清"谁、在什么场景、用这个功能、解决什么问题"？说不出=砍掉 |
| **防御规则** | 功能必须从场景推导，不能从技术能力推导 |

#### 模式 4：表面需求型（用户说的 ≠ 用户要的）
| 维度 | 内容 |
|------|------|
| **识别特征** | 直接采信用户的解决方案描述，未深挖底层问题 |
| **典型话术** | "老师说想要一个导出Word的按钮" |
| **华渔案例** | 教师说"想要更多模板" → 真实需求可能是"备课太花时间，需要更高效的起步方式" → 解决方案可能不是"更多模板"而是"AI根据教学目标自动生成课件框架" |
| **验证方法** | 连续追问Why（至少3层）：要导出Word → 为什么？→ 要打印给学生 → 为什么不直接投屏？→ 学校没有投影仪 → 真实约束是硬件环境 |
| **防御规则** | 用户反馈进入需求池前，必须经过"需求-方案分离"处理：提取底层需求，方案另议 |

#### 模式 5：全能型需求（什么都想做）
| 维度 | 内容 |
|------|------|
| **识别特征** | 需求范围不断膨胀，功能列表越来越长，优先级全是P0 |
| **典型话术** | "顺便也做一下XX"、"这个也很重要"、"用户肯定需要" |
| **华渔案例** | 101 PPT最初做"备授课工具" → 如果不克制，容易膨胀成"教学管理+学生评估+家长沟通+资源市场+AI助教"全家桶。DJ的做法是正确的——聚焦"实用、易用、好用、顶用"四用 |
| **验证方法** | 功能克制审判：核心层 ≤5个功能，每多1个都要过审；问"去掉这个功能，产品还能不能解决核心问题？"能 → 砍掉 |
| **防御规则** | 功能分层管理：核心层（MVP底线）/ 重要层（V2目标）/ 扩展层（锦上添花），每层有数量上限 |

#### 模式 6：政治正确型
| 维度 | 内容 |
|------|------|
| **识别特征** | 需求的核心驱动力是"不做显得落后/不重视"，而非真实业务价值 |
| **典型话术** | "别的公司都在做AI，我们不做不行"、"ESG要求必须有无障碍功能" |
| **华渔案例** | "元宇宙组织战略"——2022年行业热点期间推出，需审视是战略判断还是跟风。但网龙的元宇宙确实有EDA平台落地，有技术积累（3D/VR），所以这个具体案例不是伪需求，但同类模式需要警惕 |
| **验证方法** | 去掉"政治正确"光环后，纯粹用价值评估框架（D04）打分，低于阈值就不做 |
| **防御规则** | 价值评估必须脱敏——不标注需求来源（领导/竞品/用户），盲评价值 |

#### 模式 7：沉没成本型
| 维度 | 内容 |
|------|------|
| **识别特征** | "都做了这么多了，不做完浪费"——基于已投入资源而非未来价值 |
| **典型话术** | "这个3D模型已经花了3个月做了，必须用上" |
| **华渔案例** | 如果某个VR实验模块开发了很久但用户使用率极低，不能因为"已经投入了"就继续推广。应基于当前数据重新评估 |
| **验证方法** | 假设这个功能从零开始，现在还会不会做？会 → 继续；不会 → 止损 |
| **防御规则** | 所有功能定期（每季度）做"归零评估"：假设从零开始，这个功能排第几？ |

### 伪需求快速检测清单（AI自检用）

每条需求过以下5问，有2个以上回答"否"则标记为疑似伪需求：

| # | 检测问题 | 是/否 |
|---|---------|-------|
| 1 | 能说出具体是**谁**在**什么场景**下需要这个功能吗？ | |
| 2 | 有**用户原声**或**行为数据**作为证据吗？ | |
| 3 | 去掉这个功能，产品还能不能解决核心问题？（如果能 → 可能是伪需求） | |
| 4 | 这个需求对齐至少1个战略重点吗？ | |
| 5 | 实现成本和预期价值匹配吗？（ROI合理） | |

## D03 需求筛选标准

> **质量作用**：提供结构化的需求筛选依据，哪些该做、哪些不该做。

### 四维筛选矩阵

| 维度 | 权重 | 评估方法 | 阈值 |
|------|------|----------|------|
| **用户影响面** | 30% | 受影响用户占总目标用户的百分比 | ≥20% 的目标用户受影响才进入评估 |
| **痛点程度** | 25% | 痛点等级：致命（无法完成任务）/ 严重（效率极低）/ 一般（体验不佳）/ 轻微 | 至少"一般"级别 |
| **战略对齐度** | 25% | 对齐战略重点数量：3E教育 / AI+战略 / 国际化 / 国家平台 | 至少对齐1个战略重点 |
| **实现成本** | 20% | 人力×时间×技术风险的综合评估 | 成本/价值比 ≤ 3（即不能花3倍代价做1份价值的事） |

### 华渔特有筛选补充规则

| 规则 | 说明 | 来源 |
|------|------|------|
| **教育理念一票否决** | 违反DJ"尊重用户""真实学习"理念的需求，无论其他维度多高分，直接否决 | DJ教育哲学 |
| **"四用"检验** | 功能必须通过"实用、易用、好用、顶用"中至少2项检验 | 101 PPT白皮书 |
| **3E对齐** | 优先做同时满足Effective+Efficient+Engaging的需求 | DJ 2025全球智慧教育大会 |
| **反向信息同等重要** | DJ认同"告诉AI不要什么也是充分信息"——需求筛选要同时产出"做什么"和"不做什么" | DJ会议2026-03-09 |

### 需求分级标准

| 级别 | 定义 | 处理方式 | 示例 |
|------|------|----------|------|
| **P0 必做** | 不做产品无法工作或违反合规 | 立即排入当前版本 | AI课件内容审查机制（合规刚需） |
| **P1 应做** | 核心场景的主路径功能 | 排入当前或下一版本 | 教师备课AI辅助（核心场景核心功能） |
| **P2 想做** | 提升体验但非必需 | 排入Backlog，按迭代节奏评估 | 备课历史版本对比 |
| **P3 可做** | 锦上添花 | 记录但不承诺时间 | 教师社区互评功能 |

## D04 功能价值评估框架

> **质量作用**：基于价值而非感觉排序功能优先级。

### 三维价值评估

#### 维度一：用户价值（40%权重）

| 评估项 | 评分标准 | 分值 |
|--------|----------|------|
| 解决的痛点等级 | 致命=10，严重=7，一般=4，轻微=1 | 0-10 |
| 使用频率预期 | 每日=10，每周=7，每月=4，偶尔=1 | 0-10 |
| 用户感知度 | 用户能直接感知价值=10，间接感知=5，无感知=1 | 0-10 |

#### 维度二：商业价值（30%权重）

| 评估项 | 评分标准 | 分值 |
|--------|----------|------|
| 对付费/续约的影响 | 直接驱动=10，间接影响=5，无影响=1 | 0-10 |
| 对获客的影响 | 核心卖点=10，辅助卖点=5，无影响=1 | 0-10 |
| 竞争差异化程度 | 竞品无法快速复制=10，有差异=5，无差异=1 | 0-10 |

#### 维度三：战略价值（30%权重）

| 评估项 | 评分标准 | 分值 |
|--------|----------|------|
| 3E教育对齐度 | 对齐3个E=10，2个E=7，1个E=4，0个=0 | 0-10 |
| AI+战略推进 | 核心推进=10，有关联=5，无关=1 | 0-10 |
| 平台化/复用性 | 可跨业务复用=10，单业务内复用=5，一次性=1 | 0-10 |

#### 华渔特有价值加分项

| 加分项 | 条件 | 加分 |
|--------|------|------|
| DJ亲自提出或明确认可 | 有原话可追溯 | +15% |
| 基于游戏技术赋能教育 | 利用网龙3D/VR核心技术 | +10% |
| 支撑国家级平台项目 | 服务于国家智慧教育公共服务平台 | +10% |
| 国际化可复用 | 可通过Mynd.ai推向海外 | +5% |

### 评分计算
```
总分 = 用户价值平均分×0.4 + 商业价值平均分×0.3 + 战略价值平均分×0.3 + 加分项
```

| 总分区间 | 优先级建议 |
|----------|-----------|
| ≥ 8.0 | P0 — 核心功能，必须做 |
| 6.0-7.9 | P1 — 重要功能，应该做 |
| 4.0-5.9 | P2 — 有价值，看资源做 |
| < 4.0 | P3 — 暂缓或不做 |

## D05 典型情景库

> **质量作用**：覆盖华渔教育主要使用场景，为AI设计师提供场景参考。
> 情景要素标准：Who（谁）、When（什么时候）、Where（在哪里）、Why（为什么）、What（做什么）、How（怎么做）、干扰因素、期望结果。

### 情景 1：教师日常备课（核心高频场景）

| 要素 | 描述 |
|------|------|
| **Who** | 李老师，35岁，初中化学老师，教龄8年，属于"修改整合型"用户 |
| **When** | 周三晚上8点，准备明天第二节课 |
| **Where** | 家里书房，笔记本电脑，WiFi网络 |
| **Why** | 明天要讲"氧化还原反应"，需要一个有3D实验演示的课件 |
| **What** | 打开101 PPT → 搜索"氧化还原反应" → 找到模板 → 修改文字 → 插入3D实验 → 预览 → 保存 |
| **干扰因素** | 家里孩子吵闹、时间紧迫（最多45分钟完成）、不太熟悉3D功能 |
| **期望结果** | 一份包含3D实验演示的课件，明天上课直接用，学生能直观看到化学反应过程 |
| **设计要点** | 备课效率 > 花哨功能；默认模板质量要高；3D实验插入必须简单（不超过3步） |

### 情景 2：课堂实时教学（核心实时场景）

| 要素 | 描述 |
|------|------|
| **Who** | 王老师，28岁，高中物理老师，教龄3年，属于"自主设计型"用户 |
| **When** | 周一上午第三节课，45分钟课时 |
| **Where** | 智慧教室，Promethean互动显示屏，全班40个学生 |
| **Why** | 讲"力的分解"，需要用3D沙盘让学生直观理解矢量分解 |
| **What** | 打开课件 → 3D演示力的分解 → 学生用平板参与互动 → AI助教回答学生提问 → 随堂测试 |
| **干扰因素** | 课堂时间有限、部分学生网络设备卡顿、后排学生看不清屏幕 |
| **期望结果** | 学生理解力的分解概念，课堂参与率>80%，随堂测试正确率>70% |
| **设计要点** | 实时性（不能卡顿）；大屏展示清晰度；互动操作必须简单（学生2秒内能参与） |

### 情景 3：AI辅助内容生产（新兴战略场景）

| 要素 | 描述 |
|------|------|
| **Who** | 张编辑，40岁，华渔AI生产中心内容编辑 |
| **When** | 工作日，一天需要产出8-10个教学资源包 |
| **Where** | 办公室，AI生产中心工作台 |
| **Why** | 为东南亚市场定制本地化化学课程资源 |
| **What** | 输入教学大纲 → AI生成课件草稿 → 人工审核修改 → 本地化适配（语言、文化） → 发布 |
| **干扰因素** | AI生成内容可能有知识性错误、不同国家课程标准不同、翻译质量参差不齐 |
| **期望结果** | 高质量本地化教学资源，人工只需修改20%-30%，产出效率比纯人工提升5倍 |
| **设计要点** | AI生成质量（减少人工修改量）；多语言适配；内容审查自动化 |

### 情景 4：学生自主实验学习（体验驱动场景）

| 要素 | 描述 |
|------|------|

## D06 边界情况穷举清单

> **质量作用**：不遗漏边界情况，确保设计覆盖异常和极端场景。
> **特别要求**：针对教育产品的特殊场景进行穷举。

### 一、通用技术边界

| 边界情况 | 出现概率 | 处理原则 | 教育场景特殊性 |
|----------|----------|----------|----------------|
| **首次使用** | 每个新用户100% | 引导流程必须≤3步完成核心体验 | 教师通常在学期初集中首次使用，可能几十人同时注册，需支撑并发 |
| **无数据状态（空状态）** | 新用户/新学期100% | 提供有意义的默认内容，不留白 | 新学期=空课表+空资源，需预置示范课件和AI推荐 |
| **网络异常/断网** | 农村学校常见 | 核心功能支持离线或优雅降级 | **教育场景关键**：上课中断网 → 3D课件必须有本地缓存；不能在课堂上弹"网络错误"让老师尴尬 |
| **加载缓慢** | 3D/VR资源常见 | 骨架屏+进度提示，≤3秒核心可用 | 课堂45分钟，等3D加载超过10秒=浪费教学时间，学生注意力流失 |
| **权限不足** | 共享设备常见 | 明确告知缺什么权限+如何获取 | 学校电脑可能有管理员限制，教师无法安装插件，需要免安装方案 |
| **设备性能不足** | 农村学校/旧设备 | 自动检测性能并降级 | **关键**：华渔覆盖全国，经济欠发达地区学校设备老旧，3D/VR必须有2D降级方案 |
| **存储空间不足** | 课件/资源累积 | 提前预警+智能清理建议 | 教师设备存满课件很常见，需要云端同步+本地智能管理 |

### 二、教育内容边界

| 边界情况 | 出现概率 | 处理原则 | 具体场景 |
|----------|----------|----------|----------|
| **学科知识错误** | AI生成内容中偶发 | AI生成的教育内容必须有知识性审查机制 | AI生成的化学方程式不配平、物理公式单位错误 → 这是教育产品的致命错误 |
| **教学大纲不匹配** | 跨地区/跨版本 | 内容必须与目标地区课程标准对齐 | 人教版vs北师大版课程进度不同，同一知识点在不同版本出现的年级不同 |
| **内容难度偏差** | 常见 | AI推荐内容需匹配目标学段和学生水平 | 给初一学生推荐高中难度的实验 → 无法完成 → 学习挫败感 |
| **文化敏感内容** | 国际化场景 | 多文化审查机制 | 东南亚市场：宗教信仰差异、饮食禁忌出现在化学实验材料中（如酒精实验在伊斯兰国家的敏感性） |
| **内容时效过期** | 课程标准更新时 | 内容标注有效期+过期提醒 | 2024新课标改革 → 旧版课件中的知识点结构可能已调整 |
| **版权内容引用** | 资源复用时 | 版权审查+来源标注 | 教师从互联网下载图片放入101 PPT → 版权风险 |
| **争议性内容** | 敏感学科 | 标注"存在学术争议"+提供多角度解释 | 历史学科某些事件的不同解读、生物学科进化论在部分地区的敏感性 |

### 三、教育场景特有边界

| 边界情况 | 出现概率 | 处理原则 | 说明 |
|----------|----------|----------|------|
| **课堂中途设备故障** | 中等 | 必须有"一键切换到传统模式"的应急方案 | 互动显示屏死机 → 教师不能站在讲台上等重启。需要：①本地备份 ②手机热点备用 ③纸质PPT备用导出 |
| **学生注意力分散** | 高频 | 互动设计有"注意力回收"机制 | 学生在平板上用AI聊天而非做实验 → 需要教师端监控+学生端使用范围限制 |
| **教师信息化水平极低** | 农村/年长教师 | 核心功能不超过"3次点击"完成 | 50岁以上教师占比高的地区，不能假设所有教师都会使用3D工具 |
| **考试高压期** | 每学期固定 | 考试前功能需求会急剧变化（从"授课"转向"刷题"） | 期中/期末前1-2周，教师需要的不是花哨的3D课件，而是快速出题+批改 |
| **多班级差异化** | 几乎所有教师 | 支持同一课件的多版本管理 | 同一个老师教3个班，每班进度不同，课件需要微调但不能从头做 |
| **班级规模极端值** | 地区差异大 | 交互设计需适配小班(20人)到大班(60人+) | 农村合并校大班额 vs 城市小班化教学，互动方式完全不同 |
| **特殊学生需求** | 一定比例 | 无障碍设计必须覆盖 | 色盲学生看不清红绿标注的化学元素、听障学生需要字幕辅助 |
| **家长介入** | 中等 | 家长端功能边界要明确 | 家长用孩子账号查看学习记录 → 隐私边界？家长能否看到AI对话记录？ |
| **跨学期数据迁移** | 每学期一次 | 自动归档+新学期初始化 | 新学期开始时，旧学期数据怎么处理？教师的资源库要不要清空？ |
| **教师离职/调校** | 低频但影响大 | 资源归属和交接机制 | 教师调走后，他创建的课件归学校还是个人？新老师能否继承？ |
| **双师/多师课堂** | 新兴场景 | 多教师并发操作不冲突 | 全息双师课堂：主讲教师和辅导教师同时操作同一套课件 |
| **AI助教答错** | AI使用中必然 | AI回答必须标注"AI生成，仅供参考"+ 教师一键纠正 | AI助教在课堂上给学生错误答案 → 比没有AI更糟糕。教师必须有实时覆盖AI回答的权限 |
| **实验安全警告** | VR实验特有 | 虚拟实验中的危险操作必须有安全提示 | 学生在VR中"加热浓硫酸"→ 必须弹出安全警告 + 正确操作提示，不能让学生以为现实中也可以随意操作 |

### 四、提示词工程特有边界

| 边界情况 | 出现概率 | 处理原则 | 说明 |
|----------|----------|----------|------|
| **维度信息缺失** | 常见 | AI必须标注"该信息影响质量但库中没有"，不能瞎编 | 设计师查询某个维度但信息为空 → 不能沉默，要明确告知"需要补充" |
| **维度信息过时** | 中等 | 信息标注时效+过期自动提醒 | 3个月前的竞品数据可能已过时 |
| **AI上下文窗口不足** | 维度过多时 | 智能排序+按需加载，不全量堆砌 | 89个维度全量加载会溢出任何模型的上下文窗口 |
| **AI输出幻觉** | AI固有问题 | 关键判断必须可溯源，AI不能编造证据 | AI声称"87%用户反馈XX"但实际无此数据 → 需要来源校验机制 |
| **多维度矛盾** | 复杂设计中常见 | 矛盾时标注冲突+提示人工裁决 | 用户需求（要简单）vs 战略需求（要全面） → AI不能自行取舍，需提示设计师决策 |
| **设计师对维度理解偏差** | 新手常见 | 每个维度附带使用说明+正反案例 | 设计师错误地将"竞品功能"当作"用户需求"放入需求维度 |

## D07 需求优先级历史决策库

> **质量作用**：学习历史优先级决策的逻辑，避免重复犯错。
> 注：因为本项目处于初始阶段，此库以华渔已有产品的决策为蓝本建立初版，后续随项目实践持续积累。

### 历史决策案例

#### 案例 1：101 PPT 的功能优先级决策

| 维度 | 内容 |
|------|------|
| **决策场景** | 101 PPT 初期，功能候选列表100+，需要筛选MVP |
| **排序逻辑** | ①先定三类用户（依赖资源型/修改整合型/自主设计型）→ ②覆盖最大群体（依赖资源型）的核心需求优先 → ③围绕"实用、易用"先行，"好用、顶用"后续 |
| **最终排序** | P0：资源搜索+模板使用 → P1：课件编辑+3D资源插入 → P2：AI助教+语音控课 → P3：VR全沉浸体验 |
| **结果验证** | 先做资源搜索的决策正确——用户首先需要的是"快速找到能用的资源"，而非花哨功能 |
| **可复用逻辑** | **用户分群 → 覆盖最大群体 → 核心路径优先 → 增值体验后续** |

#### 案例 2：AI生产中心 vs 单体AI工具

| 维度 | 内容 |
|------|------|
| **决策场景** | AI能力要做成独立的AI工具（AI课件、AI助教分别独立），还是做成统一的AI生产中心？ |
| **排序逻辑** | DJ高复用原则："能做平台不做工具" → AI生产中心是平台级能力，可服务所有业务线 |
| **最终排序** | P0：AI生产中心（统一能力平台）→ P1：基于平台的各业务线AI应用 |
| **结果验证** | 2024年AI生产中心规模化运营，游戏R&D成本降26.7%，跨业务复用成功 |
| **可复用逻辑** | **平台优先于工具，基础设施优先于上层应用** |

#### 案例 3：国内 vs 海外市场优先级

| 维度 | 内容 |
|------|------|
| **决策场景** | 教育资源有限，先做国内市场还是海外市场？ |
| **排序逻辑** | 国内：国家级平台项目（政策确定性高+规模大）+ 职业教育（增长快）；海外：Mynd.ai独立运营（已分拆上市） |
| **最终排序** | 国内P0：国家智慧教育平台 → 国内P1：职业教育 → 海外由Mynd.ai独立排序 |
| **结果验证** | 2024年7月成功中标国家智慧教育公共服务平台升级项目 |
| **可复用逻辑** | **政策确定性高的项目优先；已独立运营的业务不抢母公司资源** |

#### 案例 4：提示词工程的子项目优先级（本项目决策）

| 维度 | 内容 |
|------|------|
| **决策场景** | DJ定义的4个子项目：信息收集、排序筛选、引擎适配、业务专用 |
| **排序逻辑** | DJ原话："先把范围缩小，做好信息收集这一步，再扩展"；"先做60分版本，快速迭代" |
| **最终排序** | P0：子项目1（维度发现+信息填充）→ P1：子项目2（排序筛选）→ P2：子项目4（AI设计师专用）→ P3：子项目3（引擎适配） |
| **结果验证** | 进行中 |
| **可复用逻辑** | **信息基础 → 信息加工 → 信息应用；基础不牢地动山摇** |

### 优先级决策规律总结

| 规律 | 说明 | 来源 |
|------|------|------|
| **用户覆盖面 > 功能深度** | 先服务最大群体的基本需求，再深入小众高级需求 | 101 PPT案例 |
| **平台 > 工具** | 可复用的平台级能力优先于单点工具 | AI生产中心案例 |
| **确定性 > 潜力** | 政策/合同确定的项目优先于市场潜力大但不确定的项目 | 国家平台案例 |
| **基础 > 应用** | 底层能力/数据/信息先行，上层应用在基础之上构建 | 提示词工程案例 |
| **60分先行** | 先出60分的版本快速验证，不追求一次性做到90分 | DJ会议原话 |
