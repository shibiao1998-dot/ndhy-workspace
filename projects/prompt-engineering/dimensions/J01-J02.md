# J01 K12教育理论库

## 1. 建构主义（皮亚杰 / 维果茨基）

**定义**：学习不是被动接收知识，而是学习者基于已有经验主动建构意义的过程。皮亚杰强调个体认知结构的同化与顺应，维果茨基强调社会交互在认知发展中的核心作用。

**核心观点**：知识不可"灌输"，只能由学习者在与环境、他人的互动中自行建构；先验知识（已有图式）决定了新知识的理解方式。

**产品设计指导**：设计探究式、项目式学习活动，而非单向讲授；提供丰富的互动情境（模拟实验、协作讨论）；用前测/诊断了解学生已有认知，据此推荐个性化学习路径。避免"看完视频就算学了"的假设。

## 2. 支架理论（Bruner）

**定义**：教学者为学习者提供临时性的结构化支持（"脚手架"），帮助其完成独立无法完成的任务，随着能力提升逐步撤除支持。

**核心观点**：支架是动态的——太多则依赖，太少则挫败；关键是"刚好够"的辅助，让学习者在支持下成功，再逐步独立。

**产品设计指导**：新手阶段提供步骤引导、提示框、示例解答；随着用户熟练度提升，逐步减少提示（自适应难度）；设计"渐进撤除"机制——第一次全引导，第二次半引导，第三次独立完成。错题解析从详细到简略，就是典型的支架撤除。

## 3. 最近发展区（维果茨基）

**定义**：学习者"已经能独立完成的水平"与"在帮助下能达到的水平"之间的区域，即 ZPD（Zone of Proximal Development）。最有效的教学发生在 ZPD 内。

**核心观点**：太简单（已掌握区）无聊，太难（超出 ZPD）挫败，只有恰好在 ZPD 内的挑战才能驱动真正的认知发展。

**产品设计指导**：自适应学习系统的理论基石——通过持续评估定位学生当前水平，推送略高于当前能力的内容；题目难度动态调整，保持"跳一跳够得着"的状态；避免统一进度的"一刀切"设计，每个学生的 ZPD 不同。

## 4. 布鲁姆认知分类修订版

**定义**：Anderson & Krathwohl（2001）修订的认知目标分类，将认知过程分为六个层级：**记忆 → 理解 → 应用 → 分析 → 评价 → 创造**，从低阶到高阶递进。

**核心观点**：教学目标应覆盖多个认知层级，而非停留在"记忆"和"理解"；高阶思维（分析、评价、创造）才是深度学习的标志；每个层级对应不同的行为动词和评估方式。

**产品设计指导**：用布鲁姆层级设计题目和活动——选择题测"记忆/理解"，应用题测"应用/分析"，开放题/项目测"评价/创造"；学习报告中标注学生在各层级的掌握度，帮助教师识别"只会背不会用"的问题；课程设计确保从低阶到高阶有完整的认知攀升路径，不止于刷题。

## 5. 多元智能理论（Gardner）

**定义**：Howard Gardner 提出人类智能不是单一的 IQ，而是至少 8 种相对独立的智能：语言、逻辑-数学、空间、音乐、身体-动觉、人际、内省、自然观察。

**核心观点**：每个学生的智能组合不同，传统教育偏重语言和逻辑-数学智能，忽视了其他维度；应为不同智能优势的学生提供多元的学习通道和表现方式。

**产品设计指导**：同一知识点提供多种呈现方式（文字、图表、动画、音频、动手操作模拟）；允许学生选择不同方式展示学习成果（写报告、画思维导图、录视频、做模型）；学习者画像中纳入智能偏好维度，用于个性化推荐。不是每种智能都要覆盖，而是提供选择权。

# J02 学习心理学知识库

## 1. 认知负荷理论（Sweller）

**定义**：工作记忆容量有限，学习材料带来的认知负荷分三类——**内在负荷**（知识本身的复杂度）、**外在负荷**（糟糕的呈现方式造成的无效负担）、**相关负荷**（促进图式建构的有效认知投入）。

**核心观点**：设计的目标是降低外在负荷、管理内在负荷、最大化相关负荷。信息过载是学习失败的首要原因。

**产品设计指导**：界面去噪——一屏一个核心信息，避免同时呈现文字+语音+动画（冗余效应）；复杂知识分步呈现（chunking），不一次性堆砌；用worked example（解题示范）降低新手的内在负荷；交互设计避免"找按钮"式的外在负荷——操作路径要直觉化。

## 2. 自我决定理论（Deci & Ryan）

**定义**：人类内在动机由三种基本心理需求驱动——**自主感**（我能选择）、**胜任感**（我能做到）、**关联感**（我被接纳、有归属）。三者满足时内在动机最强。

**核心观点**：外在奖励（积分、排名）短期有效但可能损害内在动机（过度辩护效应）；真正持久的学习动力来自满足这三种需求。

**产品设计指导**：**自主**——提供学习路径选择、内容偏好设置、学习节奏自控；**胜任**——即时反馈、进度可视化、难度自适应保持"刚好能成功"；**关联**——学习社区、同伴互助、师生连接。游戏化设计（徽章、等级）要服务于胜任感而非替代内在动机，积分体系不能变成"为积分而学"。

## 3. 心流理论（Csikszentmihalyi）

**定义**：当挑战难度与个人技能水平高度匹配时，人会进入一种全神贯注、忘我投入的最优体验状态——**心流（Flow）**。技能高于挑战则无聊，挑战高于技能则焦虑。

**核心观点**：心流的前提条件——清晰的目标、即时的反馈、挑战与技能的动态平衡。心流状态下学习效率和满足感最高。

**产品设计指导**：动态难度调节（DDA）是核心——根据用户表现实时调整题目难度，保持在"略有挑战但可达"的甜蜜区；减少打断（不在做题中途弹广告/通知）；提供清晰的阶段目标和即时反馈（对错、得分、进度条）；设计"沉浸模式"减少界面干扰，帮助用户进入心流。

## 4. 间隔重复与遗忘曲线（Ebbinghaus）

**定义**：艾宾浩斯遗忘曲线表明，新学内容在不复习的情况下会快速遗忘（20分钟后忘42%，1天后忘67%）。**间隔重复（Spaced Repetition）** 在遗忘即将发生时安排复习，可以用最少次数实现长期记忆。

**核心观点**：复习的时机比复习的次数更重要；间隔逐渐拉长（1天→3天→7天→14天→30天）效果最佳；已掌握的内容减少复习频率，未掌握的缩短间隔。

**产品设计指导**：内置 SRS（间隔重复系统）算法（如 SM-2 / FSRS），自动安排复习计划；每日推送"今日复习"任务，降低用户自主规划的认知成本；将复习融入日常学习流程而非独立模块——做新题时穿插旧知识点；复习卡片难度根据用户反馈（"简单/困难"）动态调整间隔。
