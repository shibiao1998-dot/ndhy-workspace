# H 类：设计标准与规范维度（7个）

> **目标**：确保 AI 设计师/策划的产出符合平台规范、公司标准和行业惯例，可直接交付开发实现。
> **质量守门定位**：H 类是"产出能不能用"的底线检查——不符合标准的设计即使创意再好也无法落地。
> **填充状态**：公开信息直接填充 ✅ | 需要私域信息标注 ⚠️ 待填充

## H01 平台交互规范库

| 字段 | 内容 |
|------|------|
| **维度ID** | H01 |
| **维度名称** | 平台交互规范库 |
| **质量作用** | 平台一致性检查：确保设计方案的交互模式符合目标平台的原生规范，避免开发实现时因平台差异返工。AI 设计师在产出原型/交互方案时，自动对齐平台规范。 |
| **数据来源** | Apple HIG、Google Material Design 3、W3C WCAG、各平台官方文档 |
| **更新机制** | 平台发布重大版本更新时同步（iOS/Android 年度大版本、Material Design 版本迭代）；每半年例行巡检 |

### 具体信息内容

#### 1. Web 端交互规范（华渔产品主要平台）

| 交互模式 | 规范要求 | 华渔产品适用场景 |
|----------|----------|------------------|
| **导航** | 顶部导航栏 + 侧边栏二级导航；面包屑导航深度 ≤ 4 层；当前位置高亮 | 101 教育 PPT Web 版、教师备课平台 |
| **页面加载** | 骨架屏 → 渐进加载；首屏加载 ≤ 3s；3D 资源异步加载带进度条 | 3D 课件资源库、VR 实验预览 |
| **表单交互** | 即时校验（失焦触发）；错误信息紧贴字段下方红色提示；必填项 `*` 标记 | 课件编辑、实验参数设置 |
| **反馈机制** | 操作成功 → Toast 2s 自动消失；操作失败 → 弹窗 + 明确错误原因 + 操作建议；异步操作 → Loading 状态 + 预估时间 | 课件保存、资源上传、AI 生成 |
| **拖拽交互** | 拖拽手柄明确可见；拖拽过程实时预览目标位置；释放区域高亮提示 | PPT 课件编排、实验器材拖放 |
| **快捷键** | 遵循通用快捷键（Ctrl+S 保存、Ctrl+Z 撤销）；自定义快捷键可配置且不与系统冲突 | 101 教育 PPT 桌面版 |
| **响应式** | 断点：1440px（大屏）/ 1024px（中屏）/ 768px（平板）；教育场景优先大屏体验 | 教师使用的电脑/交互屏 |

#### 2. 移动端交互规范（iOS / Android）

| 交互模式 | iOS（HIG） | Android（Material 3） | 华渔适用场景 |
|----------|-----------|---------------------|-------------|
| **返回** | 左滑返回 + 左上角返回按钮 | 系统返回键 + 顶部返回箭头 | 学生端 App、手机控课 |
| **手势** | 下拉刷新、长按预览（Peek & Pop） | 下拉刷新、长按菜单 | 课件浏览、资源预览 |
| **底部导航** | ≤ 5 个 Tab，图标 + 文字 | ≤ 5 个目的地，Material Bottom Nav | 学生端主导航 |
| **弹窗** | Action Sheet（底部弹出） | Bottom Sheet | 课件操作菜单 |
| **输入** | 键盘自适应、安全区域适配 | 键盘自适应、Material 输入框 | 搜索、笔记输入 |

#### 3. 教育专用交互设备规范

| 设备 | 交互规范 | 特殊要求 |
|------|----------|----------|
| **Promethean ActivPanel** | 多点触控（最多 20 点）；手写笔识别；手掌误触排除 | 教师站立操作 → 按钮最小 48px；学生多人同时触控 → 防干扰 |
| **VR 头显 + 手柄** | 射线指向 + 扳机选择；手势识别（抓取、释放）；6DOF 空间移动 | VR 化学实验 → 手部碰撞检测；安全边界提示 |
| **AR 眼镜（Rokid 合作）** | 语音指令 + 手势控制；HUD 信息叠加 | AR 实验辅助 → 信息层不遮挡实物；字体 ≥ 16pt |

#### 4. 无障碍标准

| 标准 | 要求 | 检查方法 |
|------|------|----------|
| **WCAG 2.1 AA** | 对比度 ≥ 4.5:1（正文）/ ≥ 3:1（大文本）；可键盘操作；屏幕阅读器兼容 | 自动化工具（Lighthouse、axe） |
| **教育无障碍** | 色盲友好色板；字体可放大（1.5x）；视频配字幕 | 人工检查 + 色盲模拟 |

### 具体信息内容

#### 1. 核心设计哲学：7C 管理设计思想

> 来源：101 教育 PPT 白皮书 —— "基于 7C 管理的设计思想"

| C | 含义 | 在设计中的体现 | 检查标准 |
|---|------|----------------|----------|
| **C1 — Customer（用户）** | 以用户为中心，理解用户真实需求 | 每个设计决策可追溯到用户需求证据 | 能回答"这个设计解决了谁的什么问题？" |
| **C2 — Context（情境）** | 理解用户使用的真实情境 | 考虑教室环境、网络条件、设备差异 | 能回答"用户在什么场景下使用？" |
| **C3 — Content（内容）** | 内容为王，功能服务于内容 | 教学内容是核心，UI 是内容的载体不是装饰 | 内容层次清晰，信息密度合理 |
| **C4 — Connection（连接）** | 建立人与知识、人与人的连接 | 社区互动、协作功能、知识关联 | 能回答"这个功能建立了什么连接？" |
| **C5 — Creativity（创造）** | 激发创造力，而非限制 | 开放式工具、灵活模板、自定义空间 | 工具是否支持用户自由创造？ |
| **C6 — Collaboration（协作）** | 支持多角色协作 | 教师协作、师生互动、家校沟通 | 协作流程是否顺畅？ |
| **C7 — Continuous（持续）** | 持续优化、迭代进化 | 数据驱动迭代、用户反馈闭环 | 设计是否预留迭代空间？ |

#### 2. 四用标准（产品设计的质量标尺）

> 来源：101 教育 PPT 白皮书 —— 围绕"实用、易用、好用、顶用"四用进行设计

| 层级 | 标准 | 定义 | 检查方法 |
|------|------|------|----------|
| **L1 实用** | 功能解决真实问题 | 不做花架子，每个功能都有明确的教学价值 | 该功能删掉后，用户的核心任务是否受影响？ |
| **L2 易用** | 学习成本低，操作自然 | 新用户 5 分钟上手核心功能；无需培训即可完成基本操作 | 新用户任务完成率 ≥ 80% |
| **L3 好用** | 体验流畅，有愉悦感 | 操作反馈及时、视觉舒适、流程顺滑 | 用户满意度评分 ≥ 4.0/5.0 |
| **L4 顶用** | 超出预期，形成口碑 | 解决用户"没想到还能这样"的需求，创造惊喜 | 用户主动推荐率（NPS）正向 |

**设计评审必问**：
- 这个功能实用吗？（解决什么真实问题？）
- 这个功能易用吗？（新用户能上手吗？）
- 这个功能好用吗？（体验流畅吗？）
- 这个功能顶用吗？（有惊喜吗？）

#### 4. 设计风格原则（已知）

> 来源：101 教育 PPT 白皮书 —— 采用"简洁通用"的设计风格

| 原则 | 说明 | 正面案例 | 反面案例 |
|------|------|----------|----------|
| **简洁** | 减少视觉噪音，突出核心内容 | 备课界面只展示当前步骤所需信息 | 界面堆满入口和功能按钮 |
| **通用** | 跨年龄段、跨学科可理解 | 图标含义不依赖特定文化背景 | 使用行业黑话或网络梗 |
| **以内容为中心** | 教学内容是主角，UI 是配角 | 课件展示区占屏幕 70%+ | 工具栏和装饰占据大量空间 |
| **温暖教育感** | 色彩亲和、不冷漠、不幼稚 | 柔和的教育蓝 + 暖色点缀 | 纯灰黑商务风或过度卡通 |

## H03 异常处理标准库

| 字段 | 内容 |
|------|------|
| **维度ID** | H03 |
| **维度名称** | 异常处理标准库 |
| **质量作用** | 异常完整性检查：确保设计方案覆盖所有可预见的异常场景，每种异常都有标准化的处理方式。AI 设计师在产出方案时自动补全异常路径。 |
| **数据来源** | 平台规范 + 华渔历史项目的异常处理经验 + 教育场景特殊异常 |
| **更新机制** | 每半年更新；新异常类型出现时实时补充 |

### 具体信息内容

#### 1. 通用异常处理标准

| 异常类型 | 触发条件 | 处理原则 | 标准话术 | 交互方式 |
|----------|----------|----------|----------|----------|
| **网络断开** | 检测到无网络连接 | 本地缓存优先 → 恢复后自动同步 | "网络已断开，已保存的内容不会丢失。恢复网络后自动同步" | 顶部条幅（黄色警告）+ 离线标记 |
| **网络超时** | 请求超过 10s 无响应 | 自动重试 1 次 → 失败后提供手动重试 | "加载较慢，正在重试…" → "加载失败，点击重试" | Loading 动画 → 错误页 + 重试按钮 |
| **服务器错误（5xx）** | 服务端返回 500/502/503 | 展示友好错误页，隐藏技术细节 | "服务器正在开小差，请稍后再试。如持续出现，请联系客服" | 全屏错误页 + 刷新按钮 + 客服入口 |
| **权限不足（403）** | 用户无权访问资源 | 明确告知原因 + 提供解决路径 | "你没有该课件的编辑权限，请联系 [课件所有者] 申请权限" | 弹窗说明 + 操作按钮（申请/返回） |
| **资源不存在（404）** | 请求的资源已被删除或链接失效 | 引导到相关内容 | "该课件不存在或已被删除。你可以搜索相关课件" | 404 页面 + 搜索框 + 推荐内容 |
| **数据为空** | 列表/搜索无结果 | 区分"从未有数据"和"筛选无结果"，引导操作 | 无数据："还没有课件，立即创建第一个" / 无结果："没有找到匹配的课件，换个关键词试试" | 空状态插画 + 引导按钮 |
| **加载失败（资源）** | 图片/3D 模型/视频加载失败 | 展示占位符 + 重试选项 | "资源加载失败" + [重试] 按钮 | 占位图（灰色块 + 图标）+ 重试 |
| **操作冲突** | 多人同时编辑同一资源 | 乐观锁 → 冲突时提示并保留两版 | "其他人正在编辑此课件，你的修改已另存为副本" | 弹窗 + 查看两版按钮 |
| **存储不足** | 设备/云端存储空间不足 | 提前预警（80% 使用率），拦截上传 | "存储空间不足，请清理后重试" + 空间管理入口 | 弹窗 + 跳转空间管理 |
| **文件格式不支持** | 上传了不支持的文件类型 | 明确告知支持的格式 | "不支持 .xxx 格式，请上传 PPT/PDF/图片（JPG, PNG）" | 上传区域错误提示 |

#### 2. 教育场景特殊异常

| 异常类型 | 场景 | 处理原则 | 标准话术 |
|----------|------|----------|----------|
| **课堂投屏断开** | 教师正在上课，投屏连接中断 | 最高优先级恢复 → 自动重连 → 本地继续展示 | "投屏连接中断，正在自动重连…内容仍在本地显示" |
| **AI 生成超时** | AI 课件/内容生成超过 30s | 后台继续生成 → 通知完成 | "AI 正在努力生成中，完成后将通知你。你可以先处理其他任务" |
| **AI 生成质量差** | AI 生成内容不符合预期 | 提供反馈 + 重新生成选项 + 手动修改入口 | "对生成结果不满意？[重新生成] [手动修改] [提交反馈]" |
| **3D 模型渲染失败** | 设备性能不足或 WebGL 不支持 | 降级为 2D 图片 + 提示 | "你的设备暂不支持 3D 预览，已切换为图片模式" |
| **VR 设备未连接** | 点击 VR 功能但未检测到设备 | 引导连接 → 提供非 VR 替代 | "未检测到 VR 设备。[连接设备帮助] [使用 3D 预览模式]" |
| **实验安全警告** | 虚拟实验中违反安全操作 | 暂停实验 + 安全提示 + 引导正确操作 | "⚠️ 安全提示：浓硫酸应缓慢加入水中。请重新操作" |
| **多学生并发操作** | 多学生在互动屏上同时操作冲突 | 分区操作 → 操作队列 | "当前区域已有同学在操作，请选择其他区域" |
| **课程时间到** | 预设课堂时间结束 | 温和提醒 → 允许延时 | "本节课时间已到。[结束课堂] [延长 5 分钟]" |

#### 3. 异常处理分级

| 级别 | 定义 | 响应时间 | 处理策略 |
|------|------|----------|----------|
| **P0 阻断** | 用户核心任务完全无法进行 | 立即 | 自动恢复 → 备用方案 → 紧急通知 |
| **P1 严重** | 核心功能部分受损 | ≤ 3s | 展示错误 + 替代方案 + 自动上报 |
| **P2 一般** | 非核心功能异常 | ≤ 5s | 友好提示 + 手动操作选项 |
| **P3 轻微** | 体验降级但不影响任务 | 被动展示 | 安静降级 + 恢复后自动回归 |

## H05 开发理解障碍库

| 字段 | 内容 |
|------|------|
| **维度ID** | H05 |
| **维度名称** | 开发理解障碍库 |
| **质量作用** | 可读性保障：识别并消除设计文档中开发人员常见的理解障碍。AI 设计师在产出文档时自动规避这些表述，确保开发能看懂、能做出来。 |
| **数据来源** | 开发-设计沟通的常见问题汇总 + 教育行业特有的沟通障碍 |
| **更新机制** | 持续积累；每次开发反馈新的理解障碍时实时补充 |

### 具体信息内容

#### 1. 模糊表述对照表

| ❌ 模糊表述 | 🤔 为什么有问题 | ✅ 应该写成 |
|------------|----------------|-------------|
| "页面要有高级感" | "高级感"没有标准定义 | "背景色 #FAFAFA，留白 ≥ 24px，字体 14px 灰色 #666" |
| "交互要流畅" | "流畅"无法量化 | "点击后 ≤ 200ms 出现反馈动画，动画时长 300ms ease-out" |
| "类似微信的效果" | 微信有几百个功能，指哪个 | "类似微信聊天列表的左滑删除效果，滑动距离 ≥ 60px 触发" |
| "智能推荐" | 推荐逻辑不清楚 | "基于用户最近 7 天浏览的学科 + 年级，推荐匹配的课件 Top 10" |
| "用户友好" | 没有可执行标准 | "新用户在无引导情况下，3 步内完成 [具体任务]" |
| "适当的动画" | 适当是多少 | "卡片入场动画：fadeIn 200ms，间隔 50ms 依次出现" |
| "响应式布局" | 哪些端？断点是？ | "≥ 1024px 三栏；768-1023px 两栏；< 768px 单栏" |
| "自适应展示" | 什么条件自适应什么 | "窗口宽度 ≥ 1200px 时展示侧边栏；< 1200px 时收起为浮层" |

#### 2. 歧义词汇库

| 歧义词 | 可能的理解 | 消歧方法 |
|--------|-----------|----------|
| "弹窗" | Modal / Dialog / Popover / Toast / Tooltip | 明确类型 + 触发条件 + 关闭方式 |
| "列表" | 简单列表 / 卡片列表 / 表格 / 瀑布流 | 明确布局 + 每条展示哪些字段 |
| "筛选" | 前端筛选 / 后端筛选 / 组合筛选 / 级联筛选 | 明确筛选维度 + 逻辑（AND/OR） + 数据量 |
| "缓存" | 浏览器缓存 / 本地存储 / 离线缓存 / CDN 缓存 | 明确缓存什么 + 有效期 + 更新策略 |
| "实时" | 毫秒级 / 秒级 / 分钟级 | 明确可接受的延迟范围 |
| "大量数据" | 100 条？1 万条？100 万条？ | 明确数据量级 + 分页策略 |

#### 3. 教育场景特有障碍

| 障碍类型 | 示例 | 解决方案 |
|----------|------|----------|
| **教学术语不理解** | "脚手架式引导"开发不知道是什么 | 翻译为具体交互："分步引导弹窗，每步高亮对应功能区" |
| **3D 交互描述不清** | "实验器材可交互"太模糊 | 明确：哪些器材可拖动、拖动范围、碰撞检测规则、物理反馈 |
| **教学流程 vs 技术流程混淆** | "先导入再实验再总结"这是教学流程，不是技术实现 | 拆分为状态机：导入状态 → 实验状态 → 总结状态，每状态的 UI/功能/数据 |
| **学段差异不清楚** | "适合中学生"具体什么年级 | 明确年级 + 认知水平 + 对应的 UI 复杂度上限 |

#### 4. 设计文档可读性检查清单

- [ ] 所有颜色都用 Hex/RGB 值，不用"浅蓝""深灰"等描述
- [ ] 所有尺寸都用 px/pt/%/rem，不用"大一点""适当间距"
- [ ] 所有动画都标明类型、时长、缓动曲线
- [ ] 所有交互都标明触发条件、反馈、结束状态
- [ ] 所有列表都标明排序规则、分页策略、数据量预估
- [ ] 所有状态都覆盖：默认/加载/空/错误/成功/禁用
- [ ] 没有使用歧义词汇库中的歧义词（或已消歧）

- **H + K 类联动**：H05（开发理解障碍）依赖 K 类术语库消除歧义
