## B03 用户细分维度库

**具体信息内容**：

### 一、教师细分（三类分型模型）

| 类型 | 特征描述 | 占比估计 | 核心需求 | 产品使用模式 |
|------|----------|----------|----------|-------------|
| **依赖资源型** | 直接使用现成课件和资源，很少修改；通常教龄较短或信息化能力较弱；对"拿来就用"有强烈期待 | ~40%（推测，基于教育信息化普及现状） | 高质量现成资源、一键生成、操作极简 | 搜索→下载→直接使用，停留时间短，编辑操作少 |
| **修改整合型** | 在现有资源基础上修改调整，整合多个来源；有一定信息化能力；追求"适合我的班级" | ~40%（推测） | 易于修改的模板、素材丰富可组合、批量调整工具 | 搜索→下载多个→拼合修改→保存，停留时间中等 |
| **自主设计型** | 从头设计课件，有独特教学理念；信息化能力强，常是学科带头人或教研组长 | ~20%（推测） | 高级编辑功能、原创素材工具、分享与展示平台 | 新建→精细设计→反复打磨→分享，停留时间长 |

> ⚠️ 占比数据为基于行业经验的推测值，需用 B02 行为数据验证

**教师二级细分维度**：

| 维度 | 分类 | 说明 |
|------|------|------|

| 学科 | 化学（重点）/物理/生物/数学/语文/英语/其他 | 理科尤其依赖实验演示和可视化 |
| 教龄 | 新手(0-3年)/成长(3-10年)/资深(10年+) | 与技术接受度和资源需求类型强相关 |
| 地域 | 一线城市/二三线城市/县镇/农村 | 影响设备条件、网络环境、资源可获取性 |
| 学校层次 | 重点/普通/薄弱 | 影响教研支撑力度和信息化预算 |

### 二、学生细分

| 维度 | 分类 | 说明 |
|------|------|------|
| 学段 | 小学(1-6)/初中(7-9)/高中(10-12) | 认知发展阶段不同，交互设计需求差异大 |
| 学科兴趣 | 理科偏好/文科偏好/均衡 | 影响对实验类产品的主动使用意愿 |
| 技术素养 | 高(自己能探索)/中(需引导)/低(需手把手) | 影响产品学习成本容忍度 |
| 学习动机 | 兴趣驱动/成绩驱动/被动型 | 影响对探索式 vs 任务式产品设计的偏好 |
| 使用场景 | 课堂跟练/课后复习/考前突击/自主探索 | 同一学生在不同场景下行为模式不同 |

### 三、教育管理者细分

| 层级 | 角色 | 核心关注点 |
|------|------|-----------|
| **校级决策层** | 校长/副校长 | 整体信息化建设方向、预算审批、政策合规 |
| **中层管理层** | 教务主任/教研组长/年级主任 | 教学质量提升、教师培训效果、使用率数据 |
| **信息化执行层** | 信息中心主任/电教管理员 | 系统部署、技术对接、日常运维、故障处理 |
| **区域管理层** | 教育局信息化科室 | 区域统一平台、数据汇总、采购招标合规 |

### 四、家长细分

| 类型 | 特征 | 对教育科技态度 |
|------|------|---------------|
| **技术拥抱型** | 主动了解教育科技，愿意让孩子尝试 | 积极支持，会主动寻找优质产品 |
| **跟随型** | 学校推荐什么就用什么，不主动选择 | 配合但不主动，关注是否影响成绩 |
| **担忧型** | 担心屏幕时间、网络安全、分散注意力 | 谨慎甚至抵触，需要安全保障和使用时间控制 |
| **无感型** | 不太关注孩子具体用什么工具学习 | 不参与决策，基本不影响产品选择 |

**质量作用**：让提示词能针对不同用户细分生成差异化内容，而不是用"教师"或"学生"的笼统标签一刀切

**数据来源**：教师三类分型基于华渔产品定位公开资料 + 教育信息化行业通识；学生/管理者/家长分型基于 K12 教育行业标准分类模型 + 逻辑推演

**更新机制**：每半年复核一次分型模型，根据 B02 行为数据验证/调整占比；新用户群体出现时即时补充

| **置信度** | 强证据/中等/初步信号 |

### 示例推测（待验证）

**推测 1：年轻教师未必是 AI 功能的主力用户**

| 字段 | 内容 |
|------|------|
| **常识预期** | 年轻教师（90后、00后）更熟悉技术，应该是 AI 辅助备课功能的主要使用者 |
| **推测情况** | 年轻教师可能因为"想证明自己的教学能力"而倾向于自主设计课件，反而是中年教师（教龄 10+ 年、事务性负担重）更愿意借助 AI 提效 |
| **推测依据** | 职业发展阶段理论——新手期更注重能力展示和认同获取；成熟期更注重效率优化。在其他行业（如医疗）也观察到类似模式：新手倾向于亲力亲为，资深人员更愿意使用辅助工具 |
| **验证方式** | 对比不同教龄段教师的 AI 功能使用率（B02 数据） |
| **产品启示** | 如果验证成立，AI 功能的推广策略不应只瞄准"技术原住民"，而应重点切入"时间贫困"的资深教师群体 |
| **置信度** | ⚠️ 初步推测，待验证 |

**推测 2：虚拟实验的最大价值可能不是"替代真实实验"而是"预习和复习"**

| 字段 | 内容 |
|------|------|
| **常识预期** | 虚拟实验的核心场景是替代因条件不足而无法开展的真实实验 |
| **推测情况** | 虚拟实验可能在"实验前预习操作流程"和"实验后复习巩固"场景中使用频率更高、学习效果更好，而不是作为真实实验的完全替代 |
| **推测依据** | 教育心理学中"预组织者"理论（Ausubel）——先建立操作心理模型再动手做，效果优于直接操作；部分高校虚拟仿真实验的使用数据显示"课前预习"模式的学生实验成绩优于"课上替代"模式 |
| **验证方式** | 分析未来实验室的使用时间分布（课前/课中/课后），对比不同使用模式下的实验完成率和正确率 |
| **产品启示** | 如果验证成立，产品定位应从"实验替代方案"转向"实验增强方案"，功能设计可增加"预习模式"和"复习闯关"模块 |
| **置信度** | ⚠️ 初步推测，待验证 |

**质量作用**：让提示词不陷入"众所周知"的思维陷阱，能基于反常识发现提供差异化的洞察和建议

**数据来源**：⚠️ 待采集（推荐方式：行为数据异常值挖掘、深度用户访谈中的意外发现、A/B 测试结果中的反直觉数据）

**更新机制**：持续积累，每次用户研究后提炼新发现；每季度回顾已有发现的验证进展

|------|------|
| **Who** | 教师（三类均涉及，尤其依赖资源型和修改整合型） |
| **When** | 工作日晚间（18:00-22:00 是备课高峰）、周末、寒暑假集中备课；通常在课前 1-3 天准备 |
| **Where** | 家中（占比最高）、学校办公室、备课室；移动端偶尔用于碎片时间浏览素材 |
| **Why** | 准备第二天/本周的教学课件，需要将教材内容转化为课堂呈现形式 |
| **What** | 搜索课件模板/素材 → 下载/选用 → 编辑调整（添加本校、本班特色内容）→ 预览 → 保存/导出为 PPT |
| **How** | 使用 101 教育 PPT，搜索与当前教学进度匹配的资源，组合编辑后导出。依赖资源型直接用，修改整合型会拼合多个来源，自主设计型用工具从头设计 |

**关键体验节点**：搜索结果是否精准匹配教材版本和章节 → 决定第一步效率

### 场景 2：课堂教学呈现

| 维度 | 描述 |
|------|------|
| **Who** | 教师（授课者）+ 学生（40-50 人的班级） |
| **When** | 正式上课时间，每节课 40-45 分钟 |
| **Where** | 教室，使用投影仪/电子白板/一体机呈现 |
| **Why** | 辅助讲解知识点，提升课堂效率和学生注意力，尤其是抽象概念的可视化 |
| **What** | 打开课件 → 按教学流程逐页展示 → 穿插互动（提问、标注、动画演示）→ 课堂练习 → 总结 |
| **How** | 投屏或直接在一体机上展示 PPT，利用动画、视频嵌入辅助讲解；部分教师会穿插使用虚拟实验做演示 |

**关键体验节点**：课件在不同设备上的兼容性和流畅度 → 投影卡顿或格式错乱会严重打断教学节奏

### 场景 3：化学实验演示（虚拟）

| 维度 | 描述 |
|------|------|
| **Who** | 教师（演示者）或学生（自操作），主要面向高中化学 |
| **When** | 课堂实验环节（约 10-20 分钟）；或实验课前的预习、实验课后的复习 |
| **Where** | 教室（教师大屏演示）、实验室（配合真实实验）、机房或家中（学生自操作） |
| **Why** | 展示危险/高成本/微观不可见的化学实验过程；让学生理解实验操作规范和反应机理 |
| **What** | 选择实验类型 → 选取器材/药品 → 按步骤操作（加热、混合、滴定等）→ 观察反应现象 → 记录数据/结论 |
| **How** | 使用未来实验室，选择预设实验或自由组合器材。教师演示时投屏讲解，学生操作时在个人设备上逐步完成。支持暂停、回放、调整参数 |

**关键体验节点**：实验现象的真实感和交互流畅度 → 不逼真的模拟反而会形成错误的认知模型

### 场景 4：学生自主学习

| 维度 | 描述 |
|------|------|
| **Who** | 学生（以兴趣驱动型和成绩驱动型为主） |
| **When** | 课后（放学后到睡前）、周末、考前复习期 |
| **Where** | 家中（占比最高）、图书馆、学校自习室 |
| **Why** | 复习课堂内容、预习新知识点、考前针对性练习、满足学科探索兴趣 |
| **What** | 打开学习平台 → 选择知识点/章节 → 观看讲解/操作虚拟实验 → 做练习题 → 查看反馈和解析 |
| **How** | 学生在平板/电脑上使用产品，自主选择学习内容。可能先看课件回顾知识点，再操作虚拟实验加深理解，最后做练习检验掌握程度 |

**关键体验节点**：能否快速定位到"我不会的那个知识点" → 学生耐心有限，3 次找不到就流失

### 场景 5：教研活动与公开课

| 维度 | 描述 |
|------|------|
| **Who** | 教研组教师群体（3-15人），通常有教研组长主导 |
| **When** | 学校固定教研活动时间（每周/每两周一次），或区域教研活动、公开课准备期 |
| **Where** | 学校会议室、教研活动室、远程教研（线上） |
| **Why** | 集体磨课、分享教学经验、准备公开课/示范课、研讨教学策略 |
| **What** | 展示课件/实验方案 → 集体研讨修改建议 → 迭代优化 → 形成教研成果（优秀课件/实验设计） |
| **How** | 某位教师展示用 101 PPT 或未来实验室设计的教学方案，组内其他教师提意见，协作修改。公开课前通常要经历 3-5 轮迭代打磨 |

**关键体验节点**：课件的分享和协作编辑能力 → 如果不支持多人协作，就只能"一人改、邮件传"的低效模式

### 场景 6：管理者数据查看与决策

| 维度 | 描述 |
|------|------|
| **Who** | 校级管理者、教务主任、信息中心主任、区域教育局管理者 |
| **When** | 学期初（规划）、学期中（检查）、学期末（总结评估）、采购决策前 |
| **Where** | 办公室，使用电脑/平板查看后台管理系统 |
| **Why** | 了解产品使用情况和效果，为续费/扩展采购提供依据，完成上级信息化考核要求 |
| **What** | 登录管理后台 → 查看整体使用数据（活跃率、使用时长、覆盖率）→ 对比分析各班级/学科差异 → 导出报告向上汇报 |
| **How** | 管理者关注宏观数据而非具体功能，需要直观的数据看板和可导出的统计报告。常需要"一张图说清楚"的汇总视图来向上级或家长委员会汇报 |

**关键体验节点**：数据是否"好看且能说明问题" → 管理者需要的不是原始数据，而是能支撑汇报的"结论性数据"

**质量作用**：让提示词能基于具体使用场景生成上下文感知的内容，而不是脱离场景的通用回答

**数据来源**：华渔产品公开资料 + K12 教育行业通识 + 教师工作模式公开研究 + 逻辑推演

**更新机制**：每半年复核场景覆盖完整性；新产品功能上线后补充新场景；用 B01/B02 数据验证场景假设

## 研究局限性声明

1. **B01（原声库）和 B02（行为数据）为空壳模板**：缺少华渔内部数据访问权限，无法填充真实数据。这两个维度的价值完全取决于后续的数据采集质量
2. **B03 占比数据为推测值**：教师三类分型的比例缺乏华渔产品内部数据支撑，40%/40%/20% 是基于行业经验的粗估
3. **B04 痛点主要基于行业通识**：虽然多数痛点有公开数据支撑，但缺少华渔特定用户的一手验证
4. **B05 反常识发现为推测性质**：两个示例均标注为"初步推测"，需要数据验证后才能作为产品决策依据
5. **B06 场景基于典型模型**：实际使用场景可能存在本文档未覆盖的边缘情况，需持续观察补充

> 📌 **优先行动建议**：最高优先级是填充 B01 和 B02——有了真实原声和行为数据，其他维度（B03 的占比、B04 的排序、B05 的验证、B06 的细节）都能得到校准和增强。
