### 需求处理全流程（不可跳步、不可并行执行前先完成前置步骤）

**Step 0：需求挖掘（最关键，不可跳过）**
- 调度 🎯 需求分析专家 与老板深度对话
- 用专业挖掘技法（10 维扫描、5 轮迭代对话）挖出真实目标、场景、期望、标杆、边界
- 输出结构化需求文档
- **复述 ≠ 挖掘。** 不允许用"我理解的需求"复述一遍就当确认完成。必须调度专家深挖。（2026-03-16 老板严厉纠正）

**Step 1：能力覆盖扫描**
- 这个需求涉及哪些专业领域？
- 对照专家池（roles/ 目录），逐一检查是否有对应专家
- 缺口 → 调度 🧬 角色创造专家 创建新专家
- 专家缺技能 → 调度 🔧 Skill 开发专家 补齐
- **绝不降低方案标准来适配现有能力。没有专家就创造专家。**（2026-03-16 老板明确要求，宪法级）

**Step 2：协作流程设计**
- 调度 🔩 流程设计专家 设计多专家协作流程
- 明确每个 Phase 由哪些专家参与、产出什么、验收标准

**Step 3：向老板汇报完整方案**
- 需求文档 + 专家阵容 + 协作流程 + 各阶段验收标准
- 老板确认后才开始执行

**Step 4：产品方案（Gate 2）**
- 调度 📐 产品定义专家 基于需求文档出产品方案
- 产品专家与老板过方案，输出产品文档
- **老板确认后**方可进入 Step 5

**Step 5：架构设计（Gate 3）**
- 调度 🏛️ 技术架构专家 基于需求文档 + 产品文档出架构设计
- 架构师与老板过方案，输出架构文档
- **老板确认后**方可进入开发

**Step 6：模块化串行开发**
- 按架构文档拆模块，一次只开发一个模块
- 每模块开发完成后单独验收，通过后才能开发下一个

**Step 7：整体验收交付**
- 全部模块完成后进行整体验收（集成测试 + 端到端验证）
- 多专家审查交付

## 行业洞察 (2026-03-19 AI 组织进化调研)

### Agent 协作趋势
- **Agent 作为工作单元**: lobehub 正在将 Agent 定义为工作交互基本单元，而非工具。支持多 Agent 协作、团队设计。
- **自组织多 Agent 平台**: OpenMOSS 实现零人工干预的计划 - 执行 - 审查 - 巡逻闭环。
- **可编程协作流**: Teamclaw 等通过 YAML 配置定义专家协作拓扑（顺序、并行、复杂结构）。
- **监控可视化**: agent-dashboard、myworktree 提供实时状态监控和输出回放。

### 上下文工程进展
- **gsd-2 系统**: 元提示 + 上下文工程 + 规范驱动开发，使 Agent 能长时间自主工作不失焦。
- **Prompt 标准化**: PromptG (spec 项目) 提供工具无关的 Prompt 存储、版本控制、分享格式。
- **全生命周期管理**: langfuse 等平台提供 Prompt 管理、Playground、数据集、评估、可观测性一体化。

### 自主 Agent 能力
- **零干预目标完成**: Autonomous-Agent-X 等项目实现完全自主迭代完成任务。
- **AI 员工概念**: steward 等项目实现自托管自主 Agent 管理基础设施。
- **多技能 Agent**: flowwink 实现 37+ 技能、工作流 DAG、A2A 委托、自愈能力。

1. 引入"Agent 单元"概念，27 位专家重构为独立工作单元

3. 建立 Agent 状态监控面板（HEARTBEAT.md 可视化）
4. 采用 PromptG 格式标准化指令包模板
5. 在低风险场景试点自主模式（文档生成、代码审查）
