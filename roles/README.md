# 角色索引与选角指南

> Leader 在 spawn 子 Agent 前，查此文件选定角色，然后读取对应目录下的 SOUL.md + STANDARDS.md 注入 task。

---

## 角色一览

| 角色 | 中文名 | 目录 | 擅长领域 | 典型任务 |
|------|--------|------|----------|----------|
| 🎯 需求分析专家 | 需求分析专家 | `需求分析专家-requirement-analyst/` | 需求挖掘与澄清 | 将模糊想法变为结构化需求文档、5轮对话挖掘、10维扫描 |
| 🔭 竞品调研专家 | 竞品调研专家 | `竞品调研专家-competitive-analyst/` | 竞品深度拆解 | 单个竞品的产品拆解、策略分析、功能对标、差异化建议 |
| 📊 市场评估专家 | 市场评估专家 | `市场评估专家-market-assessor/` | 市场可行性评估 | 市场规模分析、竞争格局全景、趋势时机判断、机会风险评估、商业可行性初判 |
| 🔍 用户研究专家 | 用户研究专家 | `用户研究专家-user-researcher/` | 用户洞察与画像 | 用户信息采集、画像构建、旅程地图、需求推演、优先级排序、洞察转化 |
| 🧭 战略定位专家 | 战略定位专家 | `战略定位专家-strategic-positioner/` | 战略方向与竞争定位 | 战略选择分析、竞争定位、价值主张设计、边界划定、战略验证、战略叙事 |
| 📐 产品定义专家 | 产品定义专家 | `产品定义专家-product-definer/` | 产品形态与功能定义 | 产品概念定义、场景拆解、功能范围界定、产品规则设计、成功标准定义、产品边界管理 |
| 🔩 流程设计专家 | 流程设计专家 | `流程设计专家-process-designer/` | 执行流程设计 | 产出物拆解、角色定义、协作拓扑、执行流程编排 |
| 🧬 角色创造专家 | 角色创造专家 | `角色创造专家-role-creator/` | AI专家角色创造 | 专家池匹配、创造角色设定（SOUL+STANDARDS）、组建项目团队 |
| 🔧 Skill 开发专家 | Skill开发专家 | `Skill开发专家-skill-developer/` | OpenClaw Skill 开发 | 创建/改进 SKILL.md、写辅助脚本、Skill 发布 |
| 📝 技术文档专家 | 技术文档专家 | `技术文档专家-technical-writer/` | 文档与方案 | 撰写文档、整理知识、设计方案、写 README |
| 🔍 代码审查专家 | 代码审查专家 | `代码审查专家-code-reviewer/` | 代码质量 | Code Review、架构评审、性能分析、安全审计 |
| 🔬 技术调研专家 | 技术调研专家 | `技术调研专家-researcher/` | 技术调研与选型 | 技术选型、可行性评估、技术方案调研 |
| 🎨 体验设计专家 | 体验设计专家 | `体验设计专家-experience-designer/` | 用户体验设计 | 信息架构设计、任务流程设计、界面布局与交互模式、体验语言定义、体验地图绘制、可用性保障、设计规范输出 |
| 🏛️ 技术架构专家 | 技术架构专家 | `技术架构专家-technical-architect/` | 系统架构设计 | 架构模式判断、系统分层、模块边界、技术选型、AI集成方案、非功能性设计、ADR |
| 🏗️ 项目管理专家 | 项目管理专家 | `项目管理专家-project-manager/` | 项目执行调度 | 指令包组装、执行排期、上下文管理、质量检查、异常处理、整合交付 |
| 🗄️ 数据库设计专家 | 数据库设计专家 | `数据库设计专家-database-architect/` | 数据模型设计 | 业务概念建模、逻辑数据模型、物理表结构设计、字段字典、约束与索引策略、schema 迁移评估、一致性风险审查 |

| ⚙️ 后端开发专家 | 后端开发专家 | `后端开发专家-backend-developer/` | 后端服务实现 | 根据契约和模型实现后端服务（Controller/Service/Repository）、输入校验、权限控制、错误处理、事务管理、日志、单元测试、联调支撑 |
| 🖥️ 前端开发专家 | 前端开发专家 | `前端开发专家-frontend-developer/` | 前端界面实现 | 根据体验方案和接口契约实现页面/组件/状态/表单/列表/异步流程/全态处理、组件测试、契约消费一致性校验、联调交接 |
| 🔗 联调集成专家 | 联调集成专家 | `联调集成专家-integration-expert/` | 跨模块集成验证 | 契约对齐验证、端到端流程跑通、字段/状态/错误语义一致性校验、集成问题归因、回归闭环验证、阶段就绪判断 |
| 🧪 测试专家 | 测试专家 | `测试专家-testing-expert/` | 测试验证与质量验收 | 测试策略制定、用例设计、功能/边界/异常验证、缺陷记录与分级、回归验证、发布就绪评估、质量风险判断 |
| 🚀 部署运维专家 | 部署运维专家 | `部署运维专家-deployment-ops/` | 发布保障与运行稳定 | 发布前提校验、环境一致性检查、发布步骤设计、回滚方案、健康检查、监控告警、发布风险评估、运行反馈闭环 |
| 📈 增长运营专家 | 增长运营专家 | `增长运营专家-growth-operator/` | 用户增长与运营 | 增长模型与指标体系、用户获取策略、激活与啊哈时刻、留存与活跃、增长实验与数据驱动、运营活动与内容运营、区域化增长策略 |
| 🔄 迭代优化专家 | 迭代优化专家 | `迭代优化专家-iteration-optimizer/` | 产品迭代管理 | 迭代方向决策、版本范围界定、优先级裁决（ABCD+ICE）、五阶段迭代循环、MVP拆解与渐进交付、迭代复盘与止损判断、教育行业迭代节奏适配 |
| 📦 发布策略专家 | 发布策略专家 | `发布策略专家-release-strategist/` | 产品发布策略与着陆 | 发布规划与版本策略、试点与灰度方案、用户培训设计、沟通与变更管理、数据迁移策略、发布执行管理、效果评估与复盘、特殊场景发布策略 |
| 📈 数据分析专家 | 数据分析专家 | `数据分析专家-data-analyst/` | 数据驱动决策 | 指标体系设计、数据采集与埋点规划、分析方法应用（描述/诊断/预测/建议）、A/B测试设计、数据可视化与叙事、教育数据特殊模型（学校健康度/教师成熟度/教学周期性） |

## 选角决策树

收到任务后，按以下逻辑匹配角色：

```
任务是什么？
│
├─ 需求模糊，需要挖掘和澄清？
│   └─→ 🎯 需求分析专家 · 需求分析专家 (需求分析专家-requirement-analyst/)
│
├─ 需要了解用户是谁 / 用户画像 / 用户旅程 / 用户需求洞察？
│   └─→ 🔍 用户研究专家 · 用户研究专家 (用户研究专家-user-researcher/)
│
├─ 市场规模 / 竞争格局全景 / 趋势时机 / 市场可行性 / 商业可行性初判？
│   └─→ 📊 市场评估专家 · 市场评估专家 (市场评估专家-market-assessor/)
│
├─ 单个竞品深度拆解 / 产品策略分析 / 功能对标 / 技术路线分析？
│   └─→ 🔭 竞品调研专家 · 竞品调研专家 (竞品调研专家-competitive-analyst/)
│
├─ 战略方向选择 / 竞争定位 / 价值主张 / 做什么不做什么 / 凭什么赢？
│   └─→ 🧭 战略定位专家 · 战略定位专家 (战略定位专家-strategic-positioner/)
│
├─ 产品到底是什么 / 产品形态 / 核心场景 / 功能范围 / 产品规则 / MVP定义？
│   └─→ 📐 产品定义专家 · 产品定义专家 (产品定义专家-product-definer/)
│
├─ 产品定义已完成，需要设计用户体验 / 信息架构 / 任务流程 / 交互模式 / 体验语言？
│   └─→ 🎨 体验设计专家 · 体验设计专家 (体验设计专家-experience-designer/)
│
├─ 方案已定，需要拆解执行流程 / 定义角色 / 设计协作？
│   └─→ 🔩 流程设计专家 · 流程设计专家 (流程设计专家-process-designer/)
│
├─ 需要创造AI专家角色设定 / 匹配专家池 / 组建项目团队？
│   └─→ 🧬 角色创造专家 · 角色创造专家 (角色创造专家-role-creator/)
│
├─ 创建/改进 OpenClaw Skill？
│   └─→ 🔧 Skill开发专家 · Skill 开发专家 (Skill开发专家-skill-developer/)
│
├─ 系统架构设计 / 技术选型 / 模块边界 / AI集成方案 / 非功能性设计？
│   └─→ 🏛️ 技术架构专家 · 技术架构专家 (技术架构专家-technical-architect/)
│
├─ 后端服务实现 / Controller-Service-Repository / 事务权限错误处理 / 契约实现？
│   └─→ ⚙️ 后端开发专家 · 后端开发专家 (后端开发专家-backend-developer/)
│
├─ 前端页面实现 / 组件开发 / 状态管理 / 表单交互 / API消费 / 全态处理？
│   └─→ 🖥️ 前端开发专家 · 前端开发专家 (前端开发专家-frontend-developer/)
│
├─ 前后端联调 / 契约对齐验证 / 端到端流程跑通 / 集成问题归因 / 回归闭环？
│   └─→ 🔗 联调集成专家 · 联调集成专家 (联调集成专家-integration-expert/)
│
├─ 测试策略 / 测试用例设计 / 功能验证 / 缺陷记录 / 回归测试 / 发布就绪评估？
│   └─→ 🧪 测试专家 · 测试专家 (测试专家-testing-expert/)
│
├─ 发布准备 / 环境校验 / 部署方案 / 回滚策略 / 监控告警 / 上线风险评估 / 运行反馈？
│   └─→ 🚀 部署运维专家 · 部署运维专家 (部署运维专家-deployment-ops/)
│
├─ 产品发布策略 / 灰度方案 / 试点计划 / 用户培训 / 沟通计划 / 变更管理 / 发布效果评估？
│   └─→ 📦 发布策略专家 · 发布策略专家 (发布策略专家-release-strategist/)
│
├─ 快速原型 / 简单 CRUD 管理后台？
│   └─→ ⚙️ 后端开发专家（后端）+ 🖥️ 前端开发专家（前端）协作，或按复杂度选单人
│
├─ 写文档 / 方案设计 / 知识整理？
│   └─→ 📝 技术文档专家 · 技术文档专家 (技术文档专家-technical-writer/)
│
├─ 审查代码 / 评审架构 / 检查质量？
│   └─→ 🔍 代码审查专家 · 代码审查专家 (代码审查专家-code-reviewer/)
│
├─ 技术选型 / 可行性评估 / 技术方案调研？
│   └─→ 🔬 技术调研专家 · 技术调研专家 (技术调研专家-researcher/)
│
├─ 多角色项目执行调度 / 指令包组装 / 全局上下文管理？
│   └─→ 🏗️ 项目管理专家 · 项目管理专家 (项目管理专家-project-manager/)
│
├─ 数据模型设计 / 实体建模 / 表结构设计 / 字段字典 / 约束与索引 / schema 迁移评估？
│   └─→ 🗄️ 数据库设计专家 · 数据库设计专家 (数据库设计专家-database-architect/)
│
├─ 接口契约设计 / API Schema / 错误模型 / 版本策略 / 幂等规则 / Mock 输出 / 契约审查？
│   └─→ 📜 API设计专家 · API 设计专家 (API设计专家-api-architect/)
│
├─ 用户增长策略 / 获客渠道 / 激活优化 / 留存提升 / 运营活动 / 增长实验 / 区域拓展？
│   └─→ 📈 增长运营专家 · 增长运营专家 (增长运营专家-growth-operator/)
│
├─ 指标体系设计 / 数据埋点规划 / 数据分析（漏斗/同期群/归因）/ A/B测试设计 / 数据看板 / 数据异常排查？
│   └─→ 📈 数据分析专家 · 数据分析专家 (数据分析专家-data-analyst/)
│
└─ 以上都不匹配？
    ├─ 偏后端构建 → ⚙️ 后端开发专家
    ├─ 偏前端构建 → 🖥️ 前端开发专家
    ├─ 偏分析/输出文字 → 📝 技术文档专家
    └─ 偏信息收集 → 🔬 技术调研专家
```

### ⚠️ 易混淆角色辨析

| 场景 | 选谁 | 理由 |
|------|------|------|
| "这个市场有多大？谁在做？值不值得入场？" | 📊 市场评估专家 | 宏观市场格局判断 |
| "这个竞品到底怎么做的？它为什么这么设计？" | 🔭 竞品调研专家 | 单个竞品深度拆解 |
| "我们到底做什么方向？跟别人有什么不同？凭什么赢？" | 🧭 战略定位专家 | 战略方向选择与竞争定位 |
| "先做一轮市场摸底 + 竞品拆解" | 📊 市场评估专家（格局）→ 🔭 竞品调研专家（深拆 Top 竞品） | 先全景后聚焦 |
| "用户是谁？他们需要什么？" | 🔍 用户研究专家 | 微观用户视角 |
| "这个项目能不能做？"（全面评估） | 📊 市场评估专家 ∥ 🔍 用户研究专家（并行）→ 联合输出 | Phase 0 双视角 |
| "市场可以做，但具体做什么方向？怎么定位？" | 🧭 战略定位专家 | 在市场可行性确认后做方向决策 |
| "差异化应该从哪个方向切入？" | 🧭 战略定位专家（主）← 📊 市场评估专家（市场数据支撑） | 定位决策由战略定位专家做，市场评估专家提供市场事实 |
| "这个产品到底是什么？长什么样？包含哪些功能？" | 📐 产品定义专家 | 战略方向已定，需要定义具体产品形态 |
| "MVP 应该包含哪些功能？" | 📐 产品定义专家 | 功能范围界定和优先级分层 |
| "定位有了，接下来怎么把它变成具体的产品？" | 📐 产品定义专家 | 战略定位→产品定义的衔接 |
| "用户用起来什么感觉？信息怎么组织？任务怎么走？" | 🎨 体验设计专家 | 产品定义已完成，需要设计使用体验 |
| "页面布局怎么安排？交互用什么模式？" | 🎨 体验设计专家 | 体验层的界面框架和交互设计 |
| "产品'说话'的风格是什么？按钮文案怎么写？" | 🎨 体验设计专家 | 体验语言和文案设计 |
| "系统怎么分层？模块怎么切？用什么技术？" | 🏛️ 技术架构专家 | 系统架构设计和技术选型 |
| "数据模型怎么设计？实体怎么拆？字段怎么定？" | 🗄️ 数据库设计专家 | 实体级数据模型设计 |
| "系统级存储策略用 SQL 还是 NoSQL？" | 🏛️ 技术架构专家 | 系统级数据架构方向 |
| "这张表的字段、约束、索引怎么设计？" | 🗄️ 数据库设计专家 | 物理表结构详细设计 |
| "这个技术能不能用？有没有替代方案？"（单项调研） | 🔬 技术调研专家 | 单项技术深度调研 |
| "整个系统的技术方案怎么定？"（系统级） | 🏛️ 技术架构专家 | 系统级架构设计和选型决策 |
| "产品定义有了，技术上怎么实现？" | 🏛️ 技术架构专家 | 产品定义→技术架构的衔接 |
| "接口怎么设计？字段叫什么？错误怎么返回？" | 📜 API设计专家 | 接口契约详细设计 |
| "系统间用 REST 还是 gRPC？版本怎么管？" | 🏛️ 技术架构专家 | 系统级接口风格方向 |
| "这个接口的请求/响应 Schema 怎么定？" | 📜 API设计专家 | 细粒度契约设计 |
| "接口字段和数据库字段怎么对齐？" | 📜 API设计专家 ∥ 🗄️ 数据库设计专家 | 字段语义联合对齐 |
| "这个页面要实现出来（有体验方案和接口文档）" | 🖥️ 前端开发专家 | 正式前端实现，有完整上游输入 |
| "快速出个 demo / 管理后台" | ⚙️ 后端开发专家（后端）+ 🖥️ 前端开发专家（前端）协作 | 后端找后端开发专家、前端找前端开发专家，简单场景可单人承接 |
| "前端组件怎么拆？状态怎么管？全态怎么覆盖？" | 🖥️ 前端开发专家 | 前端架构级实现问题 |

### 🔑 后端开发专家 vs 前端开发专家 边界说明

两者是"后端深耕"与"前端深耕"的分工关系：

| 维度 | ⚙️ 后端开发专家（后端开发） | 🖥️ 前端开发专家（前端开发） |
|------|----------------------|----------------------|
| **核心问题** | "后端服务怎么写得正确、稳定、可维护？" | "前端怎么做到体验方案一致、状态完备、契约准确？" |
| **工作性质** | 后端深度实现 + 服务工程 | 前端深度实现——组件设计、状态管理、全态覆盖、契约消费 |
| **深度** | 严格分层（Controller/Service/Repository），事务、权限、错误码、日志一个不少 | 组件树设计、状态分层、全态处理（loading/error/empty/success/disabled）、无障碍、性能优化 |
| **适用场景** | 正式后端服务、复杂业务逻辑、高可靠性要求的模块 | 正式产品前端、复杂交互、多角色界面、体验要求高的页面 |
| **上游依赖** | 数据库设计专家的数据模型 + API设计专家的接口契约 + 技术架构专家的架构约束 | 体验设计专家（体验方案）+ API设计专家（接口契约）+ 技术架构专家（架构约束） |
| **典型任务** | "实现权限系统" "实现订单事务流" "实现数据批量导入" | "实现教师端课程管理页" "做学生端交互式练习组件" "接入10个API的复杂表单" |

**易混淆场景速查**：

| 场景 | 选谁 | 理由 |
|------|------|------|
| "后端服务怎么实现？事务怎么管？权限怎么控？" | ⚙️ 后端开发专家 | 正式后端服务实现 |
| "需要快速搭个能跑的原型" | ⚙️ 后端开发专家（后端）+ 🖥️ 前端开发专家（前端）协作 | 后端和前端各司其职 |
| "这个 feature 的后端，事务/权限/错误处理要做到位" | ⚙️ 后端开发专家 | 深度后端服务工程 |
| "写个前端页面 + 简单的后端 API" | 🖥️ 前端开发专家（前端）+ ⚙️ 后端开发专家（后端） | 前后端分别由各自专家负责 |
| "实现批量导入功能，要求事务一致性" | ⚙️ 后端开发专家 | 复杂后端逻辑 |
| "做个管理后台，列表+详情+编辑" | 🖥️ 前端开发专家（前端）+ ⚙️ 后端开发专家（后端） | 前后端分别实现 |
| "这个页面要实现出来（有体验方案和接口文档）" | 🖥️ 前端开发专家 | 正式前端实现 |
| "前端组件怎么拆？状态怎么管？全态怎么覆盖？" | 🖥️ 前端开发专家 | 前端架构级实现问题 |

**协作模式**：
- **前后端分离开发** → 前端开发专家（前端）∥ 后端开发专家（后端）→ 基于相同 API 契约联调
- **纯后端服务**（如权限/事务/批量处理）→ 后端开发专家独立完成
- **正式产品前端**（体验方案完备、交互复杂）→ 前端开发专家独立完成
- **简单管理后台 CRUD** → 前端开发专家（前端）+ 后端开发专家（后端）协作，简单场景可单人承接
