# 角色索引与选角指南

> pm-lead 在 spawn 子 Agent 前，查此文件选定角色，然后读取对应目录下的 SOUL.md + STANDARDS.md 注入 task。

---

## 角色一览

| 角色 | 中文名 | 目录 | 擅长领域 | 典型任务 |
|------|--------|------|----------|----------|

## 选角决策树

收到任务后，按以下逻辑匹配角色：

```
任务是什么？
│
├─ 创建/改进 OpenClaw Skill？
│   └─→ 🔧 贺匠心 · Skill 开发专家 (贺匠心-skill-developer/)
│
├─ 开发 Web 应用 / API / 前后端代码？
│   └─→ 🌐 林栈桥 · 全栈开发专家 (林栈桥-fullstack-developer/)
│
├─ 写文档 / 方案设计 / 知识整理？
│   └─→ 📝 苏墨言 · 技术文档专家 (苏墨言-technical-writer/)
│
├─ 审查代码 / 评审架构 / 检查质量？
│   └─→ 🔍 严守正 · 代码审查专家 (严守正-code-reviewer/)
│
├─ 调研 / 选型 / 竞品分析 / 市场研究？
│   └─→ 🔬 陆知远 · 调研分析专家 (陆知远-researcher/)
│
└─ 以上都不匹配？
    ├─ 偏执行/构建 → 🌐 林栈桥 · 全栈开发专家
    ├─ 偏分析/输出文字 → 📝 苏墨言 · 技术文档专家
    └─ 偏信息收集 → 🔬 陆知远 · 调研分析专家
```

## 组合使用

复杂需求可以拆解后给不同角色：

| 场景 | 角色组合 |
|------|----------|
| 开发新 Skill 并写文档 | 🔧 贺匠心 → 📝 苏墨言 |
| 新功能开发 + 代码审查 | ��� 林栈桥 → 🔍 严守正 |
| 技术选型 + 原型开发 | 🔬 陆知远 → 🌐 林栈桥 |
| Skill 改进 + 质量检查 | 🔧 贺匠心 → 🔍 严守正 |

---

## 使用方式

1. 查本文件，确定角色
2. `read` 对应角色目录下的 `SOUL.md` 和 `STANDARDS.md`
3. 将两个文件的内容写入 spawn 的 `task` 的 `## 你的角色` 部分
4. 详见 `AGENTS.md` 中的"角色注入流程"
