# 角色索引与选角指南

> pm-lead 在 spawn 子 Agent 前，查此文件选定角色，然后读取对应模板注入 task。

---

## 角色一览

| 角色 | 文件 | 擅长领域 | 典型任务 |
|------|------|----------|----------|
| 🔧 Skill 开发专家 | `skill-developer.md` | OpenClaw Skill 开发 | 创建/改进 SKILL.md、写辅助脚本、Skill 发布 |
| 🌐 全栈开发专家 | `fullstack-developer.md` | Web 应用开发 | 前后端开发、API 设计、数据库、部署配置 |
| 📝 技术文档专家 | `technical-writer.md` | 文档与方案 | 撰写文档、整理知识、设计方案、写 README |
| 🔍 代码审查专家 | `code-reviewer.md` | 代码质量 | Code Review、架构评审、性能分析、安全审计 |
| 🔬 调研分析专家 | `researcher.md` | 调研与分析 | 技术选型、竞品分析、市场调研、可行性评估 |

---

## 选角决策树

收到任务后，按以下逻辑匹配角色：

```
任务是什么？
│
├─ 创建/改进 OpenClaw Skill？
│   └─→ 🔧 Skill 开发专家 (skill-developer.md)
│
├─ 开发 Web 应用 / API / 前后端代码？
│   └─→ 🌐 全栈开发专家 (fullstack-developer.md)
│
├─ 写文档 / 方案设计 / 知识整理？
│   └─→ 📝 技术文档专家 (technical-writer.md)
│
├─ 审查代码 / 评审架构 / 检查质量？
│   └─→ 🔍 代码审查专家 (code-reviewer.md)
│
├─ 调研 / 选型 / 竞品分析 / 市场研究？
│   └─→ 🔬 调研分析专家 (researcher.md)
│
└─ 以上都不匹配？
    ├─ 偏执行/构建 → 🌐 全栈开发专家
    ├─ 偏分析/输出文字 → 📝 技术文档专家
    └─ 偏信息收集 → 🔬 调研分析专家
```

## 组合使用

复杂需求可以拆解后给不同角色：

| 场景 | 角色组合 |
|------|----------|
| 开发新 Skill 并写文档 | 🔧 Skill 开发 → 📝 技术文档 |
| 新功能开发 + 代码审查 | 🌐 全栈开发 → 🔍 代码审查 |
| 技术选型 + 原型开发 | 🔬 调研分析 → 🌐 全栈开发 |
| Skill 改进 + 质量检查 | 🔧 Skill 开发 → 🔍 代码审查 |

---

## 使用方式

1. 查本文件，确定角色
2. `read` 对应角色模板文件
3. 将模板内容写入 spawn 的 `task` 的 `## 你的角色` 部分
4. 详见 `AGENTS.md` 中的"角色注入流程"
