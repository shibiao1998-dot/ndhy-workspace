# 竞品调研与分析专家的完整能力框架

------

## 一、竞品专家的核心能力维度（7个）

### 1. 信息搜猎能力

知道去哪里找、怎么找、怎么判断信息的可靠性。竞品调研80%的质量取决于"你能不能找到别人找不到的东西"。

### 2. 产品拆解能力

看到一个产品，能像拆机器一样把它拆成——**定位、用户、功能架构、商业模式、技术路线、运营策略**这些零件，而不是只停留在"界面长什么样"。

### 3. 行业认知能力

知道这个赛道的格局、发展阶段、关键玩家、行业共识和行业争议。没有行业认知的竞品分析就是在做功能对比表，没有灵魂。

### 4. 归因与推理能力

不只是记录"它做了什么"，而是能推断"它为什么这么做"。——**从产品决策反推背后的战略意图**，这是初级分析师和高级分析师的核心分水岭。

### 5. 比较与评价能力

能建立合理的比较维度，在同一把尺子下度量不同竞品。不是"A有这个功能B没有"这种罗列，而是能给出**"在XX场景下A的方案优于B，因为……"**这种有判断力的结论。

### 6. 洞察提炼能力

从一堆事实中提炼出**对我们有行动指导意义的结论**。竞品报告的终极价值不是"告诉你别人做了什么"，而是"告诉你我们该怎么做"。

### 7. 结构化表达能力

把海量信息组织成清晰、易读、有层次的报告，让决策者在10分钟内抓住核心结论。

## 二、竞品调研的信息来源地图（从哪里找）

### 第一圈：直接信息（最可靠）

| 来源                   | 能获得什么                       | 技巧                               |
| ---------------------- | -------------------------------- | ---------------------------------- |
| **竞品产品本身**       | 功能、流程、体验、定价           | 注册试用、购买体验、录屏拆解       |
| **官方网站**           | 定位、卖点、目标客户、定价策略   | 重点看首页话术、客户案例页、定价页 |
| **官方文档/帮助中心**  | 功能全貌、技术架构线索           | 帮助文档往往比官网更真实           |
| **App Store/应用市场** | 版本更新记录、用户评价、评分趋势 | 更新日志 = 产品优先级的时间线      |

### 第二圈：公开商业信息（较可靠）

| 来源                                            | 能获得什么                      |
| ----------------------------------------------- | ------------------------------- |
| **融资/财报信息**（Crunchbase、天眼查、IT桔子） | 资金实力、投资方、估值变化      |
| **招聘信息**（Boss直聘、LinkedIn）              | 正在招什么人 = 正在布局什么方向 |
| **专利/论文**                                   | 技术路线、未来产品方向的线索    |
| **创始人/高管公开发言**（演讲、采访、播客）     | 战略思路、价值观、对行业的判断  |
| **行业报告**（艾瑞、Gartner、CB Insights等）    | 市场规模、格局、趋势            |

### 第三圈：用户侧信息（真实但零散）

| 来源                                         | 能获得什么                 |
| -------------------------------------------- | -------------------------- |
| **用户评价**（G2、Capterra、知乎、小红书）   | 真实口碑、核心优缺点       |
| **社区/论坛**（Reddit、GitHub Issues、贴吧） | 用户痛点、功能诉求、吐槽   |
| **社交媒体**（Twitter/X、微博）              | 品牌声量、用户情绪         |
| **客户案例/成功故事**                        | 核心使用场景、价值主张验证 |

### 第四圈：隐性信息（需要推理）

| 来源                                  | 能获得什么                            |
| ------------------------------------- | ------------------------------------- |
| **SEO/SEM策略**（SimilarWeb、Ahrefs） | 竞品在抢什么关键词 = 在抢什么用户心智 |
| **内容营销策略**                      | 竞品在教育市场什么 = 在塑造什么认知   |
| **合作伙伴/生态**                     | 谁跟谁结盟 = 行业未来的势力版图       |
| **离职员工评价**（Glassdoor、脉脉）   | 内部文化、真实困境                    |

## 三、竞品调研的完整SOP（6步）

```
步骤1：明确调研目的
    │   ← 从需求文档中提取核心问题
    │   关键动作：定义"我们到底想通过竞品分析搞清楚什么"
    │   
    │   常见调研目的类型：
    │   A. 这个领域有没有人在做？做到什么程度了？（市场验证型）
    │   B. 别人怎么做的？有什么可以借鉴的？（方案参考型）
    │   C. 我们跟竞品比，优势劣势在哪？（竞争定位型）
    │   D. 行业未来往哪走？（趋势洞察型）
    │
    ▼
步骤2：建立竞品清单
    │   关键动作：找到所有相关竞品，分层分类
    │
    │   三层分类法：
    │   ┌─ 直接竞品：做同样的事，给同样的人（必须深度分析）
    │   ├─ 间接竞品：解决同样的问题，但方案不同（重点关注思路）
    │   └─ 潜在竞品：现在不是竞品，但可能切进来（保持监控）
    │
    │   筛选原则：
    │   - 直接竞品选3-5个做深度拆解（不要贪多）
    │   - 间接竞品选2-3个看思路差异
    │   - 潜在竞品列出来即可，不用深入
    │
    ▼
步骤3：信息采集
    │   关键动作：按照"信息来源地图"系统性采集
    │
    │   采集框架（每个竞品都过一遍）：
    │   ┌─ 基本信息：公司背景、团队、融资、规模
    │   ├─ 产品信息：功能、定价、技术路线、平台覆盖
    │   ├─ 用户信息：目标用户、获客方式、用户评价
    │   ├─ 商业信息：商业模式、收入规模、合作生态
    │   └─ 战略信息：近期动作、公开战略、招聘方向
    │
    │   采集纪律：
    │   - 事实和推测必须分开标注
    │   - 记录信息来源，方便回溯验证
    │   - 信息有时效性，标注采集日期
    │
    ▼
步骤4：产品深度拆解
    │   关键动作：对直接竞品做"解剖级"分析
    │
    │   拆解9个层面：
    │   ① 产品定位 → 它说自己是什么？解决什么问题？给谁用？
    │   ② 核心功能 → 主打什么功能？功能之间的逻辑关系？
    │   ③ 用户体验 → 上手难度？关键流程的体验？
    │   ④ 信息架构 → 功能怎么组织的？优先级怎么排的？
    │   ⑤ 定价策略 → 怎么收费？免费版和付费版的差异？
    │   ⑥ 技术路线 → 用了什么技术？技术选型透露了什么意图？
    │   ⑦ 运营策略 → 怎么获客？怎么留存？内容策略是什么？
    │   ⑧ 迭代轨迹 → 版本更新历史，能看出产品演进方向
    │   ⑨ 差异化点 → 它跟其他竞品最大的不同是什么？
    │
    ▼
步骤5：比较分析与洞察提炼
    │   关键动作：把拆解结果放在一起比较，提炼有价值的结论
    │
    │   四种分析方法：
    │   ┌─ 功能矩阵对比 → 谁有什么、谁没什么，一目了然
    │   ├─ 定位地图 → 在二维坐标上标出各竞品的位置（如：专业度 vs 易用性）
    │   ├─ SWOT分析 → 对每个竞品做优劣势分析
    │   └─ 策略归因 → 它为什么这么做？背后的逻辑是什么？
    │
    │   核心产出：
    │   - 行业共识是什么（大家都在做的）
    │   - 差异化方向有哪些（各家不同的选择）
    │   - 市场空白在哪里（没人做或没做好的）
    │   - 最佳实践是什么（值得借鉴的）
    │   - 常见陷阱是什么（别人踩过的坑）
    │
    ▼
步骤6：得出结论与建议
        关键动作：回到步骤1的调研目的，给出明确答案
    
        必须回答的问题：
        ① 这个市场的竞争格局是什么样的？
        ② 我们的机会在哪里？
        ③ 我们应该借鉴什么？
        ④ 我们应该避开什么？
        ⑤ 我们的差异化方向应该是什么？
```

## 四、高质量竞品分析报告的结构模板

### 报告结构（7个模块）

```
一、摘要与核心结论（1页）
    │  这份报告最重要的部分，决策者可能只看这一页
    │  包含：
    │  - 调研背景和目的（一句话）
    │  - 3-5条核心发现（最有价值的洞察）
    │  - 对我们的明确建议（该做什么/不该做什么）
    │
二、市场概况（1-2页）
    │  - 市场规模与增长趋势
    │  - 市场发展阶段判断（萌芽/成长/成熟/衰退）
    │  - 竞争格局总览图
    │  - 关键趋势与驱动力
    │
三、竞品清单与选择理由（1页）
    │  - 直接竞品 / 间接竞品 / 潜在竞品的列表
    │  - 为什么选这几个做深度分析
    │
四、竞品逐个拆解（每个竞品2-3页）
    │  每个竞品包含：
    │  - 基本画像：一句话定位 + 关键数据
    │  - 产品分析：核心功能、亮点、槽点
    │  - 商业分析：模式、定价、获客
    │  - 战略判断：它在下什么棋？
    │  - 对我们的启示：可借鉴什么？要警惕什么？
    │
五、横向对比分析（2-3页）
    │  - 功能对比矩阵表
    │  - 竞品定位地图
    │  - 定价策略对比
    │  - 核心差异化总结
    │  - 市场空白点分析
    │
六、关键洞察与机会（1-2页）
    │  不是重复前面的内容，而是提炼出：
    │  - 行业做对了什么？（最佳实践）
    │  - 行业做错了什么？（共同盲区）
    │  - 用户还有什么未被满足的需求？（机会）
    │  - 行业接下来可能往哪走？（趋势判断）
    │
七、对我们的建议（1-2页）
       这是报告的第二重要部分：
       - 我们应该进入/不进入这个方向？（如果是市场验证型）
       - 我们的差异化策略建议
       - 建议借鉴的具体做法
       - 建议避开的具体做法
       - 优先级建议
```

## 五、让竞品分析报告质量拉开差距的8条原则

| #    | 原则                                   | 说明                                                         |
| ---- | -------------------------------------- | ------------------------------------------------------------ |
| 1    | **结论前置，细节在后**                 | 先给答案，再给论据。不要让读者翻到第20页才看到结论           |
| 2    | **事实与观点分开标注**                 | "它的定价是99元/月"是事实，"它的定价偏高"是观点。混在一起会丧失可信度 |
| 3    | **不要做功能清单搬运工**               | "A有xx功能，B没有"这种表格谁都会做。高质量分析要回答"为什么有/没有"以及"这意味着什么" |
| 4    | **每个分析必须落回"对我们意味着什么"** | 竞品分析不是学术研究，没有对我们的指导意义的分析就是废话     |
| 5    | **承认信息的局限性**                   | "以下为推测""该信息未经验证"——诚实标注反而增加报告的可信度   |
| 6    | **关注竞品的变化，而非静态**           | 竞品最新3个版本更新了什么，比它现在长什么样更有价值。**变化暴露战略意图** |
| 7    | **用可视化代替大段文字**               | 定位地图、功能矩阵、时间线……一张好图胜过1000字               |
| 8    | **给出有态度的建议，而非中立的罗列**   | "综上，建议我们……"而不是"以上供参考"。不敢给建议的竞品报告是半成品 |

## 六、竞品专家常用的分析框架工具箱

| 框架             | 适用场景         | 核心用法                                                     |
| ---------------- | ---------------- | ------------------------------------------------------------ |
| **功能矩阵**     | 功能层面对比     | 横轴=竞品，纵轴=功能项，打勾/评分                            |
| **定位地图**     | 战略层面对比     | 选两个关键维度做坐标轴，标出各竞品位置                       |
| **SWOT**         | 单个竞品全面评估 | 优势/劣势/机会/威胁                                          |
| **价值曲线**     | 差异化方向寻找   | 横轴=竞争要素，纵轴=投入程度，画出各竞品的曲线，找到独特的"凹凸"形态 |
| **用户旅程对比** | 体验层面对比     | 同一个任务在不同竞品中走一遍，对比每一步的体验               |
| **商业模式画布** | 商业层面对比     | 9个模块拆解各竞品的商业逻辑                                  |
| **技术栈对比**   | 技术层面对比     | 各竞品的技术选型、架构思路                                   |
