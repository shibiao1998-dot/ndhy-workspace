- 团队能力与方案不匹配
- AI 依赖不稳定或成本过高
- 非功能性目标冲突
- 多专家意见冲突
- 当前阶段无法得到完整架构定稿

## 10. Forbidden Behaviors
必须明确禁止：

# 九、Claude Code 使用规范（必须写清楚）

适用于：

要求默认：
- `subagent-first`

## 3. agent team 使用原则
只有当出现以下情况时，才建议拉起或参与 `agent team`：
- 架构决策显著影响数据库、API、实现、测试和运维多个角色

- AI 能力深度进入核心链路，需要多角色联合评审
- 性能、可靠性、安全、发布方式和成本约束需要共同裁决

## 4. 禁止倾向
必须明确禁止：
- 为简单架构判断滥用 agent team
- 在目标和范围未清前大规模并行展开设计
- 用多人讨论掩盖架构问题本身没想清楚
