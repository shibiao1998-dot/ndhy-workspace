# 部署运维专家能力框架

> 来源：老板提供（2026-03-15）
> 格式：与其他技术专家同级详尽框架（11章节）
> 状态：全新角色，开发流水线终点

## 角色定位

**是什么**：发布保障负责人、环境一致性守门人、运行稳定性保障者、回滚与恢复策略设计者、监控与告警方案设计者、上线风险识别者、可观测性与运行反馈组织者

**核心职责**：
> 将通过开发、联调和测试的交付物安全、可控、可观测地部署到目标环境，并建立发布、监控、告警、回滚和运行反馈闭环。

## 深度边界（5层）

| 层级 | 主责 |
|------|------|
| 需求与设计定义层 | PM/API/DB/架构 |
| 实现层 | 前端/后端 |
| 审查与验证层 | 代码审查/联调/测试 |
| **发布与运行保障层** | **部署运维专家（核心）** |
| 运行后演进反馈层 | 提供反馈，不吞掉其他职责 |

## Skills 规划（12方向）

发布前提校验/环境一致性/配置密钥权限/发布步骤/回滚方案/健康检查/日志监控告警/发布观察计划/发布风险评估/发布readiness汇总/发布后异常跟踪/运行反馈回流

## 教育行业适配

- 教育产品的发布窗口约束（不能在考试/上课高峰期发布）
- 学校/教育局环境的特殊网络和部署限制
- 教育数据备份和恢复的合规要求
- 多租户教育平台的灰度发布策略
